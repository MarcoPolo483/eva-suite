    EVA DA 2.0 Approach (Motto #4 Quality): - Scalable: Support 1000+ Project Spaces - Automated: Self-service Sandbox creation - Centralized: EVA RAG Command Center for management - RBAC: Fine-grained permissions at Space + document level - Cost Attribution: Per-Space metering and cost recovery - Service Levels: Universal, Restricted, Client-Owned tiers

    ğŸ”§ Required Components (Multi-Repo Breakdown)¶

    1. eva-rag (RAG Pipeline Core)¶

    Responsibilities: - Document ingestion (Phases 1-4 complete) - Multi-tenant vector indexing (1 index per Project Space OR shared index with metadata filters) - Search/retrieval (hybrid search, reranking) - Quality metrics (retrieval accuracy, latency)

    New Requirements for Scalability: - space_id parameter on ALL operations (tenant isolation) - Dynamic index provisioning (auto-create index for new Spaces) - Cost attribution (track embedding/search costs per Space) - Health monitoring (per-Space index health)

    Handoff: HANDOFF-EVA-RAG-SCALABLE.md

    2. eva-api (API Gateway + APIM)¶

    Responsibilities: - Expose EVA Meta endpoints (Books, Spaces, Data Sources, Services) - Expose RAG endpoints (ingest, search) with space_id routing - RBAC enforcement (user â†’ Spaces â†’ permissions) - Rate limiting (per-Space quotas) - Cost metering (track API calls per Space)

    New Requirements: - Project Space CRUD endpoints - RBAC middleware (check user access to Space before routing to eva-rag) - Cost attribution middleware (log usage per Space) - APIM policies (throttling, Protected B enforcement)

    Handoff: HANDOFF-EVA-API-COMMAND-CENTER.md

    3. eva-meta (Metadata Registry)¶

    Responsibilities: - Store Project Space definitions (Sandbox vs Production, cost center, owner) - Store Data Source catalog (per-Space document locations) - Store Service registry (RAG endpoints, health URLs) - Store RBAC mappings (users â†’ Spaces â†’ roles)

    New Requirements: - eva_project_spaces table (space_id, name, tier [sandbox/production], cost_center, status, owner_id) - eva_space_users table (space_id, user_id, role [admin/contributor/reader]) - eva_space_config table (space_id, rag_params [JSON], ui_config [JSON]) - eva_cost_attribution table (space_id, date, cost, usage_tokens, documents_ingested)

    Handoff: HANDOFF-EVA-META-PROJECT-SPACES.md

    4. eva-da-2 (Dashboard UI)¶

    Responsibilities: - Platform Command Center (global health, all Spaces, cost recovery) - RAG Command Center (ingestion health, vector status, quality metrics) - Project Space Command Center (per-Space uptime, users, cost, start/stop) - EVA Library Panel (Books navigation with public/private tiers) - Sandbox Creator (self-service wizard for PS-A to create new Spaces)

    New Requirements: - <eva-platform-command-center> component (global view) - <eva-rag-command-center> component (RAG-specific view) - <eva-space-command-center space-id="jurisprudence"> component (per-Space view) - <eva-space-creator> wizard (guided Sandbox creation) - RBAC-aware navigation (show Spaces user has access to)

    Handoff: HANDOFF-EVA-DA-2-COMMAND-CENTERS.md

    5. eva-infra (Azure Infrastructure)¶

    Responsibilities: - Provision Azure AI Search (with multi-index support OR shared index with filters) - Provision Cosmos DB (metadata, provenance, cost attribution) - Provision Key Vault (connection strings, API keys) - Provision Application Insights (telemetry per-Space) - Provision Cost Management (cost tags per-Space)

    New Requirements: - Terraform module for Project Space provisioning (reusable for 1000+ Spaces) - Resource tagging strategy (space_id, tier, cost_center tags) - Cost allocation reports (per-Space cost breakdowns) - Monitoring dashboards (per-Space health metrics)

    Handoff: HANDOFF-EVA-INFRA-MULTI-TENANT.md
