    üö® Error Recovery Procedures¶

    Philosophy: Fail fast on critical errors, skip gracefully on non-critical failures, ALWAYS log for post-mortem analysis.

    Failure Modes & Recovery Actions¶

    Failure Mode Severity Recovery Action Example
    Pre-flight check fails (missing tools) CRITICAL Abort entire workflow TypeDoc not installed
    TypeDoc generation fails (1 repo) WARNING Skip repo, log error, continue with others Syntax error in one .ts file
    MkDocs build fails (broken link) CRITICAL Abort that repo, continue with others Missing file in nav
    RAG extraction fails (encoding issue) WARNING Skip file, log error, extract others Non-UTF-8 file
    Validation fails (missing metadata) CRITICAL Abort deployment for that repo No _metadata.json
    GitHub Pages deploy fails (network) CRITICAL Retry 3x with exponential backoff, then fail Timeout pushing to gh-pages

    Retry Logic¶

    3-Attempt Strategy: 

    $maxRetries = 3
$attempt = 1
while ($attempt -le $maxRetries) {
    try {
        # Run operation
        break  # Success, exit loop
    }
    catch {
        if ($attempt -eq $maxRetries) {
            Write-Error "CRITICAL: Failed after $maxRetries attempts"
            exit 1
        }
        Start-Sleep -Seconds ([math]::Pow(2, $attempt))  # Exponential backoff: 2s, 4s, 8s
        $attempt++
    }
}

    Retry Targets: - Network operations (git push, gh workflow trigger) - External tool calls (TypeDoc, MkDocs) - File I/O (large file writes)

    Never Retry: - Syntax errors (won't fix themselves) - Missing files (deterministic) - Configuration errors (require human fix)

    Abort Threshold¶

    50% Rule: If >50% of repos fail, abort entire operation and escalate to Marco.

    $totalRepos = 3  # eva-orchestrator, EVA-Sovereign-UI, eva-da-2
$failedRepos = 0
$successRepos = 0

foreach ($repo in $repos) {
    # Run P22 workflow
    if ($LASTEXITCODE -ne 0) {
        $failedRepos++
    }
    else {
        $successRepos++
    }

    # Check abort threshold
    $failureRate = $failedRepos / $totalRepos
    if ($failureRate -gt 0.5) {
        Write-Error "CRITICAL: Failure rate $failureRate exceeds 50% threshold. Aborting."
        exit 1
    }
}

    Graceful Degradation¶

    Principle: Deliver SOMETHING rather than ALL-OR-NOTHING.

    Example: If EVA DA Book fails but EVA Agentic Book and EVA Sovereign UI Book succeed: - Deploy the 2 successful books - Log EVA DA failure for manual fix - Report partial success to Marco

    Status Report Format: 

    ‚úÖ eva-orchestrator - EVA Agentic Book deployed successfully
‚úÖ EVA-Sovereign-UI - EVA Sovereign UI Book deployed successfully

    Rollback Strategy¶

    GitHub Pages Rollback: 

    # If deploy succeeds but validation fails
gh workflow run publish-docs.yml --ref $(git rev-parse HEAD~1)  # Deploy previous commit

    Manual Rollback: 1. Identify last known-good commit: git log --oneline gh-pages 2. Reset gh-pages branch: git reset --hard <commit-sha> 3. Force push: git push origin gh-pages --force
