    Agile Crew (P15) Inbox¶

    Purpose: Staging area for sprint-ready P02 feature artifacts awaiting Agile Crew review and implementation.

    Overview¶

    This inbox receives automated deliveries from the P02 Intake Pipeline. Each feature includes:

    1. Manifest JSON: Metadata about the feature (key, slug, priority, source)
    2. P02 Artifacts: Link to comprehensive requirements documentation in docs/features/<slug>/

    Workflow¶

    ┌─────────────────┐
│ Intake Inbox    │  ← Marco drops feature ideas
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ P02 Pipeline    │  ← Automated requirements gathering
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Agile Crew Inbox│  ← YOU ARE HERE
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Sprint Backlog  │  ← Agile Crew converts to tasks
└─────────────────┘

    Manifest Format¶

    Each feature has a JSON manifest: <feature-slug>-manifest.json

    {
  "feature_key": "P02-FINOPS-007",
  "feature_slug": "azure-cost-anomaly-detection",
  "title": "Azure Cost Anomaly Detection System",
  "source_intake_file": "cost-anomaly-detection.md",
  "p02_artifacts_path": "docs/features/azure-cost-anomaly-detection",
  "created_timestamp": "2025-11-25T10:30:00.0000000Z",
  "status": "ready-for-crew",
  "priority": "HIGH"
}

    P02 Artifacts¶

    Each feature includes 10 comprehensive documents in docs/features/<slug>/:

    Artifact Purpose
    README.md Feature overview, quick start, usage examples
    P02-Overview.md P02 pattern execution summary
    requirements.md Functional and non-functional requirements with priorities
    feature-map.md Capability mapping, scope definition
    backlog.md Sprint tasks with acceptance criteria
    use-cases.md User scenarios with examples
    risks.md Risks and mitigations
    architecture-notes.md Technical design, data schemas, scalability
    adr-<slug>.md Architectural decision records
    tests.md Testing strategy, test scenarios

    Agile Crew Actions¶

    1. Review Feature¶

    # List all features in inbox
Get-ChildItem "docs\agile-crew\inbox" -Filter "*-manifest.json" | 
  ForEach-Object {
    $manifest = Get-Content $_.FullName | ConvertFrom-Json
    [PSCustomObject]@{
      Priority = $manifest.priority
      FeatureKey = $manifest.feature_key
      Title = $manifest.title
      Artifacts = $manifest.p02_artifacts_path
    }
  } | Format-Table -AutoSize

    2. Read P02 Artifacts¶

    # Example: Review backlog for a feature
$slug = "azure-cost-anomaly-detection"
Get-Content "docs\features\$slug\backlog.md"

    3. Convert to Sprint Tasks¶

    Manual Process (for now): 1. Review backlog.md for sprint tasks 2. Create GitHub Issues or Azure DevOps work items 3. Assign to sprint and team members 4. Link to P02 feature directory for reference

    Future Automation (P15 pattern): - Auto-create ADO work items from backlog.md - Auto-link to P02 feature artifacts - Auto-apply tags, priority, effort estimates

    4. Move to Processed¶

    Once Agile Crew has converted to sprint tasks:

    # Move manifest to processed folder
$slug = "azure-cost-anomaly-detection"
Move-Item "docs\agile-crew\inbox\$slug-manifest.json" `
          "docs\agile-crew\processed\$slug-manifest.json"

    Priority Levels¶

    Priority Meaning Typical Action
    HIGH Critical feature, high business value Review within 24 hours, sprint next iteration
    MEDIUM Important feature, moderate value Review within 1 week, sprint within 1-2 iterations
    LOW Nice-to-have, low urgency Review as capacity allows, backlog for future

    Status Tracking¶

    Manifests include status field for workflow tracking:

    Status Meaning
    ready-for-crew Just delivered from P02, awaiting Agile Crew review
    in-review Agile Crew is reviewing and planning
    sprint-planned Converted to sprint tasks, assigned to team
    in-progress Active development
    completed Feature implemented and deployed

    Update status: 

    $manifest = Get-Content "docs\agile-crew\inbox\$slug-manifest.json" | ConvertFrom-Json
$manifest.status = "in-review"
$manifest | ConvertTo-Json -Depth 10 | Set-Content "docs\agile-crew\inbox\$slug-manifest.json"

    Queries¶

    High Priority Features¶

    Get-ChildItem "docs\agile-crew\inbox" -Filter "*-manifest.json" | 
  ForEach-Object {
    $manifest = Get-Content $_.FullName | ConvertFrom-Json
    if ($manifest.priority -eq "HIGH") { $manifest }
  } | Format-List

    Features by Date¶

    Get-ChildItem "docs\agile-crew\inbox" -Filter "*-manifest.json" | 
  Sort-Object LastWriteTime -Descending | 
  Select-Object Name, LastWriteTime

    Ready Features (not yet in sprint)¶

    Get-ChildItem "docs\agile-crew\inbox" -Filter "*-manifest.json" | 
  ForEach-Object {
    $manifest = Get-Content $_.FullName | ConvertFrom-Json
    if ($manifest.status -eq "ready-for-crew") { $manifest }
  }

    Monitoring¶

    Pipeline Health¶

    Check last P02 intake run: 

    Get-Content "docs\_state\P02-Intake-State.json" | ConvertFrom-Json | 
  Select-Object lastRunTimestamp, @{N='ProcessedCount';E={$_.processedFiles.Count}}

    Usage Costs¶

    View P02 costs for intake features: 

    .\scripts\p02-usage-report.ps1 -Last 7 -AsTable

    Troubleshooting¶

    Issue: Manifest missing or malformed¶

    Check: Validate JSON syntax 

    Test-Json -Path "docs\agile-crew\inbox\$slug-manifest.json"

    Fix: Manually edit or regenerate via P02 pipeline

    Issue: P02 artifacts incomplete¶

    Check: Verify all 10 files exist 

    $expectedFiles = @("README.md", "P02-Overview.md", "requirements.md", 
                   "feature-map.md", "backlog.md", "use-cases.md", 
                   "risks.md", "architecture-notes.md", "tests.md")

$slug = "azure-cost-anomaly-detection"
$missingFiles = $expectedFiles | Where-Object {
  -not (Test-Path "docs\features\$slug\$_")
}

if ($missingFiles) {
  Write-Host "Missing files: $($missingFiles -join ', ')"
}

    Fix: Re-run P02 or manually create missing artifacts
