    EVA-Sovereign-UI Migration & Deprecation Plan¶

    Date: November 28, 2025
    Product Owner: Marco Presta
    Scrum Master: Copilot (GitHub Copilot)
    Status: VS IDE Session 0 (Orchestrator planning)

    Executive Summary¶

    Context: EVA-Sovereign-UI is the new unified library consolidating: - eva-i11y (accessibility components) - eva-i18n (internationalization) - eva-gc-look-n-feel (Government of Canada visual identity)

    Current Status: - Phase 5 (Canada Profile): COMPLETE ✅ (VS IDE Session 1) - Missing: 53 components across Sprint 1.2 → Sprint 16 (VS IDE Session 2) - Blockers: Tests + Storybook stories for each component, QA-found bugs

    Goal: Deprecate legacy libraries once migration complete, refactor existing UI/UX code to use EVA-Sovereign-UI exclusively.

    What EVA-Sovereign-UI Consolidates¶

    Legacy Libraries (TO BE DEPRECATED)¶

    1. eva-i11y (Accessibility)¶

    Purpose: WCAG 2.1 AA compliance, ARIA roles, keyboard navigation, screen reader support

    Components: - SkipToMainContent - Skip navigation link - FocusTrap - Trap focus within modals/dialogs - LiveRegion - Announce dynamic content changes to screen readers - KeyboardNavigation - Arrow key navigation for lists/grids - ColorContrastChecker - Validate color contrast ratios - AltTextValidator - Ensure images have alt text - FormFieldLabels - Associate labels with form inputs

    Migration Status: All components migrated to eva-sovereign-ui/accessibility/

    2. eva-i18n (Internationalization)¶

    Purpose: EN/FR bilingual support, locale switching, date/number formatting

    Components: - LanguageToggle - Switch between EN/FR - LocalizedText - Display translated strings from i18n keys - DateFormatter - Format dates per locale (en-CA: MM/DD/YYYY, fr-CA: DD/MM/YYYY) - NumberFormatter - Format numbers/currency (EN: $1,000.00, FR: 1 000,00 $) - TranslationProvider - Context provider for i18n state - TranslationMissing - Display warnings for missing translation keys

    Migration Status: All components migrated to eva-sovereign-ui/i18n/

    3. eva-gc-look-n-feel (GC Visual Identity)¶

    Purpose: Government of Canada Web Experience Toolkit (WET) styling, Canada.ca design patterns

    Components: - GCHeader - Bilingual header with Canada wordmark, language toggle, search - GCFooter - Footer with social media links, terms, privacy policy - GCBreadcrumb - Navigation breadcrumb trail - GCAlert - Alert banners (info, warning, error, success) - GCButton - Primary, secondary, danger, link button styles - GCCard - Content card with header, body, footer - GCGrid - Responsive grid system (12-column) - GCTypography - Heading levels, body text, captions

    Migration Status: All components migrated to eva-sovereign-ui/gc-components/

    Loss in Data Loss Incident (Nov 28, 2025): - ❌ Repository: eva-gc-look-n-feel was NEVER pushed to GitHub - ❌ Impact: Full repository lost in morning data loss incident (~5 hours work) - ✅ Recovery: Can be reconstructed from EVA-Sovereign-UI (which has the migrated components)

    Refactoring Approach: Gradual Migration¶

    Strategy: Page-by-page migration (not all-at-once rewrite)

    Step 1: Identify High-Value Pages¶

    Criteria: Most-used pages, pages with accessibility issues, pages needing i18n fixes

    EVA DA High-Value Pages: - /dashboard - Main user dashboard - /jp/search - Jurisprudence search (complex UI) - /applications/new - Start new CPP-D application - /chat - Chat with EVA assistant

    Priority: Start with /dashboard (most-used)

    Step 2: Create Layout Metadata (WYSIWYG Tool)¶

    Process: 1. Open page in WYSIWYG Tool 2. Tool analyzes existing Svelte code 3. Generate initial layouts/eva-da/dashboard.json 4. Dev reviews and tweaks in visual editor

    Step 3: Replace Page with LayoutRenderer¶

    Before (Hand-coded Svelte): 

    <!-- routes/dashboard/+page.svelte -->

    Result: Page complexity drops from 200+ lines to 5 lines

    Step 4: Test & Validate¶

    Automated Tests: - Visual regression (Playwright screenshots before/after) - A11y audit (Axe, Pa11y) - must pass with no violations - I18n coverage (all keys resolved for EN/FR) - Unit tests (Vitest) for any custom components

    Manual QA: - Marco tests page in dev environment - Windows Guy tests in test environment - User acceptance testing (UAT) with real users (if available)

    Acceptance Criteria: - Visual appearance identical to original - All functionality preserved (no regressions) - Accessibility improved (fewer violations) - I18n coverage 100% (all keys translated)

    Step 5: Deploy & Monitor¶

    Deployment: - Deploy to dev → test → prod (standard pipeline) - Blue/green deployment (zero downtime) - Monitor for errors (Azure Application Insights)

    Monitoring: - Error rate (must be < 1%) - Page load time (must be < 2 seconds) - Bounce rate (must not increase) - User feedback (collect via in-app surveys)

    Rollback Trigger: - If error rate > 5%, auto-rollback - If page load time > 5 seconds, investigate - If user feedback negative, schedule re-design

    Bug Fixes from QA (VS IDE Session 2)¶

    Context: VS IDE Session 2 found bugs during component development

    Critical Bugs (Fix in Sprint 1.2)¶

    1. LanguageToggle not persisting selection
    2. Issue: Page reload resets to default language (EN)
    3. Root Cause: Missing localStorage persistence

    4. Fix: Add localStorage.setItem('locale', newLocale) on toggle

    5. GCHeader search not submitting on Enter key

    6. Issue: Keyboard users can't submit search (mouse-only)
    7. Root Cause: Missing onKeyDown handler for Enter key

    8. Fix: Add keyboard event listener: if (e.key === 'Enter') { handleSearch(); }

    9. FocusTrap not working in modal dialogs

    10. Issue: Tab key escapes modal, focus moves to background page
    11. Root Cause: Focus trap activated after content loaded (race condition)
    12. Fix: Use onMount lifecycle to ensure trap activates after DOM ready

    High Priority Bugs (Fix in Sprint 2)¶

    1. DateFormatter showing wrong format in FR locale
    2. Issue: FR shows MM/DD/YYYY (US format), should be DD/MM/YYYY
    3. Root Cause: Hardcoded format string, not respecting locale

    4. Fix: Use Intl.DateTimeFormat with locale parameter

    5. ColorContrastChecker failing on valid colors

    6. Issue: Reports contrast ratio 4.3:1 as failing (WCAG AA requires 4.5:1)
    7. Root Cause: Rounding error in calculation

    8. Fix: Use higher precision math (3 decimal places)

    9. GCAlert not announcing to screen readers

    10. Issue: New alerts not read by NVDA/JAWS
    11. Root Cause: Missing role="alert" and aria-live="assertive"
    12. Fix: Add ARIA attributes + ensure content updates trigger announcement

    Medium Priority Bugs (Fix in Sprint 3-4)¶

    1. ComponentPalette drag-and-drop broken on Firefox
    2. Issue: Drag works on Chrome, not on Firefox
    3. Root Cause: Firefox requires explicit dataTransfer.setData() call

    4. Fix: Add e.dataTransfer.setData('text/plain', componentId)

    5. PropsEditor not validating required props

    6. Issue: Can save layout with missing required props (causes runtime errors)
    7. Root Cause: No validation schema enforcement

    8. Fix: Add Zod schema validation, show error messages

    9. BreakpointPreview not responsive on actual mobile devices

    10. Issue: Emulation mode works in desktop browser, but real mobile layout broken
    11. Root Cause: Viewport meta tag not set correctly
    12. Fix: Add <meta name="viewport" content="width=device-width, initial-scale=1.0">

    Testing Requirements (Per Component)¶

    Unit Tests (Vitest)¶

    Coverage Target: 80%+ per component

    Test Cases: 1. Props validation: Test all prop combinations (valid and invalid) 2. Event handlers: Test all user interactions (click, keypress, focus, blur) 3. Conditional rendering: Test all branches (if/else, show/hide) 4. Edge cases: Empty data, null values, very long strings 5. Accessibility: ARIA attributes present, keyboard navigation works

    Example Test: 

    // tests/GCHeader.test.ts
import { render, fireEvent } from '@testing-library/svelte';
import { describe, it, expect } from 'vitest';
import GCHeader from '../src/gc-components/GCHeader.svelte';

describe('GCHeader', () => {
  it('renders with default props', () => {
    const { getByRole } = render(GCHeader);
    expect(getByRole('banner')).toBeInTheDocument();
  });

  it('toggles language on click', async () => {
    const { getByText } = render(GCHeader, { props: { showLanguageToggle: true } });
    const toggle = getByText('FR');
    await fireEvent.click(toggle);
    expect(getByText('EN')).toBeInTheDocument(); // Now showing EN option
  });

  it('submits search on Enter key', async () => {
    const { getByRole } = render(GCHeader, { props: { showSearch: true } });
    const searchInput = getByRole('searchbox');
    await fireEvent.keyDown(searchInput, { key: 'Enter' });
    // Assert search event fired (use event spy)
  });
});

    Mitigation: Blue/green deployment, automated rollback, gradual page-by-page migration (not big-bang rewrite)
