    Senior Advisor Testing & Findings Report¶

    Date: December 15, 2025 Tester: GitHub Copilot (Scrum Master) Test Subject: Senior Advisor integration via OpenAI API Script: scripts/call-senior-advisor.ps1

    Executive Summary¶

    Status: ✅ Script created and tested, documentation reviewed, integration pattern understood Blocker: ⚠️ OpenAI API key not set in environment (expected for Marco's environment) Finding: Senior Advisor is a well-documented external service persona (SP01) with clear governance boundaries

    1. Senior Advisor Identity (Per Registry & Documentation)¶

    Official Classification¶

    Code: SP01 Name: Senior Advisor Type: Service Persona (external) Layer: External consultant (not part of EVA DevTools) Home Repo: eva-lwo (Living Working Office) Registry Status: ✅ Documented in agents/registry.yaml as SP00 (overview)

    Role Definition¶

    From eva-meta/docs/agents/personas/SP01-senior-advisor.md:

    "High-level framing, patterns, options and critique for complex problems"

    NOT a DevTools pattern (P01-P22) - DevTools are internal automation IS a Service Persona (SP00-SPxx) - External-facing advisor services

    Key Characteristics¶

    1. External Consultant Pattern
    2. Lives OUTSIDE EVA DevTools
    3. Accessed via OpenAI API (ChatGPT)
    4. Manual invocation only (no autonomous background runs)

    5. All outputs are ADVISORY (must be validated by humans)

    6. Intended Use

    7. Early framing of ideas, strategies, designs
    8. Drafting narratives (emails, memos, slide outlines)
    9. Identifying risks, edge cases, stakeholders
    10. Synthesizing non-sensitive reference material

    11. Pattern exploration and option analysis

    12. Non-Use (Forbidden)

    13. Direct access to Protected B+ data
    14. Formal compliance/ATO sign-off
    15. Final wording of legally binding texts
    16. Authoritative answers from protected systems

    3. Integration Pattern: ChatGPT Bridge¶

    Current Manual Pattern¶

    File: docs/chatgpt-bridge.md Process: 1. GitHub Copilot (SM) writes requests in bridge file 2. Marco copies requests to ChatGPT manually 3. ChatGPT (Senior Advisor) provides response 4. Marco pastes response back to bridge file 5. GitHub Copilot (SM) processes response into actionable plans

    New Automated Pattern (Our Script)¶

    File: scripts/call-senior-advisor.ps1 Process: 1. Script reads Request #N from bridge file 2. Script calls OpenAI API directly (no manual copy/paste) 3. OpenAI (ChatGPT) generates response 4. Script inserts response back into bridge file automatically 5. GitHub Copilot (SM) processes response (same as before)

    Improvement: Eliminates manual copy/paste, saves Marco 5-10 minutes per consultation

    5. Governance & Safety Boundaries¶

    Classification Rules (Per SP01 Documentation)¶

    Allowed Content: - ✅ Unclassified information only - ✅ Sanitized/de-identified summaries - ✅ Public or shareable documents - ✅ EVA documentation approved for external LLMs

    Forbidden Content: - ❌ Protected B/C data - ❌ Personal information - ❌ Credentials or production data - ❌ Internal logs or case files

    Autonomy Level¶

    Classification: Advisory Only Cannot: - Modify repositories - Update backlogs - Change CI/CD - Read protected databases

    Must: - Clearly indicate outputs are suggestions - Be validated by humans before use - Be cross-checked against EVA/ESDC sources

    Audit Trail¶

    When used in departmental contexts: - Invocations should be logged - Prompt metadata captured - Response metadata captured - Follows EVA Chat logging patterns

    6. Integration with DevTools Patterns¶

    Relationship to P01-P22¶

    From SP01 documentation:

    "Jarvis and DevTools treat the Senior Advisor as a pluggable, external 'advisor of advisors' whose outputs must be validated against internal specs, CDDs, and governance docs before they affect real backlogs or code."

    Pattern: DevTools agents (P02-P22) may incorporate Senior Advisor ideas via humans, but must ground final actions on internal data.

    Example Workflow¶

    1. P02 Requirements Engine generates CDD (internal data)
    2. Human sanitizes CDD, sends to Senior Advisor for options analysis
    3. Senior Advisor suggests 3 architectural approaches (external pattern exploration)
    4. Human selects approach, validates against internal governance
    5. P02 continues with validated approach (internal execution)

    Key: Human is the bridge. Senior Advisor never directly modifies EVA systems.

    8. Script Implementation Details¶

    File Structure¶

    scripts/
  call-senior-advisor.ps1         # PowerShell wrapper
  call-senior-advisor-internal.py # Python execution (temp, deleted after run)

src/
  eva_suite_orchestrator/
    openai_agent.py                # OpenAI API client (existing)

    Key Features¶

    1. Automatic Python Script Generation
    2. PowerShell creates Python script on-the-fly
    3. Uses existing OpenAIAgent class from orchestrator

    4. Cleans up temp script after execution

    5. Request Extraction

    6. Regex pattern: ### Request #N:.*?(?=### Request #|\Z)
    7. Extracts full request section from bridge file

    8. Parses title, status, full text

    9. Response Insertion

    10. Finds request section via regex
    11. Removes existing response if present
    12. Inserts new response with timestamp

    13. Preserves all other content

    14. Error Handling

    15. Python detection (exit if missing)
    16. Library check (auto-install requests)
    17. API key validation (clear error message)
    18. File validation (bridge file exists)

    10. Comparison: Manual vs Automated Pattern¶

    Aspect Manual ChatGPT Bridge Automated Script
    Setup Open ChatGPT, copy/paste Run one PowerShell command
    Time 10-15 minutes 1-2 minutes (mostly API wait)
    Error Risk High (manual copy/paste errors) Low (automated extraction/insertion)
    Consistency Variable (depends on copy accuracy) High (same regex every time)
    Audit Trail Manual notes Automatic timestamp in file
    System Prompt Must paste manually each time Embedded in script (consistent)
    Response Format Varies (depends on ChatGPT UI) Structured markdown (predictable)

    Recommendation: Use automated script when API key is available. Fallback to manual for one-off quick questions.
