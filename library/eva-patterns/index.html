    EVA Patterns (P00-P15) - Internal Development Patterns¶

    Purpose: Internal development workflow patterns used by EVA Agile Crew for coordinated work

    Status: Extracted from EVA Agentic folder, adapted for eva-orchestrator Date: 2025-12-05

    What Are EVA Patterns?¶

    EVA Patterns (P00-P15) are INTERNAL DEVELOPMENT PATTERNS, not live service personas.

    They define how EVA Agile Crew (coordinated development agents) structures work: - Sprint planning and execution - Code review and testing - Security and compliance checks - Documentation and onboarding - Metrics and observability

    Critical Distinction: - P00-P15: Internal workflows (how we build) - Personas: Live service agents (what we deploy, e.g., EVA Data Ingestion)

    Pattern Catalog¶

    P00 - DevTools Agents Spec¶

    Purpose: Foundation specification for all DevTools agents Scope: Agent runtime, tool registry, memory model, orchestration patterns Owner: POD-F (Foundation)

    P02 - Requirements (REQ)¶

    Purpose: Requirements refinement and intake pipeline Workflow: CDDs → Epics → Stories → Acceptance Criteria Tools: EVA Requirements Agent, Senior Advisor (ChatGPT), Agent Concept Canvas Automation: .github/workflows/p02-intake-pipeline.yml (runs every 6 hours) Owner: Product Owner + Scrum Master

    P03 - Scrum (SCR)¶

    Purpose: Sprint ceremonies, backlog management, velocity tracking Ceremonies: Planning, stand-ups, reviews, retrospectives Artifacts: Sprint board, burn charts, velocity metrics Automation: .github/workflows/psm-1-autonomous.yml (4x daily) Owner: Scrum Master (GitHub Copilot)

    P04 - Librarian (LIB)¶

    Purpose: Repo documentation consistency, README maintenance, ADRs Deliverables: README templates, architecture docs, cross-linking Quality: Docs are first-class artifacts, auditable Owner: POD-F (Foundation)

    P05 - Scaffolder (SCA)¶

    Purpose: Bootstrap repos, modules, pipelines from templates Tools: templates/ folder (eva-core, eva-api, eva-ui, etc.) Automation: scripts/agent-bootstrap.mjs Deliverables: New repo with: README, .devcontainer, CI/CD, tests Owner: POD-F (Foundation)

    P06 - Review (REV)¶

    Purpose: First-pass PR analysis, code quality checks Checks: Correctness, security vulnerabilities, accessibility, bilingual support, test coverage Automation: GitHub Actions PR checks (ESLint, Prettier, Jest, CodeQL) Owner: Dev Masters (ai2, windowsGuy)

    P07 - Testing (TST)¶

    Purpose: Testing strategy, failure explanation, coverage tracking Scope: Unit tests (80% coverage target), integration tests (60%), E2E tests (critical flows) Tools: Jest, Playwright, axe-core (accessibility) Owner: QA Master (GitHub Copilot)

    P08 - CI/CD Guardian (CICD)¶

    Purpose: CI/CD pipeline guidance, deployment automation Pipelines: Build, test, lint, security scans, deploy (dev → pilot → prod) Guardrails: Fail on violations, rollback on health check failures Owner: POD-O (Operations)

    P09 - Runbook & SRE (RUN)¶

    Purpose: Operational runbooks, incident response, kill switches Deliverables: Runbooks in docs/runbooks/, incident playbooks, rollback procedures Scope: SEV1-4 classification, kill switches via EVA Ops Owner: POD-O (Operations)

    P10 - Metrics & Insights (MET)¶

    Purpose: Metrics collection, dashboards, performance analysis Dashboards: EVA Ops (LiveOps), FinOps (cost attribution), velocity/burn charts Tools: Application Insights, Cosmos DB, Power BI Owner: POD-O (Operations)

    P11 - Security & Compliance (SEC)¶

    Purpose: Security scanning, compliance checks, vulnerability management Checks: Dependency scanning (npm audit), secrets detection, SAST (CodeQL), DAST Compliance: Protected B, ITSG-33, WCAG 2.1 AA, bilingual (EN/FR) Owner: POD-F (Foundation) + Security Office

    P12 - UX, Copy & Accessibility (UXA)¶

    Purpose: UX patterns, copy guidelines, accessibility audits Standards: WCAG 2.1 AA (considering 2.2), GC Design System, bilingual i18n Tools: Lighthouse CI, axe DevTools, EVA i11y library Deliverables: Component specs, accessibility audit reports Owner: POD-X (Experience)

    P13 - Onboarding & Sandbox Coach (ONB)¶

    Purpose: New developer onboarding, sandbox environments, training materials Deliverables: Onboarding guides, Dev Container setup, playground repos Lifecycle: Sandbox tier (experimentation, no real data) Owner: Scrum Master (GitHub Copilot)

    P14 - LiveOps Companion (LVO)¶

    Purpose: Live operations support, monitoring, alerting Scope: SLO tracking (99% uptime, <300ms P95 latency), incident response, rollbacks Tools: EVA Ops console, Azure Monitor, Application Insights Owner: POD-O (Operations)

    P15 - Dev Master Orchestrator (DVM / Jarvis)¶

    Purpose: Orchestrate development work across agents (ai2, windowsGuy) Workflows: GitHub Actions workflows (jarvis-refine, jarvis-plan, jarvis-execute, jarvis-daily, jarvis-review, jarvis-release, jarvis-uat, jarvis-retro) Coordination: Sprint YAML configs, .eva-pod.yaml per repo, pods-map.yaml registry Owner: Dev Masters (ai2, windowsGuy)

    P16 - Directory Mapping Pattern (DMP)¶

    Purpose: Pre-computed workspace/repo trees for 1600x faster navigation Scope: Global tree (eva-suite-full-tree.txt), per-repo cache (.eva-cache/repo-tree.txt), session trees Benefits: Instant file discovery, zero LLM cost, 29x more context available Automation: Nightly refresh (global), bootstrap (per-repo), session start (quick) Owner: POD-O (Orchestrator) + Scrum Master

    Pattern Usage¶

    Sprint-Based Delivery¶

    All patterns follow weekly sprint cadence: - P03 (Scrum) runs ceremonies - P02 (REQ) refines backlog for next sprint - P06 (Review) + P07 (Test) validate work - P10 (Metrics) tracks velocity and burn

    Quality Gates¶

    Work moves through states: Ready → In Progress → QA Gate → Done - P06 (Review) + P07 (Test) enforce QA gate - P08 (CICD) enforces CI/CD quality gates - P11 (SEC) enforces security gates

    Definition of Done¶

    All patterns enforce Definition of Done (docs/DEFINITION-OF-DONE.md): - Code reviewed (P06) - Tests pass (P07) - Security scans pass (P11) - Documentation updated (P04) - Accessibility verified (P12) - Deployed to dev (P08)

    Integration with GC Agentic Framework¶

    EVA Patterns implement Volume 2 (Engineering & Delivery): - P02: Requirements refinement (maps to Agent Concept Canvas in Vol4) - P03: Sprint ceremonies (maps to Agile delivery in Vol2) - P04: Documentation as first-class artifact (Vol2 principle) - P05: Scaffolding (maps to repo organization in Vol2) - P06: Code review (maps to PR workflow in Vol2) - P07: Testing (maps to quality gates in Vol2) - P08: CI/CD (maps to continuous integration/deployment in Vol2) - P09: Runbooks (maps to operational readiness in Vol3) - P10: Metrics (maps to observability in Vol3) - P11: Security/compliance (maps to security scanning in Vol2, compliance in Vol1) - P12: Accessibility (maps to WCAG 2.1 AA in Vol1/Vol2) - P13: Onboarding (maps to standardized dev environments in Vol2) - P14: LiveOps (maps to monitoring/incident management in Vol3) - P15: Orchestration (maps to team structure and coordination in Vol2)
