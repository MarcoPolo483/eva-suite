    üõ†Ô∏è Tools I've Built (Lessons Learned)¶

    Automation Health Monitoring¶

    Problem: Manual checks slow down login, workflows fail silently
    Solution: Scheduled task + cached JSON report (5s background, <1s login)
    Files: - scripts/Update-AutomationHealthReport.ps1 - Comprehensive health check - scripts/Setup-ScheduledHealthCheck.ps1 - One-time task creation - scripts/Invoke-AutomationHealthCheck.ps1 - Manual on-demand checks - login.ps1 - Reads cache, shows instant status - logoff.ps1 - Auto-summarizes accomplishments

    Lesson: Cache expensive operations, run heavy lifting in background

    Project Board Sync Process¶

    Problem: Issues created via GitHub UI don't appear on project board
    Solution: Always add to markdown backlog first, then sync runs automatically
    Files: - docs/backlog/user-request-backlog.md - Source of truth - .github/workflows/project-board-sync.yml - Runs every 30 min - sm-qa-toolbox/project/sync-project-board.ps1 - Sync engine

    Lesson: Markdown ‚Üí GitHub Projects (one-way sync). Manual trigger with gh workflow run project-board-sync.yml when urgent.

    Workflow Failure Resolution¶

    Problem: 4 critical workflows failing simultaneously
    Solution: Pattern-based detection + actionable fix suggestions
    Fixes Applied: - Auth conflicts (removed peter-evans action, used gh CLI) - YAML syntax (github.ref vs github.ref_name) - Missing dependencies (setup.py, GITHUB_TOKEN) - Array handling (PowerShell @() wrapping)

    Lesson: Keep failure pattern database, build auto-fix library

    üéì Lessons Learned¶

    Process Lessons¶

    1. Cache expensive checks - Don't make login slow
    2. Markdown is source of truth - GitHub Projects syncs FROM markdown
    3. Manual triggers exist - Use gh workflow run when urgent
    4. Background tasks scale - Hourly scheduled task > every login check
    5. Document everything - Future Marco needs instructions

    Technical Lessons¶

    1. PowerShell arrays need @() - Single objects aren't arrays by default
    2. github.ref vs github.ref_name - Context variables matter
    3. Peter-evans action conflicts - Use gh CLI for PR management
    4. GITHUB_TOKEN scoping - Need explicit env var for some actions
    5. Event Viewer exists - Windows Task Scheduler logs are accessible

    Communication Lessons¶

    1. @mentions don't auto-notify - Need explicit team assignments
    2. Project boards need manual sync - Or automation via markdown
    3. Issue creation ‚â† board addition - Two separate systems
    4. Comments ‚â† assignments - Comments don't move issues
