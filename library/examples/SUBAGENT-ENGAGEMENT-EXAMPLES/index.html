    1Ô∏è‚É£ Context Engineering¶

    Read memory, specs, requirements BEFORE coding

    SUBAGENT START CHECKLIST:

    2Ô∏è‚É£ Workspace Management¶

    Housekeeping, quality gates, cleanup

    DURING EXECUTION:
‚ñ° Commit after EACH task completion (LESSON-026)
‚ñ° Verify build success before moving to next task
‚ñ° Run quality gate checks (lint, test, accessibility)
‚ñ° Clean up temporary files and artifacts
‚ñ° Maintain git hygiene (meaningful commits, no large files)
‚ñ° Update documentation as features are added

    Why: Maintains code quality, prevents corruption, enables rollback.

    Example: EVA DA 2.0 subagent: - 6 commits (one per task + final report) - Build verified after each: 13-28s range, zero errors - Quality gates enforced: 11/11 passed - No temporary files committed

    NAVIGATION TOOLS:
‚ñ° Read eva-suite-full-tree.txt FIRST (workspace structure)
‚ñ° Read .eva-cache/repo-tree.txt (current repo structure)
‚ñ° Use grep_search on tree files (1600x faster than file_search)
‚ñ° Know where files live before creating/editing
‚ñ° Follow existing folder conventions

    Why: Instant file discovery, consistent organization, prevents duplication.

    Example: Instead of: 

    // ‚ùå SLOW: Blind search

    When to Use Subagents¶

    ‚úÖ USE runSubagent for: - Multi-task sequences (3+ related tasks) - Long-duration work (6+ hours) - Complex integrations requiring sustained focus - When fresh context prevents drift - When all context is documented (no ambiguity)

    ‚ùå DON'T USE runSubagent for: - Single isolated tasks - Quick fixes (<1 hour) - Interactive exploratory work - When Marco needs real-time input - When requirements are unclear
