    EVA FinOps - Pricing Catalog & API¶

    Azure service pricing data for EVA Suite cost attribution and FinOps dashboards.

    Overview¶

    The EVA FinOps module provides strongly-typed access to Azure service pricing data used throughout the EVA Suite for:

    • Cost Estimation - Calculate estimated costs based on usage (tokens, pages, storage, compute hours)
    • Cost Attribution - Track costs per project/app/feature/environment using EVA headers
    • FinOps Dashboards - Power LiveOps dashboards with real-time cost metrics
    • Budget Planning - Forecast costs for new projects or features

    Structure¶

    eva-finops/
├── config/
│   ├── eva-finops-pricing-v0.1.csv    # Human-editable pricing reference
│   └── eva-finops-pricing-v0.1.json   # Machine-readable pricing data
├── src/
│   ├── Pricing/
│   │   ├── PricingEntry.ts            # Pricing entry model
│   │   └── PricingCatalog.ts          # Catalog loader and lookup
│   └── api/
│       └── finops-router.ts           # Express API endpoints
└── tests/
    └── PricingCatalog.test.ts         # Unit tests

    Pricing Data¶

    Categories¶

    • LLM - GPT-4.1, GPT-4o, mini variants (input/output tokens)
    • OCR - Azure AI Document Intelligence (prebuilt, custom extract)
    • Search - Azure AI Search (S1, S2 tiers)
    • Cache - Azure Cache for Redis (Standard C0, C1)
    • Storage - Blob Storage, Data Lake (hot tier)
    • App - App Service / Azure Container Apps (small, medium instances)

    Pricing Entry Format¶

    interface PricingEntry {
  evaKey: string;        // "aoai_gpt4_1_input"
  category: string;      // "LLM"
  service: string;       // "GPT-4.1"
  skuOrModel: string;    // "Standard"
  meterType: string;     // "Input"
  unit: string;          // "per 1M tokens"
  priceUsd: number;      // 5.00
  notes?: string;        // Optional notes
}

    Usage¶

    Basic Lookup¶

    import { PricingCatalog } from './src/Pricing/PricingCatalog.js';

const catalog = new PricingCatalog();

// Get all entries
const allPrices = catalog.getAll();

// Lookup by key
const entry = catalog.tryGetByKey("aoai_gpt4_1_input");
console.log(`${entry.service}: $${entry.priceUsd} ${entry.unit}`);
// Output: GPT-4.1: $5.00 per 1M tokens

// Get just the price
const price = catalog.tryGetPrice("aoai_gpt4o_mini_output");
console.log(price); // 0.60

    Cost Calculation Example¶

    // Calculate cost for 2.5M input tokens + 500K output tokens (GPT-4.1)
const inputPrice = catalog.tryGetPrice("aoai_gpt4_1_input")!;    // 5.00 per 1M
const outputPrice = catalog.tryGetPrice("aoai_gpt4_1_output")!;  // 15.00 per 1M

const inputTokens = 2_500_000;
const outputTokens = 500_000;

const totalCost = 
  (inputTokens / 1_000_000) * inputPrice +
  (outputTokens / 1_000_000) * outputPrice;

console.log(`Total cost: $${totalCost.toFixed(2)}`);
// Output: Total cost: $20.00

    Dependency Injection (Express)¶

    import express from 'express';
import { PricingCatalog } from './src/Pricing/PricingCatalog.js';
import { createFinOpsRouter } from './src/api/finops-router.js';

const app = express();

// Register PricingCatalog as singleton
const pricingCatalog = new PricingCatalog();

// Mount FinOps API routes
app.use('/api/finops', createFinOpsRouter(pricingCatalog));

app.listen(3000);

    API Endpoints¶

    GET /api/finops/pricing¶

    Returns all pricing entries.

    Response: 200 OK 

    [
  {
    "evaKey": "aoai_gpt4_1_input",
    "category": "LLM",
    "service": "GPT-4.1",
    "skuOrModel": "Standard",
    "meterType": "Input",
    "unit": "per 1M tokens",
    "priceUsd": 5.00,
    "notes": ""
  },
  ...
]

    GET /api/finops/pricing/:evaKey¶

    Returns a specific pricing entry by key.

    Example: /api/finops/pricing/aoai_gpt4_1_input

    Response: 200 OK 

    {
  "evaKey": "aoai_gpt4_1_input",
  "category": "LLM",
  "service": "GPT-4.1",
  "skuOrModel": "Standard",
  "meterType": "Input",
  "unit": "per 1M tokens",
  "priceUsd": 5.00,
  "notes": ""
}

    Response: 404 Not Found 

    {
  "error": "Pricing entry not found",
  "evaKey": "invalid_key"
}

    Testing¶

    Run unit tests:

    npm test -- eva-finops/tests/PricingCatalog.test.ts

    Tests verify: - ✅ All 19 pricing entries load correctly - ✅ Known keys return expected values - ✅ Prices match CSV/JSON source data - ✅ Error handling for missing files/keys - ✅ Reload functionality works - ✅ All 6 categories present

    Cost Attribution Headers¶

    EVA Suite uses standard headers for cost attribution:

    // Example HTTP request headers
{
  "x-eva-project": "hr-assistant",
  "x-eva-app": "benefits-qa",
  "x-eva-feature": "eligibility-check",
  "x-eva-env": "prod",
  "x-eva-user": "marco@example.com"
}

    These headers are logged with telemetry to enable per-project/app/feature cost breakdowns in FinOps dashboards.

    Updating Pricing Data¶

    1. Edit config/eva-finops-pricing-v0.1.csv (human-friendly)
    2. Convert to JSON (or manually update eva-finops-pricing-v0.1.json)
    3. Call catalog.reload() to pick up changes without restart
    4. Commit and push changes

    Integration Points¶

    • EVA Orchestrator - Calculate request costs based on token usage
    • EVA DA 2.0 - Track per-session costs
    • EVA Suite LiveOps - Power cost dashboards and alerts
    • EVA Metering - Log costs alongside usage metrics
