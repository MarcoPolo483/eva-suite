    j GitHub Actions Jurisprudence ingestion

    Nice, this is the fun glue layer. Here’s J — a complete GitHub Actions workflow for the Jurisprudence ingestion + RAG pipeline, wired to your Project Space structure from step I and the P02 pack from step G.

    You can drop this file in:

    .github/workflows/jurisprudence-ingestion.yml

    and then let Copilot flesh out the details.

    /.github/workflows/jurisprudence-ingestion.yml¶

    name: Jurisprudence Ingestion & RAG Pipeline

on:
  # Manual runs (backfill or ad-hoc)
  workflow_dispatch:
    inputs:
      mode:
        description: "Ingestion mode"
        required: true
        default: "delta"
        type: choice
        options:
          - delta
          - backfill
      source:
        description: "Data source (scc|fc|fca|sst|all)"
        required: true
        default: "all"
        type: string
      from_date:
        description: "Backfill start date (YYYY-MM-DD, optional for delta)"
        required: false
        type: string
      to_date:
        description: "Backfill end date (YYYY-MM-DD, optional for delta)"
        required: false
        type: string

  # Daily delta ingestion (e.g., 06:00 UTC)
  schedule:
    - cron: "0 6 * * *"

env:
  PROJECT_SPACE: jurisprudence
  PROJECT_ROOT: projects/jurisprudence
  PYTHON_VERSION: "3.11"
  NODE_VERSION: "22"
  AZURE_REGION: "canadaeast"
  # Vector index name(s) – adjust as needed
  JURISPRUDENCE_INDEX: "jurisprudence-main"

  # These should be set as GitHub Encrypted Secrets
  # AZURE_CLIENT_ID
  # AZURE_TENANT_ID
  # AZURE_SUBSCRIPTION_ID
  # AZURE_RAG_RESOURCE_GROUP
  # AZURE_SEARCH_SERVICE
  # AZURE_OPENAI_ENDPOINT
  # AZURE_OPENAI_KEY

jobs:
  # 1) Validate the P02 requirements and constraints before running anything else
  validate-p02-config:
    name: Validate P02 Requirements & Constraints
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: ${{ env.PROJECT_ROOT }}/p02
    steps:
      - name: Checkout repo
        uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: ${{ env.PYTHON_VERSION }}

      - name: Install tooling (jsonschema, yamllint, etc.)
        run: |
          python -m pip install --upgrade pip
          pip install jsonschema

      - name: Validate requirements.json schema
        run: |
          # Optional: place a schema at p02/requirements-schema.json
          if [ -f requirements-schema.json ]; then
            jsonschema -i requirements.json requirements-schema.json
          else
            echo "No requirements-schema.json; basic JSON validation only."
            python -m json.tool requirements.json > /dev/null
          fi

      - name: Validate constraints.json schema
        run: |
          if [ -f constraints-schema.json ]; then
            jsonschema -i constraints.json constraints-schema.json
          else
            echo "No constraints-schema.json; basic JSON validation only."
            python -m json.tool constraints.json > /dev/null
          fi

  # 2) Run unit tests on extractors, chunking, metadata mapping, etc.
  unit-tests:
    name: Run Unit Tests
    runs-on: ubuntu-latest
    needs: [validate-p02-config]
    defaults:
      run:
        working-directory: ${{ env.PROJECT_ROOT }}
    steps:
      - name: Checkout repo
        uses: actions/checkout@v4

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}

      - name: Install Node dependencies (if using Jest/Vitest)
        run: |
          if [ -f package.json ]; then
            npm ci
          else
            echo "No package.json found; skipping Node deps."
          fi

      - name: Run unit tests
        run: |
          if [ -d "tests/unit" ]; then
            # Adjust to your test runner
            npm test -- --runInBand || echo "No JS tests configured; ignoring."
          else
            echo "No unit tests directory; skipping."
          fi

      - name: Run Python unit tests (extractors, transformers)
        run: |
          if [ -d "ingestion/extractors" ] || [ -d "ingestion/transformers" ]; then
            python -m pip install pytest
            pytest -q || echo "No Python tests configured; ignoring."
          else
            echo "No Python extractors/transformers found; skipping Python tests."
          fi

  # 3) Determine ingestion mode (delta vs backfill) & build parameters
  prepare-ingestion-context:
    name: Prepare Ingestion Context
    runs-on: ubuntu-latest
    needs: [unit-tests]
    outputs:
      MODE: ${{ steps.set-mode.outputs.MODE }}
      SOURCE: ${{ steps.set-source.outputs.SOURCE }}
      FROM_DATE: ${{ steps.set-dates.outputs.FROM_DATE }}
      TO_DATE: ${{ steps.set-dates.outputs.TO_DATE }}
    steps:
      - name: Determine mode (schedule vs manual)
        id: set-mode
        run: |
          if [ "${{ github.event_name }}" = "schedule" ]; then
            echo "MODE=delta" >> $GITHUB_OUTPUT
          else
            echo "MODE=${{ github.event.inputs.mode }}" >> $GITHUB_OUTPUT
          fi

      - name: Determine source
        id: set-source
        run: |
          if [ "${{ github.event_name }}" = "schedule" ]; then
            # Daily run – default to all sources
            echo "SOURCE=all" >> $GITHUB_OUTPUT
          else
            echo "SOURCE=${{ github.event.inputs.source }}" >> $GITHUB_OUTPUT
          fi

      - name: Determine date range
        id: set-dates
        run: |
          if [ "${{ github.event_name }}" = "schedule" ]; then
            # Delta ingestion by default: previous 24h
            FROM_DATE=$(date -u -d "yesterday" +%F)
            TO_DATE=$(date -u +%F)
          else
            FROM_DATE="${{ github.event.inputs.from_date }}"
            TO_DATE="${{ github.event.inputs.to_date }}"
          fi
          echo "FROM_DATE=$FROM_DATE" >> $GITHUB_OUTPUT
          echo "TO_DATE=$TO_DATE" >> $GITHUB_OUTPUT

  # 4) Execute ingestion (HTML/PDF/OCR) based on mode + source
  run-ingestion:
    name: Run Ingestion Pipeline
    runs-on: ubuntu-latest
    needs: [prepare-ingestion-context]
    env:
      MODE: ${{ needs.prepare-ingestion-context.outputs.MODE }}
      SOURCE: ${{ needs.prepare-ingestion-context.outputs.SOURCE }}
      FROM_DATE: ${{ needs.prepare-ingestion-context.outputs.FROM_DATE }}
      TO_DATE: ${{ needs.prepare-ingestion-context.outputs.TO_DATE }}
    defaults:
      run:
        working-directory: ${{ env.PROJECT_ROOT }}
    steps:
      - name: Checkout repo
        uses: actions/checkout@v4

      - name: Set up Python for ingestion scripts
        uses: actions/setup-python@v5
        with:
          python-version: ${{ env.PYTHON_VERSION }}

      - name: Install ingestion dependencies
        run: |
          if [ -f "ingestion/requirements.txt" ]; then
            python -m pip install --upgrade pip
            pip install -r ingestion/requirements.txt
          else
            echo "No ingestion/requirements.txt; please configure as needed."
          fi

      - name: Print ingestion context
        run: |
          echo "MODE: $MODE"
          echo "SOURCE: $SOURCE"
          echo "FROM_DATE: $FROM_DATE"
          echo "TO_DATE: $TO_DATE"

      - name: Run backfill ingestion
        if: env.MODE == 'backfill'
        run: |
          python ingestion/run_ingestion.py \
            --mode backfill \
            --source "$SOURCE" \
            --from-date "$FROM_DATE" \
            --to-date "$TO_DATE" \
            --project-space "$PROJECT_SPACE"

      - name: Run delta ingestion
        if: env.MODE == 'delta'
        run: |
          python ingestion/run_ingestion.py \
            --mode delta \
            --source "$SOURCE" \
            --from-date "$FROM_DATE" \
            --to-date "$TO_DATE" \
            --project-space "$PROJECT_SPACE"

      - name: Archive ingestion logs
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: ingestion-logs
          path: |
            ${{ env.PROJECT_ROOT }}/ingestion/logs/*.log
          if-no-files-found: ignore

  # 5) Update RAG Index (embeddings + vector store)
  update-rag-index:
    name: Update RAG Index
    runs-on: ubuntu-latest
    needs: [run-ingestion]
    defaults:
      run:
        working-directory: ${{ env.PROJECT_ROOT }}
    steps:
      - name: Checkout repo
        uses: actions/checkout@v4

      - name: Set up Python for RAG tools
        uses: actions/setup-python@v5
        with:
          python-version: ${{ env.PYTHON_VERSION }}

      - name: Install RAG dependencies
        run: |
          if [ -f "rag/requirements.txt" ]; then
            python -m pip install --upgrade pip
            pip install -r rag/requirements.txt
          else
            echo "No rag/requirements.txt; please configure as needed."
          fi

      - name: Login to Azure
        uses: azure/login@v2
        with:
          client-id: ${{ secrets.AZURE_CLIENT_ID }}
          tenant-id: ${{ secrets.AZURE_TENANT_ID }}
          subscription-id: ${{ secrets.AZURE_SUBSCRIPTION_ID }}

      - name: Run embedding + index update
        run: |
          python rag/update_index.py \
            --project-space "$PROJECT_SPACE" \
            --index-name "$JURISPRUDENCE_INDEX" \
            --search-service "$AZURE_SEARCH_SERVICE" \
            --resource-group "$AZURE_RAG_RESOURCE_GROUP"

  # 6) QA / Evaluation (RAG quality, hallucination, bilingual parity)
  qa-evaluation:
    name: QA Evaluation (RAG, Accuracy, Bilingual)
    runs-on: ubuntu-latest
    needs: [update-rag-index]
    defaults:
      run:
        working-directory: ${{ env.PROJECT_ROOT }}
    steps:
      - name: Checkout repo
        uses: actions/checkout@v4

      - name: Set up Python for evaluation
        uses: actions/setup-python@v5
        with:
          python-version: ${{ env.PYTHON_VERSION }}

      - name: Install eval dependencies
        run: |
          if [ -f "tests/eval/requirements.txt" ]; then
            python -m pip install --upgrade pip
            pip install -r tests/eval/requirements.txt
          else
            echo "No tests/eval/requirements.txt; please configure as needed."
          fi

      - name: Run RAG quality checks
        run: |
          if [ -f "tests/eval/run_rag_quality.py" ]; then
            python tests/eval/run_rag_quality.py \
              --project-space "$PROJECT_SPACE" \
              --index-name "$JURISPRUDENCE_INDEX"
          else
            echo "RAG eval script not present; skip for now."
          fi

      - name: Run bilingual parity checks
        run: |
          if [ -f "tests/eval/run_bilingual_parity.py" ]; then
            python tests/eval/run_bilingual_parity.py \
              --project-space "$PROJECT_SPACE"
          else
            echo "Bilingual parity eval script not present; skip for now."
          fi

      - name: Upload evaluation reports
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: jurisprudence-eval-reports
          path: |
            ${{ env.PROJECT_ROOT }}/tests/eval/*.json
            ${{ env.PROJECT_ROOT }}/tests/eval/*.md
          if-no-files-found: ignore

  # 7) Optional: Notify LiveOps / Teams / Email
  notify:
    name: Notify LiveOps / Stakeholders
    runs-on: ubuntu-latest
    needs: [qa-evaluation]
    steps:
      - name: Compose summary
        id: summary
        run: |
          echo "status=success" >> $GITHUB_OUTPUT
          echo "message=Jurisprudence ingestion & RAG pipeline completed." >> $GITHUB_OUTPUT

      # Wire this step to your notification channel (Teams, email, etc.)
      - name: Placeholder notification
        run: |
          echo "Notification: ${{ steps.summary.outputs.message }}"
          # TODO: curl/logic app/Teams webhook once available

    How this fits everything you’ve already built¶

    • P02 pack is validated in validate-p02-config.
    • Ingestion scripts live under projects/jurisprudence/ingestion.
    • RAG index update lives under projects/jurisprudence/rag.
    • Eval & RA/IDE evidence live under projects/jurisprudence/tests/eval.

    • RA/IDE + ATO are supported by:

    • logs/artifacts upload

    • explicit steps with clear evidence
    • scheduled + manual modes
