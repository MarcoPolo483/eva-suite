    H — Jurisprudence ATO Package (DRAFT)¶

    For deployment on the EVA Foundation (Protected B)¶

    Prepared for: AICOE, ESDC Cyber, Privacy Office, DGPOC¶

    Version 1.0 — Ready for refinement after SI Gap Analysis¶

    1. Executive Summary¶

    The Jurisprudence AI system will be deployed on the EVA Foundation, a Protected-B, GC-compliant GenAI platform managed by AICOE.

    The system provides:

    • secure ingestion of public jurisprudence decisions
    • retrieval-augmented reasoning (RAG)
    • citation-based answers
    • bilingual summaries
    • non-binding analytical support for adjudicators
    • strict human-in-the-loop oversight

    The system does not:

    • make decisions
    • interpret law
    • store or process personal information
    • operate autonomously
    • fetch data from live internet sources

    The system operates at GC Agentic Maturity Level 2.5 with EVA-enforced Level 4 autonomy prevention.

    ATO Category: Unclassified / Protected B DADM Classification: Low-Impact, Non-Decisional Privacy Impact: No PI expected; PADI recommended Security Framework: ITSG-33 Medium PB profile

    2. System Description¶

    2.1 Architecture Overview¶

    The Jurisprudence system operates as a Project Space inside EVA:

    EVA Core ─────────── Authentication, RBAC, guardrails, logging
EVA RAG ───────────  Ingestion, embeddings, vector search, evidence binding
EVA Agentic ───────  P01–P15 multi-agent patterns (bounded autonomy)
EVA Sovereign UI ───  GC Design System, WCAG 2.2 AA, bilingualization
LiveOps ───────────  Telemetry, audit, RA/IDE evidence, drift detection
Azure OpenAI ──────  GPT-4-class models (Canada region)
Azure Resources ───  Storage, Functions, APIM, Search/Qdrant

    Deployment Model: Containerized / Serverless hybrid Hosting: Azure Canada Central / Canada East (GC-protected) Isolation Boundary: EVA Project Space: /jurisprudence Access Control: AICOE-enforced RBAC

    3. Assets & Data Classification¶

    Asset Classification Justification
    Publicly available jurisprudence Unclassified Public domain
    Parsed/cleaned jurisprudence Protected B Because of system security boundary
    Metadata Protected B System-generated, internal use
    Embeddings Protected B Derived from processed content
    Logs/Telemetry Protected B Contains technical traces
    Model outputs Protected B System-generated

    No personal information is collected, stored, or processed. If PI is detected in a rare historical decision → automatic redaction.

    4. Privacy (PADI-aligned)¶

    4.1 PI Determination¶

    • Source data is 100% public jurisprudence.
    • Accidental PI is possible in older rulings (names, addresses). Mitigation: P08 ingestion agent performs PI redaction before RAG.

    4.2 Privacy Controls¶

    • No personal identifiers permitted in RAG corpora.
    • No personal identifiers permitted in agent outputs.
    • Reviewer Agent (P03) blocks PI-containing content.
    • Logs contain no user content (token hashing).
    • System access is authenticated, role-based, and monitored.

    4.3 Privacy Notice¶

    Included via Sovereign UI banner.

    Conclusion: A PADI (not a full PIA) is sufficient and expected.

    5. Threat Model (STRIDE for GC Context)¶

    Threat Risk Mitigation
    Spoofing Unauthorized access AAD + RBAC + APIM keys
    Tampering Data alteration Immutable logs, hashing, ingestion validation
    Repudiation Denying actions Full telemetry + audit trails
    Information Disclosure PI leakage Redaction pipeline, Reviewer Agent
    Denial of Service Overload Rate limiting, APIM throttling
    Elevation of Privilege Tool misuse EVA Core tool whitelist

    Extra GC-specific mitigations:

    • No external internet access
    • No self-modifying agents
    • No ingestion outside explicit URLs
    • No Level 4 autonomy features

    7. RA/IDE — Responsible AI Evaluation¶

    RA/IDE Pillar Implementation
    Responsible Retrieval-only, citation required, guardrails
    Impact Non-decisional support only
    Design WCAG + bilingual + privacy-by-design
    Evaluation Continuous RAG drift + hallucination rate monitoring

    Metrics collected:

    • Chunk coherence score
    • Citation completeness
    • Output accuracy vs. ground truth
    • Drift detection on embeddings
    • Retrieval recall
    • Number of agent escalations to human oversight

    8. DADM (Automated Decision-Making Directive) Assessment¶

    Decision Context¶

    The system does not make decisions; it supports human adjudicators.

    Impact Level:¶

    Low (Tier 1) → No algorithmic decision-making, no scoring.

    Required Evidence:¶

    • Human-in-the-loop enforcement
    • Non-binding outputs
    • Retrieval-based results only
    • No automated determinations

    All provided.

    12. Residual Risks¶

    Low-Level Risks¶

    • Potential OCR inaccuracies
    • Occasional incomplete metadata
    • Rare hallucination in model output
    • Temporary drift after ingestion updates

    Mitigation:

    • Reviewer Agent (P03) corrections
    • Drift detection + automated QA
    • Human validation for complex outputs

    No unacceptable risks remain after EVA guardrails.¶
