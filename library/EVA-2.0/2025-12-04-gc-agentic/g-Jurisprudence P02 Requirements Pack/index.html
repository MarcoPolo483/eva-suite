    3. Data Ingestion Specification (Blueprint for P02 → P08 → P05)¶

    Save as:

    /projects/jurisprudence/p02/ingestion-spec.md

    3.1 Ingestion Stages¶

    1. Source Discovery (HTML/PDF)

    2. Traverse only approved tribunal URLs

    3. Detect language (EN/FR)

    4. Queue each decision for extraction

    5. Parsing & Extraction

    6. HTML stripping (retain headings, tables)

    7. PDF → OCR (when needed)
    8. Apply structural markers

    9. Extract citations

    10. Metadata Extraction

    11. Map metadata to canonical EVA schema

    12. Tribunal, docket, dates, parties, keywords

    13. Content Cleaning

    14. Remove banners, ads, sidebars

    15. Normalize Unicode

    16. Detect bilingual variants

    17. Chunking

    18. Semantic chunking with heading anchors

    19. Table chunking (grid → JSON)

    20. Embedding

    21. text-embedding-3-large

    22. Hybrid indexing

    23. Vector Store Upload

    24. Maintain evidence links

    25. Store metadata alongside chunks

    26. Validation (P03 Reviewer)

    27. Validate chunk coherence

    28. Confirm bilingual linking
    29. Validate metadata fields

    30. Reject any PI detection

    31. Readiness for RAG

    32. Promote dataset to Project Space index

    33. Notify LiveOps

    4.1 P02 → P03 Review Template¶

    Review Checklist for Jurisprudence Ingestion
--------------------------------------------
1. Are all metadata fields populated?
2. Do extracted texts preserve:
   - headings
   - paragraphs
   - citations
   - tables
3. Are chunk boundaries semantic?
4. Is bilingual linking correct?
5. Are any personal details present?
6. Is provenance captured for each chunk?
7. Does the vector entry include tribunal filters?
8. Does the output meet accessibility metadata?
