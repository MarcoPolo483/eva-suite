    i EVA Project Space Folder Structure for Jurisprudence Copy

    Nice, this is a fun one. Let’s give Jurisprudence a clean, EVA-native Project Space you can drop straight into your EVA Suite root.

    Below is I — Complete EVA Project Space Folder Structure for Jurisprudence, aligned with everything we’ve already defined in P02, the ATO package, and the agentic governance matrix.

    You can adapt paths to your actual monorepo layout, but this is a solid reference.

    1. Top-Level Placement¶

    Assuming EVA Suite root:

    eva-suite/
  projects/
    jurisprudence/
      README.md
      project-config.json
      docs/
      p02/
      ingestion/
      rag/
      agents/
      ui/
      tests/
      ops/
      security/
      data/

    3. How This Lines Up With What We Already Built¶

    • P02 pack → lives under p02/ and drives:

    • ingestion/pipelines/

    • ingestion/metadata/
    • rag/config/

    • agents/workflows/

    • ATO package → lives under docs/ATO/ and security/.

    • Agentic governance matrix (P01–P15) → docs/Governance/ and agents/policies/.

    • Sovereign UI → ui/sovereign/ using your GC-ready component library.

    • RA/IDE evidence → ops/monitoring/, tests/eval/, and docs/ATO/RA-IDE-Evidence-Index.md.

    4. Minimal “Day 1” Subset (if you want to start small)¶

    If you want a thinner first commit and let P02/agents grow it:

    projects/jurisprudence/
  README.md
  project-config.json
  docs/{CDD,ATO,Governance}/...
  p02/{requirements.json, constraints.json, ingestion-spec.md}
  ingestion/{pipelines, extractors, metadata}
  rag/{config, schemas}
  agents/{registry.json, prompts, policies}
  ui/sovereign/{pages, components, i18n}
  tests/{unit, integration}
  ops/{monitoring, runbooks}
  security/{threat-model.md, risk-register.md}
