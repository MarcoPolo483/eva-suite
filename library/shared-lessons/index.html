    EVA Shared Lesson Database¶

    Purpose: Centralized catalog of lessons learned across all EVA Suite repos for cross-repo learning

    Status: Active (nightly sync at 3:00 AM EST)

    Owner: P01-PAT (Pattern Recognition & Documentation)

    Task: TASK-012 - Shared Lesson Database

    Overview¶

    The Shared Lesson Database consolidates lessons[] from all 26 EVA Suite repos' .eva-memory.json files into a single searchable catalog. This enables:

    1. Cross-Repo Learning: Agents can access lessons from any repo without manual search
    2. Pattern Recognition: Identify recurring lessons across multiple repos
    3. Knowledge Compound Effect: Each lesson adds to institutional knowledge base
    4. Search by Category: Find lessons by Performance, Architecture, Automation, etc.

    Quick Start¶

    Generate Lessons Catalog¶

    # Run consolidation script (scans all repos with .eva-memory.json)
.\scripts\consolidate-lessons.ps1

# Output: docs/shared-lessons/lessons-catalog.md

    Search Lessons¶

    # Search by keyword
.\scripts\Search-Lesson.ps1 -Query "folder tree"

# Filter by category
.\scripts\Search-Lesson.ps1 -Query "performance" -Category "Performance"

# Find specific lesson by ID
.\scripts\Search-Lesson.ps1 -Query "LESSON-001"

    Automatic Sync¶

    Lessons catalog is automatically updated nightly at 3:00 AM EST via .github/workflows/lesson-sync.yml.

    Manual trigger: 

    gh workflow run lesson-sync.yml --repo MarcoPolo483/eva-orchestrator

    Lesson Categories¶

    Lessons are automatically categorized by keywords in title/insight:

    Category Keywords
    Performance faster, performance, speed, latency, optimization

    Automation automation, workflow, monitoring, cicd, pipeline
    Memory & Context memory, context, lesson, bootstrap, recall

    Security & Compliance security, compliance, rbac, auth, audit

    Usage Patterns¶

    For Agents (Copilot/ChatGPT)¶

    On session start: 1. Read .eva-memory.json (local repo lessons) 2. Query shared catalog for related lessons from other repos 3. Apply lessons to current task

    Example: 

    # Agent needs to implement monitoring system
.\scripts\Search-Lesson.ps1 -Query "monitoring" -Category "Automation"

# Returns LESSON-006: Automation Monitoring - No Silent Failures
# Agent applies PATTERN-003: Session Start Health Check

    For Developers (Marco)¶

    Discover patterns: 

    # What performance lessons exist?
.\scripts\Search-Lesson.ps1 -Category "Performance"

# Find lessons about specific feature
.\scripts\Search-Lesson.ps1 -Query "APIM"

    Add new lessons: 1. Update .eva-memory.json in any repo with new lesson 2. Wait for nightly sync (3:00 AM EST) or trigger manually 3. New lesson automatically added to shared catalog

    Lesson Lifecycle¶

    1. Capture (Per Repo)¶

    Lessons documented in .eva-memory.json: 

    {
  "lessons": [
    {
      "id": "LESSON-001",
      "title": "Folder Trees 1600x Faster",
      "insight": "grep_search on folder tree is 1600x faster...",
      "evidence": "Quantitative measurement...",
      "impact": "Use folder tree for all file discovery",
      "dateRecorded": "2025-12-06"
    }
  ]
}

    2. Consolidation (Nightly)¶

    consolidate-lessons.ps1 runs at 3:00 AM EST: - Scans all 26 repos with .eva-memory.json - Extracts lessons with repo context - Deduplicates by title (keeps most detailed) - Categorizes by keywords - Generates lessons-catalog.md

    3. Discovery (On Demand)¶

    Search-Lesson.ps1 queries catalog: - Keyword search (title, insight, evidence) - Category filtering - Lesson ID lookup

    4. Application (Real Work)¶

    Agents apply lessons to tasks: - Cross-repo learning (apply lessons from other repos) - Pattern recognition (identify recurring patterns) - Avoid repeated mistakes (learn from past failures)

    Integration¶

    With Memory System (TASK-011)¶

    • Shared lessons complement per-repo memory
    • Per-repo: Context-specific lessons (e.g., APIM config for eva-foundation)
    • Shared: Universal lessons (e.g., folder trees 1600x faster)

    With Context Engineering (LESSON-007)¶

    • Lessons provide "Context" block for 4-Block prompts
    • Search lessons before starting task (avoid reinventing patterns)
    • Reference lesson IDs in task descriptions

    With Autonomous Queue (TASK-016)¶

    • Agents query shared lessons before executing tasks
    • Apply relevant patterns automatically
    • Document new lessons in .eva-memory.json after task completion

    Maintenance¶

    Add New Category¶

    Edit consolidate-lessons.ps1: 

    $categories = @{
    "Performance" = @()
    "Architecture" = @()
    ...
    "NewCategory" = @()  # Add here
}

# Add categorization logic
if ($title -match "new|keyword") {
    $categories["NewCategory"] += $lesson
}

    Fix Deduplication Logic¶

    Edit deduplication criteria in consolidate-lessons.ps1: 

    # Current: Keep lesson with longest insight
$group | Sort-Object -Property { $_.Insight.Length } -Descending | Select-Object -First 1

# Alternative: Keep most recent
$group | Sort-Object -Property DateRecorded -Descending | Select-Object -First 1
