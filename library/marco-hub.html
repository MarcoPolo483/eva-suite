üõ†Ô∏è GitHub Basics ‚Äì Personal Access Tokens

Use these steps whenever you need to create or refresh a Personal Access Token (PAT) for EVA automation. Copy the token immediately after creation‚ÄîGitHub only shows it once.

  1. Sign in to GitHub.com.
  2. Select your avatar (top-right) ‚ûú Settings.
  3. In the left sidebar, scroll to the bottom and choose Developer settings.
  4. Open Personal access tokens ‚Üí Fine-grained tokens, then click Generate new token.
  5. Name the token (example: EVA Orchestrator Automation), set an expiry, and pick the target repository.
  6. Select the required permissions (Projects, Contents, Issues as needed) and press Generate token.
  7. Copy the token value and store it in a password manager.
  8. Navigate to the EVA repository ‚ûú Settings ‚Üí Secrets and variables ‚Üí Actions ‚Üí New repository secret to store the token (e.g., PROJECT_SYNC_TOKEN).

Tip: Repeat from step 4 whenever a token expires or scopes change.
