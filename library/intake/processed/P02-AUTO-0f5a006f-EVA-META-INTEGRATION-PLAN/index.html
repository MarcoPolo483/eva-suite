    2. Triggers: On push to eva-meta config/agents-registry.yaml
    3. Action: Update orchestrator's agents/registry.yaml

    4. Notifications: PR or direct commit with changelog

    5. Pre-commit Hook: eva-meta/.pre-commit-config.yaml

    6. Validates agent registry YAML schema
    7. Checks for duplicate agent IDs

    8. Ensures all required fields present

    9. CLI Command: eva-meta/scripts/sync-registries.ps1

    10. Manual sync option
    11. Dry-run mode
    12. Conflict detection and resolution

    üö® Risks & Mitigations¶

    Risk Impact Mitigation
    Registry drift Inconsistent agent behavior Automated sync (Phase 6)
    Over-centralization Meta becomes bottleneck Hybrid model - only specs in meta
    Broken references Repos can't find meta files Validation scripts + CI checks
    Adoption resistance Devs skip meta integration Clear value proposition + easy tooling
    Version conflicts Different repos need different agent versions Semantic versioning in meta registry

    üìä Metrics & KPIs¶

    Success Metrics: - Time to discover agent spec: < 30 seconds - Registry sync time: < 2 minutes - Meta reference adoption: 100% of active repos - Agent spec completeness: 80%+ documented

    Quality Metrics: - Registry schema validation: 100% pass rate - Cross-repo reference integrity: 0 broken links - Audit log completeness: 90%+ of ingestions logged

    üîÑ Rollback Plan¶

    If integration causes issues:

    1. Workspace: Remove eva-meta from EVA-2.0.code-workspace
    2. References: Remove meta sections from .eva-memory.json files
    3. Profiles: Keep local copies in repos until ready to migrate
    4. Sync: Disable GitHub Action, use manual updates

    No production impact - this is developer tooling only.
