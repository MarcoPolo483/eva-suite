    CDD: EVA Email Draft Assistant Tool¶

    Last Updated: 2025-11-19 Status: Backlog Priority: MEDIUM Epic: EVA Tools Suite

    Vision¶

    Provide context-aware email composition assistance with AI-powered drafting, tone adjustment, grammar checking, and template suggestions—enabling users to write professional emails faster and with greater confidence.

    Business Value: Reduce email drafting time by 50%, improve communication quality, ensure bilingual compliance, and lower writing anxiety for non-native speakers.

    Core Capabilities¶

    • Context-Aware Drafting: Generate emails based on brief prompts ("Reply to meeting request", "Decline politely")
    • Tone Adjustment: Rewrite emails in different tones (formal, casual, empathetic, assertive)
    • Grammar & Style: Real-time spelling, grammar, and style suggestions
    • Template Library: Pre-built templates for common scenarios (meeting requests, status updates, apologies)
    • Bilingual Support: Draft in English or French, translate between languages
    • Smart Replies: Suggest 3 quick reply options based on received email
    • Length Control: Expand brief notes into full emails or condense long drafts
    • Citation Mode: Auto-cite policies, documents when composing formal emails

    User Stories¶

    US-1: As a PS-A, I want email drafts from prompts so that I write faster¶

    Acceptance Criteria:

    • Enter prompt: "Reply to Marco's meeting request, propose Tuesday 2pm"
    • EVA generates full email with greeting, body, closing
    • Review and edit draft before sending
    • Insert draft into Outlook compose window (add-in)
    • Generate draft in <5 seconds

    US-2: As a manager, I want tone adjustment so that emails match context¶

    Acceptance Criteria:

    • Write email draft (any tone)
    • Select tone: Formal, Casual, Empathetic, Assertive, Direct
    • EVA rewrites email preserving meaning, changing tone
    • Preview side-by-side comparison (original vs. rewritten)
    • Apply rewrite or revert to original

    US-3: As a non-native speaker, I want grammar help so that emails are professional¶

    Acceptance Criteria:

    • Type email in compose window
    • Real-time underlines for spelling, grammar, style issues
    • Hover over underline to see suggestion
    • Click to apply fix (one-click correction)
    • Bilingual support (en-US/fr-FR)

    US-4: As a user, I want templates so that I start with best practices¶

    Acceptance Criteria:

    • Browse template library (20+ templates)
    • Categories: Meetings, Status Updates, Apologies, Announcements
    • Select template, fill placeholders (name, date, details)
    • Preview and customize before inserting
    • Save custom templates for reuse

    US-5: As a bilingual user, I want translation so that I draft in one language, send in another¶

    Acceptance Criteria:

    • Draft email in English
    • Click "Translate to French" button
    • EVA translates preserving tone and formatting
    • Review translation, make edits
    • Send bilingual email (both languages in body)

    Functional Requirements¶

    FR-1: Email Generation¶

    System SHALL generate emails from prompts using GPT-4o with context (sender, recipient, subject).

    FR-2: Tone Adjustment¶

    System SHALL rewrite emails in 5 tones: Formal, Casual, Empathetic, Assertive, Direct.

    FR-3: Grammar Check¶

    System SHALL provide real-time spelling, grammar, style suggestions using Azure AI Language.

    FR-4: Template Library¶

    System SHALL provide 20+ pre-built templates organized by category (Meetings, Status, Apologies, Announcements).

    FR-5: Translation¶

    System SHALL translate emails between en-US and fr-FR using Azure Translator with tone preservation.

    FR-6: Smart Replies¶

    System SHALL suggest 3 quick reply options based on received email content (Accept, Decline, Ask for Details).

    FR-7: Length Control¶

    System SHALL expand brief notes (50 words) into full emails (200 words) or condense long drafts by 50%.

    FR-8: Citation Mode¶

    System SHALL auto-cite policies, documents when keywords detected (e.g., "Policy 123" → insert link).

    FR-9: Outlook Integration¶

    System SHALL integrate as Outlook add-in (compose pane, insert draft button).

    FR-10: Privacy¶

    System SHALL process emails locally (client-side) unless user opts-in to send to backend for advanced features.

    FR-11: Custom Templates¶

    System SHALL allow users to save custom templates (10 per user) for reuse.

    FR-12: Export Options¶

    System SHALL allow exporting drafts as plain text, HTML, or copying to clipboard.

    FR-13: Bilingual Interface¶

    System SHALL provide UI in en-US and fr-FR with language switcher.

    FR-14: Access Control¶

    System SHALL enforce RBAC: users access own drafts only.

    FR-15: Audit Logging¶

    System SHALL log all generations, tone adjustments, translations to Application Insights.

    Non-Functional Requirements¶

    Performance¶

    • Email generation: <5 seconds (P95)
    • Tone adjustment: <3 seconds (P95)
    • Grammar check: <1 second (real-time)
    • Translation: <5 seconds for 500-word email
    • Support 5000 concurrent users

    Reliability¶

    • Uptime: 99.9% SLA
    • Auto-retry failed API calls (3 attempts)
    • Graceful degradation (basic grammar only if backend unavailable)
    • Cache templates locally (offline mode)

    Security¶

    • Encrypt drafts at rest (AES-256) and in transit (TLS 1.3)
    • RBAC: users access own drafts only
    • PII masking in logs (email addresses, names)
    • Compliance: PIPEDA, TBS Directive on Privacy
    • No email content stored on backend (opt-in only)

    Accessibility¶

    • WCAG 2.1 AA compliance (Outlook add-in UI)
    • Keyboard navigation (Tab, Enter, Esc)
    • Screen reader support (suggestions, templates)
    • High-contrast mode for UI

    i18n¶

    • Bilingual support (en-US/fr-FR)
    • Auto-detect email language or manual selection
    • ICU formatting for dates, times
    • Bilingual templates and suggestions

    Observability¶

    • Application Insights custom events (generate, tone adjust, translate)
    • Track prompt types, template usage, tone preferences
    • Alerts for high failure rates or slow response times
    • Metrics: emails generated/day, avg processing time

    Scalability¶

    • Horizontal scaling (Azure Functions for API)
    • Partition storage by /tenantId/userId (HPK)
    • Azure CDN for template library
    • Support 50K users per tenant

    Technical Architecture¶

    Backend (Composition Engine)¶

    • Generation: Azure OpenAI GPT-4o (prompt templates for emails)
    • Grammar: Azure AI Language (spelling, grammar, style)
    • Translation: Azure Translator API (en-US ↔ fr-FR)
    • Processing: Azure Functions HTTP triggers (REST API)

    Frontend (Outlook Add-in)¶

    • Framework: Office.js (Outlook Add-in SDK)
    • UI: React components (compose pane, side panel)
    • Editor: Draft.js or ProseMirror (rich text editing)
    • Communication: REST API calls to Azure Functions

    Integration Points¶

    • eva-openai: GPT-4o for email generation and tone adjustment
    • eva-auth: RBAC for user access
    • eva-safety: Content moderation for generated emails
    • eva-api: REST API for generate, tone adjust, translate

    Data Models¶

    Draft (Cosmos DB, HPK /tenantId/userId):

    {
  "draftId": "uuid",
  "tenantId": "ESDC",
  "userId": "marco@esdc",
  "prompt": "Reply to meeting request, propose Tuesday 2pm",
  "content": "Hi Marco,\n\nThank you for the meeting request...",
  "tone": "formal",
  "language": "en-US",
  "createdAt": "2025-11-19T10:00:00Z"
}

    Template (Cosmos DB, HPK /tenantId/global):

    {
  "templateId": "meeting-request",
  "category": "Meetings",
  "name": "Meeting Request",
  "content": "Hi {recipient},\n\nI'd like to schedule a meeting to discuss {topic}. Are you available on {date} at {time}?\n\nBest regards,\n{sender}",
  "placeholders": ["recipient", "topic", "date", "time", "sender"],
  "language": "en-US"
}

    Implementation Phases¶

    Phase 1: Core Generation (Sprint 7)¶

    • GPT-4o email generation from prompts
    • Outlook add-in scaffold (Office.js)
    • React compose UI (text area, buttons)
    • Template library (20 templates)
    • Basic export (plain text, HTML)

    Phase 2: Tone & Grammar (Sprint 8)¶

    • Tone adjustment (5 tones)
    • Azure AI Language integration (grammar)
    • Real-time suggestions UI (underlines, tooltips)
    • Side-by-side comparison (original vs. adjusted)
    • Apply/revert buttons

    Phase 3: Translation & Smart Replies (Sprint 9)¶

    • Azure Translator integration (en-US ↔ fr-FR)
    • Smart replies (3 quick options)
    • Length control (expand, condense)
    • Citation mode (auto-link policies)
    • Bilingual email composer

    Phase 4: Polish & Publish (Sprint 10)¶

    • Custom templates (user-created)
    • WCAG 2.1 AA audit (add-in UI)
    • Load testing (5000 concurrent users)
    • Publish to Microsoft AppSource
    • UAT with pilot users

    Success Criteria¶

    • ✅ Generate emails in <5 seconds (P95)
    • ✅ 50% reduction in email drafting time (user survey)
    • ✅ 95%+ grammar accuracy (bilingual)
    • ✅ Tone adjustment preserves meaning (95% user satisfaction)
    • ✅ WCAG 2.1 AA compliance (axe DevTools passes)
    • ✅ 10K+ emails generated per month
    • ✅ Published to Microsoft AppSource

    Open Questions¶

    Q1: Privacy Mode (COMPLIANCE)¶

    Question: Process emails client-side or send to backend? Impact: Privacy vs. feature richness Recommendation: Client-side by default, opt-in for advanced features (tone, translation) Decision Owner: Security Sentinel

    Q2: Template Curation (UX)¶

    Question: Fixed template library or allow user submissions? Impact: Quality vs. variety Recommendation: Fixed library (Sprint 7), user submissions with moderation (Sprint 11) Decision Owner: UX Partner

    Q3: Tone Accuracy (TECH)¶

    Question: How to validate tone adjustment preserves meaning? Impact: User trust vs. complexity Recommendation: Side-by-side preview, user feedback loop for corrections Decision Owner: ai2

    Q4: Gmail Support (SCOPE)¶

    Question: Build Outlook add-in only or support Gmail? Impact: Time-to-market vs. reach Recommendation: Outlook MVP (Sprint 10), Gmail extension (Sprint 12) Decision Owner: Marco

    Q5: Citation Sources (FEATURES)¶

    Question: Auto-cite from internal knowledge base or external sources? Impact: Accuracy vs. scope Recommendation: Internal KB only (Sprint 9), external sources (Sprint 13) Decision Owner: ai2
