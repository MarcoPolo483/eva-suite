    CDD: EVA OS Deployment Models¶

    Last Updated: 2025-11-19 Status: Backlog Priority: LOW (Moonshot) Epic: EVA OS Integration (Windows Copilot+ Replacement)

    Vision¶

    Provide enterprise-grade deployment options for EVA OS integration—including Copilot+ SDK partnership with Microsoft, Intune (MDM/MAM) integration for centralized management, and Microsoft Store distribution for consumer-friendly installation—enabling ESDC to deploy EVA at scale across 100K+ Windows 11 devices with minimal IT overhead.

    Business Value: Enable enterprise deployment (Intune policies, zero-touch provisioning), reduce deployment costs by 80% (vs. manual installation), establish EVA as a first-class Windows 11 AI assistant (Microsoft Store presence), and position EVA as the strategic partner for Microsoft's Copilot+ ecosystem.

    Core Capabilities¶

    • Copilot+ SDK Partnership: EVA as certified Copilot+ AI (Microsoft hardware integration, NPU optimization)
    • Intune Integration: Deploy EVA via Microsoft Endpoint Manager (MDM/MAM policies)
    • Microsoft Store Distribution: Consumer-friendly installation (one-click, auto-update)
    • Group Policy Objects (GPO): Enterprise configuration (AD/SCCM deployment)
    • Zero-Touch Provisioning: Pre-installed on Windows 11 devices (OEM partnership)
    • Auto-Update Mechanism: Silent updates (security patches, feature releases)
    • Offline Installer: Standalone MSI/MSIX (air-gapped environments)
    • Licensing & Activation: Per-user licensing (Entra ID integration, volume licensing)

    User Stories¶

    US-1: As an IT admin, I want to deploy EVA via Intune so that I manage all devices centrally¶

    Acceptance Criteria:

    • Intune admin console: Add EVA app (MSIX package from Microsoft Store)
    • Assign to group: "All Windows 11 users" (or specific departments)
    • Deployment: EVA auto-installs on next device check-in (within 24 hours)
    • Configuration: Push settings via Intune (e.g., disable wake word, enable DLP)
    • Monitoring: Intune reports installation status (success, failed, pending)

    US-2: As a user, I want to install EVA from Microsoft Store so that I get it easily¶

    Acceptance Criteria:

    • Open Microsoft Store → search "EVA Assistant"
    • EVA app listing: Screenshots, description, ratings, reviews
    • Click "Get" → EVA installs (within 2 minutes, auto-launch on completion)
    • Auto-update: New versions install automatically (silent, no user action)
    • Uninstall: Right-click EVA in Start menu → "Uninstall" (clean removal)

    US-3: As an enterprise, I want to pre-install EVA on devices so that users have it out-of-box¶

    Acceptance Criteria:

    • OEM partnership: Dell, HP, Lenovo pre-install EVA (Windows 11 Pro/Enterprise)
    • First boot: EVA setup wizard runs (sign in with Entra ID, configure preferences)
    • No bloatware: EVA removable if user declines (not forced, opt-in)
    • Licensing: EVA included in Windows 11 Pro/Enterprise (no extra cost for ESDC)
    • Branding: EVA app icon in Start menu, taskbar (default pinned)

    US-4: As an IT admin, I want to configure EVA via GPO so that I enforce policies across AD¶

    Acceptance Criteria:

    • GPO template: Download ADMX/ADML files (from EVA portal)
    • Import to AD: Group Policy Management Console (GPMC)
    • Configure policies: Wake word (on/off), context capture (clipboard, files), DLP rules
    • Apply to OU: "Windows 11 Users" (all devices in organizational unit)
    • Policy enforcement: EVA respects GPO settings (user cannot override if admin-disabled)

    US-5: As an IT admin, I want offline installer so that I deploy in air-gapped environments¶

    Acceptance Criteria:

    • Download MSI/MSIX installer (from EVA portal, authenticated download)
    • Transfer to air-gapped network (USB drive, internal file share)
    • Silent install: msiexec /i EVA.msi /quiet TENANTID=esdc
    • Configuration: Pre-configure settings in MSI (no cloud required for first run)
    • Activation: Manual license key (or offline activation, grace period: 30 days)

    Functional Requirements¶

    1. FR-1: SHALL provide MSIX package for Microsoft Store distribution (consumer-friendly)
    2. FR-2: SHALL integrate with Intune (MDM/MAM deployment, configuration policies)
    3. FR-3: SHALL provide ADMX/ADML templates for Group Policy Objects (GPO)
    4. FR-4: SHALL support OEM pre-installation (Dell, HP, Lenovo partnerships)
    5. FR-5: SHALL provide offline installer (MSI/MSIX, air-gapped environments)
    6. FR-6: SHALL support zero-touch provisioning (Windows Autopilot integration)
    7. FR-7: SHALL provide auto-update mechanism (silent updates, security patches)
    8. FR-8: SHALL integrate with Entra ID for licensing and activation (per-user)
    9. FR-9: SHALL support volume licensing (enterprise agreements, site licenses)
    10. FR-10: SHALL provide uninstall wizard (clean removal, no residual files)
    11. FR-11: SHALL log all deployment events to Application Insights (install, update, uninstall)
    12. FR-12: SHALL enforce Intune policies (admin settings override user preferences)
    13. FR-13: SHALL support Copilot+ SDK (Microsoft hardware integration, NPU optimization)
    14. FR-14: SHALL provide deployment documentation (IT admin guide, troubleshooting)
    15. FR-15: SHALL comply with Microsoft Store policies (privacy, security, accessibility)

    Non-Functional Requirements¶

    Performance¶

    • Microsoft Store Install: <2 min (download + install, 50 Mbps connection)
    • Intune Deployment: <5 min (policy push → EVA installed on device)
    • Auto-Update: <1 min (background download, silent install)
    • Offline Installer: <30s (MSI install, local media)

    Reliability¶

    • Installation Success: >99% (all deployment methods)
    • Auto-Update Success: >98% (silent updates, no user disruption)
    • Uninstall Clean: 100% (no residual files, registry entries)
    • Rollback: Auto-rollback on failed update (restore previous version)

    Security¶

    • Signed Packages: Code-signed (Microsoft Authenticode, trusted certificate)
    • Integrity Verification: SHA-256 hash validation (detect tampering)
    • Secure Channel: Downloads via HTTPS only (no HTTP fallback)
    • Privilege Escalation: No admin rights required (user-level install)

    Accessibility¶

    • WCAG 2.1 AA: Installer wizard keyboard-accessible (no mouse required)
    • Screen Reader Support: ARIA labels for installer UI, progress indicators
    • High Contrast: Installer respects Windows High Contrast themes

    Internationalization¶

    • Locales: en-US (default), fr-FR
    • Installer UI: Translated (setup wizard, buttons, error messages)
    • ADMX/ADML: Localized templates (English and French policy descriptions)

    Observability¶

    • Metrics: Application Insights tracks install count, update success rate, uninstall reasons
    • Alerts: Installation failure rate >2%, auto-update failure >3%
    • Dashboards: Power BI dashboard for deployment stats, update trends, device coverage

    Scalability¶

    • Concurrent Installs: Support 10K+ simultaneous installs (Microsoft Store CDN)
    • Update Bandwidth: 1 PB/month (100K users, 10 MB updates, monthly cadence)
    • Intune Scale: Support 100K+ managed devices (enterprise deployment)

    Technical Architecture¶

    Microsoft Store Package¶

    MSIX Manifest:

    <Package xmlns="http://schemas.microsoft.com/appx/manifest/foundation/windows10">
  <Identity Name="ESDC.EVA" Publisher="CN=ESDC" Version="1.0.0.0" />
  <Properties>
    <DisplayName>EVA Assistant</DisplayName>
    <PublisherDisplayName>Employment and Social Development Canada</PublisherDisplayName>
    <Logo>Assets\StoreLogo.png</Logo>
  </Properties>
  <Dependencies>
    <TargetDeviceFamily Name="Windows.Desktop" MinVersion="10.0.22000.0" />
  </Dependencies>
  <Applications>
    <Application Id="EVA" Executable="EVA.exe" EntryPoint="EVA.App">
      <uap:VisualElements DisplayName="EVA" Square150x150Logo="Assets\Logo.png" />
    </Application>
  </Applications>
</Package>

    Intune Deployment¶

    Intune App Configuration:

    {
  "appType": "win32",
  "displayName": "EVA Assistant",
  "publisher": "ESDC",
  "installCommandLine": "msiexec /i EVA.msi /quiet TENANTID={TenantId}",
  "uninstallCommandLine": "msiexec /x {ProductCode} /quiet",
  "detectionRules": [
    {
      "ruleType": "registry",
      "keyPath": "HKLM\\Software\\ESDC\\EVA",
      "valueName": "Version",
      "operator": "greaterThanOrEqual",
      "value": "1.0.0"
    }
  ],
  "requirements": {
    "osMinimumVersion": "10.0.22000",
    "architecture": "x64"
  }
}

    GPO Template¶

    ADMX Policy Definition:

    <policy name="EVA_WakeWord_Enabled" class="User" displayName="$(string.EVA_WakeWord_Enabled)" 
        explainText="$(string.EVA_WakeWord_Enabled_Explain)" 
        key="Software\Policies\ESDC\EVA" valueName="WakeWordEnabled">
  <parentCategory ref="EVA_Category" />
  <supportedOn ref="windows:SUPPORTED_Windows_11" />
  <enabledValue><decimal value="1" /></enabledValue>
  <disabledValue><decimal value="0" /></disabledValue>
</policy>

    Implementation Phases¶

    Phase 1: Microsoft Store Distribution (Sprint 28, Week 1-3)¶

    • MSIX package creation (manifest, assets, code signing)
    • Microsoft Store submission (app listing, screenshots, description)
    • Auto-update mechanism (background download, silent install)
    • Store review process (privacy policy, security audit)
    • Public release (Microsoft Store availability)

    Phase 2: Intune Integration (Sprint 29, Week 4-6)¶

    • Win32 app packaging (MSI with Intune wrapper)
    • Intune app configuration (install/uninstall commands, detection rules)
    • Policy enforcement (admin settings override user preferences)
    • Deployment testing (pilot group, rollback on failure)
    • Documentation (IT admin guide, troubleshooting)

    Phase 3: GPO & Offline Installer (Sprint 30, Week 7-9)¶

    • ADMX/ADML templates (Group Policy definitions)
    • GPO deployment guide (import to AD, configure policies)
    • Offline installer (MSI/MSIX, air-gapped environments)
    • Silent install parameters (tenant ID, configuration presets)
    • License activation (offline activation, grace period)

    Phase 4: OEM Partnership & Copilot+ SDK (Sprint 31, Week 10-12)¶

    • OEM partnership agreements (Dell, HP, Lenovo pre-installation)
    • Zero-touch provisioning (Windows Autopilot integration)
    • Copilot+ SDK integration (Microsoft hardware, NPU optimization)
    • Volume licensing (enterprise agreements, site licenses)
    • Power BI dashboard for deployment stats, update trends, device coverage

    Success Criteria¶

    • ✅ Microsoft Store install <2 min (download + install)
    • ✅ Installation success >99% (all deployment methods)
    • ✅ Auto-update success >98% (silent updates)
    • ✅ Intune deployment <5 min (policy push → installed)
    • ✅ Device coverage >80% of Windows 11 ESDC devices within 1 year
    • ✅ OEM partnership (at least 1 vendor pre-installs EVA)
    • ✅ IT admin satisfaction >85% (deployment survey)
