    Power BI Dashboard Suite - Architecture & Implementation¶

    Last Updated: 2025-11-19
    Status: In Progress
    Priority: HIGH (Q1 2026 Demo)
    Owner: Scrum Master (GitHub Copilot) + Marco Presta

    ðŸ“Š Executive Summary¶

    Design and implement 3 Power BI dashboards to support EVA 2.0 operations, cost tracking, and learning analytics.

    Dashboards: 1. Command Center Dashboard - Platform health, spaces, costs, quality metrics 2. Space Dashboard - Individual space performance, budgets, user activity 3. Learning Analytics Dashboard - Engagement, completions, ROI tracking

    Dependencies: Azure Cosmos DB (data storage), sample data generators (Python), Power BI Desktop (installed)

    Timeline: Sprint 2-3 (data model + generators in Sprint 2, visuals in Sprint 3)

    1. Command Center Dashboard¶

    Vision¶

    Provide platform operators with real-time visibility into EVA 2.0's overall health, including active spaces, cost trends, quality metrics, and system performance.

    Target Audience¶

    • Platform Operators (daily monitoring)
    • AICOE Leadership (weekly reviews)
    • Marco (Product Owner) - strategic decisions

    Key Metrics¶

    Platform Health (Top Row)¶

    • Total Spaces: Count of all project spaces (sandbox + production)
    • Active Users: Unique users in last 30 days
    • Total Sessions: Session count (last 30 days)
    • Platform Availability: % uptime (99.9% SLA target)

    Cost Overview (Middle Section)¶

    • Monthly Spend: Total Azure costs (current month)
    • Cost by Service: Breakdown (Cosmos DB, OpenAI, Search, APIM, etc.)
    • Cost Trend: Line chart (last 6 months)
    • Top 5 Spaces by Cost: Bar chart with tenant names + spend

    Quality Metrics (Bottom Section)¶

    • Average CSAT: Platform-wide satisfaction score (1-5)
    • P95 Latency: API response time (target: <300ms)
    • Error Rate: % of failed requests (target: <1%)
    • Safety Filter Triggers: Count of moderation/injection blocks

    Data Refresh Strategy¶

    Cosmos DB Change Feed â†’ Real-Time Sync¶

    • Use Azure Functions with Cosmos DB Change Feed trigger
    • Push incremental updates to Power BI dataset via REST API
    • Latency: <1 minute for new data

    Scheduled Refresh (Fallback)¶

    • Power BI Service scheduled refresh: Every 1 hour
    • Incremental refresh for tables >1M rows
    • Full refresh nightly (2 AM UTC)

    Performance Optimization¶

    • Partition Pruning: Query only relevant date ranges (HPK on date)
    • Aggregation Tables: Pre-aggregate daily/weekly/monthly metrics in Cosmos DB
    • DirectQuery vs. Import: Use Import mode for dashboards (faster), DirectQuery for ad-hoc queries

    Sample Data Generators (Python)¶

    Generator 1: Platform Metrics¶

    File: dashboards/sample-data/generate-platform-metrics.py

    import json
import random
from datetime import datetime, timedelta
from azure.cosmos import CosmosClient

def generate_platform_metrics(days=180):
    """Generate 6 months of platform metrics"""
    metrics = []
    base_spaces = 30
    base_users = 800

    for day in range(days):
        date = (datetime.now() - timedelta(days=days-day)).strftime("%Y-%m-%d")

        # Simulate growth over time
        growth_factor = 1 + (day / days) * 0.5

        metric = {
            "id": f"platform-{date}",
            "metricType": "platform.health",
            "date": date,
            "timestamp": f"{date}T14:30:00Z",
            "metrics": {
                "totalSpaces": int(base_spaces * growth_factor),
                "activeUsers": int(base_users * growth_factor * random.uniform(0.9, 1.1)),
                "totalSessions": int(5000 * growth_factor * random.uniform(0.8, 1.2)),
                "availability": round(random.uniform(99.85, 99.99), 2),
                "monthlyCost": round(10000 * growth_factor * random.uniform(0.9, 1.1), 2),
                "avgCSAT": round(random.uniform(4.0, 4.5), 1),
                "p95Latency": int(random.uniform(200, 350)),
                "errorRate": round(random.uniform(0.1, 0.8), 1),
                "safetyTriggers": int(random.uniform(5, 20))
            },
            "costByService": {
                "cosmosDB": round(random.uniform(3000, 5000), 2),
                "openAI": round(random.uniform(5000, 10000), 2),
                "search": round(random.uniform(1500, 3000), 2),
                "apim": round(random.uniform(700, 900), 2),
                "other": round(random.uniform(500, 1000), 2)
            }
        }
        metrics.append(metric)

    return metrics

def upload_to_cosmos(metrics, connection_string, database, container):
    """Upload metrics to Cosmos DB"""
    client = CosmosClient.from_connection_string(connection_string)
    db = client.get_database_client(database)
    cont = db.get_container_client(container)

    for metric in metrics:
        cont.upsert_item(metric)
        print(f"Uploaded: {metric['date']}")

if __name__ == "__main__":
    metrics = generate_platform_metrics(days=180)

    # Save to JSON for Power BI import (no Azure required)
    with open("platform-metrics.json", "w") as f:
        json.dump(metrics, f, indent=2)
    print(f"âœ… Generated {len(metrics)} platform metrics")

    File: dashboards/sample-data/generate-space-metrics.py

    import json
import random
from datetime import datetime, timedelta

SPACES = [
    {"id": "tenant-jurisprudence", "name": "Jurisprudence Research", "budget": 5000},
    {"id": "tenant-hr", "name": "HR Assistant", "budget": 3000},
    {"id": "tenant-benefits", "name": "Benefits Finder", "budget": 4000},
    {"id": "tenant-km", "name": "Knowledge Management", "budget": 6000},
    {"id": "tenant-training", "name": "Training Bot", "budget": 2500}
]

USERS = [
    "Alice Johnson", "Bob Smith", "Carol White", "David Brown", "Eve Davis",
    "Frank Wilson", "Grace Lee", "Henry Martinez", "Ivy Taylor", "Jack Anderson"
]

def generate_space_metrics(days=30):
    """Generate 30 days of space metrics"""
    metrics = []

    for space in SPACES:
        for day in range(days):
            date = (datetime.now() - timedelta(days=days-day)).strftime("%Y-%m-%d")

            # Simulate budget consumption over month
            budget_consumed = space["budget"] * (day / days) * random.uniform(0.9, 1.1)

            metric = {
                "id": f"{space['id']}-{date}",
                "tenantId": space["id"],
                "date": date,
                "spaceName": space["name"],
                "spaceStatus": random.choice(["green", "green", "green", "yellow"]),  # Mostly green
                "budget": {
                    "allocated": space["budget"],
                    "consumed": round(budget_consumed, 2),
                    "remaining": round(space["budget"] - budget_consumed, 2),
                    "percentUsed": round((budget_consumed / space["budget"]) * 100, 1)
                },
                "usage": {
                    "sessions": int(random.uniform(300, 500)),
                    "uniqueUsers": int(random.uniform(50, 100)),
                    "avgSessionDuration": round(random.uniform(8.0, 15.0), 1),
                    "ragQueries": int(random.uniform(800, 1500)),
                    "toolInvocations": int(random.uniform(200, 400))
                },
                "cost": {
                    "total": round(budget_consumed, 2),
                    "byUser": [
                        {"userId": f"user-{i:03d}", "userName": random.choice(USERS), "cost": round(random.uniform(50, 500), 2)}
                        for i in range(10)
                    ]
                },
                "quality": {
                    "csat": round(random.uniform(4.0, 4.8), 1),
                    "thumbsUp": int(random.uniform(300, 450)),
                    "thumbsDown": int(random.uniform(10, 30)),
                    "p50Latency": int(random.uniform(150, 250)),
                    "p95Latency": int(random.uniform(280, 400)),
                    "p99Latency": int(random.uniform(450, 600))
                }
            }
            metrics.append(metric)

    return metrics

if __name__ == "__main__":
    metrics = generate_space_metrics(days=30)

    Power BI Data Connectors¶

    Option 1: Azure Cosmos DB Connector (Recommended)¶

    Pros: - Native connector in Power BI Desktop - DirectQuery support (real-time data) - Automatic schema detection

    Setup: 1. Power BI Desktop â†’ Get Data â†’ Azure Cosmos DB 2. Enter Cosmos DB endpoint URL and access key 3. Select database and containers 4. Transform data (flatten JSON, filter columns) 5. Load to model

    Performance: - Use HPK in queries (e.g., WHERE c.metricType = 'platform.health' AND c.date >= '2025-10-01') - Enable query folding (push filters to Cosmos DB)

    Option 2: REST API (Custom Connector)¶

    Use Case: When advanced logic needed (aggregations, joins across containers)

    Setup: 1. Build Azure Function that queries Cosmos DB and returns JSON 2. Power BI â†’ Get Data â†’ Web â†’ Function URL 3. Parse JSON response 4. Load to model

    Pros: Full control over query logic, caching, rate limiting

    Option 3: Export to CSV (Development/Testing)¶

    Use Case: No Azure access yet, need to prototype dashboards

    Setup: 1. Run Python generators â†’ JSON files 2. Convert JSON to CSV (pandas: pd.read_json().to_csv()) 3. Power BI â†’ Get Data â†’ CSV 4. Load to model

    Migration Path: Replace CSV with Cosmos DB connector when Azure ready

    Open Questions¶

    Q1: Power BI Licensing (BUDGET)¶

    Question: Power BI Pro (\(10/user/month) or Premium (\)5000/month, unlimited users)?
    Impact: Budget vs. user scalability
    Recommendation: Start with Pro (5 users = $50/month), upgrade to Premium if >50 users
    Decision Owner: Marco + Finance

    Q2: Cosmos DB Costs (COST CONTROL)¶

    Question: What throughput (RU/s) for Cosmos DB? 400 RU/s ($24/month) or auto-scale?
    Impact: Performance vs. cost
    Recommendation: Start with 400 RU/s manual, monitor for throttling, enable auto-scale if P95 >50ms
    Decision Owner: windowsGuy

    Q3: Real-Time vs. Scheduled Refresh (ARCHITECTURE)¶

    Question: Use Change Feed (real-time, complex) or scheduled refresh (hourly, simple)?
    Impact: Data freshness vs. implementation complexity
    Recommendation: Scheduled refresh for MVP (hourly is sufficient), add Change Feed in v2 if <1 min latency needed
    Decision Owner: ai2

    Q4: Row-Level Security Implementation (SECURITY)¶

    Question: How to enforce tenant isolation in Power BI? RLS based on Entra ID claims?
    Impact: Security vs. usability
    Recommendation: RLS with tenantId filter, map Entra ID roles to tenants (PS-A sees own space, platform admins see all)
    Decision Owner: Security Sentinel

    Q5: Dashboard Embedding Strategy (INTEGRATION)¶

    Question: Embed in EVA Ops portal (iframe, Power BI Embedded API) or link to Power BI Service?
    Impact: User experience vs. development effort
    Recommendation: Link to Power BI Service for MVP (simple), embed in portal for v2 (seamless UX)
    Decision Owner: Marco
