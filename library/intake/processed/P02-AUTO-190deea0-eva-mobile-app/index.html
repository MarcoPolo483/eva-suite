    CDD: EVA Mobile App (iOS/Android)¶

    Last Updated: 2025-11-19 Status: Backlog Priority: MEDIUM Epic: Cross-Cutting Concerns

    Vision¶

    Provide native iOS and Android mobile applications for EVA Chat, Learning, and Tools—enabling voice-first interactions, offline mode, push notifications, and biometric authentication for on-the-go productivity and accessibility.

    Business Value: Extend EVA reach to 60% of users who prefer mobile, increase engagement by 45% with push notifications, improve accessibility with voice-first design, and enable offline productivity during travel or connectivity gaps.

    Core Capabilities¶

    • Voice-First Interface: Primary interaction mode with Azure Speech Services (hands-free)
    • Offline Mode: Cache last 10 conversations, sync when online
    • Push Notifications: Certifications, incidents, course reminders, chat responses
    • Biometric Authentication: Face ID (iOS), fingerprint (Android), seamless login
    • Camera Integration: Document scan with OCR, QR code reader for course enrollment
    • Responsive Layouts: Optimized for phone (375×667) and tablet (768×1024)
    • Dark Mode: System-aware dark/light theme with manual toggle
    • Native Performance: 60fps scrolling, <100ms touch response

    User Stories¶

    US-1: As a mobile user, I want voice queries so that I can interact hands-free¶

    Acceptance Criteria:

    • Tap microphone icon → speak query → transcribed via Azure Speech Services
    • Support wake phrase "Hey EVA" when app is open (background mode deferred to Sprint 7)
    • Voice input works in en-US and fr-FR (auto-detect language)
    • Transcription appears in chat input (editable before sending)
    • Voice response playback optional (toggle in settings)

    US-2: As a mobile user, I want offline mode so that I can access recent conversations without connectivity¶

    Acceptance Criteria:

    • Last 10 conversations cached locally (SQLite database)
    • Offline banner appears when no connectivity
    • User can read cached conversations, copy text, share content
    • New queries queued (sent when connectivity restored)
    • Sync indicator shows "Syncing 3 messages..." when online

    US-3: As a mobile user, I want push notifications so that I stay informed on the go¶

    Acceptance Criteria:

    • Push notification when: certification earned, course reminder (1 day before), incident alert (P0/P1), chat response (if away from app >5 min)
    • Tap notification → open app to specific content (deep link)
    • Notification preferences in settings (toggle per notification type)
    • Badge count shows unread notifications (red dot on app icon)
    • Works on iOS 16+ and Android 13+ (Firebase Cloud Messaging)

    US-4: As a mobile user, I want biometric login so that I access EVA securely without typing passwords¶

    Acceptance Criteria:

    • First login: Entra ID OAuth 2.0 (redirect to browser, SSO)
    • Enable Face ID (iOS) or fingerprint (Android) in settings
    • Subsequent logins: biometric prompt (fallback to PIN if biometric fails)
    • Biometric data stored locally (never sent to server)
    • Session expires after 7 days (re-authenticate with biometrics)

    US-5: As a mobile user, I want document scanning so that I can quickly add documents to spaces¶

    Acceptance Criteria:

    • Camera button in space → scan document (auto-detect edges, crop)
    • OCR via Azure AI Document Intelligence (extract text from scan)
    • Preview scanned text (editable before uploading)
    • Upload to space (progress indicator, <10 seconds for 5-page document)
    • QR code reader for course enrollment (scan QR → auto-enroll in course)

    Functional Requirements¶

    1. FR-1: SHALL support iOS 16+ and Android 13+ platforms
    2. FR-2: SHALL integrate Azure Speech Services for voice input/output (en-US, fr-FR)
    3. FR-3: SHALL cache last 10 conversations in SQLite (offline mode)
    4. FR-4: SHALL sync queued messages when connectivity restored (<5 seconds)
    5. FR-5: SHALL send push notifications via Azure Notification Hubs (iOS APNs, Android FCM)
    6. FR-6: SHALL support biometric authentication (Face ID, Touch ID, fingerprint)
    7. FR-7: SHALL integrate camera for document scanning (auto-crop, OCR via Azure AI Document Intelligence)
    8. FR-8: SHALL support QR code scanning for course enrollment
    9. FR-9: SHALL render markdown chat messages with syntax highlighting (code blocks)
    10. FR-10: SHALL support dark mode (system-aware, manual toggle)
    11. FR-11: SHALL deep link from push notifications to specific content
    12. FR-12: SHALL maintain 60fps scrolling performance (React Native FlatList)
    13. FR-13: SHALL log telemetry to Application Insights (screen views, voice queries, offline sync)
    14. FR-14: SHALL support accessibility (VoiceOver on iOS, TalkBack on Android, font scaling)
    15. FR-15: SHALL handle session expiry (re-authenticate after 7 days)

    Non-Functional Requirements¶

    Performance¶

    • App Launch: <2s cold start, <500ms warm start
    • Screen Transitions: <300ms navigation between screens
    • Voice Transcription: <1s latency (Azure Speech Services)
    • Offline Sync: <5s to sync queued messages when online

    Reliability¶

    • Crash Rate: <0.1% (industry standard for production apps)
    • Offline Mode: 100% reliability for cached content access
    • Push Delivery: >95% success rate (Azure Notification Hubs SLA)
    • Background Sync: Retry failed uploads with exponential backoff

    Security¶

    • Authentication: Entra ID OAuth 2.0 with PKCE (browser redirect)
    • Biometric Storage: Local device keychain (iOS Keychain, Android Keystore)
    • Data Encryption: SQLite database encrypted (AES-256), API calls TLS 1.3
    • Certificate Pinning: Prevent MITM attacks (pin Azure API certificates)

    Accessibility¶

    • WCAG 2.1 AA: Support VoiceOver (iOS), TalkBack (Android), dynamic font scaling
    • Voice Control: Primary interaction mode (hands-free operation)
    • Color Contrast: 4.5:1 text, 3:1 UI components (both light and dark mode)
    • Touch Targets: Minimum 44×44pt (iOS), 48×48dp (Android)

    Internationalization¶

    • Locales: en-US (default), fr-FR
    • Resource Catalogs: React Native i18n library (JSON translation files)
    • Voice Input: Azure Speech Services multilingual models
    • Date/Time: Localized formatting (user's device settings)

    Observability¶

    • Metrics: Application Insights tracks app launches, screen views, voice queries, offline syncs, crashes
    • Alerts: Crash rate >0.5%, push delivery <90%, voice transcription errors >5%
    • Dashboards: Power BI dashboard for mobile adoption, feature usage, platform breakdown (iOS vs. Android)

    Scalability¶

    • User Base: Support 10K+ concurrent mobile users
    • Push Notifications: Azure Notification Hubs (100K+ devices)
    • Offline Storage: SQLite (max 50 MB per user, auto-prune old conversations)

    Technical Architecture¶

    Frontend¶

    • Framework: React Native 0.73+ with TypeScript
    • Build Tool: Expo SDK 50+ (managed workflow)
    • UI Library: React Native Paper (Material Design), custom components
    • Navigation: React Navigation v6 (stack, tab, drawer navigators)
    • State Management: Zustand (global state), React Query (server state)
    • Offline Storage: @react-native-async-storage/async-storage + SQLite (expo-sqlite)

    Backend¶

    • API Gateway: Azure APIM (same endpoints as web app, mobile-optimized responses)
    • Push Notifications: Azure Notification Hubs (iOS APNs, Android FCM)
    • Voice Services: Azure Speech Services (speech-to-text, text-to-speech)
    • Document Intelligence: Azure AI Document Intelligence (OCR for camera scans)
    • Authentication: Entra ID OAuth 2.0 (react-native-app-auth library)

    Data Models¶

    Offline Conversation (SQLite):

    CREATE TABLE conversations (
  id TEXT PRIMARY KEY,
  space_id TEXT,
  messages TEXT, -- JSON array
  last_synced_at INTEGER,
  created_at INTEGER
);

    Push Notification (Azure Notification Hubs):

  "notification": {
    "title": "Certification Earned",
    "body": "Congratulations! You've earned RAG Fundamentals.",
    "sound": "default",
    "badge": 1
  },
  "data": {
    "type": "certification",
    "certId": "cert-123",
    "deepLink": "eva://learning/certifications/cert-123"
  }
}

    Implementation Phases¶

    Phase 1: Core Mobile App (Sprint 4, Week 1-2)¶

    • React Native project setup (Expo managed workflow)
    • Entra ID OAuth 2.0 authentication (browser redirect)
    • Chat interface (send/receive messages, markdown rendering)
    • SQLite offline storage (cache last 10 conversations)
    • iOS and Android builds (TestFlight, Google Play Console beta)

    Phase 2: Voice & Push (Sprint 5, Week 3-4)¶

    • Azure Speech Services integration (voice input/output)
    • Voice UI (microphone button, transcription display)
    • Azure Notification Hubs setup (iOS APNs, Android FCM)
    • Push notification handling (deep links, badge counts)
    • Notification preferences screen

    Phase 3: Camera & Biometrics (Sprint 6, Week 5-6)¶

    • Camera integration (document scan, auto-crop)
    • Azure AI Document Intelligence OCR
    • QR code scanning (course enrollment)
    • Biometric authentication (Face ID, fingerprint)
    • Session management (7-day expiry, biometric refresh)

    Phase 4: Polish & Observability (Sprint 7, Week 7-8)¶

    • Dark mode (system-aware, manual toggle)
    • Accessibility audit (VoiceOver, TalkBack, font scaling)
    • Application Insights telemetry (screen views, voice queries)
    • Performance optimization (FlatList, image caching, lazy loading)
    • App Store and Google Play production release

    Success Criteria¶

    • ✅ App launch <2s cold start
    • ✅ Voice transcription <1s latency
    • ✅ Offline mode 100% reliability for cached content
    • ✅ Push delivery >95% success rate
    • ✅ Crash rate <0.1% in production
    • ✅ User adoption >40% of total EVA users within 3 months
    • ✅ WCAG 2.1 AA compliance (accessibility audit passed)

    Open Questions¶

    1. Should the mobile app support background voice activation ("Hey EVA" when app is closed)?

    2. Impact: User convenience vs. battery drain and iOS/Android restrictions

    3. Recommendation: App-open only for MVP, background mode deferred to Sprint 9

    4. Decision Owner: Marco (PO)

    5. What is the offline storage limit per user?

    6. Impact: User experience vs. device storage constraints

    7. Recommendation: 50 MB max (auto-prune conversations older than 30 days)

    8. Decision Owner: Marco (PO)

    9. Should the mobile app support Learning courses with video playback?

    10. Impact: Feature parity with web vs. bandwidth/storage concerns

    11. Recommendation: Video playback for Sprint 6, offline video deferred to Sprint 8

    12. Decision Owner: Marco (PO)

    13. Should push notifications include message preview (security risk)?

    14. Impact: User convenience vs. privacy (message visible on lock screen)

    15. Recommendation: Generic notifications for MVP ("New message in EVA"), preview opt-in for Sprint 7

    16. Decision Owner: Marco (PO)

    17. Should the mobile app support iPad/tablet-optimized layouts?

    18. Impact: Broader device support vs. development effort
    19. Recommendation: Phone-optimized for MVP, tablet layouts Sprint 8
    20. Decision Owner: Marco (PO)
