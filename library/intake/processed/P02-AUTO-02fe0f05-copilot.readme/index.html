    üõ†Ô∏è Technical Architecture¶

    Siebel Audit Columns (Enterprise Standard)¶

    • 8 Siebel Standard: ROW_ID (GUID "1-XXXXXXXX-..."), CREATED, CREATED_BY, LAST_UPD, LAST_UPD_BY, MODIFICATION_NUM, DB_LAST_UPD, DB_LAST_UPD_SRC
    • 5 EVA Extensions: TENANT_ID, VERSION, IS_DELETED, DELETED_AT, DELETED_BY
    • Purpose: Complete traceability for all entities
    • Implementation: EVA-DAL auto-manages (developers never touch directly)

    EVA-DAL (Data Abstraction Layer)¶

    • Repository pattern with automatic audit columns
    • Soft deletes only (physical deletes FORBIDDEN)
    • Multi-tenancy enforcement (TENANT_ID auto-filter)
    • Optimistic locking (VERSION check on update)
    • Audit trail integration (EVA-Audit-Log.jsonl)

    EVA-Meta (Schema Registry)¶

    • Single source of truth for: schemas, repositories, governance, agents, products
    • Validation tools: validate-schemas.ts, generate-types.ts
    • Integration: EVA-DAL, GoC.scorecard, EVA Matrix all consume

    EVA Directory (Canadian Context)¶

    • Central registry for GC departments, provinces, municipalities, Crown Corps
    • No cross-border data or intelligence
    • Multi-tenant: gc-esdc, gc-tbs, on-health, qc-revenue, etc.

    üéì Lessons Learned (Dec 3-7, 2025)¶

    1. Three Concepts Pattern is the Foundation¶

    What Happened: Dec 7 - shipped BOLT 1-6, established CE + HK + WM as daily operating system Lesson: All three concepts must work together - not "Context Engineering only" Decision: Eva-orchestrator is reference implementation, EVA-Sovereign-UI is first client Impact: 59 PowerShell scripts to audit/transform, transformation plan ready (843 lines)

    2. External Templates Solve Backtick Hell¶

    What Happened: P02 used here-strings, causing endless backtick escaping issues Lesson: External .md files = zero escaping, readable templates, easier maintenance Action: BOLT 6 complete - all P02 templates external Tool: templates/*.md.template read at runtime

    3. Orchestrator Queue Not Working¶

    What Happened: Dec 4 - GitHub issue #7 sat idle 4+ hours, no autonomous pickup Lesson: Hub Update workflow + Autonomous Queue Executor NOT monitoring GitHub issues Decision: Manual intervention required OR fix queue monitoring Impact: Scrum Master completed work manually (Canada-first scrub)

    4. Scope Management is Critical¶

    What Happened: Dec 3 - revealed massive Five Eyes ambitions (471M people, 5 nations, $2M certs) Lesson: Trying to serve everyone = serving no one well Decision: Focus on Canada (38M), perfect the model, let others adapt Impact: Eliminated complexity, risks, costs while maintaining quality

    5. Ethics Before Features¶

    What Happened: Dec 3 - deep discussion about dual-use AI (surveillance risks, weaponization) Lesson: Building powerful tools without safeguards = inevitable abuse Decision: Canada-first reduces weaponization risk, establishes ethical leadership Impact: Positioned EVA as responsible AI example

    6. Validation Mandate¶

    What Happened: Dec 3 - user corrected P02 column count (17 ‚Üí 10), demanded fact-checking Lesson: "Don't fool yourself and trust everything that you read... check for facts" Action: Validate all documentation before proceeding Tool: Test against real data, check references, verify claims

    7. Memory Architecture Matters¶

    What Happened: Found eva-lwo with Senior Advisor's 3-year memory, P02 reports, heartbeat logs Lesson: Persistent memory across sessions = continuity and context Action: Created copilot.readme.md (this file) for Scrum Master memory Tool: Both agents maintain memory files, sync context
