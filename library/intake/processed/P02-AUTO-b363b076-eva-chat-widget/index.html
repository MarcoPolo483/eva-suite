    CDD: EVA Chat Widget (AskMe Embeddable)¶

    Last Updated: 2025-11-19 Status: Backlog Priority: HIGH Epic: EVA Chat

    Vision¶

    Provide embeddable chat widget ("AskMe" button) that can be integrated into any webpage, SharePoint site, or application—allowing users to access EVA's conversational AI without leaving their workflow. Think Intercom/Drift style widget with context awareness.

    Business Value: Increase adoption by 60%, reduce context switching, enable just-in-time assistance, and expand EVA's reach across ESDC digital properties.

    Core Capabilities¶

    • Embeddable Widget: Lightweight JavaScript SDK (<50KB)
    • Floating Button: Customizable "AskMe" button (color, position, text)
    • Context Awareness: Pass page context to AI (current URL, page title, selected text)
    • Persistent Sessions: Resume conversations across page navigations
    • Proactive Prompts: "Need help with this form?" based on page context
    • Multi-Channel: Works on public websites, SharePoint, Teams, Outlook
    • Brand Customization: Custom colors, logo, greeting message
    • Privacy Controls: Opt-out, data retention policies

    User Stories¶

    US-1: As a web admin, I want to embed EVA so that users can get help without leaving the site¶

    Acceptance Criteria:

    • Install widget via single <script> tag (CDN hosted)
    • Configure via JavaScript: EVA.init({ tenantId: 'ESDC', theme: 'blue' })
    • Widget loads async (non-blocking page load)
    • "AskMe" button appears in bottom-right (customizable position)
    • Click opens chat panel (slide-in animation)

    US-2: As a user, I want context-aware help so that EVA understands what I'm working on¶

    Acceptance Criteria:

    • Widget captures current URL, page title, selected text
    • User can select text → right-click → "Ask EVA about this"
    • EVA prefixes query with page context: "On page 'Benefits Application', user asks..."
    • Context sent only if user grants permission (privacy consent)
    • Context enrichment improves answer relevance by 40%

    US-3: As a user, I want persistent sessions so that I can resume conversations after page refresh¶

    Acceptance Criteria:

    • Session ID stored in localStorage (expires after 24 hours)
    • Chat history persists across page navigations (same domain)
    • Widget auto-opens if user was mid-conversation
    • Session syncs to Cosmos DB (backup, multi-device)
    • Clear conversation button (deletes session)

    US-4: As a SharePoint admin, I want proactive prompts so that users discover EVA¶

    Acceptance Criteria:

    • Widget detects page context (e.g., Benefits form)
    • After 30 seconds inactivity, shows proactive prompt: "Need help with Benefits?"
    • User can dismiss or engage ("Yes, tell me more")
    • Prompts configurable per tenant (enable/disable, delay, message)
    • Engagement tracked (Application Insights)

    US-5: As a PS-A, I want branded widget so that it matches my space's identity¶

    Acceptance Criteria:

    • Configure theme: primary color, logo, greeting message
    • Override default "AskMe" button text (e.g., "Ask HR Bot")
    • Custom avatar for bot (upload image URL)
    • Preview widget before deploying
    • Reset to default theme button

    Functional Requirements¶

    FR-1: Embeddable SDK¶

    System SHALL provide JavaScript SDK (<50KB gzipped) embeddable via single <script> tag, loaded async from Azure CDN.

    FR-2: Initialization¶

    SDK SHALL initialize via EVA.init({ tenantId, theme, position, language }) with default values if not provided.

    FR-3: Floating Button¶

    Widget SHALL display floating "AskMe" button (customizable position: bottom-right, bottom-left, top-right, top-left).

    FR-4: Chat Panel¶

    Click button SHALL open chat panel (slide-in animation from right) with greeting message, input box, and chat history.

    FR-5: Context Capture¶

    Widget SHALL capture page context (URL, title, selected text) if user grants permission (consent dialog on first use).

    FR-6: Session Persistence¶

    System SHALL store session ID in localStorage (24-hour expiry) and sync chat history to Cosmos DB.

    FR-7: Multi-Page Resume¶

    Widget SHALL resume conversations across page navigations (same domain) using localStorage session ID.

    FR-8: Proactive Prompts¶

    Widget SHALL detect page context and show proactive prompts (configurable delay, message) after inactivity.

    FR-9: Privacy Consent¶

    Widget SHALL request user consent before capturing page context (PIPEDA compliance), with opt-out option.

    FR-10: Brand Customization¶

    System SHALL allow PS-A to configure theme (colors, logo, greeting, button text) via Admin UI.

    FR-11: Multi-Channel Support¶

    SDK SHALL work on public websites, SharePoint (Modern/Classic), Microsoft Teams (tab app), Outlook (add-in).

    FR-12: Performance¶

    Widget SHALL load in <1 second (async), not block page load, and achieve Lighthouse performance score ≥90.

    FR-13: Accessibility¶

    Widget SHALL comply with WCAG 2.1 AA (keyboard navigation, screen reader, focus management).

    FR-14: Analytics¶

    System SHALL track widget interactions (opens, messages, proactive engagement) in Application Insights.

    FR-15: Error Handling¶

    Widget SHALL display user-friendly error messages (bilingual) and degrade gracefully if backend unavailable.

    Non-Functional Requirements¶

    Performance¶

    • Widget load time: <1 second (P95)
    • Chat response latency: <3 seconds (P95)
    • Lighthouse performance score: ≥90 (page with widget)
    • Bundle size: <50KB gzipped
    • Non-blocking async load (no impact on page metrics)

    Reliability¶

    • Uptime: 99.9% SLA (CDN + Azure Front Door)
    • Graceful degradation (show static help link if backend down)
    • Auto-retry failed messages (3 attempts with exponential backoff)
    • Offline mode: Queue messages, send when online

    Security¶

    • Content Security Policy (CSP) compatible
    • HTTPS only (TLS 1.3)
    • OAuth 2.0 authentication (Entra ID SSO)
    • XSS protection (sanitize user inputs)
    • PII masking before logging (SIN, email, phone)
    • CORS policies (whitelist allowed domains)

    Accessibility¶

    • WCAG 2.1 AA compliance (verified via axe DevTools)
    • Keyboard navigation (Tab, Enter, Esc)
    • Screen reader announcements (live regions for messages)
    • Focus trap when panel open (Esc to close)
    • High-contrast mode support

    i18n¶

    • Bilingual support (en-US/fr-FR)
    • Auto-detect browser language or manual toggle
    • ICU formatting for dates/times
    • Bilingual error messages

    Observability¶

    • Application Insights custom events (widget load, open, message, proactive engagement)
    • Real User Monitoring (RUM) for performance metrics
    • Error tracking with stack traces
    • Alerts for high load times or error rates

    Scalability¶

    • CDN distribution (Azure Front Door)
    • Support 10K concurrent users per tenant
    • API rate limiting (100 requests/hour per user)
    • Horizontal scaling (Azure Functions backend)

    Technical Architecture¶

    Frontend (Widget SDK)¶

    • Framework: Vanilla JavaScript (no dependencies for lightweight SDK)
    • UI: Web Components (custom elements, shadow DOM for isolation)
    • Build: Rollup (tree shaking, minification)
    • CDN: Azure CDN for SDK distribution
    • Communication: EventSource (SSE) for real-time chat streaming

    Backend¶

    • API: Azure Functions HTTP triggers (Node.js)
    • Chat Service: eva-chat-service (GPT-4o integration)
    • Session Storage: Cosmos DB (HPK /tenantId/sessionId)
    • Context Enrichment: Extract page metadata, append to LLM prompt

    Integration Points¶

    • eva-chat-service: GPT-4o conversational AI
    • eva-auth: User authentication (Entra ID SSO)
    • eva-safety: Content moderation, PII masking
    • eva-api: API Gateway (APIM)

    Data Models¶

    Widget Configuration (Cosmos DB, HPK /tenantId):

    {
  "tenantId": "ESDC",
  "theme": {
    "primaryColor": "#0078D4",
    "logo": "https://cdn.esdc.gc.ca/logo.png",
    "greeting": "Hi! I'm EVA. How can I help?",
    "buttonText": "AskMe"
  },
  "proactive": {
    "enabled": true,
    "delay": 30000,
    "message": "Need help with this page?"
  },
  "privacy": {
    "captureContext": true,
    "dataRetention": "24h"
  }
}

    Session (localStorage + Cosmos DB, HPK /tenantId/sessionId):

    {
  "sessionId": "uuid",
  "userId": "marco@esdc",
  "tenantId": "ESDC",
  "pageContext": {
    "url": "https://intranet.esdc/benefits",
    "title": "Benefits Application",
    "selectedText": "maternity leave duration"
  },
  "messages": [
    { "role": "user", "content": "How long is maternity leave?" },
    { "role": "assistant", "content": "In Canada..." }
  ],
  "createdAt": "2025-11-19T10:00:00Z",
  "expiresAt": "2025-11-20T10:00:00Z"
}

    Open Questions¶

    Q1: Authentication Flow (TECH)¶

    Question: Embed widget before or after user logs in? Impact: UX vs. security Recommendation: Allow anonymous mode (basic Q&A), prompt SSO for personalized features Decision Owner: ai2 + Marco

    Q2: Context Privacy (COMPLIANCE)¶

    Question: Capture page context by default or require explicit consent? Impact: UX friction vs. PIPEDA compliance Recommendation: Request consent on first use, remember preference Decision Owner: Security Sentinel

    Q3: Multi-Tenant Isolation (SECURITY)¶

    Question: Single SDK for all tenants or tenant-specific SDKs? Impact: Simplicity vs. security Recommendation: Single SDK with tenantId parameter, tenant data isolated in Cosmos DB Decision Owner: ai2

    Q4: Proactive Prompt Frequency (UX)¶

    Question: Show proactive prompts on every page or limit to once per session? Impact: Helpfulness vs. annoyance Recommendation: Once per session, user can re-enable in settings Decision Owner: UX Partner

    Q5: Offline Mode (FEATURES)¶

    Question: Queue messages when offline or show "No connection" error? Impact: UX richness vs. complexity Recommendation: Queue messages in localStorage, send when online (retry UI) Decision Owner: ai2
