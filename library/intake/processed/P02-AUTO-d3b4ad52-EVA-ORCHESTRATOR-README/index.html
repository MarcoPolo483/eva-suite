    EVA Suite Orchestrator¶

    A Python-based orchestration engine that reads YAML plans and generates AI-ready prompts for building and maintaining EVA Suite features.

    ğŸš€ Quick Start¶

    1. Install Dependencies¶

    pip install -r requirements.txt

    This installs: - pyyaml - For parsing YAML plan files - requests - For HTTP requests (OpenAI API in future)

    2. Run Dry-Run Mode¶

    See plan summary and task list:

    cd eva-orchestrator
set PYTHONPATH=%CD%\src
python -m eva_suite_orchestrator.cli --plan plans/eva-suite/sprint-heroes-demo.yml --mode dry-run

    3. Generate AI Prompts¶

    Generate prompts for all tasks:

    cd eva-orchestrator
set PYTHONPATH=%CD%\src
python -m eva_suite_orchestrator.cli --plan plans/eva-suite/sprint-heroes-demo.yml --mode print-prompts

    Or simply double-click tools\run_eva_plan.bat on Windows.

    ğŸ“‹ Plan Format¶

    Plans are YAML files with this structure:

    plan_id: unique-plan-id
label: "Human-readable plan name"
owner: "plan-owner"
description: >
  Multi-line description of the plan

default_repo:
  name: "repo-name"
  local_path: "../relative/path"
  main_branch: "main"

tags:
  - "tag1"
  - "tag2"

tasks:
  - id: "task-id"
    title: "Task title"
    repo: "repo-name"
    path_hint: "src/components, src/data"
    kind: "feature"
    status: "planned"
    instructions: |
      Multi-line task instructions
      with implementation details

    ğŸ¯ Modes¶

    dry-run (default)¶

    Shows plan summary and task list without generating prompts. Good for quick validation.

    Output: - Plan metadata (ID, owner, tags, task count) - Task table (ID, title, repo, status)

    print-prompts¶

    Generates complete AI-ready prompts for all tasks.

    Output: - Plan summary - One prompt per task with: - Context about the plan - Repository information - Task details and instructions - Implementation guidelines - Important reminders

    ğŸ“ Using Generated Prompts¶

    1. Run orchestrator in print-prompts mode
    2. Copy a task prompt
    3. Paste into ChatGPT, Copilot, or your AI tool
    4. The AI will have full context to implement the task

    The included plans/eva-suite/sprint-heroes-demo.yml defines 6 tasks:

    2. EVA DA - Decision support with domain selection and structured answers

    Each task has: - Clear instructions - Path hints for file locations - Repository target - Feature type and status

    ğŸ’¡ Tips¶

    • Use --mode dry-run first to validate your plan
    • Review generated prompts before sending to AI
    • Keep task instructions focused and specific
    • Include path hints to guide file placement
    • Use consistent status values across tasks

    ğŸ› Troubleshooting¶

    Module not found error: 

    # Make sure PYTHONPATH includes src/
set PYTHONPATH=%CD%\src

    Encoding issues on Windows: The CLI automatically handles UTF-8 encoding on Windows. If you see garbled characters, the output is still valid - just the terminal display is affected.

    YAML parsing errors: - Check for proper indentation (use spaces, not tabs) - Validate required fields are present - Ensure multi-line strings use | or >
