    ğŸ“ Learning Paths¶

    Path 1: Executive / Decision Maker (30 minutes)¶

    1. Read "Executive Summary" in GC-AGENTIC-FRAMEWORK-SUMMARY.md
    2. Review "5-Step Department Adoption" section
    3. Check "Key Insights" section
    4. Review Volume 0 (adoption guide) from archive

    Goal: Understand strategic value and adoption approach

    Path 2: Architect / Technical Lead (2-3 hours)¶

    1. Read full GC-AGENTIC-FRAMEWORK-SUMMARY.md
    2. Deep dive Volume 1 (platform requirements)
    3. Study 7 major epics in detail
    4. Review EVA implementation in README.md
    5. Examine WEEK-1-AGENT-2-GC-COMPONENTS.md

    Goal: Design platform architecture aligned with GC templates

    Path 3: Developer / Engineer (4 hours)¶

    1. Read GC-AGENTIC-FRAMEWORK-QUICK-REF.md
    2. Study Volume 2 (engineering guide)
    3. Review Three Concepts Pattern in any script (e.g., scripts/*.ps1)
    4. Examine bootstrap process (scripts/bootstrap-repo.ps1)
    5. Review EVA Memory System (.eva-memory.json in any repo)

    Goal: Implement GC-compliant agents and components

    Path 4: Governance / Compliance (2 hours)¶

    1. Read Volume 1 sections on autonomy levels and lifecycle tiers
    2. Study Volume 3 (LiveOps & governance)
    3. Review compliance checklist in GC-AGENTIC-FRAMEWORK-QUICK-REF.md
    4. Examine Volume 4 patterns and anti-patterns

    Goal: Establish governance framework and review processes

    Path 5: Product Owner / Program Lead (1-2 hours)¶

    1. Read Volume 0 (adoption guide)
    2. Review Volume 4 (use cases & patterns)
    3. Study Agent Concept Canvas examples
    4. Examine EVA Crew Charter alignment

    Goal: Define pilot use cases and agent scope
