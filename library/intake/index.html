    EVA Suite Intake Process¶

    Purpose: Drop feature ideas here for automated P02 Requirements Engine processing.

    How It Works¶

    1. Drop a file (.md or .txt) in docs/intake/inbox/
    2. Wait for automation (runs every 6 hours via GitHub Action, or trigger manually)
    3. Pipeline executes:
    4. Parses your intake document
    5. Runs P02 Requirements Engine with telemetry
    6. Generates 10 sprint-ready artifacts
    7. Delivers to Agile Crew (P15) inbox
    8. Moves intake file to processed/

    Intake Document Format¶

    Option 1: With Front-Matter (Recommended)¶

    ---
FeatureKey: P02-FINOPS-005
FeatureSlug: cost-optimization-dashboard
Title: Real-time Cost Optimization Dashboard
Description: Add Power BI dashboard for FinOps cost tracking
Priority: HIGH
---

## Feature Overview

Build a real-time dashboard that shows...

[rest of your feature description]

    Option 2: Plain Markdown (Auto-generates metadata)¶

    # My Cool Feature

Description of the feature...

## Requirements

- Requirement 1
- Requirement 2

    Auto-generated: - FeatureKey: P02-AUTO- - FeatureSlug: my-cool-feature (from filename) - Title: First heading or filename

    Front-Matter Fields¶

    Field Required Description Example
    FeatureKey No P02 feature identifier P02-FINOPS-005
    FeatureSlug No URL-friendly slug cost-dashboard
    Title No Feature title Cost Dashboard
    Description No Short description Add FinOps dashboard
    Priority No HIGH / MEDIUM / LOW HIGH

    If not provided, the pipeline auto-generates these from the filename and content.

    What Happens Next¶

    1. P02 Execution (Automatic)¶

    The pipeline runs scripts/p02-run.ps1 with your feature metadata:

    .\scripts\p02-run.ps1 `
  -FeatureKey "P02-FINOPS-005" `
  -FeatureSlug "cost-dashboard" `
  -Caller "github-action" `
  -Description "Real-time Cost Optimization Dashboard"

    2. Artifacts Generated¶

    P02 creates 10 documentation artifacts in docs/features/<slug>/:

    • README.md
    • P02-Overview.md
    • requirements.md
    • feature-map.md
    • backlog.md
    • use-cases.md
    • risks.md
    • architecture-notes.md
    • adr-.md
    • tests.md

    3. Agile Crew Delivery¶

    Pipeline creates a manifest in docs/agile-crew/inbox/<slug>-manifest.json:

    {
  "feature_key": "P02-FINOPS-005",
  "feature_slug": "cost-dashboard",
  "title": "Real-time Cost Optimization Dashboard",
  "source_intake_file": "cost-dashboard.md",
  "p02_artifacts_path": "docs/features/cost-dashboard",
  "created_timestamp": "2025-11-25T10:30:00.0000000Z",
  "status": "ready-for-crew",
  "priority": "HIGH"
}

    4. Telemetry Logged¶

    All P02 execution details logged to docs/_audit/EVA-Audit-Log.jsonl: - p02-execution event (run ID, status, duration, artifacts count) - p02-openai-usage events (model, tokens, cost estimate)

    5. Intake File Moved¶

    Your original file moves to docs/intake/processed/ with naming: - P02-FINOPS-005-cost-dashboard.md

    Manual Execution¶

    If you don't want to wait for the scheduled run:

    Option 1: GitHub UI (Recommended)¶

    1. Go to Actions tab
    2. Select P02 Intake Pipeline
    3. Click Run workflow
    4. Choose branch and dry-run option
    5. Click Run workflow

    Option 2: Local CLI¶

    # Process all new intake files
.\scripts\p02-intake-processor.ps1 -Caller "cli"

# Dry run (preview only)
.\scripts\p02-intake-processor.ps1 -Caller "cli" -DryRun

    Examples¶

    Example 1: Simple Feature Idea¶

    File: docs/intake/inbox/ai-code-review.md

    # AI-Powered Code Review Assistant

Help developers review pull requests using GPT-4.

## Goals
- Automated PR analysis
- Security vulnerability detection
- Style and best practice recommendations

## Tech Stack
- OpenAI API
- GitHub API
- Azure Functions

    Result: Auto-generates P02-AUTO-<hash> and ai-code-review slug.

    Example 2: Structured Feature Request¶

    File: docs/intake/inbox/finops-alerts.md

    ---
FeatureKey: P02-FINOPS-006
FeatureSlug: cost-alert-system
Title: FinOps Cost Alert System
Description: Real-time alerts when cloud spending exceeds thresholds
Priority: HIGH
---

## Business Case

Finance team needs immediate notification when:
- Daily spend > $1000
- Weekly spend > $5000
- Any single resource > $500/day

## Requirements

1. Real-time monitoring of Azure costs
2. Configurable alert thresholds
3. Multi-channel notifications (email, Slack, Teams)
4. Alert history and analytics

## Constraints

- Must integrate with existing Azure Cost Management API
- Latency < 15 minutes for alerts
- Support 100+ cost rules

    Result: Uses explicit metadata, generates comprehensive P02 artifacts.

    Pipeline Schedule¶

    Automatic runs: Every 6 hours (00:00, 06:00, 12:00, 18:00 UTC)

    Manual trigger: Anytime via GitHub Actions UI

    Monitoring & Reporting¶

    Check Pipeline Status¶

    GitHub Actions: - Navigate to Actions P02 Intake Pipeline - View execution logs, artifacts, and status

    View P02 Usage¶

    # Last 24 hours
.\scripts\p02-usage-report.ps1 -Last 1 -AsTable

# Specific feature
.\scripts\p02-usage-report.ps1 -FeatureKey "P02-FINOPS-005" -AsTable

# Export CSV for analysis
.\scripts\p02-usage-report.ps1 -Last 7 -AsCsv

    Check Agile Crew Inbox¶

    Get-ChildItem "docs\agile-crew\inbox" | Select-Object Name, LastWriteTime

    Troubleshooting¶

    Issue: File not processed¶

    Check: 1. Is file in docs/intake/inbox/? 2. Is file extension .md or .txt? 3. Has pipeline run since file was added? 4. Check state file: docs/_state/P02-Intake-State.json

    Solution: Run manually or wait for next scheduled execution.

    Issue: P02 execution failed¶

    Check: 1. GitHub Actions logs for error details 2. Validate front-matter syntax (YAML format) 3. Check if scripts/p02-run.ps1 exists

    Solution: Fix syntax errors and retry manually.

    Issue: Dry run shows unexpected behavior¶

    Action: Review parsed metadata in dry-run output, adjust front-matter or filename.

    State File¶

    Location: docs/_state/P02-Intake-State.json

    Purpose: Tracks processed files to avoid duplicates.

    Structure: 

    {
  "lastRunTimestamp": "2025-11-25T10:30:00.0000000Z",
  "processedFiles": [
    {
      "path": "docs/intake/processed/P02-FINOPS-005-cost-dashboard.md",
      "hash": "a1b2c3d4...",
      "feature_key": "P02-FINOPS-005",
      "feature_slug": "cost-dashboard",
      "processed_at": "2025-11-25T10:30:00.0000000Z",
      "original_filename": "cost-dashboard.md"
    }
  ]
}

    Reset state (reprocess all files): 

    Remove-Item "docs\_state\P02-Intake-State.json"

    Best Practices¶

    1. Use descriptive filenames: feature-name.md becomes feature-name slug
    2. Include front-matter: Provides better control over P02 execution
    3. Write clear descriptions: Helps P02 generate better requirements
    4. Set priority: HIGH/MEDIUM/LOW for Agile Crew inbox sorting
    5. One file per feature: Don't combine multiple features in one intake doc
    6. Review P02 artifacts: Check generated docs and refine as needed
