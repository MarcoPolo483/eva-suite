    [LESSON-005] User Feedback Loop + Self-Learning Content Updates¶

    Pod: Pod Gamma (UI/UX) + Pod Alpha (RAG/Content) Role: All roles (cross-functional feature) Sprint: Sprint-01 (Week 0 - Planning) Date: 2025-11-17 Tags: user-feedback, rag, self-learning, content-governance, ui, quality

    ğŸ“‹ Context¶

    During EVA 2.0 planning, Marco identified a major missing feature from initial architecture:

    User feedback mechanism - Thumbs up/down on chat responses with:

    1. Immediate feedback capture (satisfaction signal)
    2. Dissatisfaction diagnostics (checkboxes + free text)
    3. Review workflow (team reviews and acts on feedback)
    4. Self-learning content loop (feedback triggers RAG content review/update)

    Reference implementation: AssistMe (ESDC) has this in production (see AssistMe Go Live document)

    Key insight from Marco's clients:

    "Would the thumbs up or down also trigger review of the content used to compose the answer? That would allow a self-learning, content updating process. This is major..."

    Business value: Closes the loop between user satisfaction and content quality - system improves itself based on real usage patterns.

    âŒ Problem¶

    Current State: No Feedback Mechanism¶

    EVA 2.0 initial design (as of Nov 2025):

    • âœ… Chat interface with responses
    • âœ… RAG retrieval with citations
    • âœ… Safety filters and guardrails
    • âŒ No user satisfaction capture
    • âŒ No feedback diagnostics
    • âŒ No content quality loop

    Impact:

    • âŒ No visibility into user satisfaction (which responses are helpful? which fail?)
    • âŒ No systematic quality improvement (team doesn't know what to fix)
    • âŒ Stale RAG content (outdated documents never flagged for update)
    • âŒ Missed learning opportunities (same bad responses repeated forever)
    • âŒ No compliance evidence (governance requires user feedback tracking)

    Example Failure Scenario¶

    Situation: User asks "What's the claims processing policy for XYZ?"

    EVA response: Retrieves outdated policy document from 2023, provides incorrect guidance

    Without feedback loop:

    1. User frustrated (incorrect answer)
    2. No signal captured (EVA doesn't know it failed)
    3. Same mistake repeated for next 100 users
    4. Team discovers issue months later via complaints
    5. Manual content audit required (expensive, slow)

    With feedback loop (desired state):

    1. User clicks thumbs down
    2. Flags "Outdated information" + note: "Policy changed in Jan 2025"
    3. Feedback queued for content team review
    4. Content team updates RAG index with latest policy
    5. Future users get correct answer
    6. Continuous improvement proven for governance

    DissatisfactionModal.tsx (based on AssistMe pattern):

    interface DissatisfactionModalProps {
  messageId: string;
  tenantId: string;
  onSubmit: (feedback: UserFeedback) => Promise<void>;
  onClose: () => void;
}

const DISSATISFACTION_CATEGORIES = [
  { id: "incorrect", label: "Incorrect or outdated information" },
  { id: "irrelevant", label: "Answer not relevant to my question" },
  { id: "incomplete", label: "Incomplete or missing details" },
  { id: "unclear", label: "Confusing or unclear explanation" },
  { id: "sources", label: "Missing or wrong source citations" },
  { id: "language", label: "Language quality issues" },
  { id: "other", label: "Other (please describe)" },
];

export const DissatisfactionModal: React.FC<DissatisfactionModalProps> = ({
  messageId,
  tenantId,
  onSubmit,
  onClose,
}) => {
  const [selectedCategories, setSelectedCategories] = useState<string[]>([]);
  const [detailText, setDetailText] = useState("");
  const [submitting, setSubmitting] = useState(false);

  const handleSubmit = async () => {
    setSubmitting(true);
    await onSubmit({
      messageId,
      tenantId,
      rating: "negative",
      categories: selectedCategories,
      detail: detailText,
      timestamp: new Date().toISOString(),
    });
    setSubmitting(false);
    onClose();
  };

  return (
    <Modal
      isOpen
      onClose={onClose}
      title="Help us improve"
      aria-labelledby="feedback-modal-title"
    >
      <h2 id="feedback-modal-title">What went wrong?</h2>
      <p>Select all that apply:</p>

      <fieldset>
        <legend className="sr-only">Dissatisfaction reasons</legend>
        {DISSATISFACTION_CATEGORIES.map((cat) => (
          <label key={cat.id} className="checkbox-label">
            <input
              type="checkbox"
              value={cat.id}
              checked={selectedCategories.includes(cat.id)}
              onChange={(e) => {
                if (e.target.checked) {
                  setSelectedCategories([...selectedCategories, cat.id]);
                } else {
                  setSelectedCategories(
                    selectedCategories.filter((c) => c !== cat.id)
                  );
                }
              }}
            />
            {cat.label}
          </label>
        ))}
      </fieldset>

      <label htmlFor="feedback-detail">Additional details (optional):</label>
      <textarea
        id="feedback-detail"
        value={detailText}
        onChange={(e) => setDetailText(e.target.value)}
        placeholder="Please provide more context to help us improve..."
        rows={4}
        maxLength={1000}
      />

      <div className="modal-actions">
        <button onClick={onClose} disabled={submitting}>
          Cancel
        </button>
        <button
          onClick={handleSubmit}
          disabled={submitting || selectedCategories.length === 0}
        >
          {submitting ? "Submitting..." : "Submit Feedback"}
        </button>
      </div>
    </Modal>
  );
};

    API handler (simplified):

    export const submitFeedback = async (req: Request, res: Response) => {
  const feedback: UserFeedback = req.body;

  // 1. Validate
  if (!feedback.messageId || !feedback.tenantId || !feedback.rating) {
    return res.status(400).json({ error: "Missing required fields" });
  }

  // 2. Retrieve message context (for enrichment)
  const message = await getMessageById(feedback.messageId);
  if (!message) {
    return res.status(404).json({ error: "Message not found" });
  }

  // 3. Build enriched feedback record
  const feedbackRecord = {
    id: `feedback-${uuidv4()}`,
    ...feedback,
    conversationId: message.conversationId,
    retrievedDocuments: message.retrievedDocuments,
    modelUsed: message.modelUsed,
    promptVersion: message.promptVersion,
    guardrailsApplied: message.guardrailsApplied,
    reviewStatus: "new",
    pk: feedback.tenantId,
  };

  // 4. Store in Cosmos DB
  await cosmosClient
    .database("eva")
    .container("feedback")
    .items.create(feedbackRecord);

  // 5. Emit telemetry event (for Governance MTI)
  await telemetry.track("user_feedback_submitted", {
    feedbackId: feedbackRecord.id,
    tenantId: feedback.tenantId,
    rating: feedback.rating,
    categories: feedback.categories,
    retrievedDocCount: message.retrievedDocuments?.length || 0,
  });

  // 6. Check if content review needed (self-learning trigger)
  if (
    feedback.rating === "negative" &&
    feedback.categories?.includes("incorrect")
  ) {
    await checkContentReviewThreshold(
      message.retrievedDocuments,
      feedback.tenantId
    );
  }

  res.status(201).json({
    id: feedbackRecord.id,
    message: "Thank you for your feedback",
  });
};

    Layer 4: Self-Learning Content Loop (Pod Alpha - RAG)¶

    Location: eva-rag/src/feedback-analysis/content-review-trigger.ts

    Auto-trigger logic:

    interface ContentReviewThreshold {
  docId: string;
  source: string;
  negativeFeedbackCount: number;
  threshold: number; // e.g., 3 negative feedbacks
  windowDays: number; // e.g., last 30 days
}

export const checkContentReviewThreshold = async (
  retrievedDocs: RetrievedDocument[],
  tenantId: string
) => {
  const windowDays = 30;
  const threshold = 3; // Auto-flag if 3+ negative feedbacks

  for (const doc of retrievedDocs) {
    // Query feedback collection for this doc
    const query = `
      SELECT COUNT(1) as negativeCount
      FROM feedback f
      WHERE f.tenantId = @tenantId
        AND ARRAY_CONTAINS(f.retrievedDocuments, { "docId": @docId }, true)
        AND f.rating = 'negative'
        AND f.timestamp >= @windowStart
    `;

    const windowStart = new Date();
    windowStart.setDate(windowStart.getDate() - windowDays);

    const result = await cosmosClient
      .database("eva")
      .container("feedback")
      .items.query({
        query,
        parameters: [
          { name: "@tenantId", value: tenantId },
          { name: "@docId", value: doc.docId },
          { name: "@windowStart", value: windowStart.toISOString() },
        ],
      })
      .fetchAll();

    const negativeCount = result.resources[0]?.negativeCount || 0;

    // Threshold exceeded - auto-create content review task
    if (negativeCount >= threshold) {
      await createContentReviewTask({
        docId: doc.docId,
        source: doc.source,
        tenantId,
        reason: `${negativeCount} negative feedbacks in ${windowDays} days`,
        priority: "high",
        assignedTo: "content-team",
        autoCreated: true,
      });

      // Emit alert
      await telemetry.track("content_review_auto_triggered", {
        docId: doc.docId,
        tenantId,
        negativeCount,
        threshold,
        windowDays,
      });
    }
  }
};

    Content review task workflow:

    1. Auto-created task appears in LiveOps dashboard
    2. Content team reviews flagged document
    3. Update document in source system (SharePoint, wiki, etc.)
    4. Trigger RAG re-ingestion for updated doc
    5. Track improvement: Monitor feedback for same doc post-update
    6. Close task when positive feedback increases

    Success metrics:

    • % of flagged documents updated within 7 days
    • % reduction in negative feedback post-update
    • User satisfaction trend (rolling 30-day average)

    ğŸ“š Lesson Learned¶

    Key Principles¶

    1. "User Feedback is Gold"

    2. Direct signal of what's working vs. failing

    3. More valuable than synthetic evals (real user intent)

    4. Required for governance compliance (evidence of quality monitoring)

    5. "Close the Loop"

    6. Feedback without action = wasted opportunity

    7. Systematic review workflow prevents feedback pile-up

    8. Self-learning automation scales beyond human review capacity

    9. "Content Quality is Dynamic"

    10. RAG content goes stale (policies change, products evolve)

    11. Feedback-driven updates keep content current

    12. Proactive pattern detection (3+ flags) prevents widespread issues

    13. "Transparency Builds Trust"

    14. Show users their feedback matters (thank you message)
    15. Display content update dates in citations
    16. Report feedback metrics in governance dashboards

    When to Apply¶

    Critical for:

    • âœ… Production AI assistants (user-facing chat)
    • âœ… RAG-based systems (content quality varies over time)
    • âœ… Multi-tenant platforms (different content per tenant)
    • âœ… Regulated environments (governance requires evidence)

    Lower priority for:
