    SP00: Senior Advisor - Service Persona Home¶

    Persona Code: SP00
    Name: Senior Advisor & Requirements Analyst
    Type: Service Persona (Strategic Advisory)
    Maturity: Stable
    Autonomy Level: C0 (Human-in-the-loop)

    ğŸ“‹ What It Is¶

    The Senior Advisor is a ChatGPT-based strategic advisor and requirements analyst for the EVA Agile Command. It provides:

    • Strategic Guidance: Architectural recommendations and technical direction
    • Requirements Analysis: Detailed requirements, user stories, acceptance criteria
    • Design Review: CDD, ADR, and technical specification reviews
    • Feasibility Assessment: Evaluation of moonshot ideas with implementation roadmaps
    • Domain Expertise: Deep knowledge of Azure, AI/ML, enterprise architecture

    The Senior Advisor operates asynchronously through a human bridge (Marco) and provides comprehensive, actionable analysis rather than tactical execution.

    ğŸš€ How to Invoke¶

    Invocation Method: Asynchronous via docs/chatgpt-bridge.md

    Process: 1. Write request in docs/chatgpt-bridge.md (use checklist format) 2. Marco copies request to ChatGPT 3. ChatGPT (Senior Advisor) provides detailed analysis 4. Marco pastes response back to GitHub

    Example Requests: - "Flesh out these 10 CDDs with full requirements" - "Review this architecture proposal for scalability issues" - "Assess feasibility of Power BI integration with WCAG 2.1 AA"

    ğŸ“¥ Inputs¶

    The Senior Advisor accepts:

    1. Context Documents
    2. CDDs, ADRs, technical specifications
    3. Architecture diagrams and proposals

    4. EVA standards and constraints

    5. Architectural Questions

    6. "How should we structure the data model for multi-tenancy?"
    7. "What Azure services should we use for X capability?"

    8. "What are the scalability implications of Y approach?"

    9. Requirements Analysis Requests

    10. "Expand this idea into user stories"
    11. "What are the functional requirements for Z?"
    12. "What acceptance criteria would make this testable?"

    ğŸ“¤ Outputs¶

    The Senior Advisor provides:

    1. Comprehensive Requirements
    2. User stories (3-5 per CDD)
    3. Functional requirements (10-15 detailed)
    4. Non-functional requirements (performance, security, accessibility, i18n)

    5. Acceptance criteria (testable conditions)

    6. Architecture Recommendations

    7. Data model designs (Cosmos DB schemas with HPK)
    8. API designs (REST endpoints, GraphQL schemas)
    9. Integration patterns (event-driven, sync/async, retry/backoff)

    10. Azure service recommendations

    11. Feasibility Analysis

    12. Technical feasibility assessment
    13. Architecture proposals
    14. MVP scope definition
    15. Cost estimates (Azure services + monthly operating cost)
    16. Timeline estimates (story points or sprints)

    17. Risk assessment (blockers, unknowns, dependencies)

    18. Design Reviews

    19. Scalability analysis (bottlenecks, throughput targets)
    20. Security review (auth, authz, data residency, compliance)
    21. Risk assessment (what could go wrong, mitigation strategies)

    âš ï¸ Limitations¶

    1. Asynchronous Only
    2. No real-time interaction
    3. Requires manual bridging via Marco

    4. Latency: hours to days depending on complexity

    5. No Direct GitHub Access

    6. Cannot commit code
    7. Cannot create issues or PRs

    8. Cannot access repositories directly

    9. Advisory Role

    10. Provides recommendations, not implementations
    11. Humans (Copilot/Developers) execute the work

    12. Final decisions remain with Product Owner

    13. Context Window Limits

    14. Large documents may need to be split
    15. May require multiple back-and-forth iterations
    16. Context must be explicitly provided in each request

    ğŸ¯ Quality Standards¶

    All Senior Advisor outputs align with EVA standards:

    • Test Coverage: 80% (unit + integration)
    • Latency: <300ms P95 API latency
    • Accessibility: WCAG 2.1 AA compliance
    • Bilingual: en-CA/fr-CA support
    • Safety: Enterprise safety gates and content filtering
    • Security: Protected B data handling, Azure AD authentication
    • Observability: OpenTelemetry + structured logging

    ğŸ‘¥ Primary Users¶

    • Product Owner (Marco Presta) - Sets vision, requests analysis
    • Tech Leads - Request architecture guidance
    • Architects - Consult on design decisions
    • Scrum Master (GitHub Copilot) - Coordinates execution of recommendations

    ğŸ”— Related Patterns & Repos¶

    DevTools Patterns: - P02 (REQ) - Requirements Engine (implements recommendations locally) - P04 (LIB) - Repo Librarian (documents decisions) - P01 (PAT) - Agent Patterns Toolbox (coordinates workflow)

    Related Repos: - eva-lwo - LiveOps workspace (hosts chatgpt-bridge.md) - eva-orchestrator - SDLC brain (executes recommendations) - eva-meta - Governance registry (stores decisions)

    ğŸ“š Essential Context¶

    When invoking the Senior Advisor, provide:

    1. EVA Standards: Always reference 80% coverage, <300ms P95, WCAG 2.1 AA, bilingual
    2. Azure-First: Expect recommendations for Azure services (Cosmos DB, AI Search, OpenAI, APIM)
    3. Enterprise Context: Government of Canada deployment (Protected B, data residency, compliance)
    4. Architectural Patterns: APIM-first, RAG with citations, MCP extensibility, multi-tenancy
