    Agent Request & Output Template Generator¶

    Purpose: Generate standardized request templates for agent services Version: 1.0.0 Usage: P02 TEMPLATE [service-name]

    How to Use¶

    Generate Input Template¶

    # Get input template for a specific service
P02 TEMPLATE schema-design
P02 TEMPLATE migration
P02 TEMPLATE performance

# Get output template (what agent will deliver)
P02 TEMPLATE OUTPUT schema-design
P02 TEMPLATE OUTPUT migration

    Fill & Submit Template¶

    1. Request template: P02 TEMPLATE migration
    2. Copy template to your editor
    3. Fill in the values
    4. Submit: Paste filled template in chat

    Template Example: Migration Request¶

    Command¶

    P02 TEMPLATE migration

    Generated Template¶

    P02 REQUEST: Migration Planning
  SOURCE:
    database: [Database name]
    container: [Container name]
    schema_version: [e.g., "v1.0"]
    partition_key: [Current partition key]
  TARGET:
    database: [Database name]
    container: [Container name]
    schema_version: [e.g., "v2.0"]
    partition_key: [New partition key]
  DATA_VOLUME:
    record_count: [e.g., "50000"]
    total_size: [e.g., "2GB"]
  REQUIREMENTS:
    - downtime: [e.g., "Zero downtime" | "Max 1 hour"]
    - rollback: [YES | NO]
    - verification: [YES | NO]
    - backup: [YES | NO]
  TIMELINE: [e.g., "2 weeks"]

    Filled Example¶

    P02 REQUEST: Migration Planning
  SOURCE:
    database: eva-rag
    container: documents
    schema_version: v1.0
    partition_key: /tenant_id
  TARGET:
    database: eva-rag
    container: documents_hpk
    schema_version: v2.0
    partition_key: [/space_id, /tenant_id, /user_id]
  DATA_VOLUME:
    record_count: 50000
    total_size: 2GB
  REQUIREMENTS:
    - downtime: Zero downtime
    - rollback: YES
    - verification: YES
    - backup: YES
  TIMELINE: 2 weeks

    Output Template Example¶

    Command¶

    P02 TEMPLATE OUTPUT migration

    Generated Output Template¶

    P02 DELIVERABLE: Migration Package

FILE_STRUCTURE:
  deliverables/migration-[feature]/
   migrate_[feature].py
   rollback_[feature].py
   verify_[feature].py
   MIGRATION-GUIDE.md
   migration_config.yaml
   DUA-MIGRATION-[DATE].md
   tests/
       test_migration.py

DOCUMENTATION:
  - MIGRATION-GUIDE.md: Step-by-step procedures
  - migration_config.yaml: Configuration settings
  - DUA-MIGRATION-[DATE].md: Archive document

SCRIPTS:
  - migrate_[feature].py: Main migration script with dry-run
  - rollback_[feature].py: Emergency rollback
  - verify_[feature].py: Post-migration validation

QUALITY_CHECKS:
   Dry-run mode implemented
   Rollback tested
   Data integrity verified (checksums)
   Unit tests written (>80% coverage)
   Documentation complete

    Template Validation¶

    Before submitting a filled template, validate it:

    # Validate template format
validate-agent-template migration_request.yaml

# Check required fields
validate-agent-template --check-required migration_request.yaml

    Validation Checks: - All required fields filled - Values match expected types - Constraints satisfied - Enum values valid - References exist

    Creating New Templates¶

    When adding a new agent service, create templates:

    1. Input Template¶

    # templates/[agent-id]/[service-name]-input.yaml
[AGENT_ID] REQUEST: [Service Name]
  [SECTION_1]:
    field1: [description or example]
    field2: [description or example]
  [SECTION_2]:
    - [item 1]
    - [item 2]

    2. Output Template¶

    # templates/[agent-id]/[service-name]-output.yaml
[AGENT_ID] DELIVERABLE: [Package Name]

FILE_STRUCTURE: |
  [directory tree]

DOCUMENTATION:
  - [file]: [description]

QUALITY_CHECKS:
   [check 1]
   [check 2]

    3. Register Template¶

    # templates/registry.yaml
templates:
  - agent_id: P02
    service: migration
    input_template: templates/P02/migration-input.yaml
    output_template: templates/P02/migration-output.yaml
    version: 1.0.0

    Template Benefits¶

    For Users (Marco)¶

    • Clarity: Know exactly what to provide
    • Completeness: Don't forget critical details
    • Speed: Copy-paste template, fill values
    • Consistency: Same format every time

    For Agents¶

    • Structure: Receive well-organized requests
    • Validation: Check required fields upfront
    • Efficiency: Less back-and-forth for clarifications
    • Quality: Standardized deliverable format

    Best Practices¶

    Filling Templates¶

    1. Don't skip fields - Use "N/A" if not applicable
    2. Be specific - "High performance" "< 50ms p95 latency"
    3. Include examples - Add sample data when possible
    4. Link context - Reference related PRs, docs, issues

    Using Templates¶

    1. Start simple - Use simple request format for quick tasks
    2. Use templates for complex work - Migrations, architecture, etc.
    3. Customize as needed - Templates are starting points
    4. Validate before submit - Check required fields
