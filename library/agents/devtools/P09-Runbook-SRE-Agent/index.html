    Pattern P09: Runbook & SRE Agent¶

    Status: Draft Owner: POD-O (Operations) + SRE Version: 1.0.0 Last Updated: 2025-12-12

    Overview¶

    The Runbook & SRE Agent drafts and updates operational runbooks from incidents, logs, and alerts to improve incident response and reduce mean time to recovery (MTTR).

    Purpose¶

    • Generate runbooks from incident postmortems
    • Update existing runbooks with new learnings
    • Document alert response procedures
    • Reduce MTTR through standardized playbooks

    Incident-to-Runbook Automation¶

    Incident Analysis¶

    Script: tools/runbooks/analyze-incident.ps1

    # Analyze incident and generate runbook draft
param(
    [Parameter(Mandatory)]
    [string]$IncidentId,

    [string]$OutputPath = "runbooks/incidents"
)

Write-Host "ðŸ“‹ Analyzing incident: $IncidentId" -ForegroundColor Cyan

# Fetch incident details from GitHub Issues
$issue = gh issue view $IncidentId --json title,body,labels,comments --jq .

if (!$issue) {
    Write-Error "Incident not found: $IncidentId"
    exit 1
}

$issueData = $issue | ConvertFrom-Json

# Parse incident timeline
$timeline = @()
$issueData.comments | ForEach-Object {
    if ($_.body -match "T\+(\d+)") {
        $timeline += @{
            offset = $Matches[1]
            event = $_.body
            author = $_.author.login
            timestamp = $_.createdAt
        }
    }
}

# Extract key information
$title = $issueData.title -replace "INCIDENT-\d+:\s*", ""
$severity = ($issueData.labels | Where-Object { $_ -match "priority-" })[0] -replace "priority-", ""
$rootCause = $issueData.body | Select-String -Pattern "Root Cause:(.*?)##" | ForEach-Object { $_.Matches.Groups[1].Value.Trim() }
$resolution = $issueData.body | Select-String -Pattern "Resolution:(.*?)##" | ForEach-Object { $_.Matches.Groups[1].Value.Trim() }

# Generate runbook
$runbookPath = "$OutputPath/incident-$IncidentId-runbook.md"
New-Item -Path $runbookPath -ItemType File -Force | Out-Null

@"
# Runbook: $title

**Generated from**: Incident #$IncidentId
**Severity**: $severity
**Date**: $(Get-Date -Format "yyyy-MM-dd")

## Incident Overview

$($issueData.body)

## Symptoms

- Alert triggered in monitoring system
- Error rate spike detected
- User reports of service degradation

## Root Cause

$rootCause

## Resolution Steps

$resolution

## Detailed Playbook

### 1. Initial Triage

**Estimated Time**: 5 minutes

\`\`\`bash
# Check service health
curl https://<app-name>.azurewebsites.net/health

# Query recent errors
az monitor app-insights query \
  --app <app-insights-id> \
  --analytics-query "traces | where timestamp > ago(1h) | where severityLevel >= 3"
\`\`\`

**Expected Output**:
- Health endpoint returns 200 OK or error details
- Recent error logs show root cause pattern

### 2. Identify Impact Scope

**Estimated Time**: 5 minutes

\`\`\`bash
# Check error rate
az monitor app-insights query \
  --app <app-insights-id> \
  --analytics-query "requests | where timestamp > ago(1h) | summarize ErrorRate = countif(success == false) * 100.0 / count()"

# Identify affected users
az monitor app-insights query \
  --app <app-insights-id> \
  --analytics-query "requests | where success == false | summarize Users = dcount(user_Id)"
\`\`\`

**Expected Output**:
- Error rate percentage
- Count of unique affected users

### 3. Apply Mitigation

**Estimated Time**: 10 minutes

\`\`\`bash
# Rollback to last known good version (if deployment-related)
az webapp deployment slot swap \
  --name <app-name> \
  --resource-group <rg-name> \
  --slot staging

# OR restart service (if transient issue)
az webapp restart \
  --name <app-name> \
  --resource-group <rg-name>
\`\`\`

**Verification**:
- Error rate returns to baseline (<1%)
- Health endpoint returns 200 OK

### 4. Post-Mitigation Validation

**Estimated Time**: 10 minutes

\`\`\`bash
# Run smoke tests
curl -X POST https://<app-name>.azurewebsites.net/api/smoke-test

# Monitor error rate for 10 minutes
watch -n 60 'az monitor app-insights query --app <app-insights-id> --analytics-query "requests | where timestamp > ago(5m) | summarize ErrorRate = countif(success == false) * 100.0 / count()"'
\`\`\`

**Success Criteria**:
- Smoke tests pass
- Error rate stable at <1% for 10+ minutes

    # Alert: {{ALERT_NAME}}

**Severity**: {{SEVERITY}}
**Owner**: {{TEAM}}
**Last Updated**: {{DATE}}

## Alert Definition

**Trigger Condition**: {{CONDITION}}

Example:
```kusto
requests
| where timestamp > ago(5m)
| summarize ErrorRate = countif(success == false) * 100.0 / count()
| where ErrorRate > 5

    What This Alert Means¶

    {{DESCRIPTION}}

    Example: "This alert fires when the error rate exceeds 5% over a 5-minute window, indicating a widespread service degradation affecting users."

    Immediate Actions (5 minutes)¶

    1. Verify Alert¶

    Purpose: Confirm alert is not a false positive

    # Check current error rate
az monitor app-insights query \
  --app {{APP_INSIGHTS_ID}} \
  --analytics-query "requests | where timestamp > ago(5m) | summarize ErrorRate = countif(success == false) * 100.0 / count()"

    Expected: Error rate > 5% confirms alert is valid

    2. Assess Impact¶

    Purpose: Determine scope of impact

    # Count affected users
az monitor app-insights query \
  --app {{APP_INSIGHTS_ID}} \
  --analytics-query "requests | where timestamp > ago(15m) | where success == false | summarize AffectedUsers = dcount(user_Id)"

# Identify error patterns
az monitor app-insights query \
  --app {{APP_INSIGHTS_ID}} \
  --analytics-query "exceptions | where timestamp > ago(15m) | summarize Count = count() by type, outerMessage | order by Count desc"

    3. Communicate¶

    Purpose: Notify stakeholders

    • Update status page (if user-facing)
    • Post in #incidents Slack channel
    • Page on-call engineer (if after hours)

    Root Cause Investigation (15 minutes)¶

    Common Causes¶

    1. Recent Deployment
    2. Check: Last deployment time vs alert trigger time

    3. Action: Consider rollback if within 1 hour

    4. Dependency Failure

    5. Check: Downstream service health

    6. Action: Escalate to dependent team

    7. Resource Exhaustion

    8. Check: CPU, memory, disk, connection pool metrics

    9. Action: Scale up or restart service

    10. Database Issues

    11. Check: Query performance, connection count
    12. Action: Restart connections or failover to replica

    Investigation Commands¶

    # Check recent deployments
az webapp deployment list \
  --name {{APP_NAME}} \
  --resource-group {{RG_NAME}} \
  --query "[0]" -o table

# Check resource metrics
az monitor metrics list \
  --resource {{RESOURCE_ID}} \
  --metric "CpuPercentage,MemoryPercentage" \
  --start-time $(date -u -d '1 hour ago' '+%Y-%m-%dT%H:%M:%SZ')

# Check database performance
az sql db show-connection-string \
  --client ado.net \
  --name {{DB_NAME}}

    Mitigation Steps¶

    Option 1: Rollback Deployment¶

    When to use: Recent deployment correlates with incident

    az webapp deployment slot swap \
  --name {{APP_NAME}} \
  --resource-group {{RG_NAME}} \
  --slot production \
  --target-slot previous

    Verification: Error rate drops to <1% within 5 minutes

    Option 2: Restart Service¶

    When to use: Resource exhaustion or transient issue

    az webapp restart \
  --name {{APP_NAME}} \
  --resource-group {{RG_NAME}}

    Verification: Service recovers within 2 minutes

    Option 3: Scale Out¶

    When to use: High load causing degradation

    az appservice plan update \
  --name {{PLAN_NAME}} \
  --resource-group {{RG_NAME}} \
  --number-of-workers 5

    Verification: Error rate decreases as load distributes

    Post-Incident¶

    Validation¶

    • Error rate returned to baseline (<1%)
    • No new errors in past 15 minutes
    • Smoke tests passing
    • User-facing features operational

    Documentation¶

    • Create incident ticket with timeline
    • Update this runbook if new learnings
    • Schedule postmortem (for P0/P1 incidents)

    Follow-up Actions¶

    • Review monitoring thresholds
    • Improve alerting to reduce noise
    • Implement permanent fix for root cause

    Escalation Path: 1. On-call engineer (primary) 2. Team lead (if >30 min) 3. Engineering manager (if P0/P1 >1 hour) 

    ## Runbook Generation from Logs

### Log Pattern Extraction

```python
# tools/runbooks/extract_patterns.py
"""
Extract common error patterns from Application Insights logs
to generate runbook troubleshooting steps
"""
import re
from collections import Counter
from typing import List, Dict

class LogPatternExtractor:
    def __init__(self, logs: List[str]):
        self.logs = logs

    def extract_error_patterns(self) -> Dict[str, int]:
        """Extract and count common error patterns"""
        patterns = []

        for log in self.logs:
            # Extract exception types
            if match := re.search(r'(\w+Exception|Error):', log):
                patterns.append(match.group(1))

            # Extract HTTP status codes
            if match := re.search(r'status[_\s]code[:\s]+(\d{3})', log, re.IGNORECASE):
                patterns.append(f"HTTP {match.group(1)}")

            # Extract common error keywords
            keywords = ['timeout', 'connection refused', 'not found', 'unauthorized', 'forbidden']
            for keyword in keywords:
                if keyword in log.lower():
                    patterns.append(keyword.title())

        return dict(Counter(patterns).most_common(10))

    def generate_troubleshooting_steps(self, patterns: Dict[str, int]) -> str:
        """Generate troubleshooting steps based on error patterns"""
        steps = "## Troubleshooting Steps\n\n"

        for pattern, count in patterns.items():
            if 'Exception' in pattern:
                steps += f"### {pattern} ({count} occurrences)\n\n"
                steps += f"**Possible Causes**:\n"
                steps += f"- Code bug in error handling\n"
                steps += f"- Invalid input data\n"
                steps += f"- Configuration error\n\n"
                steps += f"**Investigation**:\n"
                steps += f"```bash\n"
                steps += f"# Find full stack traces\n"
                steps += f'az monitor app-insights query --app <id> --analytics-query "exceptions | where type == \\'{pattern}\\' | take 5"\n'
                steps += f"```\n\n"

            elif 'HTTP 5' in pattern:
                steps += f"### {pattern} ({count} occurrences)\n\n"
                steps += f"**Possible Causes**:\n"
                steps += f"- Server-side error\n"
                steps += f"- Unhandled exception\n"
                steps += f"- Dependency failure\n\n"

            elif 'Timeout' in pattern:
                steps += f"### {pattern} ({count} occurrences)\n\n"
                steps += f"**Possible Causes**:\n"
                steps += f"- Slow database queries\n"
                steps += f"- Downstream service latency\n"
                steps += f"- Resource contention\n\n"

        return steps

# Example usage
if __name__ == "__main__":
    # Load logs from Application Insights
    logs = [
        "ValueError: Invalid configuration parameter",
        "HTTP status code: 500",
        "Connection timeout after 30 seconds",
        # ... more logs
    ]

    extractor = LogPatternExtractor(logs)
    patterns = extractor.extract_error_patterns()
    troubleshooting = extractor.generate_troubleshooting_steps(patterns)

    print(troubleshooting)
