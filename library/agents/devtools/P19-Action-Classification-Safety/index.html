    P19: Action Classification & Safety¶

    Pattern Code: P19
    Short Code: ACS
    Name: Action Classification & Safety
    Type: DevTools Pattern
    Scope: Safety & Autonomy Governance
    Maturity: Concept

    ðŸ“‹ What It Is¶

    P19 Action Classification & Safety (ACS) classifies actions by autonomy level (C0-C3) and enforces safety guardrails per the GC Agentic AI Framework. It provides:

    • Action classification: Label all agent actions with autonomy level (C0/C1/C2/C3)
    • Safety guardrails: Enforce appropriate controls for each autonomy level
    • Human oversight mapping: Define when human approval/monitoring/review is required
    • Risk assessment: Flag high-risk actions requiring additional validation
    • Compliance validation: Ensure actions meet GC Agentic AI Framework requirements

    P19 works with P16 (AWR Awareness) to make autonomy levels explicit and P21 (PRV Provenance) to audit autonomy decisions.

    ðŸŽ¯ Mission¶

    P19 Action Classification & Safety's mission:

    1. Classify all actions: Every DevTools agent action labeled C0/C1/C2/C3
    2. Enforce safety guardrails: Appropriate controls for autonomy level
    3. Enable human oversight: Clear triggers for human intervention
    4. Support ATO compliance: Evidence for governance panels
    5. Reduce AI risk: Prevent unsafe autonomous actions

    ðŸš€ Autonomy Levels (C0-C3)¶

    Per GC Agentic AI Framework:

    C0: Human-in-the-loop¶

    • Description: AI suggests, human approves every action
    • Human Role: Active decision-maker for each action
    • Examples:
    • P06 Review Agent (suggests changes, human merges)
    • SP02 ATO Auditor (generates report, human decides)
    • P02 Requirements Engine (suggests user stories, PO approves)
    • Safety Guardrails:
    • Human approval required before execution
    • Clear explanation of proposed action
    • Explicit accept/reject mechanism
    • No time-based auto-approval

    C1: Human-on-the-loop¶

    • Description: AI acts autonomously, human monitors and can intervene
    • Human Role: Monitor, can pause/stop/override
    • Examples:
    • P04 Librarian (auto-updates docs, human reviews)
    • P10 Metrics Agent (auto-generates reports)
    • P15 Dev Master Orchestrator (auto-coordinates agents, human can override)
    • Safety Guardrails:
    • Real-time monitoring dashboard
    • Clear pause/stop mechanism
    • Alert on anomalous behavior
    • Human review within 24 hours

    C2: Human-over-the-loop¶

    • Description: AI acts autonomously with constraints, human reviews periodically
    • Human Role: Review periodically (weekly/monthly), adjust constraints
    • Examples:
    • P07 Testing Agent (generates tests, runs in CI)
    • SP01 Data Ingestion (auto-processes approved sources)
    • P08 CI/CD Guardian (auto-optimizes pipelines)
    • Safety Guardrails:
    • Strict constraints (approved sources, safe operations)
    • Periodic human review (weekly/monthly)
    • Automatic rollback on failure
    • Audit trail for all actions (P21)

    C3: Full autonomy¶

    • Description: AI acts fully autonomously within well-defined boundaries
    • Human Role: Set boundaries, review outcomes quarterly
    • Examples:
    • Not yet implemented in EVA Suite
    • Requires extensive safety validation
    • Requires governance panel approval
    • Safety Guardrails:
    • Extensive safety validation required
    • Governance panel approval mandatory
    • Continuous monitoring with automatic kill switch
    • Quarterly human review of outcomes

    1. Action Classification¶

    Classification Criteria: - Human approval required? â†’ C0 - Human monitoring real-time? â†’ C1 - Human reviews periodically? â†’ C2 - Human sets boundaries only? â†’ C3

    Classification Outputs: - Autonomy level (C0/C1/C2/C3) - Rationale (why this level?) - Safety guardrails required - Human oversight requirements - Required documentation (PIA, AIIA, risk register)

    2. Safety Guardrails¶

    C0 Guardrails: - Human approval UI/CLI - Explanation of proposed action - Accept/reject mechanism - No auto-approval

    C1 Guardrails: - Real-time monitoring dashboard - Pause/stop mechanism - Alert on anomalies - 24-hour human review

    C2 Guardrails: - Strict constraints (approved sources, safe operations) - Automatic rollback on failure - Periodic human review (weekly/monthly) - Audit trail (P21)

    C3 Guardrails: - Extensive safety validation - Governance panel approval - Continuous monitoring - Automatic kill switch - Quarterly human review

    3. Compliance Validation¶

    GC Agentic AI Framework Requirements: - C0: No special requirements - C1: Risk assessment, monitoring dashboard - C2: AIIA required, periodic review cadence - C3: AIIA required, governance panel approval, extensive safety validation

    Validation Checks: - âœ… PIA exists (if processes personal data) - âœ… AIIA exists (if C2/C3 autonomy) - âœ… Risk register updated - âœ… Human oversight documented - âœ… Safety guardrails implemented
