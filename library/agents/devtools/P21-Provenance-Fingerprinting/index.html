    P21: Provenance & Fingerprinting¶

    Pattern Code: P21
    Short Code: PRV
    Name: Provenance & Fingerprinting
    Type: DevTools Pattern
    Scope: Traceability & Audit
    Maturity: Concept

    ðŸ“‹ What It Is¶

    P21 Provenance & Fingerprinting (PRV) captures provenance and fingerprints for all actions to support audits and ATO (Authority to Operate). It provides:

    • Provenance fingerprints: Who did what, when, why, with what inputs/outputs
    • Audit trail: 7-year retention for Government of Canada compliance
    • ATO evidence: Support for governance panels and external audits
    • Action traceability: Link actions to users, requirements, approvals
    • Cost attribution: Track costs by user, project, domain

    P21 works with SP02 (ATO Auditor) to validate compliance and P19 (ACS Action Classification) to capture autonomy levels.

    ðŸŽ¯ Mission¶

    P21 Provenance & Fingerprinting's mission:

    1. Capture all actions: Every significant action gets a provenance fingerprint
    2. Enable audits: Provide evidence for internal/external audits
    3. Support ATO: Supply governance panels with compliance evidence
    4. Enable traceability: Link actions to requirements, approvals, costs
    5. Enforce retention: 7-year retention per GC requirements

    2. Query Provenance by User/Project/Domain¶

    Prompt: "P21 â€“ PRV, query provenance for [user | project | domain]"

    Example Query: 

    -- Find all data ingestion actions by Data Steward in Q4 2025
SELECT *
FROM provenance
WHERE user_role = 'Data Steward'
  AND action_type = 'data_ingestion'
  AND timestamp >= '2025-10-01'
  AND timestamp <= '2025-12-31'
ORDER BY timestamp DESC

    Result: List of provenance fingerprints with action details

    ðŸ“¦ Key Capabilities (Concept)¶

    1. Action Capture¶

    Captured Actions: - Data ingestion (SP01) - API calls (eva-api, Azure OpenAI, Azure AI Search) - Deployments (P08, P15) - Configuration changes (eva-infra, Azure Policy) - Security events (P11) - Compliance audits (SP02)

    Capture Method: - Middleware in eva-api (auto-capture all API calls) - Event hooks in ingestion pipelines (SP01) - Deployment scripts (P08, P15) - Azure Monitor integration (infrastructure changes)

    2. Audit Trail¶

    Storage: - Cosmos DB container: provenance - Partition key: user_id - Time-to-live: 7 years (2557 days) - Backup: Geo-redundant (Canada Central + Canada East)

    Retention Policy: - 7 years per Government of Canada requirements - Automatic purge after 7 years - Export to cold storage if longer retention needed

    3. ATO Evidence¶

    Evidence Types: - Action inventory (who did what, when) - Autonomy level distribution (C0/C1/C2/C3) - Data residency compliance (100% Canada) - Cost attribution (by user/project/service) - Non-compliance events (and resolutions)

    Evidence Reports: - On-demand generation via SP02 (ATO Auditor) - Quarterly reports for governance panels - Annual reports for external audits (SSC, TBS, OAG)

    4. Cost Attribution¶

    Attribution Dimensions: - User (who triggered the action) - Project (which project/domain) - Service (which Azure service) - Pod (which pod owns the action) - Autonomy level (C0/C1/C2/C3)

    Use Cases: - Chargeback (bill projects for their usage) - Cost optimization (identify expensive actions) - Budget planning (forecast costs by project)

    ðŸ”— Related Patterns¶

    Works With: - SP02 (ATO Auditor) - Uses provenance for ATO evidence - P19 (ACS) - Captures autonomy level in fingerprints - P16 (AWR) - Awareness blocks included in fingerprints - P11 (SEC) - Security events generate fingerprints - P10 (MET) - Metrics derived from provenance data

    Feeds: - P20 (CSI) - Continuous Self-Improvement (learns from provenance) - SP01 (Data Ingestion) - Ingestion reports reference provenance

    ðŸ‘¥ Primary Users¶

    • SP02 (ATO Auditor): Query provenance for ATO evidence
    • Security SMEs: Investigate security events
    • Platform Owner: Cost attribution and budget planning
    • External Auditors: Review compliance evidence (SSC, TBS, OAG)

    ðŸŽ¯ Autonomy Level¶

    P21 Itself: C1 (Human-on-loop) - Captures fingerprints automatically - Human reviews fingerprints periodically (weekly/monthly) - Human generates ATO evidence reports on-demand

    ðŸ“Š Fingerprint Retention & Privacy¶

    Retention Period¶

    • Default: 7 years (Government of Canada requirement)
    • Configurable: Can be extended for specific action types
    • Automatic purge: After retention period expires

    Privacy & Data Protection¶

    • Classification: Protected B or lower
    • Data residency: Canada Central + Canada East only
    • Encryption: At rest (CMK) and in transit (TLS 1.2+)
    • Access control: RBAC enforced (Security SMEs, Auditors only)
    • PII handling: No PII in fingerprints (only user IDs, not names)

    Compliance¶

    • TBS Policy on Service and Digital: 7-year retention
    • SSC Cloud Security Controls: Audit trail required
    • GC Agentic AI Framework: Provenance required for C2/C3 autonomy
