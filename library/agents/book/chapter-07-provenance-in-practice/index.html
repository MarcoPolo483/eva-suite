    Chapter 07: Provenance in Practice¶

    Focus: Real-world P21 Implementation & ATO Compliance Scope: Production audit trails, cost attribution, 7-year retention Audience: Auditors, Security Teams, FinOps, DevOps Maturity: Stable (Production-ready patterns)

    üìñ Chapter Overview¶

    This chapter demonstrates how P21 Provenance & Fingerprinting works in production across real EVA Suite operations. You'll see actual provenance chains, ATO evidence generation workflows, cost attribution using fingerprints, 7-year audit trail implementation, and compliance reporting automation.

    While Chapter P21 explained the theory of provenance, this chapter shows the practice‚Äîreal fingerprint IDs, actual audit queries, working Azure configurations, and production compliance reports that satisfy SSC, TBS, and OAG auditors.

    By the end of this chapter, you'll understand: - How to generate and validate provenance fingerprints - How to trace a production deployment from requirement to outcome - How to attribute Azure costs to specific products/features - How to query the 7-year audit trail for compliance evidence - How to automate ATO reporting for quarterly reviews

    This is the hands-on operations guide for P21 implementation.

    Cost Attribution Query¶

    -- Azure Cost Management query via Azure Resource Graph
SELECT
  r.tags.product,
  r.tags.feature,
  r.tags.tier,
  r.type,
  SUM(c.cost) as total_cost_cad,
  COUNT(r.id) as resource_count
FROM resources r
JOIN costs c ON r.id = c.resourceId
WHERE c.date >= '2025-11-01' AND c.date < '2025-12-01'
  AND r.tags.product = 'eva-da-2'
GROUP BY
  r.tags.product,
  r.tags.feature,
  r.tags.tier,
  r.type
ORDER BY total_cost_cad DESC

    Archive Retrieval Workflow¶

    Scenario: Auditor requests evidence from 5 years ago (2020 deployment).

    Process: 1. Query Cosmos DB (fails‚Äîdata aged out to Archive tier) 2. Query Azure Blob Archive tier (12-hour retrieval SLA) 3. Azure Function rehydrates blob to Hot tier 4. Return JSON provenance fingerprints to auditor 5. Re-archive blob after 7 days

    Code Example: 

    # Azure Function: Retrieve archived provenance
import azure.functions as func
from azure.storage.blob import BlobServiceClient
import json

def retrieve_archived_provenance(fingerprint_id: str) -> dict:
    """
    Retrieve provenance fingerprint from Archive tier.
    SLA: 12 hours for rehydration.
    """
    blob_service = BlobServiceClient.from_connection_string(
        os.environ["AZURE_STORAGE_CONNECTION_STRING"]
    )

    container_client = blob_service.get_container_client("provenance-archive")

    # Archive blob naming: YYYY/MM/fingerprint_id.json
    blob_name = f"2020/12/{fingerprint_id}.json"
    blob_client = container_client.get_blob_client(blob_name)

    # Rehydrate blob (change tier from Archive to Hot)
    blob_client.set_standard_blob_tier("Hot")

    # Wait for rehydration (async, typically 12 hours)
    # Return job ID to auditor for polling
    return {
        "status": "rehydrating",
        "fingerprint_id": fingerprint_id,
        "estimated_completion": "2025-12-13T10:00:00Z",
        "poll_url": f"/api/provenance/status/{fingerprint_id}"
    }

    Automated ATO Report Generation¶

    Azure Function: Generate quarterly ATO evidence package.

    # Azure Function: Generate ATO Evidence Package
import azure.functions as func
from azure.cosmos import CosmosClient
from datetime import datetime, timedelta
import json

def generate_ato_evidence_package(
    ato_number: str,
    quarter: str,
    year: int
) -> dict:
    """
    Generate ATO evidence package for quarterly review.
    Queries P21 provenance database for compliance evidence.
    """
    cosmos_client = CosmosClient.from_connection_string(
        os.environ["COSMOS_CONNECTION_STRING"]
    )
    db = cosmos_client.get_database_client("provenance")
    container = db.get_container_client("fingerprints")

    # Calculate quarter date range
    quarter_start, quarter_end = get_quarter_dates(quarter, year)

    # Query 1: All Production deployments in quarter
    deployments = list(container.query_items(
        query="""
        SELECT c.id, c.metadata.created, c.who.approvers, c.compliance
        FROM c
        WHERE c.metadata.environment = 'production'
          AND c.compliance.ato_number = @ato_number
          AND c.metadata.created >= @start_date
          AND c.metadata.created < @end_date
        """,
        parameters=[
            {"name": "@ato_number", "value": ato_number},
            {"name": "@start_date", "value": quarter_start.isoformat()},
            {"name": "@end_date", "value": quarter_end.isoformat()}
        ]
    ))

    # Query 2: Security control evidence
    security_events = list(container.query_items(
        query="""
        SELECT c.id, c.metadata.action_type, c.security_validation
        FROM c
        WHERE c.metadata.action_type IN ('security_scan', 'penetration_test', 'vulnerability_assessment')
          AND c.metadata.created >= @start_date
          AND c.metadata.created < @end_date
        """,
        parameters=[
            {"name": "@start_date", "value": quarter_start.isoformat()},
            {"name": "@end_date", "value": quarter_end.isoformat()}
        ]
    ))

    # Query 3: Access control audit (Azure AD logs)
    access_control_events = query_azure_ad_logs(quarter_start, quarter_end)

    # Query 4: Encryption compliance (Key Vault logs)
    encryption_events = query_key_vault_logs(quarter_start, quarter_end)

    # Generate evidence package
    evidence_package = {
        "ato_number": ato_number,
        "quarter": f"Q{quarter} {year}",
        "generated_date": datetime.utcnow().isoformat(),
        "generated_by": "P21-PRO (Provenance Agent)",

        "summary": {
            "total_deployments": len(deployments),
            "security_scans": len([e for e in security_events if e["metadata"]["action_type"] == "security_scan"]),
            "penetration_tests": len([e for e in security_events if e["metadata"]["action_type"] == "penetration_test"]),
            "access_control_events": len(access_control_events),
            "encryption_events": len(encryption_events)
        },

        "controls": {
            "AC-2": {
                "status": "PASS" if verify_ac2_compliance(access_control_events) else "FAIL",
                "evidence_count": len(access_control_events)
            },
            "AU-2": {
                "status": "PASS",
                "evidence_count": len(deployments) + len(security_events)
            },
            "SC-8": {
                "status": "PASS" if verify_sc8_compliance(deployments) else "FAIL",
                "evidence_count": len(deployments)
            },
            "SC-28": {
                "status": "PASS" if verify_sc28_compliance(encryption_events) else "FAIL",
                "evidence_count": len(encryption_events)
            },
            "SI-10": {
                "status": "PASS" if verify_si10_compliance(security_events) else "FAIL",
                "evidence_count": len([e for e in security_events if "OWASP" in str(e)])
            }
        },

        "artifacts": {
            "deployments": deployments,
            "security_events": security_events,
            "access_control_events": access_control_events[:100],  # Truncate for size
            "encryption_events": encryption_events[:100]
        }
    }

    # Upload to Azure Blob Storage for auditor access
    upload_to_blob_storage(
        container_name="ato-evidence",
        blob_name=f"{ato_number}/Q{quarter}-{year}/evidence-package.json",
        data=json.dumps(evidence_package, indent=2)
    )

    return evidence_package

def verify_ac2_compliance(events: list) -> bool:
    """Verify AC-2 (Account Management) compliance."""
    # Check: 100% MFA enforcement, no shared accounts
    for event in events:
        if not event.get("mfa_enforced"):
            return False
        if event.get("account_type") == "shared":
            return False
    return True

def verify_sc8_compliance(deployments: list) -> bool:
    """Verify SC-8 (Transmission Confidentiality) compliance."""
    # Check: TLS 1.2+ on all deployments
    for deployment in deployments:
        if not deployment.get("compliance", {}).get("tls_version", "").startswith("1.2"):
            return False
    return True

def verify_sc28_compliance(events: list) -> bool:
    """Verify SC-28 (Protection at Rest) compliance."""
    # Check: All storage encrypted with CMK
    for event in events:
        if not event.get("cmk_encrypted"):
            return False
    return True

def verify_si10_compliance(events: list) -> bool:
    """Verify SI-10 (Information Input Validation) compliance."""
    # Check: Zero critical/high vulnerabilities in OWASP scans
    for event in events:
        if event.get("critical_vulnerabilities", 0) > 0:
            return False
        if event.get("high_vulnerabilities", 0) > 0:
            return False
    return True

    üéì Key Takeaways¶

    Provenance Chains Enable Traceability¶

    • Every Production deployment traces back to originating requirement (CDD)
    • Multi-layer approval chains (PO ‚Üí Security ‚Üí ATO Authority) clearly documented
    • Cryptographic fingerprints prevent tampering

    Cost Attribution Requires Discipline¶

    • Tag ALL Azure resources with product/feature/tier/fingerprint
    • Use Azure Cost Management queries to track actual vs budget
    • Automate variance reporting to catch overruns early

    7-Year Retention is Non-Negotiable¶

    • GC ATO requirements mandate 7-year audit trail
    • Use tiered storage (Hot ‚Üí Cool ‚Üí Archive) to optimize costs
    • Immutable blob storage prevents accidental deletion

    Automation Reduces Audit Burden¶

    • Quarterly ATO evidence packages generated automatically
    • Compliance queries pre-built and validated
    • Auditors get JSON packages instead of manual evidence gathering

    P21 is the Foundation of Governance¶

    • Without provenance, AI decisions are opaque
    • With provenance, every action is accountable and explainable
    • Enables EVA to achieve and maintain ATO compliance
