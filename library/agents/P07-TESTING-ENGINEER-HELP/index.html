    Agent Service Catalog - P07 (HELP)¶

    Agent ID: P07 Agent Name: Testing Engineer
    POD: POD-Q (Quality) Status: ACTIVE Version: 1.0.0

    Service Overview¶

    P07 is the Testing Engineer agent responsible for writing tests, implementing test automation, achieving code coverage targets, and ensuring test infrastructure reliability across the EVA Suite.

    Core Services¶

    1. Unit Test Development¶

    • Description: Write unit tests for functions, classes, and modules
    • Input: Code to test, coverage targets, edge cases
    • Output: Unit tests with ≥80% coverage, test documentation
    • Example: P07: Write unit tests for Cosmos DB service methods in eva-api

    Input Template: 

    P07 REQUEST: Unit Tests
  MODULE: [Module/file path]
  FUNCTIONS:
    - [Function 1 to test]
    - [Function 2 to test]
  COVERAGE_TARGET: [e.g., "80%"]
  EDGE_CASES:
    - [Case 1, e.g., "Empty input"]
    - [Case 2, e.g., "Large dataset"]
  MOCKING:
    - [Service 1 to mock, e.g., "Azure Cosmos DB"]
    - [Service 2 to mock, e.g., "Azure OpenAI"]

    2. Integration Test Implementation¶

    • Description: Write integration tests for multi-component workflows
    • Input: Integration points, data flows, test environments
    • Output: Integration tests, test data, environment setup
    • Example: P07: Create integration tests for eva-api document upload workflow

    Input Template: 

    P07 REQUEST: Integration Tests
  WORKFLOW: [Workflow name]
  COMPONENTS:
    - [Component 1, e.g., "API endpoint"]
    - [Component 2, e.g., "Blob Storage"]
    - [Component 3, e.g., "Cosmos DB"]
  TEST_SCENARIOS:
    - [Scenario 1, e.g., "Upload document → Store blob → Create metadata"]
    - [Scenario 2, e.g., "Error handling when storage fails"]
  ENVIRONMENT: [Dev | Test | CI/CD]

    3. Test Coverage Improvement¶

    • Description: Increase test coverage for modules below target
    • Input: Current coverage report, target coverage, priority modules
    • Output: New tests, coverage delta report, improvement plan
    • Example: P07: Increase eva-api test coverage from 52.92% to 80%

    Input Template: 

    P07 REQUEST: Coverage Improvement
  REPOSITORY: [Repo name]
  CURRENT_COVERAGE: [e.g., "52.92%"]
  TARGET_COVERAGE: [e.g., "80%"]
  PRIORITY_MODULES:
    - [Module 1 with low coverage]
    - [Module 2 with low coverage]
  TIMELINE: [e.g., "2 weeks"]
  APPROACH: [Quick wins first | Systematic | Highest risk]

    4. Test Automation & CI/CD Integration¶

    • Description: Automate test execution in CI/CD pipelines
    • Input: Test suite, CI/CD platform, quality gates
    • Output: Automated test pipeline, quality gate config, reports
    • Example: P07: Integrate pytest with GitHub Actions for eva-api with coverage reporting

    Input Template: 

    P07 REQUEST: Test Automation
  TEST_SUITE: [Test suite name]
  CI_CD_PLATFORM: [GitHub Actions | Azure Pipelines]
  TRIGGERS:
    - [On push to main]
    - [On pull request]
  QUALITY_GATES:
    - min_coverage: [e.g., "80%"]
    - max_failures: [e.g., "0"]
  REPORTING:
    - [Coverage report to Codecov]
    - [Test results to GitHub]

    5. Test Fixture & Mock Development¶

    • Description: Create reusable test fixtures and mocks
    • Input: Services to mock, test data requirements
    • Output: Pytest fixtures, mock implementations, test utilities
    • Example: P07: Create pytest fixtures for Azure services (Cosmos, Blob, OpenAI)

    Input Template: 

    P07 REQUEST: Test Fixtures
  SERVICES_TO_MOCK:
    - service: [e.g., "Azure Cosmos DB"]
      operations: [List, Create, Update, Delete]
    - service: [e.g., "Azure Blob Storage"]
      operations: [Upload, Download, Delete]
  TEST_DATA:
    - [Type 1, e.g., "Sample documents"]
    - [Type 2, e.g., "User credentials"]
  SCOPE: [Module | Repository-wide]

    Service Dependencies¶

    Upstream (P07 depends on): - P01 (System Architect): Architecture to understand integration points - P02 (Data Architect): Data models for test data - P03 (Security Architect): Security test requirements - P04 (Library Manager): Test dependencies, pytest plugins

    Downstream (Depends on P07): - P09 (QA Engineer): Tests validated by QA before acceptance - P11 (DevOps): Tests integrated into CI/CD - SM (Scrum Master): Test completion tracked in sprint

    Service Standards¶

    Test Coverage Standards¶

    • Unit test coverage: ≥80% for all modules
    • Critical path coverage: 100%
    • Public API coverage: 100%
    • Test-to-code ratio: 1:1 or higher

    Test Quality Standards¶

    • Each test tests one thing (single responsibility)
    • Tests are deterministic (no flaky tests)
    • Tests run in <1 minute (unit), <5 minutes (integration)
    • Tests independent (can run in any order)

    Test Naming Standards¶

    def test_<function>_<scenario>_<expected_result>():
    # Example: test_create_document_with_valid_data_returns_201()
    pass

    Test Organization Standards¶

    tests/
├── unit/                    # Fast, isolated tests
│   ├── test_models.py
│   ├── test_services.py
│   └── test_utils.py
├── integration/             # Multi-component tests
│   ├── test_api_endpoints.py
│   └── test_database_operations.py
├── fixtures/                # Shared fixtures
│   ├── conftest.py
│   └── mock_services.py
└── data/                    # Test data
    └── sample_documents.json

    Common Patterns¶

    Pattern 1: AAA Test Structure¶

    def test_create_document_success():
    # Arrange - Set up test data and mocks
    mock_cosmos = Mock()
    service = CosmosService(mock_cosmos)
    document = {"id": "123", "content": "test"}

    # Act - Execute the function
    result = service.create_document(document)

    # Assert - Verify the outcome
    assert result.status_code == 201
    mock_cosmos.create_item.assert_called_once()

    Pattern 2: Pytest Fixtures¶

    # conftest.py
@pytest.fixture
def mock_cosmos_client():
    """Mock Azure Cosmos DB client"""
    mock = Mock(spec=CosmosClient)
    mock.get_database_client.return_value = Mock()
    return mock

@pytest.fixture
def sample_document():
    """Sample document for testing"""
    return {
        "id": "doc-123",
        "space_id": "space-1",
        "content": "Test content"
    }

# test_cosmos_service.py
def test_create_document(mock_cosmos_client, sample_document):
    service = CosmosService(mock_cosmos_client)
    result = service.create_document(sample_document)
    assert result.id == "doc-123"

    Pattern 3: Parametrized Tests¶

    @pytest.mark.parametrize("input_data,expected", [
    ({"name": "test"}, 201),
    ({"name": ""}, 400),      # Empty name
    ({}, 400),                # Missing name
    ({"name": "x" * 1000}, 400),  # Too long
])
def test_create_document_validation(input_data, expected):
    result = create_document(input_data)
    assert result.status_code == expected

    Quality Checklist¶

    Before marking P07 work complete:

    • Test coverage meets target (≥80%)
    • All tests pass (0 failures)
    • Tests follow AAA pattern
    • Test names descriptive and clear
    • Fixtures reusable and well-documented
    • Mocks properly configured
    • Edge cases covered
    • Integration tests validate workflows
    • Tests run in CI/CD pipeline
    • Coverage report generated and reviewed
