    EVA Artifacts Generator¶

    Feature ID: P02-ARTIFACTS-001
    Status: Requirements Complete (Implementation pending)
    Owner: Marco
    Sprint: Sprint 02
    Created: 2025-11-25

    What is this?¶

    The EVA Artifacts Generator is an intelligent documentation assistant that scans recent repository activity (via EVA-New-Artifacts-Map.md and EVA-Audit-Log.jsonl), identifies missing or incomplete documentation artifacts, and generates structured placeholders or full documentation using AI assistance. It acts as a "meta-feature" that helps maintain comprehensive, high-quality documentation across the EVA Suite ecosystem.

    Why does it exist?¶

    1. Documentation Debt Reduction: Automatically identifies files/features lacking proper documentation (README, P02 packages, CDDs, ADRs)
    2. P02 Bootstrap Acceleration: Helps launch new P02 feature packages by analyzing recent code changes and generating requirement stubs
    3. Executive Reporting: Summarizes recent EVA Suite activity for stakeholder updates and sprint reviews
    4. Knowledge Transfer: Ensures every significant artifact has accompanying documentation for team onboarding

    Key capabilities¶

    • ✅ Artifact Scanning: Parse EVA-New-Artifacts-Map.md (21-day window) and EVA-Audit-Log.jsonl
    • ✅ Gap Detection: Identify files missing documentation (scripts without README, features without P02 packages)
    • ⏳ Documentation Generation: Create structured stubs (README templates, P02 skeletons, CDD outlines)
    • ⏳ AI-Assisted Authoring: Use OpenAI to generate comprehensive docs based on code analysis
    • ⏳ P02 Integration: Log own OpenAI usage via p02-telemetry.ps1 (feature_key: P02-ARTIFACTS-001)
    • ⏳ Multi-Repo Support: Extend to all 23 EVA Suite repos (Phase 2)

    Quick start¶

    # Phase 1: Scan and report gaps
cd "C:\Users\marco\Documents\_AI Dev\EVA Suite\eva-orchestrator"
.\scripts\eva-artifacts-scan.ps1

# Example output:
# Found 127 files in last 21 days
# Missing documentation:
#   - scripts/eva-audit-bootstrap.ps1 (no README, no tests)
#   - docs/features/eva-safety/ (incomplete P02 package: missing tests.md, backlog.md)
#   - docs/EVA-2.0/CDDs/ (3 CDDs with no status tracking)

# Phase 2: Generate doc stubs
.\scripts\eva-artifacts-generate.ps1 -Target "scripts/eva-audit-bootstrap.ps1" -DocType "README"

# Output: scripts/README-eva-audit-bootstrap.md (template ready for review)

# Phase 3: AI-assisted P02 package creation
.\scripts\eva-artifacts-generate.ps1 -Target "docs/features/eva-safety" -DocType "P02-Package" -UseOpenAI

# Output: Full 10-file P02 package generated, logged via P02 telemetry

    Integration with EVA Audit Trail¶

    The EVA Artifacts Generator is a consumer of audit trail data:

    Audit Trail Output Generator Usage
    EVA-New-Artifacts-Map.md Primary data source: lists all files modified in last 21 days
    EVA-Audit-Log.jsonl Complementary source: provides exact timestamps, change types, file sizes
    EVA-Audit-State.json Optional: detect newly created files vs modified files

    The Generator also produces audit events via P02 telemetry:

    {
  "type": "p02-openai-usage",
  "project": "eva-orchestrator",
  "feature_key": "P02-ARTIFACTS-001",
  "run_id": "a1b2c3d4-...",
  "model": "gpt-4.1",
  "input_tokens": 8500,
  "output_tokens": 2400,
  "total_tokens": 10900,
  "cost_estimate": 0.381,
  "caller": "cli",
  "timestamp": "2025-11-25T14:30:00.000Z"
}

    For IT shops & research labs¶

    This pattern is designed to be generic and reusable:

    • Replace EVA-specific paths with your own repository structure
    • Customize templates to match your documentation standards
    • Adapt P02 integration to your requirements gathering process
    • Extend to other doc types: ADRs, RFCs, design docs, API specs

    The EVA Artifacts Generator demonstrates how AI-assisted documentation can scale without sacrificing quality or auditability.
