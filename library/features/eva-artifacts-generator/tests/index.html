    Phase 1 Tests (Sprint 02)¶

    T-001: Parse EVA-New-Artifacts-Map.md¶

    Requirement: FR-1
    Type: Unit Test
    Priority: HIGH

    Objective: Verify parser extracts file list from EVA-New-Artifacts-Map.md correctly

    Setup: 1. Create sample map file: test-artifacts-map.md (100 file entries) 2. Include directory headers: ## eva-orchestrator\scripts 3. Include file entries: - scripts/test-script.ps1 *(Last modified: 2025-11-25 14:30)* 4. Include edge cases: Files with spaces, special chars, no timestamp

    Execution steps: 1. Call Parse-ArtifactsMap -Path "test-artifacts-map.md" 2. Inspect returned array

    Expected results: - Returns array of 100 objects - Each object has properties: path, lastModified, directory - Timestamps parsed correctly: "2025-11-25 14:30" → DateTime object - Files with spaces parsed: "docs/My Feature/README.md" - Malformed lines skipped with warning logged

    Pass criteria: - [ ] All 100 files extracted - [ ] Timestamps accurate (±0 seconds) - [ ] Edge cases handled gracefully - [ ] Execution time <2 seconds

    Test data: Located in scripts/tests/fixtures/test-artifacts-map.md

    T-002: Parse EVA-Audit-Log.jsonl¶

    Requirement: FR-2
    Type: Unit Test
    Priority: MEDIUM

    Objective: Verify parser extracts events from EVA-Audit-Log.jsonl correctly

    Setup: 1. Create sample log file: test-audit-log.jsonl (500 events) 2. Include various event types: file-change, p02-openai-usage, scan-complete 3. Include date range: Last 30 days 4. Include corrupted line: Malformed JSON on line 250

    Execution steps: 1. Call Parse-AuditLog -Path "test-audit-log.jsonl" -Days 21 2. Inspect returned array

    Expected results: - Returns array of ~350 events (21 days out of 30) - Each object has properties: scanTime, filePath, changeType, sizeBytes - Corrupted line skipped with warning - Date filtering accurate

    Pass criteria: - [ ] Correct event count (±5 events for date boundary) - [ ] All properties present - [ ] Corrupted line handled gracefully - [ ] Execution time <5 seconds

    Test data: Located in scripts/tests/fixtures/test-audit-log.jsonl

    T-003: Detect MissingREADME gaps¶

    Requirement: FR-3
    Type: Integration Test
    Priority: HIGH

    Objective: Verify scanner detects scripts without README files

    Setup: 1. Create test directory structure: 

    test-repo/
  scripts/
    has-readme.ps1
    has-readme.README.md
    missing-readme.ps1  # No README
    2. Generate test artifacts map referencing these files

    Execution steps: 1. Call Detect-Gaps -Files $testFiles -Rules @("MissingREADME") 2. Inspect returned gaps array

    Expected results: - Gaps array contains 1 gap: - category: "MissingREADME" - filePath: "scripts/missing-readme.ps1" - severity: "medium" - recommendation: "Create README explaining script purpose..."

    Pass criteria: - [ ] Exactly 1 gap detected (no false positives) - [ ] Gap properties correct - [ ] has-readme.ps1 NOT flagged (has adjacent README)

    Test data: Located in scripts/tests/fixtures/test-repo/
