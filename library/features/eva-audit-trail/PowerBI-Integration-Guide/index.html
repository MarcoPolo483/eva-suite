    Power BI Integration Guide - P02 OpenAI Usage Analytics¶

    Created: 2025-11-25
    Purpose: Configure Power BI dashboards for EVA Suite P02 Requirements Engine usage analytics

    Overview¶

    The P02 Usage Report (scripts/p02-usage-report.ps1) exports Power BI-optimized CSV files that can be directly imported into Power BI Desktop for advanced analytics dashboards.

    Use Case: Analyze P02 OpenAI usage patterns, costs, and trends across EVA Suite projects (separate from EVA Suite LiveOps & FinOps dashboards which go through APIM).

    Data Source: P02 Usage Report CSV¶

    Generate CSV Export¶

    cd eva-orchestrator
.\scripts\p02-usage-report.ps1 -AsCsv

# With filters (optional)
.\scripts\p02-usage-report.ps1 -From (Get-Date).AddDays(-30) -AsCsv
.\scripts\p02-usage-report.ps1 -FeatureKey "P02-AUDIT-001" -AsCsv

    Output: docs\_audit\P02-Usage-Report.csv

    CSV Schema (Power BI Optimized)¶

    Column Type Description Power BI Use
    Timestamp DateTime Full timestamp (yyyy-MM-dd HHss) Time series analysis
    Date Date Date only (yyyy-MM-dd) Date-based filtering
    Time Time Time only (HHss) Time-of-day analysis
    Year Integer Year (2025) Year grouping
    Month Integer Month (1-12) Month grouping
    MonthName String Month name (January, February, etc.) Month display
    DayOfWeek String Day name (Monday, Tuesday, etc.) Day-of-week analysis
    Hour Integer Hour (0-23) Hour-of-day analysis
    Project String Repository name (eva-orchestrator) Project filtering
    FeatureKey String P02 feature ID (P02-AUDIT-001) Feature comparison
    RunId String Execution UUID Run correlation
    Model String OpenAI model (gpt-4.1, gpt-4.1-mini) Model comparison
    InputTokens Integer Prompt tokens Token metrics
    OutputTokens Integer Completion tokens Token metrics
    TotalTokens Integer Combined tokens Token metrics
    CostEstimate Decimal USD cost Cost metrics
    Caller String Execution mode (cli, manual, github-action) Usage pattern analysis

    Power BI Setup (Step-by-Step)¶

    1. Import CSV Data¶

    1. Open Power BI Desktop
    2. Get Data â†’ Text/CSV
    3. Navigate to: eva-orchestrator\docs\_audit\P02-Usage-Report.csv
    4. Click Open
    5. Power Query Editor opens with data preview
    6. Click Transform Data (recommended) or Load (skip transforms)

    2. Configure Data Types (Transform Data)¶

    Power BI usually auto-detects types correctly, but verify:

    Column Power BI Type
    Timestamp Date/Time
    Date Date
    Time Time
    Year, Month, Hour Whole Number
    MonthName, DayOfWeek Text
    Project, FeatureKey, RunId, Model, Caller Text
    InputTokens, OutputTokens, TotalTokens Whole Number
    CostEstimate Decimal Number (2-3 decimal places)

    Recommended transforms: - Set CostEstimate format to Currency (USD) with 3 decimal places - Set Date format to yyyy-MM-dd - Set Time format to HHss

    Click Close & Apply when done.

    3. Create Date Hierarchy (Optional)¶

    Power BI may auto-create a date hierarchy. If not:

    1. In Fields pane, right-click Date column
    2. Select New Hierarchy
    3. Drag Year, MonthName, Date into hierarchy
    4. Rename to Date Hierarchy

    Dashboard Visualizations¶

    Page 1: Executive Summary¶

    KPI Cards (4 cards at top): 1. Total Cost (USD): SUM of CostEstimate, format as Currency 2. Total Events: COUNT of RunId (or FeatureKey) 3. Total Tokens: SUM of TotalTokens, format as Thousands (K) 4. Avg Cost/Event: AVERAGE of CostEstimate, format as Currency

    Bar Chart (Feature Cost Comparison): - Axis: FeatureKey - Values: SUM of CostEstimate - Sort: Descending by cost - Data Labels: On, Currency format

    Line Chart (Cost Over Time): - Axis: Date (or Timestamp) - Values: SUM of CostEstimate - Legend: Model - Data Labels: Off (cluttered)

    Donut Chart (Cost by Model): - Legend: Model - Values: SUM of CostEstimate - Data Labels: Percentage + Currency

    Page 2: Token Usage Analysis¶

    Stacked Bar Chart (Tokens by Feature): - Axis: FeatureKey - Values: SUM of InputTokens, SUM of OutputTokens - Legend: Auto (Input vs Output) - Sort: Descending by TotalTokens

    Scatter Chart (Cost vs Tokens): - X-Axis: TotalTokens - Y-Axis: CostEstimate - Legend: Model - Size: (leave blank or use constant) - Tooltip: FeatureKey, RunId

    Table (Top 10 Most Expensive Runs): - Columns: Timestamp, FeatureKey, Model, TotalTokens, CostEstimate - Sort: Descending by CostEstimate - Top N Filter: 10 rows

    Page 3: Time-Based Patterns¶

    Matrix (Cost by Day of Week & Hour): - Rows: DayOfWeek (custom sort: Mon-Sun) - Columns: Hour (0-23) - Values: SUM of CostEstimate - Conditional Formatting: Heatmap (low=green, high=red)

    Column Chart (Events by Day of Week): - Axis: DayOfWeek - Values: COUNT of RunId - Sort: Custom (Mon, Tue, Wed, Thu, Fri, Sat, Sun)

    Line Chart (Events by Hour of Day): - Axis: Hour (0-23) - Values: COUNT of RunId - Data Labels: Off

    Slicer (Month Filter): - Field: MonthName - Style: Dropdown or List - Cross-filters: All visuals on page

    Page 4: Feature Deep Dive¶

    Slicer (Feature Selector): - Field: FeatureKey - Style: Dropdown (single-select recommended)

    KPI Cards (4 cards, filtered by slicer): 1. Feature Cost: SUM of CostEstimate (filtered) 2. Feature Events: COUNT of RunId (filtered) 3. Avg Tokens/Event: AVERAGE of TotalTokens (filtered) 4. Most Used Model: Mode of Model (filtered)

    Line Chart (Feature Cost Over Time): - Axis: Date - Values: SUM of CostEstimate - Filters: Apply FeatureKey slicer

    Table (Feature Run Details): - Columns: Timestamp, RunId, Model, TotalTokens, CostEstimate, Caller - Filters: Apply FeatureKey slicer - Sort: Descending by Timestamp

    Page 5: Model Efficiency Comparison¶

    Clustered Bar Chart (Cost per 1K Tokens by Model): - Axis: Model - Values: CALCULATE measure: SUM(CostEstimate) / (SUM(TotalTokens) / 1000) - Data Labels: On, 3 decimal places

    Formula for measure: 

    CostPer1KTokens = 
DIVIDE(
    SUM(P02_Usage[CostEstimate]),
    SUM(P02_Usage[TotalTokens]) / 1000,
    0
)

    Donut Chart (Token Distribution by Model): - Legend: Model - Values: SUM of TotalTokens - Data Labels: Percentage

    Line Chart (Model Usage Trend): - Axis: Date - Values: COUNT of RunId - Legend: Model

    DAX Measures (Recommended)¶

    Add these calculated measures in Power BI:

    1. Total Cost¶

    Total Cost = SUM(P02_Usage[CostEstimate])

    2. Total Events¶

    Total Events = COUNTROWS(P02_Usage)

    3. Total Tokens¶

    Total Tokens = SUM(P02_Usage[TotalTokens])

    4. Average Cost per Event¶

    Avg Cost per Event = 
DIVIDE(
    SUM(P02_Usage[CostEstimate]),
    COUNTROWS(P02_Usage),
    0
)

    5. Cost per 1K Tokens¶

    Cost per 1K Tokens = 
DIVIDE(
    SUM(P02_Usage[CostEstimate]),
    SUM(P02_Usage[TotalTokens]) / 1000,
    0
)

    6. Input/Output Token Ratio¶

    Input Output Ratio = 
DIVIDE(
    SUM(P02_Usage[InputTokens]),
    SUM(P02_Usage[OutputTokens]),
    0
)

    7. Most Expensive Feature¶

    Most Expensive Feature = 
CALCULATE(
    FIRSTNONBLANK(P02_Usage[FeatureKey], 1),
    TOPN(1, ALL(P02_Usage[FeatureKey]), [Total Cost], DESC)
)

    8. Weekend vs Weekday Cost¶

    Weekend Cost = 
CALCULATE(
    [Total Cost],
    P02_Usage[DayOfWeek] IN {"Saturday", "Sunday"}
)

Weekday Cost = 
CALCULATE(
    [Total Cost],
    NOT(P02_Usage[DayOfWeek] IN {"Saturday", "Sunday"})
)

    Refresh Strategy¶

    Option 1: Manual Refresh (Development)¶

    1. Run P02 usage report script: .\scripts\p02-usage-report.ps1 -AsCsv
    2. Open Power BI Desktop
    3. Home â†’ Refresh
    4. CSV data reloads automatically

    Use case: Ad-hoc analysis, testing, development

    Option 2: Scheduled Refresh (Production)¶

    Prerequisites: - Power BI Pro or Premium license - Publish report to Power BI Service (app.powerbi.com) - Configure data gateway (if CSV is on network drive)

    Steps: 1. Publish report: File â†’ Publish â†’ Select workspace 2. In Power BI Service, go to dataset Settings 3. Scheduled Refresh â†’ Configure frequency (daily, weekly) 4. Gateway Connection â†’ Configure if needed 5. Data Source Credentials â†’ Update file path if changed

    Automation: - Add GitHub Actions workflow to run p02-usage-report.ps1 -AsCsv nightly - Commit CSV to repo (or upload to Azure Blob Storage) - Power BI refreshes from updated CSV

    Option 3: Azure Data Lake Integration (Enterprise)¶

    For large-scale deployments:

    1. Export to Azure Blob: 

      .\scripts\p02-usage-report.ps1 -AsCsv
az storage blob upload --account-name evasuite --container-name p02-usage --file docs\_audit\P02-Usage-Report.csv

    2. Power BI connects to Azure Blob:

    3. Get Data â†’ Azure â†’ Azure Blob Storage
    4. Authenticate with Azure AD

    5. Select CSV file

    6. Scheduled refresh:

    7. GitHub Actions uploads CSV to Azure Blob nightly
    8. Power BI refreshes from Azure Blob (no gateway needed)

    Filters & Slicers (Recommended)¶

    Add these slicers to enable interactive filtering:

    1. Date Range Slicer: Date column (between style)
    2. Feature Slicer: FeatureKey (dropdown or list)
    3. Model Slicer: Model (list style with checkboxes)
    4. Caller Slicer: Caller (list style)
    5. Month Slicer: MonthName (dropdown)
    6. Day of Week Slicer: DayOfWeek (list style)

    Best practice: Place common slicers on first page, sync to all pages: - Right-click slicer â†’ Sync slicers â†’ Select pages to sync

    Troubleshooting¶

    Issue: CSV not updating in Power BI¶

    Solution: Refresh data source 1. Home â†’ Refresh 2. If still stale: Transform Data â†’ Data Source Settings â†’ Refresh Preview

    Issue: Date columns showing as text¶

    Solution: Change data type 1. Transform Data 2. Select Date column 3. Transform â†’ Data Type â†’ Date 4. Repeat for Timestamp (Date/Time) and Time (Time)

    Issue: DayOfWeek sorting alphabetically instead of Mon-Sun¶

    Solution: Create custom sort column 1. Transform Data â†’ Add Column â†’ Custom Column 2. Formula: 

    if [DayOfWeek] = "Monday" then 1
else if [DayOfWeek] = "Tuesday" then 2
else if [DayOfWeek] = "Wednesday" then 3
else if [DayOfWeek] = "Thursday" then 4
else if [DayOfWeek] = "Friday" then 5
else if [DayOfWeek] = "Saturday" then 6
else 7
    3. Name column: DayOfWeekSort 4. Modeling â†’ Select DayOfWeek â†’ Sort by Column â†’ DayOfWeekSort

    Issue: Cost values showing too many decimals¶

    Solution: Format as currency 1. Select CostEstimate column 2. Column Tools â†’ Format â†’ Currency (USD) 3. Decimal Places â†’ 2 or 3

    Data Governance¶

    Security¶

    • Row-level security (RLS): Filter by Project if multiple teams share dashboard
    • Sensitivity labels: Mark as "Internal" or "Confidential" in Power BI Service
    • Access control: Restrict dashboard to DevTools team, FinOps analysts

    Compliance¶

    • Data retention: CSV files in docs\_audit\ follow EVA Audit Trail retention policy (90 days default)
    • PII: No personal identifiable information in P02 usage logs (only system metadata)
    • Cost data: Estimated costs only, not actual Azure/OpenAI billing data

    Audit Trail¶

    • P02 usage report generation is itself logged to EVA Audit Trail (script executions)
    • Power BI refresh logs available in Power BI Service (Settings â†’ Refresh History)
