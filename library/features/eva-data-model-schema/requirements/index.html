    Accessibility (WCAG 2.1 AA Compliance)¶

    • Keyboard Navigation: All interactive elements accessible via Tab/Enter/Space/Arrow keys
    • Screen Reader Support: Proper ARIA labels, roles, live regions
    • Color Contrast: 4.5:1 minimum for normal text, 3:1 for large text
    • Focus Indicators: Visible focus states for all interactive elements
    • No Keyboard Traps: Users can navigate in/out of all UI sections

    Reference: WCAG 2.1 Guidelines

    Bilingual Support (EN-CA / FR-CA)¶

    • No Hardcoded Text: All UI text externalized via @lit/localize or EVA i18n service
    • Complete Translation: Both languages supported from day 1 (not "TODO: translate")
    • Dynamic Switching: Language selector functional without page reload
    • Date/Number Formatting: Locale-aware (EN-CA vs FR-CA conventions)
    • Right-to-Left Ready: Layout supports future RTL languages (if applicable)

    Reference: eva-i18n/README.md, Official Languages Act compliance

    GC Design System (canada-gc Theme)¶

    • Theme Only: Use canada-gc theme via EVA Sovereign UI, no hardcoded colors
    • Component Reuse: Check EVA-Sovereign-UI/packages/eva-sovereign-ui-wc/src/components/ FIRST
    • Spacing/Typography: Use theme tokens (--spacing-md, --font-body, etc.)
    • No Custom Styling: If theme doesn't support it, propose theme enhancement (don't override)
    • Responsive Design: Mobile-first, breakpoints at 640px/768px/1024px

    Reference: EVA-Sovereign-UI/docs/THEME-SYSTEM.md, GC Design System

    RBAC-Aware (Role-Based Access Control)¶

    • Server-Side Authorization: All authorization logic in backend (Azure Functions, EVA Foundation)
    • UI Reflects Permissions: Components show/hide based on RBACUserContext (read from API)
    • No UI-Level Security: UI hiding is UX only, not a security control
    • Graceful Degradation: If user lacks permission, show helpful message (not blank screen)
    • Audit Logging: All state-changing operations log to EVA Audit Trail

    Reference: eva-foundation/shared/security/, docs/EVA-2.0/rbac-architecture.md

    Protected B Data Handling¶

    • Encryption at Rest: All PII/sensitive data encrypted in Cosmos DB (automatic via Azure)
    • Encryption in Transit: HTTPS only, TLS 1.2+ (enforced by Azure App Service)
    • No Secrets in Code: Use Azure Key Vault for connection strings, API keys
    • Data Minimization: Only collect/store data required for feature functionality
    • Audit Trail: All data access logged to .eva-audit/EVA-Audit-Log.jsonl

    Reference: Treasury Board Directive on Security Management, ITSG-33

    Context Engineering Pattern¶

    • 4-Block Prompts: Use Context ‚Üí Task ‚Üí Output ‚Üí Self-check when generating code with Copilot
    • Spec Anchoring: Reference this requirements.md + architecture-notes.md in every prompt

    • Reuse First: Check feature-map.md for existing components before building new

    Reference: docs/context-engineering.md, docs/eva-patterns/P16-DMP.md
