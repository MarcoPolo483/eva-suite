    Sprint Backlog: Azure API Management Deployment¶

    Feature: P02-APIM-DEPLOYMENT Sprint: Sprint 1 (2-3 hours total) Generated: 2025-12-09 02:17:09 UTC Status: ‚úÖ READY FOR EXECUTION Pattern: Three Concepts (Testable + Incremental + Reversible)

    üß© CONTEXT¶

    Mission: Deploy Azure API Management Consumption tier in incremental, testable, reversible steps. Each task includes execution evidence, rollback procedures, and validation criteria.

    Constraints: - Maximum 3-hour implementation window - Zero backend code changes - Zero downtime for existing services - All changes must be reversible

    Reuses: - Existing backend services (4 App Services running) - Existing Azure infrastructure (Entra ID, App Insights) - Existing documentation (2,900+ lines)

    Validates: - Each task completion before proceeding - Rollback procedures tested - Performance metrics tracked

    üìã Sprint Tasks¶

    Task 1: APIM Instance Deployment ‚≠ê CRITICAL¶

    Story Points: 2 Time Estimate: 15-20 minutes Owner: POD-F DevOps Dependencies: None

    User Story:

    As a DevOps engineer, I need to deploy APIM Consumption tier so that I can configure API gateway capabilities.

    Acceptance Criteria: - [ ] APIM instance created in Canada Central - [ ] Gateway URL accessible: https://eva-apim-demo.azure-api.net - [ ] Management API responds to health check - [ ] Resource tags applied (Environment: demo, Owner: Marco, Cost-Center: POD-F)

    Execution Steps: 

    # Step 1: Verify prerequisites
az account show  # Verify logged in
az provider show --namespace Microsoft.ApiManagement --query "registrationState"  # Must be "Registered"

# Step 2: Create APIM instance (Consumption tier)
az apim create `
  --name eva-apim-demo `
  --resource-group rg-evada2 `
  --publisher-name "EVA Suite" `
  --publisher-email marco.presta@example.com `
  --sku-name Consumption `
  --location canadacentral `
  --tags Environment=demo Owner=Marco CostCenter=POD-F `
  --no-wait

# Step 3: Monitor deployment (5-15 minutes)
az apim show --name eva-apim-demo --resource-group rg-evada2 --query "provisioningState"
# Wait until: "Succeeded"

# Step 4: Get gateway URL
az apim show --name eva-apim-demo --resource-group rg-evada2 --query "gatewayUrl" -o tsv
# Expected: https://eva-apim-demo.azure-api.net

    Validation: 

    # Test 1: Health check
curl https://eva-apim-demo.azure-api.net/status-0123456789abcdef
# Expected: HTTP 404 (gateway responds, no APIs configured yet)

# Test 2: Management API
az apim api list --resource-group rg-evada2 --service-name eva-apim-demo
# Expected: [] (empty list, no APIs imported yet)

    Rollback Procedure: 

    # Delete APIM instance (if needed)
az apim delete --name eva-apim-demo --resource-group rg-evada2 --yes --no-wait
# Time to rollback: <2 minutes

    Definition of Done: - ‚úÖ APIM instance status = "Succeeded" - ‚úÖ Gateway URL accessible (returns HTTP 404) - ‚úÖ Azure Portal shows instance - ‚úÖ Cost tracking enabled (tags applied)

    Task 2: OpenAPI Import (eva-api) ‚≠ê CRITICAL¶

    Story Points: 3 Time Estimate: 20-30 minutes Owner: POD-F DevOps Dependencies: Task 1

    User Story:

    As a developer, I need eva-api endpoints available via APIM so that I can access API with cost tracking.

    Acceptance Criteria: - [ ] OpenAPI spec exported from eva-api FastAPI - [ ] All endpoints imported to APIM - [ ] Endpoints accessible via APIM gateway - [ ] Backend URL configured (points to eva-api App Service)

    Execution Steps: 

    # Step 1: Export OpenAPI from eva-api (if not running, start it)
cd "c:\Users\marco\Documents\_AI Dev\EVA Suite\eva-api"
$env:PYTHONPATH = "src"
# Start eva-api if needed: uvicorn eva_api.main:app --host 0.0.0.0 --port 8000

# Step 2: Download OpenAPI spec
curl http://localhost:8000/openapi.json -o eva-api-openapi.json
# Verify: File size >10KB, valid JSON

# Step 3: Import to APIM
az apim api import `
  --resource-group rg-evada2 `
  --service-name eva-apim-demo `
  --path eva `
  --specification-path eva-api-openapi.json `
  --specification-format OpenApiJson `
  --display-name "EVA API" `
  --protocols https `
  --subscription-required true `
  --api-id eva-api-v1

# Step 4: Configure backend (point to running App Service)
az apim api update `
  --resource-group rg-evada2 `
  --service-name eva-apim-demo `
  --api-id eva-api-v1 `
  --service-url https://eva-api-marco-5346.azurewebsites.net

# Alternative: Use local backend for testing
# --service-url http://localhost:8000

    Validation: 

    # Test 1: List imported endpoints
az apim api operation list `
  --resource-group rg-evada2 `
  --service-name eva-apim-demo `
  --api-id eva-api-v1
# Expected: List of operations (/health, /graphql, etc.)

# Test 2: Test health endpoint (no auth required)
curl https://eva-apim-demo.azure-api.net/eva/health
# Expected: {"status":"ready","uptime_seconds":...}

# Test 3: Test authenticated endpoint (should fail without key)
curl https://eva-apim-demo.azure-api.net/eva/graphql
# Expected: HTTP 401 (subscription key required)

    Rollback Procedure: 

    # Delete imported API
az apim api delete `
  --resource-group rg-evada2 `
  --service-name eva-apim-demo `
  --api-id eva-api-v1 `
  --yes
# Time to rollback: <1 minute

    Definition of Done: - ‚úÖ All eva-api endpoints imported - ‚úÖ Health check accessible via APIM - ‚úÖ Backend URL configured correctly - ‚úÖ Subscription key requirement enforced

    Task 3: Product & Subscription Setup ‚≠ê HIGH¶

    Story Points: 2 Time Estimate: 15-20 minutes Owner: POD-F DevOps Dependencies: Task 2

    User Story:

    As Marco, I need API keys for demo users so that they can access EVA API with rate limiting.

    Acceptance Criteria: - [ ] "EVA Demo Free" product created - [ ] Rate limit: 100 req/min configured - [ ] Daily quota: 10,000 req/day configured - [ ] Test subscription created for Marco - [ ] API key generated and tested

    Execution Steps: 

    # Step 1: Create product
az apim product create `
  --resource-group rg-evada2 `
  --service-name eva-apim-demo `
  --product-id eva-demo-free `
  --product-name "EVA Suite - Demo Free" `
  --description "Free tier for demo: 100 req/min, 10K req/day" `
  --subscription-required true `
  --approval-required false `
  --state published

# Step 2: Add API to product
az apim product api add `
  --resource-group rg-evada2 `
  --service-name eva-apim-demo `
  --product-id eva-demo-free `
  --api-id eva-api-v1

# Step 3: Create rate limit policy
$policy = @"
<policies>
    <inbound>
        <base />
        <rate-limit-by-key calls="100" renewal-period="60" counter-key="@(context.Subscription.Id)" />
        <quota-by-key calls="10000" renewal-period="86400" counter-key="@(context.Subscription.Id)" />
    </inbound>
    <backend><base /></backend>
    <outbound><base /></outbound>
    <on-error><base /></on-error>
</policies>
"@
$policy | Out-File -FilePath product-policy.xml -Encoding utf8

az apim product policy create `
  --resource-group rg-evada2 `
  --service-name eva-apim-demo `
  --product-id eva-demo-free `
  --policy-file product-policy.xml

# Step 4: Create subscription for Marco
az apim subscription create `
  --resource-group rg-evada2 `
  --service-name eva-apim-demo `
  --subscription-id marco-demo-001 `
  --display-name "Marco Demo Subscription" `
  --scope /products/eva-demo-free `
  --state active

# Step 5: Get subscription key
az apim subscription show `
  --resource-group rg-evada2 `
  --service-name eva-apim-demo `
  --subscription-id marco-demo-001 `
  --query "primaryKey" -o tsv
# Save this key securely!

    Validation: 

    # Get subscription key
$subKey = az apim subscription show `
  --resource-group rg-evada2 `
  --service-name eva-apim-demo `
  --subscription-id marco-demo-001 `
  --query "primaryKey" -o tsv

# Test 1: Valid request with key
curl https://eva-apim-demo.azure-api.net/eva/health `
  -H "Ocp-Apim-Subscription-Key: $subKey"
# Expected: HTTP 200, {"status":"ready"...}

# Test 2: Rate limit enforcement (send 101 requests in 1 minute)
for ($i=1; $i -le 101; $i++) {
    curl https://eva-apim-demo.azure-api.net/eva/health `
      -H "Ocp-Apim-Subscription-Key: $subKey" -s -o $null -w "%{http_code}\n"
}
# Expected: First 100 return 200, 101st returns 429 (Too Many Requests)

# Test 3: Check rate limit headers
curl https://eva-apim-demo.azure-api.net/eva/health `
  -H "Ocp-Apim-Subscription-Key: $subKey" -i
# Expected headers:
#   X-RateLimit-Limit: 100
#   X-RateLimit-Remaining: 99 (or lower)

    Rollback Procedure: 

    # Delete subscription
az apim subscription delete `
  --resource-group rg-evada2 `
  --service-name eva-apim-demo `
  --subscription-id marco-demo-001 `
  --yes

# Delete product
az apim product delete `
  --resource-group rg-evada2 `
  --service-name eva-apim-demo `
  --product-id eva-demo-free `
  --yes --delete-subscriptions
# Time to rollback: <1 minute

    Definition of Done: - ‚úÖ Product created and published - ‚úÖ Rate limiting enforced (100 req/min) - ‚úÖ Daily quota enforced (10K req/day) - ‚úÖ Marco's subscription active with valid key - ‚úÖ Rate limit headers present in responses

    Task 4: Application Insights Integration üéØ MEDIUM¶

    Story Points: 2 Time Estimate: 15-20 minutes Owner: POD-F DevOps Dependencies: Task 3

    User Story:

    As Marco, I need usage metrics logged so that I can track costs per user.

    Acceptance Criteria: - [ ] Application Insights connected to APIM - [ ] All API calls logged - [ ] Custom metrics tracked (user ID, department) - [ ] Real-time dashboard accessible

    Execution Steps: 

    # Step 1: Get Application Insights instrumentation key
$appInsightsKey = az monitor app-insights component show `
  --resource-group rg-evada2 `
  --app eva-apim-insights `
  --query "instrumentationKey" -o tsv

# If doesn't exist, create it
if (-not $appInsightsKey) {
    az monitor app-insights component create `
      --resource-group rg-evada2 `
      --app eva-apim-insights `
      --location canadacentral `
      --application-type web

    $appInsightsKey = az monitor app-insights component show `
      --resource-group rg-evada2 `
      --app eva-apim-insights `
      --query "instrumentationKey" -o tsv
}

# Step 2: Create APIM logger
az apim logger create `
  --resource-group rg-evada2 `
  --service-name eva-apim-demo `
  --logger-id app-insights-logger `
  --logger-type applicationInsights `
  --description "Application Insights Logger" `
  --credentials instrumentationKey=$appInsightsKey

# Step 3: Enable diagnostics
az apim diagnostic create `
  --resource-group rg-evada2 `
  --service-name eva-apim-demo `
  --diagnostic-id applicationinsights `
  --logger-id app-insights-logger `
  --always-log allErrors `
  --sampling-type fixed `
  --sampling-percentage 100

    Validation: 

    # Test 1: Make API calls
for ($i=1; $i -le 10; $i++) {
    curl https://eva-apim-demo.azure-api.net/eva/health `
      -H "Ocp-Apim-Subscription-Key: $subKey" -s -o $null
}

# Wait 2-3 minutes for data to appear in Application Insights

# Test 2: Query Application Insights
az monitor app-insights metrics show `
  --resource-group rg-evada2 `
  --app eva-apim-insights `
  --metric requests/count `
  --interval PT1M
# Expected: Shows request count for last minute

# Test 3: Open Azure Portal dashboard
Start-Process "https://portal.azure.com/#@/resource/subscriptions/.../resourceGroups/rg-evada2/providers/Microsoft.Insights/components/eva-apim-insights"
# Verify: Request chart shows 10+ requests

    Definition of Done: - ‚úÖ Application Insights connected - ‚úÖ API calls logged (100% coverage) - ‚úÖ Metrics visible in Azure Portal - ‚úÖ Dashboard accessible to Marco
