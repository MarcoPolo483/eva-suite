    Cost-Optimized Autonomous Agile Crew - Final Architecture¶

    Date: November 28, 2025
    Sprint Model: 168-hour autonomous operation
    Budget: $23/week (¼ of $100/month baseline)
    Optimization Goal: Maximize autonomy, minimize cost

    Executive Summary¶

    Previous Cost: $144/month (GPT-4 for all agents) ❌
    Optimized Cost: \(9.45/week = **\)37.80/month** ✅
    Savings: $106.20/month (74% reduction)

    GitHub Actions:
    Previous: 7,290 min/month ($42/month overage) ❌

    Savings: $42/month

    Total Monthly Cost: $37.80 (well under $100 budget, leaves $62 for burst capacity)

    Autonomous Crew Architecture¶

    1. Master Scrum Master (MSM)¶

    Role: Daily standup generation, overall sprint coordination

    Schedule: Daily at 7:00 AM EST
    Model: GPT-4o-mini (was GPT-4)
    Cost: $0.15/run × 1 run/day × 30 days = $4.50/month

    What it does: - Collects POD reports from eva-lwo - Gets recent commits (last 24h) - Generates daily standup for Marco - Tracks costs, updates heartbeat - Escalates strategic decisions only

    Optimization: - Switched from GPT-4 ($2.00/run) to GPT-4o-mini ($0.15/run) - Savings: $1.85/day = $55.50/month

    2. PSM-1 (POD 1 Scrum Master)¶

    Role: Sprint progress tracking for POD 1

    Schedule: Every 4 hours (9 AM, 1 PM, 5 PM, 9 PM EST)
    Model: GPT-4o-mini
    Cost: $0.10/run × 6 runs/day × 30 days = $18.00/month

    What it does: - Tracks story points, velocity, blockers - Monitors task status (completed, in-progress, ready, blocked) - Generates progress reports - Calculates sprint completion % - Updates heartbeat with velocity metrics

    NEW: Fully implemented autonomous workflow

    3. Dev Master (DM-1)¶

    Role: Task assignment, AI expert spawning via GitHub Spark

    Schedule: Every 6 hours (8 AM, 2 PM, 8 PM, 2 AM EST)
    Model: GPT-4o-mini
    Cost: $0.15/run × 4 runs/day × 30 days = $18.00/month

    What it does: - Gets POD 1 backlog (labeled "pod-1" + "ready") - Checks capacity (max 3 concurrent experts) - Generates expert specification (GPT-4o-mini) - Creates GitHub Spark prompt - Posts instructions to issue - Updates issue labels (ready → in-progress)

    Optimization: - Reduced from every 4h to every 6h (6 runs/day → 4 runs/day) - Savings: $0.30/day = $9.00/month

    GitHub Spark Integration: - Marco deploys manually via https://github.com/spark - Expert creates repo, implements solution, opens PR - Full Sovereign-UI context in prompt (Svelte 5, Tailwind, GC Design System, a11y, i18n)

    4. Heartbeat Monitor¶

    Role: Health checks for all autonomous agents

    Schedule: Every 2 hours
    Model: None (GitHub Actions API only)
    Cost: FREE

    What it does: - Checks MSM health (last run <48h = healthy) - Checks PSM-1 health (last run <4h = healthy) - Auto-respawns stale agents - Creates GitHub issue if critical failure - Logs to eva-lwo/heartbeat-logs/

    Optimization: - Reduced from every 30 min to every 2 hours - GitHub Actions: 12 runs/day × 1 min = 12 min/day = 360 min/month (FREE tier) - Savings: 2,520 min/month = saved from overage

    5. P02 Daily Backlog Analysis (PLANNED)¶

    Role: Analyze backlog, suggest priorities

    Schedule: Daily at 6:00 AM EST (1h before MSM)
    Model: GPT-4o-mini
    Cost: $0.15/run × 1 run/day × 30 days = $4.50/month

    What it does: - Analyzes POD 1 backlog - Suggests task priorities - Identifies dependencies, blockers - Feeds recommendations to MSM standup - Tracks technical debt

    Status: Not yet implemented (next sprint)

    GitHub Spark Integration Strategy¶

    Expert Spawning Workflow¶

    1. Dev Master detects task (every 6 hours)
    2. Queries backlog for tasks labeled "pod-1" + "ready"

    3. Checks capacity (max 3 concurrent experts = open PRs)

    4. Generate expert specification (GPT-4o-mini)

    5. Task analysis
    6. Tech stack selection
    7. Acceptance criteria

    8. Estimated hours

    9. Create Spark prompt

    10. Full EVA-Sovereign-UI context
    11. Svelte 5 + Tailwind
    12. GC Design System patterns
    13. Bilingual support (EN/FR)
    14. a11y requirements (ARIA labels)

    15. Tool import structure

    16. Post to GitHub issue

    17. Spark prompt in issue comment
    18. Instructions for Marco

    19. Update labels (ready → in-progress)

    20. Marco deploys via Spark

    21. Go to https://github.com/spark
    22. Paste prompt
    23. Review generated app

    24. Click "Create Repository"

    25. Expert implements solution

    26. Creates repo with Actions + Dependabot
    27. Implements full-stack solution
    28. Adds tests
    29. Deploys immediately
    30. Opens PR to eva-orchestrator

    Capacity Management¶

    Max Concurrent Experts: 3
    Why? Balance parallel work with code review capacity

    Check: Count open PRs with label "spark-import"
    If capacity full: Dev Master defers task assignment until slot opens

    Risk Mitigation¶

    Risk 1: OpenAI API Cost Spike¶

    Likelihood: Medium (if prompt engineering inefficient)
    Impact: High (budget overrun)
    Mitigation: - Monitor cost-tracking/ daily - Set up alerts at $50/month, $75/month thresholds - Cap token limits (max_tokens: 2000 for MSM, 800 for PSM-1) - Review prompts monthly, optimize for conciseness

    Risk 2: GitHub Actions Overage¶

    Likelihood: Low (current: 1,740 min/month, buffer: 260 min)
    Impact: Medium ($0.008/min × overage)
    Mitigation: - Monitor GitHub Actions usage weekly - Set up self-hosted runner if approaching 2,000 min/month - Reduce workflow durations (optimize git operations, cache dependencies)

    Risk 3: Spark Expert Capacity Exceeded¶

    Likelihood: Medium (if backlog grows faster than experts complete)
    Impact: Low (tasks deferred, not lost)
    Mitigation: - Capacity limit: max 3 concurrent experts - DM-1 checks capacity before spawning - Marco can force-spawn if urgent (workflow_dispatch input)

    Risk 4: Heartbeat False Negatives¶

    Likelihood: Low (2h interval gives buffer)
    Impact: Medium (missed failures)
    Mitigation: - Health check windows: MSM=48h, PSM-1=4h (2× heartbeat interval) - Auto-respawn on stale detection - Manual override: Marco can run workflows via workflow_dispatch
