    CDD: EVA Vision Database Drill-Down¶

    Last Updated: 2025-11-19 Status: Backlog Priority: LOW (Moonshot) Epic: EVA Vision (Power BI Plugin)

    Vision¶

    Enable EVA to execute interactive database queries and what-if scenarios directly from Power BI dashboards—allowing users to drill down to row-level data, run ad-hoc SQL queries, simulate business scenarios, and explore data relationships beyond pre-built visuals, transforming dashboards from static reports into dynamic data exploration tools.

    Business Value: Reduce time to insight by 80% (vs. manual SQL queries), empower business users to explore data without SQL knowledge, enable real-time scenario planning (budget forecasting, risk analysis), and position EVA as the intelligent query layer between users and enterprise databases.

    Core Capabilities¶

    • Row-Level Drill-Down: Click dashboard aggregation → view underlying row-level data
    • Natural Language to SQL: "Show me all orders from Canada over $10K" → generates SQL query
    • What-If Scenarios: "What if sales increase by 15%?" → simulates impact across metrics
    • Data Relationship Exploration: "Which products are bought together?" → discovers patterns
    • Ad-Hoc Queries: Execute custom SQL (validated, read-only) without leaving Power BI
    • Query Optimization: EVA suggests indexes, partitions, or query rewrites for performance
    • Data Export: Download query results (CSV, Excel, JSON) for offline analysis
    • Query History: Save and replay favorite queries (personal library)

    User Stories¶

    US-1: As an analyst, I want to drill down to row-level data so that I investigate anomalies¶

    Acceptance Criteria:

    • Click bar in "Sales by Region" chart (e.g., Ontario region)
    • EVA asks: "Show details for Ontario?" (confirmation)
    • Displays table with row-level data: customer name, order ID, amount, date (top 100 rows)
    • Paginate through results (100 rows per page)
    • Export to Excel (all rows, not just displayed)

    US-2: As a business user, I want to ask data questions in plain language so that I avoid writing SQL¶

    Acceptance Criteria:

    • Ask "Show me all high-value orders from Q4"
    • EVA translates to SQL: SELECT * FROM Orders WHERE Quarter = 'Q4' AND Amount > 10000
    • Displays query preview (user confirms or edits)
    • Executes query, returns results in table format (within 3 seconds)
    • Follow-up: "Only from Canada" → EVA adds filter to existing query

    US-3: As a planner, I want to run what-if scenarios so that I forecast business outcomes¶

    Acceptance Criteria:

    • Ask "What if sales increase by 15% in Ontario?"
    • EVA calculates impact: "Total revenue: $1.38M (vs. current $1.2M), YoY growth: 27%"
    • Shows comparison: Current vs. Scenario (side-by-side charts)
    • Adjust scenario: "Make it 20%" → EVA recalculates instantly
    • Save scenario (name: "Optimistic Q1", persist for future reference)

    US-4: As a data engineer, I want EVA to optimize my queries so that I improve performance¶

    Acceptance Criteria:

    • Ask "Show me all orders from the last year"
    • EVA executes query, detects slow performance (>5 seconds)
    • Suggests optimization: "Add index on Order_Date column (estimated 80% speedup)"
    • Preview optimized query (highlighted changes)
    • Apply suggestion → query runs faster (validation: execution time logged)

    US-5: As a user, I want to save favorite queries so that I reuse them later¶

    Acceptance Criteria:

    • Execute query → click "Save query"
    • Name query: "High-Value Orders", add description (optional)
    • Query saved to personal library (visible in EVA chat sidebar)
    • Replay query: Click saved query → re-executes with current data
    • Share query with team (generate shareable link)

    Functional Requirements¶

    1. FR-1: SHALL support row-level drill-down (click aggregation → view underlying rows)
    2. FR-2: SHALL translate natural language to SQL using Azure OpenAI GPT-4o
    3. FR-3: SHALL validate SQL queries (syntax, security: block DROP/DELETE/UPDATE)
    4. FR-4: SHALL execute read-only queries within 3 seconds P95 (simple queries)
    5. FR-5: SHALL display query results in paginated tables (100 rows per page)
    6. FR-6: SHALL support what-if scenarios (simulate metric changes, show impact)
    7. FR-7: SHALL detect data relationships (discover patterns, suggest correlations)
    8. FR-8: SHALL optimize slow queries (suggest indexes, partitions, rewrites)
    9. FR-9: SHALL export query results (CSV, Excel, JSON formats)
    10. FR-10: SHALL save queries to personal library (name, description, replayable)
    11. FR-11: SHALL share queries with team (generate shareable links)
    12. FR-12: SHALL log all queries to Application Insights (audit trail, performance tracking)
    13. FR-13: SHALL enforce database permissions (users only query tables they have access to)
    14. FR-14: SHALL support multiple database types (SQL Server, Azure SQL, Synapse, Snowflake)
    15. FR-15: SHALL provide error messages in bilingual format (en-US, fr-FR)

    Non-Functional Requirements¶

    Performance¶

    • Query Execution: <3s P95 (simple queries: single table, <1M rows)
    • Query Translation: <2s (natural language → SQL)
    • What-If Calculation: <1s (scenario adjustment → impact calculation)
    • Data Export: <5s (10K rows → CSV/Excel download)

    Reliability¶

    • SQL Validation: >99% (syntax errors caught before execution)
    • Query Success Rate: >95% (excluding user permission errors)
    • Failover: Graceful degradation if database unavailable (show cached results if available)
    • Timeout Handling: Queries >30s auto-canceled (prevent resource exhaustion)

    Security¶

    • SQL Injection Prevention: Parameterized queries only (no string concatenation)
    • Read-Only Enforcement: Block all write operations (DROP, DELETE, UPDATE, INSERT)
    • Permission Enforcement: Users only query tables they have database-level access to
    • Audit Logging: All queries logged to Cosmos DB (user ID, query text, execution time, 1-year retention)

    Accessibility¶

    • WCAG 2.1 AA: Query results table keyboard-accessible, screen reader support
    • Keyboard Shortcuts: Alt+D (drill-down), Alt+S (save query), Alt+E (export results)
    • Focus Management: Logical tab order (query input → results table → export button)

    Internationalization¶

    • Locales: en-US (default), fr-FR
    • SQL Translation: Support bilingual queries (EVA understands both languages)
    • Error Messages: Translated (e.g., "Query failed" → "La requête a échoué")

    Observability¶

    • Metrics: Application Insights tracks query count, execution time, error rate, optimization suggestions
    • Alerts: Query latency >5s, error rate >5%, timeout rate >2%
    • Dashboards: Power BI dashboard for query usage, top queries, slow queries, error trends

    Scalability¶

    • Concurrent Queries: Support 200+ simultaneous queries (queued if database connection limit reached)
    • Query Volume: Handle 10K+ queries/day (cached results reduce database load)
    • Result Set Size: Support queries returning up to 100K rows (paginated, exportable)

    Technical Architecture¶

    Backend¶

    • API Gateway: Azure APIM (rate limiting, JWT validation, query routing)
    • Functions: Azure Functions (Node.js 20) for query orchestration, SQL translation, what-if calculations
    • AI Service: Azure OpenAI GPT-4o (natural language → SQL translation)
    • Database Connectors: ODBC/JDBC drivers (SQL Server, Azure SQL, Synapse, Snowflake)
    • Query Optimizer: Azure SQL Database Tuning Advisor (suggest indexes, partitions)
    • Cache: Azure Redis (query results, 5 min - 1 hour TTL)

    SQL Translation¶

    Natural Language → SQL Pipeline:

    1. User Query → "Show me all orders from Canada over $10K"
    2. Intent Recognition → Azure OpenAI GPT-4o (identify tables, columns, filters)
    3. SQL Generation → SELECT * FROM Orders WHERE Country = 'Canada' AND Amount > 10000
    4. Validation → Syntax check, security validation (block write operations)
    5. Execution → Run query against database (parameterized, read-only)
    6. Results → Return data in paginated table format

    Implementation Phases¶

    Phase 1: Core Query Execution (Sprint 14, Week 1-3)¶

    • Database connector setup (SQL Server, Azure SQL)
    • Natural language → SQL translation (Azure OpenAI GPT-4o)
    • SQL validation (syntax check, security validation: block writes)
    • Query execution (parameterized, read-only, timeout: 30s)
    • Results table display (paginated, 100 rows per page)

    Phase 2: Drill-Down & Export (Sprint 15, Week 4-6)¶

    • Row-level drill-down (click aggregation → view underlying rows)
    • Data export (CSV, Excel, JSON formats)
    • Query history (save executed queries to personal library)
    • Query replay (re-execute saved queries with current data)
    • Error handling (timeout, permission denied, syntax errors)

    Phase 3: What-If Scenarios (Sprint 16, Week 7-9)¶

    • Scenario simulation (adjust metrics, calculate impact)
    • Comparison view (current vs. scenario, side-by-side charts)
    • Scenario persistence (save named scenarios for future reference)
    • Multi-metric impact (ripple effect across related metrics)
    • Scenario sharing (generate shareable links)

    Phase 4: Optimization & Advanced Features (Sprint 17, Week 10-12)¶

    • Query optimization suggestions (indexes, partitions, rewrites)
    • Data relationship discovery (patterns, correlations)
    • Multi-database support (Synapse, Snowflake connectors)
    • Power BI dashboard for query usage, slow queries, optimization wins
    • Documentation and user guides

    Success Criteria¶

    • ✅ Query execution <3s P95 (simple queries)
    • ✅ SQL translation accuracy >95% (natural language → valid SQL)
    • ✅ Row-level drill-down functional (click aggregation → view rows)
    • ✅ What-if scenarios <1s (scenario adjustment → impact calculation)
    • ✅ User adoption >35% of Power BI users within 6 months
    • ✅ Query success rate >95% (excluding user permission errors)
    • ✅ User satisfaction >80% (post-query survey)
