    CDD: EVA Tools Marketplace¶

    Last Updated: 2025-11-19
    Status: Backlog
    Priority: MEDIUM
    Epic: EVA Tools Suite

    Vision¶

    Create an internal marketplace for productivity agents that employees can discover, install, and use with one click. EVA Tools Marketplace democratizes AI tool creation by allowing teams to publish their own agents, rate existing tools, and share best practices—fostering innovation and productivity across ESDC.

    Business Value: Accelerate AI adoption by 10x, reduce duplicate tool development, enable grassroots innovation, and build internal AI expertise through tool sharing.

    Core Capabilities¶

    • Agent Registry: Catalog of available productivity agents (Document Summarizer, Meeting Transcription, Email Draft, Code Review, Data Analysis, etc.)
    • One-Click Install: Deploy agents to user workspace with single click
    • Ratings & Reviews: User feedback, star ratings, usage statistics
    • Agent Templates: Starter kits for building custom agents (LangChain, Semantic Kernel, Autogen)
    • Publishing Workflow: Submit → Review → Approve → Publish (governance)
    • Search & Discovery: Filter by category, rating, language, author
    • Usage Analytics: Track installs, active users, session metrics per agent
    • Versioning: Agent updates with changelog, rollback capability

    User Stories¶

    US-1: As an employee, I want to discover productivity agents so that I can automate repetitive tasks¶

    Acceptance Criteria: - Marketplace displays agent cards (icon, title, description, rating) - Filter by category (document processing, meeting tools, email, code, data) - Search by keyword (e.g., "summarize", "transcribe") - Sort by rating, installs, recent updates - Preview agent details (features, screenshots, reviews) before installing

    US-2: As an employee, I want one-click install so that I can start using agents immediately¶

    Acceptance Criteria: - Click "Install" button on agent card - Agent added to user workspace (EVA Portal "My Tools") - Permissions requested upfront (e.g., "Access your OneDrive") - Installation status notification (success/failure) - Agent ready to use within 10 seconds

    US-3: As a power user, I want to rate and review agents so that others benefit from my experience¶

    Acceptance Criteria: - 5-star rating system per agent - Optional text review (max 500 chars) - Review visible to all users (with author name) - Upvote/downvote reviews for helpfulness - Report inappropriate reviews (moderation)

    US-4: As a team lead, I want to publish custom agents so that my team can reuse our solutions¶

    Acceptance Criteria: - Submit agent via web form (name, description, code repo link, category) - Approval workflow (security review → functional test → publish) - Agent versioning (v1.0, v1.1, changelog) - Usage analytics (installs, active users, session metrics) - Update/unpublish agent after publishing

    US-5: As a developer, I want agent templates so that I can build custom agents quickly¶

    Acceptance Criteria: - Marketplace provides starter kits (LangChain, Semantic Kernel, Autogen) - Templates include sample code, README, tests, deployment scripts - One-click fork to personal repo (GitHub) - Documentation links to agent development guides - Community Discord/Teams channel for support

    Functional Requirements¶

    FR-1: Agent Registry¶

    Marketplace SHALL display catalog of agents with icon, title, description, rating, installs, author, category.

    FR-2: One-Click Install¶

    Users SHALL install agents with single click, adding them to "My Tools" workspace within 10 seconds.

    FR-3: Ratings & Reviews¶

    Users SHALL rate agents (1-5 stars) and write reviews (max 500 chars) visible to all users.

    FR-4: Search & Filter¶

    Users SHALL search agents by keyword and filter by category, rating, language, author.

    FR-5: Publishing Workflow¶

    Team leads SHALL submit agents via web form with approval workflow (submit → review → approve → publish).

    FR-6: Agent Versioning¶

    Published agents SHALL support versioning (v1.0, v1.1) with changelogs and rollback capability.

    FR-7: Usage Analytics¶

    Marketplace SHALL track installs, active users, session metrics, and errors per agent.

    FR-8: Agent Templates¶

    Marketplace SHALL provide starter kits (LangChain, Semantic Kernel, Autogen) with sample code and deployment scripts.

    FR-9: Permissions¶

    Agents SHALL request permissions upfront (e.g., "Access OneDrive", "Send emails") with user consent.

    FR-10: Security Review¶

    All submitted agents SHALL undergo security review (code scan, dependency audit, secret detection) before approval.

    FR-11: Moderation¶

    Marketplace SHALL allow users to report inappropriate reviews, with admin moderation tools.

    FR-12: Bilingual Support¶

    Agent names, descriptions, and UI text SHALL support en-US and fr-FR.

    FR-13: Export/Import¶

    Users SHALL export their installed agents as JSON (backup) and import to another workspace.

    FR-14: Featured Agents¶

    Marketplace SHALL highlight featured agents (curated by admins) on homepage.

    FR-15: Notifications¶

    Users SHALL receive notifications for agent updates, new installs, and review responses.

    Non-Functional Requirements¶

    Performance¶

    • Marketplace page load: <2 seconds (P95)
    • Agent install: <10 seconds (deployment + config)
    • Search: <1 second for 1000+ agents
    • Usage analytics dashboard: <3 seconds

    Reliability¶

    • Uptime: 99.9% SLA (8.76 hours downtime/year)
    • Graceful degradation if agent unavailable (show cached data)
    • Rollback capability for failed agent updates
    • Retry logic for transient install failures

    Security¶

    • Code scanning for all submitted agents (GitHub Advanced Security)
    • Dependency audit (npm audit, pip audit)
    • Secret detection (no hardcoded API keys)
    • OAuth 2.0 for agent permissions (Entra ID)
    • Tenant isolation (agents can't access cross-tenant data)

    Accessibility¶

    • WCAG 2.1 AA compliance (keyboard navigation, screen reader)
    • High-contrast mode for agent cards
    • Focus indicators visible (≥3:1 contrast ratio)
    • ARIA labels for dynamic content

    i18n¶

    • Bilingual UI (en-US/fr-FR)
    • ICU formatting for dates, numbers
    • Agent descriptions translated (author responsibility)

    Observability¶

    • Application Insights custom events (installs, searches, reviews)
    • Distributed tracing (OpenTelemetry)
    • Usage metrics per agent (installs, active users, sessions)
    • Alerts for high error rates, failed installs

    Scalability¶

    • Support 10,000+ users, 500+ agents
    • Horizontal scaling (stateless API, state in Cosmos DB)
    • CDN for agent assets (icons, screenshots)
    • Rate limiting (100 installs/hour per user)

    Technical Architecture¶

    Frontend¶

    • Framework: React 18+ with TypeScript
    • Agent Cards: Custom component with lazy loading
    • Search: Algolia or Azure AI Search (instant search)
    • State Management: React Query (caching), Zustand (UI state)
    • Build: Vite

    Backend¶

    • API: Node.js/TypeScript (Express.js)
    • Database: Cosmos DB (HPK: /agentId, /userId/agentId)
    • File Storage: Azure Blob Storage (agent assets, templates)
    • Security Scanning: GitHub Advanced Security API
    • Deployment: Azure Functions (agent install orchestration)

    Data Models¶

    • Agents: { id, name, description, author, category, rating, installs, version, codeRepoUrl, icon }
    • Installs: { userId, agentId, installedAt, version, status }
    • Reviews: { agentId, userId, rating, comment, createdAt, upvotes }
    • Analytics: { agentId, date, installs, activeUsers, sessions, errors }

    Implementation Phases¶

    Phase 1: Marketplace MVP (Sprint 4)¶

    • Build marketplace UI (agent cards, search, filter)
    • Store agents in Cosmos DB
    • Implement one-click install (add to user workspace)
    • Add ratings & reviews UI
    • Deploy to dev environment

    Phase 2: Publishing & Security (Sprint 5)¶

    • Build agent submission form
    • Implement approval workflow (submit → review → publish)
    • Integrate GitHub Advanced Security (code scanning)
    • Agent versioning and changelogs
    • Usage analytics dashboard

    Phase 3: Templates & Community (Sprint 6)¶

    • Publish agent templates (LangChain, Semantic Kernel, Autogen)
    • Build developer documentation portal
    • Add community forum (agent development Q&A)
    • Export/import installed agents (JSON backup)
    • Featured agents homepage section

    Phase 4: Production Launch (Sprint 7)¶

    • WCAG 2.1 AA audit (keyboard, screen reader)
    • Security audit (code scanning, dependency audit)
    • Load testing (10,000 users, 500 agents)
    • User acceptance testing (UAT)
    • Documentation and training materials

    Success Criteria¶

    • ✅ 500+ agents published within 12 months
    • ✅ 10,000+ installs across ESDC
    • ✅ 80% agent approval rate (submitted → published)
    • ✅ CSAT >4.0/5.0 (user satisfaction)
    • ✅ WCAG 2.1 AA compliance (axe DevTools passes)
    • ✅ 99.9% uptime (monthly SLA)
    • ✅ <2 second marketplace page load (P95)

    Open Questions¶

    Q1: Agent Approval Process (GOVERNANCE)¶

    Question: Who approves submitted agents? Security team, platform admins, or automated?
    Impact: Quality control vs. approval bottleneck
    Recommendation: Automated security scan + manual functional test (platform admin)
    Decision Owner: Security Sentinel + Marco

    Q2: Agent Monetization (FUTURE)¶

    Question: Should internal marketplace support paid agents or credits?
    Impact: Innovation incentive vs. procurement complexity
    Recommendation: Free for MVP, explore internal credits in Q3 2026
    Decision Owner: Marco + Finance

    Q3: External Agents (POLICY)¶

    Question: Can users install agents from external marketplaces (ChatGPT plugins, Copilot extensions)?
    Impact: Feature richness vs. security risk
    Recommendation: Blocked for MVP, evaluate allowlist approach in Q2 2026
    Decision Owner: Security Sentinel + Marco

    Q4: Agent Sandboxing (SECURITY)¶

    Question: How to prevent malicious agents from accessing sensitive data?
    Impact: Security vs. functionality
    Recommendation: OAuth 2.0 scopes (granular permissions), tenant isolation, code scanning
    Decision Owner: Security Sentinel

    Q5: Agent Analytics Privacy (COMPLIANCE)¶

    Question: What usage data can be collected from installed agents?
    Impact: Insights vs. privacy
    Recommendation: Aggregate metrics only (no PII), opt-in for detailed telemetry
    Decision Owner: Privacy Officer + Marco
