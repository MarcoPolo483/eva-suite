    1. Bulk Story Templates – Use the P01–P21 table as issue templates (copy the row contents as issue body: description, acceptance, QA checks, dependencies). Add pod-1, pod-f-priority, and pattern-specific labels.
    2. Agentic Book Stories – Treat each chapter row as a discrete story; assign bilingual writing + accessibility reviews (P12) and dev doc sync (P04). Generate GitHub issues with tasks like Draft chapter 03 lifecycle diagrams & Translate chapter 04 safety guidance.
    3. Sprint Alignment – After TASK-037 completes, convert the templates into sprint stories for Sprint 5. Block P01–P10 into Sprint 5, P11–P16 into Sprint 6, and schedule chapters/batch cleanup for Sprint 7.

    1. Priority order: P01–P15 must land before P16–P21.
    2. Owners: POD-F leads (Pxx) should claim issues via GitHub Projects (label pod-f-priority).
    3. Dependencies: P02/REQ should link from backlog to prevent context drift; P15 should oversee instrumentation (ASA scoreboard).
    4. Timing: Slot P01–P10 into Sprint 5 (Dec 16–22), P11–P16 into Sprint 6, and documentation batches into Sprint 7.
