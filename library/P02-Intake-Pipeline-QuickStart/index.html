    ğŸ”„ Pipeline Behavior¶

    What Happens Automatically¶

    1. Scan: Pipeline checks docs/intake/inbox/ for new .md or .txt files
    2. Parse: Extracts front-matter (or auto-generates metadata)
    3. Execute P02: Runs p02-run.ps1 with telemetry
    4. Generate: Creates 10 P02 artifacts in docs/features/<slug>/
    5. Deliver: Creates manifest in docs/agile-crew/inbox/
    6. Move: Moves intake file to docs/intake/processed/
    7. Track: Updates docs/_state/P02-Intake-State.json
    8. Log: All telemetry to docs/_audit/EVA-Audit-Log.jsonl

    P02 Artifacts Generated (10 files)¶

    File Content
    README.md Overview, quick start
    P02-Overview.md Pattern execution summary
    requirements.md FR + NFR with priorities
    feature-map.md Capabilities, scope
    backlog.md Sprint tasks + acceptance criteria
    use-cases.md User scenarios
    risks.md Risks + mitigations
    architecture-notes.md Technical design
    adr-<slug>.md Architectural decisions
    tests.md Testing strategy

    Schedule¶

    Default: Every 6 hours (00:00, 06:00, 12:00, 18:00 UTC)

    Change schedule: Edit .github/workflows/p02-intake-pipeline.yml

    on:
  schedule:
    - cron: '0 */6 * * *'  # Every 6 hours

    Paths¶

    Customize paths via parameters:

    .\scripts\p02-intake-processor.ps1 `
  -IntakeInboxPath "custom/inbox" `
  -IntakeProcessedPath "custom/processed" `
  -AgileCrewInboxPath "custom/agile-crew" `
  -StateFilePath "custom/state.json" `
  -Caller "cli"

    ğŸ› Troubleshooting¶

    Issue: File not processed¶

    Symptoms: File in docs/intake/inbox/ but not processed

    Check: 1. Is file extension .md or .txt? 2. Has pipeline run since file was added? 3. Is file already in state (check P02-Intake-State.json)?

    Solutions: - Wait for next scheduled run (every 6 hours) - Trigger manually via GitHub Actions - Run locally: .\scripts\p02-intake-processor.ps1 -Caller "cli"

    Issue: P02 execution failed¶

    Symptoms: Error in GitHub Actions logs or CLI output

    Common causes: 1. Invalid YAML front-matter syntax 2. Special characters in FeatureKey or FeatureSlug 3. Missing scripts/p02-run.ps1 or dependencies

    Solutions: - Validate YAML syntax (use VS Code YAML extension) - Use kebab-case for slugs: my-feature-name - Check FeatureKey format: P02-XXX-NNN - Review error logs for specific failure

    Issue: Duplicate processing¶

    Symptoms: Feature processed multiple times

    Cause: State file corrupted or deleted

    Solution: 

    # Check state file
Get-Content "docs\_state\P02-Intake-State.json" | ConvertFrom-Json | 
  Select-Object -ExpandProperty processedFiles | 
  Format-Table -AutoSize

# If corrupted, recreate from processed folder
# (Manual step: rebuild state from docs/intake/processed/)

    ğŸ“ Best Practices¶

    1. Use descriptive filenames: azure-cost-alerts.md â†’ azure-cost-alerts slug
    2. Include front-matter: Better control over P02 execution
    3. Set priority: Helps Agile Crew prioritize sprint planning
    4. One feature per file: Don't combine multiple features
    5. Clear descriptions: Better P02 artifact quality
    6. Review before commit: Use dry-run to preview behavior
    7. Monitor costs: Check usage reports weekly
