    EVA Automation - Logging & Transparency Guide¶

    üìã Overview¶

    The EVA automation system is fully transparent with multiple logging layers for audit, debugging, and troubleshooting.

    üîç Logging Locations¶

    1. Cached Health Report (Primary)¶

    Location: eva-orchestrator/scripts/automation-health-report.json

    What it contains: - Complete health status snapshot - All workflow statuses with run IDs - GitHub Pages status - Repository cleanliness - Garbage collection findings - Timestamp and execution time

    Updated: Every hour by scheduled task

    View it: 

    # Pretty print the current report
Get-Content .\scripts\automation-health-report.json | ConvertFrom-Json | ConvertTo-Json -Depth 5

# Quick summary
Get-Content .\scripts\automation-health-report.json | ConvertFrom-Json | Select OverallHealth, TotalIssues, TimestampFriendly, ExecutionTimeSeconds

    Sample output: 

    {
  "Timestamp": "2025-11-27T14:16:42.8263619-05:00",
  "TimestampFriendly": "2025-11-27 14:16:42",
  "OverallHealth": "Degraded",
  "TotalIssues": 2,
  "CriticalIssues": 0,
  "ExecutionTimeSeconds": 4.96,
  "Workflows": { ... },
  "GitHubPages": { ... },
  "Repositories": { ... },
  "Garbage": { ... }
}

    2. Windows Task Scheduler Logs (System Level)¶

    Location: Windows Event Viewer

    View task execution history: 

    # See last run details
Get-ScheduledTaskInfo -TaskName "EVA-AutomationHealthReport" | Format-List

# Key fields:
# - LastRunTime: When task last executed
# - LastTaskResult: Exit code (0 = success)
# - NextRunTime: When it will run next
# - NumberOfMissedRuns: If task failed to run on schedule

    View detailed event logs: 

    # Get task execution events from Event Viewer
Get-WinEvent -FilterHashtable @{
    LogName = 'Microsoft-Windows-TaskScheduler/Operational'
    ID = 102, 103, 106, 107  # Task started, completed, registered, triggered
} | Where-Object { $_.Message -like "*EVA-AutomationHealthReport*" } | 
    Select-Object TimeCreated, Id, Message | 
    Format-Table -Wrap

    Common Event IDs: - 102 - Task completed successfully - 103 - Task failed to start - 106 - Task registered/updated - 107 - Task triggered - 111 - Task terminated (timeout/error)

    3. Standard Output Logs (Console Output)¶

    When the scheduled task runs, PowerShell output is captured by Task Scheduler.

    View task output: 

    # Enable task history first (if not already enabled)
# Open Task Scheduler GUI > View > Enable All Tasks History

# Then view in GUI:
# Task Scheduler > Task Scheduler Library > EVA-AutomationHealthReport
# > History tab shows all executions with output

    What's logged to console: - Section headers (Workflows, GitHub Pages, Repositories, Garbage) - Each workflow check result (‚úÖ/‚ö†Ô∏è/‚ùå) - Issues found with descriptions - Summary statistics - Execution time

    Example console output:

Checking: EVA Audit Trail Update
...

=== GitHub Pages Deployment ===

  ‚úÖ Accessible (updated 5 hours ago)

    4. Housekeeping Reports (Long-term Archive)¶

    Location: eva-orchestrator/reports/housekeeping-report-*.md

    Created by: Invoke-EvaHousekeeping.ps1 (run manually)

    What it contains: - Full workspace health snapshot - Git repository statuses - Test results (if run with tests) - Timestamped markdown report

    View reports: 

    # List all reports
Get-ChildItem .\reports\housekeeping-report-*.md | Select Name, LastWriteTime

# View most recent
Get-Content .\reports\housekeeping-report-*.md | Select-Object -Last 1 | Get-Content

    Note: These are manual reports, not automated hourly.

    5. GitHub Actions Logs (Workflow Execution)¶

    Location: https://github.com/MarcoPolo483/eva-orchestrator/actions

    What's logged: - Every workflow run with full console output - Success/failure status - Run duration and timestamps - Commit that triggered the run - Artifacts and job summaries

    View from command line: 

    # List recent runs for a workflow
gh run list --workflow "P02 Usage Snapshot" --limit 10

# View detailed logs for a specific run
gh run view RUN_ID --log

# Download logs locally
gh run download RUN_ID --name "logs"

    Run IDs are included in health report: 

    # Get run ID from health report for investigation
$report = Get-Content .\scripts\automation-health-report.json | ConvertFrom-Json
$report.Workflows.Details | Select Name, Status, LastRunId

    6. Git Activity Logs (Audit Trail)¶

    Location: eva-orchestrator/.git/logs/ and GitHub

    View local git history: 

    # See all automation-related commits
git log --oneline --grep="automation" --grep="workflow" --grep="health" -i

# See who changed automation files
git log --follow scripts/Update-AutomationHealthReport.ps1

# View specific commit details
git show COMMIT_HASH

    GitHub audit trail: - All commits: https://github.com/MarcoPolo483/eva-orchestrator/commits/main - Workflow runs: https://github.com/MarcoPolo483/eva-orchestrator/actions - Settings changes: https://github.com/MarcoPolo483/eva-orchestrator/settings/actions

    üìä Audit & Debugging Commands¶

    Check System Health¶

    # Current health status
Get-Content .\scripts\automation-health-report.json | ConvertFrom-Json | Select OverallHealth, TotalIssues, CriticalIssues, TimestampFriendly

# Report age
$report = Get-Content .\scripts\automation-health-report.json | ConvertFrom-Json
$age = (Get-Date) - [DateTime]::Parse($report.Timestamp)
Write-Host "Report age: $([Math]::Round($age.TotalMinutes, 0)) minutes"

    Check Scheduled Task¶

    # Task status and schedule
Get-ScheduledTask -TaskName "EVA-AutomationHealthReport" | Select Name, State, @{N="NextRun";E={(Get-ScheduledTaskInfo -TaskName $_.TaskName).NextRunTime}}

# Task execution history
Get-ScheduledTaskInfo -TaskName "EVA-AutomationHealthReport" | Format-List LastRunTime, LastTaskResult, NumberOfMissedRuns

# View task definition
Export-ScheduledTask -TaskName "EVA-AutomationHealthReport"

    Check Workflow History¶

    # Get run IDs from report
$report = Get-Content .\scripts\automation-health-report.json | ConvertFrom-Json
$report.Workflows.Details | Format-Table Name, Status, LastRunId, LastRunAge

# View specific workflow logs
gh run view 19744533224 --log  # Replace with actual run ID

    Trace an Issue¶

    # 1. Check current report
Get-Content .\scripts\automation-health-report.json | ConvertFrom-Json | ConvertTo-Json -Depth 5

# 2. Check when task last ran
Get-ScheduledTaskInfo -TaskName "EVA-AutomationHealthReport"

# 3. View task event logs
Get-WinEvent -FilterHashtable @{LogName='Microsoft-Windows-TaskScheduler/Operational'} -MaxEvents 20 | Where-Object {$_.Message -like "*EVA-AutomationHealthReport*"}

# 4. Run manual check for comparison
.\scripts\Invoke-AutomationHealthCheck.ps1 -Detailed

# 5. Check GitHub Actions directly
gh run list --workflow "WORKFLOW_NAME" --limit 5

    üõ°Ô∏è Privacy & Security¶

    What's NOT Logged¶

    • ‚ùå Secrets or tokens
    • ‚ùå Personal data
    • ‚ùå File contents
    • ‚ùå Sensitive configuration

    What IS Logged¶

    • ‚úÖ Public workflow statuses
    • ‚úÖ Public GitHub Pages URLs
    • ‚úÖ File paths (public repo)
    • ‚úÖ Execution timestamps
    • ‚úÖ Health status indicators

    Data Storage¶

    • Local only: JSON report stored locally in repo (not committed)
    • GitHub: Workflow run logs (public repo actions)
    • Windows: Event Viewer logs (local system)

    üìù Report Archiving (Optional)¶

    The current system overwrites the report every hour. To keep history:

    Enable Report Archiving¶

    # Modify Update-AutomationHealthReport.ps1 to save timestamped copies
$OutputPath = "$PSScriptRoot\automation-health-report.json"
$ArchivePath = "$PSScriptRoot\archive\automation-health-report-$(Get-Date -Format 'yyyyMMdd-HHmmss').json"

# Save current version
$Report | ConvertTo-Json -Depth 10 | Set-Content -Path $OutputPath -Encoding UTF8

# Archive a copy (optional)
if (-not (Test-Path (Split-Path $ArchivePath))) {
    New-Item -ItemType Directory -Path (Split-Path $ArchivePath) -Force | Out-Null
}
Copy-Item $OutputPath $ArchivePath

    Cleanup Old Archives¶

    # Delete archives older than 7 days
Get-ChildItem .\scripts\archive\automation-health-report-*.json | 
    Where-Object { $_.LastWriteTime -lt (Get-Date).AddDays(-7) } | 
    Remove-Item -Force

    üö® Troubleshooting Logs¶

    Task is not running¶

    # Check task is enabled
Get-ScheduledTask -TaskName "EVA-AutomationHealthReport" | Select State

# Check for errors
Get-WinEvent -FilterHashtable @{
    LogName = 'Microsoft-Windows-TaskScheduler/Operational'
    ID = 103, 111  # Failed to start, terminated
} | Where-Object { $_.Message -like "*EVA-AutomationHealthReport*" } | 
    Select-Object TimeCreated, Message -First 5

    Report not updating¶

    # Check file timestamp
Get-Item .\scripts\automation-health-report.json | Select LastWriteTime

# Check file permissions
icacls .\scripts\automation-health-report.json

# Run task manually and watch output
Start-ScheduledTask -TaskName "EVA-AutomationHealthReport"

    Login shows stale report¶

    # Verify report age
$report = Get-Content .\scripts\automation-health-report.json | ConvertFrom-Json
Write-Host "Generated: $($report.TimestampFriendly)"
Write-Host "Age: $([Math]::Round(((Get-Date) - [DateTime]::Parse($report.Timestamp)).TotalMinutes, 0)) minutes"

# Force fresh run
Start-ScheduledTask -TaskName "EVA-AutomationHealthReport"
Start-Sleep -Seconds 6
.\login.ps1

    ‚úÖ Verification Checklist¶

    Run these to verify logging is working:

    # 1. Report exists and is recent
Test-Path .\scripts\automation-health-report.json
Get-Item .\scripts\automation-health-report.json | Select LastWriteTime

# 2. Task is active and scheduled
Get-ScheduledTask -TaskName "EVA-AutomationHealthReport" | Select State, @{N="NextRun";E={(Get-ScheduledTaskInfo -TaskName $_.TaskName).NextRunTime}}

# 3. Task has run successfully
(Get-ScheduledTaskInfo -TaskName "EVA-AutomationHealthReport").LastTaskResult -eq 0

# 4. Report contains expected data
$report = Get-Content .\scripts\automation-health-report.json | ConvertFrom-Json
$report.Workflows.TotalCount -eq 5  # Should be 5 workflows
$report.OverallHealth -ne "Unknown"  # Should have health status
