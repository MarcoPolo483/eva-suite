    Azure Deployment Architecture for EVA Hero Demos¶

    Context: Hero demos graduate from EVA Suite Portal â†’ Standalone Azure deployments Current Example: EVA DA Chat at https://evasuitestoragedev.z9.web.core.windows.net/ Decision Date: 2025-12-13

    Hybrid Deployment Model¶

    EVA Suite Portal (GitHub Pages)¶

    URL: https://marcopolo483.github.io/eva-suite/ Purpose: Product catalog, navigation, documentation hub Hosting: GitHub Pages (free, static content) Content: - Product hero cards (40+ products) - Hero demo launcher (links/iframes to Azure demos) - EVA Library (Books collection) - EVA Agile Scrum Portal (sprint tracking)

    Hero Demos (Azure Static Web Apps)¶

    URL Pattern: https://eva-{product}-{env}.z9.web.core.windows.net/ Purpose: Live interactive demos (product incubators) Hosting: Azure Static Web Apps (scalable, HTTPS, custom domains) Examples: - https://evasuitestoragedev.z9.web.core.windows.net/ - EVA DA Chat demo (current) - https://eva-liveops-dev.z9.web.core.windows.net/ - EVA LiveOps demo (future) - https://eva-da-dev.z9.web.core.windows.net/ - EVA Domain Assistant demo (future)

    Access: Public (demo users, stakeholders) Data: Synthetic/static demo data SLA: None (best-effort)

    Stage 2: Alpha (Early Access)¶

    Environment: Azure Static Web Apps - ALPHA URL Pattern: https://eva-{product}-alpha.z9.web.core.windows.net/ Access: Restricted (internal ESDC users, invited testers) Data: Mix of real + synthetic data (limited scope) SLA: None (support via email)

    Stage 3: Beta (Pilot)¶

    Environment: Azure App Service - BETA URL Pattern: https://eva-{product}-beta.service.canada.ca/ Access: Pilot departments (controlled rollout) Data: Real production data (pilot scope) SLA: Best-effort (8x5 support)

    Azure Resource Naming Convention¶

    Static Web Apps (Hero Demos)¶

    eva-{product}-{env}-swa
    Examples: - eva-chat-dev-swa â†’ witty-river-0aabfad0f.3.azurestaticapps.net - eva-da-dev-swa â†’ delightful-sky-085ed100f.3.azurestaticapps.net - eva-liveops-dev-swa â†’ icy-mushroom-0292f750f.3.azurestaticapps.net

    Note: Azure assigns random-name hostnames to Static Web Apps automatically. Custom domains can be configured after deployment.

    App Service (Production)¶

    eva-{product}-{env}-app
    Examples: - eva-chat-prod-app - EVA Chat production - eva-da-prod-app - EVA Domain Assistant production

    Storage Accounts¶

    eva{product}{env}storage
    Examples: - evachatdevstorage - EVA Chat dev storage - evadadevstorage - EVA DA dev storage

    GitHub Actions Workflow (.github/workflows/deploy-dev.yml): 

    name: Deploy EVA Chat Demo to Azure (DEV)

on:
  push:
    branches: [main]
  workflow_dispatch:

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install dependencies
        run: npm ci

      - name: Build
        run: npm run build
        env:
          VITE_ENV: dev
          VITE_API_URL: https://eva-chat-dev-api.azurewebsites.net

      - name: Deploy to Azure Static Web Apps
        uses: Azure/static-web-apps-deploy@v1
        with:
          azure_static_web_apps_api_token: ${{ secrets.AZURE_STATIC_WEB_APPS_API_TOKEN_DEV }}
          repo_token: ${{ secrets.GITHUB_TOKEN }}
          action: "upload"
          app_location: "/"
          output_location: "dist"

    Graduation Checklist¶

    Graduating from Hero Demo â†’ Alpha¶

    • Dedicated Azure Static Web App created (eva-{product}-alpha-swa)
    • GitHub Actions workflow configured (.github/workflows/deploy-alpha.yml)
    • Access control enabled (Azure AD authentication)
    • Synthetic data replaced with limited real data
    • Hero card updated in Portal (status: 'alpha')
    • Demo URL removed from public hero demos section
    • Documentation Book published in EVA Library

    Graduating from Alpha â†’ Beta¶

    • Azure App Service created (eva-{product}-beta-app)
    • Multi-region deployment (Canada Central + Canada East)
    • Full backend APIs deployed
    • Real production data (pilot scope)
    • Pilot user onboarding process established
    • SLA monitoring dashboards configured
    • Hero card updated (status: 'beta')

    Graduating from Beta â†’ Production¶

    • Azure App Service Production created (eva-{product}-prod-app)
    • Custom domain configured (eva-{product}.service.canada.ca)
    • SSL certificate (TBS managed)
    • Full RBAC enforcement (Azure AD integration)
    • Protected B data compliance validated
    • 99.9% SLA monitoring + alerting
    • 24x7 support plan established
    • Hero card updated (status: 'production')
    • Hero demo archived/deleted

    Cost Estimation (Azure Resources)¶

    Hero Demo (Dev)¶

    • Azure Static Web Apps: Free tier (100 GB bandwidth/month)
    • Storage Account: ~$0.50/month (demo data)
    • Total: ~$0.50/month per demo

    Alpha¶

    • Azure Static Web Apps: Standard tier (~$9/month)
    • Storage Account: ~$5/month (limited real data)
    • Total: ~$14/month per product

    Beta¶

    • Azure App Service: B1 tier (~$13/month)
    • Azure SQL Database: S1 tier (~$30/month)
    • Application Insights: ~$5/month
    • Total: ~$48/month per product

    Production¶

    • Azure App Service: P1v3 tier (~$70/month x 2 regions = $140/month)
    • Azure SQL Database: S3 tier (~$75/month)
    • Application Insights: ~$20/month
    • Azure Front Door: ~$35/month
    • Total: ~$270/month per product

    6 Products in Production: ~$1,620/month
