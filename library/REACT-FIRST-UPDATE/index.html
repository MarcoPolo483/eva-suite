    EVA-Sovereign-UI Architecture Update: React-First Approach¶

    Date: November 28, 2025
    Status: Decision finalized, migration in progress
    ADR: ADR-006-react-first-ui-framework.md

    Summary of Changes¶

    From: Svelte 5-first approach
    To: React-first with Web Components distribution

    Rationale¶

    1. GitHub Spark Integration: Spark has stronger React support and generates higher-quality React code
    2. Visual Builder Ecosystem: TeleportHQ, Plasmic, Locofy all have mature React support, limited/no Svelte
    3. AI Code Generation: ChatGPT, Copilot, Spark all have better React training data
    4. Framework Agnostic: Compile React → Web Components for use in any framework
    5. Enterprise Adoption: React is most widely adopted in GC departments, larger talent pool

    Updated Architecture¶

    ┌──────────────────────────────────────────────────┐
│  EVA-Sovereign-UI (React + TypeScript)            │
│                                                    │
│  - Author components in React                      │
│  - GC Design System patterns                       │
│  - React ARIA / Radix (a11y)                      │
│  - react-i18next (EN/FR bilingual)                │
│  - CSS Custom Properties (design tokens)          │
└────────────────┬───────────────────────────────────┘
                 │
                 │ Build (via @lit-labs/react or similar)
                 ▼
┌──────────────────────────────────────────────────┐
│  Web Components Distribution                      │
│                                                    │
│  - Custom Elements (standards-based)              │
│  - Works in React, Vue, Angular, Svelte, HTML    │
│  - Sovereign profiles via attributes              │
└──────────────────────────────────────────────────┘

    Technology Stack¶

    Primary Development¶

    • React 18+ (with Server Components where appropriate)
    • TypeScript (strict mode)
    • Tailwind CSS (GC-compliant config)
    • React ARIA or Radix UI (accessibility)
    • react-i18next (i18n/l10n)

    Distribution¶

    • Web Components (Custom Elements + Shadow DOM)
    • @lit-labs/react or @microsoft/fast-element (React → WC compiler)

    Tooling¶

    • Vite (build tool)
    • Storybook (component dev)
    • Vitest + Testing Library (tests)
    • axe-core (a11y testing)

    DevMaster Spark Prompts¶

    Old Context: 

    Use Svelte 5 + Tailwind utility classes
Follow GC Design System patterns (canada.ca)
Ensure bilingual support (EN/FR via i18n keys)
Include ARIA labels and roles for a11y
Structure for Tool import (clear sections/components)

    New Context: 

    Use React + TypeScript with Web Components build
Follow GC Design System patterns (canada.ca)
Use React ARIA or Radix UI for accessibility primitives
Ensure bilingual support (EN/FR via react-i18next)
Include ARIA labels, roles, and keyboard navigation
Structure for Tool import (clear sections/components)
Compile to Web Components for framework-agnostic use

    Spark Integration Pattern (Updated)¶

    Step 1: DevMaster generates Spark prompt¶

    Build an EVA Suite expert agent for: Login Screen

Context: EVA-Sovereign-UI Design System (React-first)
- Use React + TypeScript
- Tailwind CSS with GC-compliant config
- Follow GC Design System patterns (canada.ca)
- Use React ARIA or Radix UI for accessible components
- Ensure bilingual support (EN/FR via react-i18next)
- Include proper ARIA labels, roles, keyboard navigation
- Structure for Tool import (clear component boundaries)

Description:
Create a secure login screen with username/password fields...

Tech Stack:
- React 18+
- TypeScript (strict mode)
- Tailwind CSS
- React ARIA (accessibility)
- react-i18next (i18n)

Acceptance Criteria:
- WCAG 2.1 AA compliant
- Keyboard-only navigation works
- Screen reader accessible
- Bilingual (EN/FR) toggle
- Mobile responsive

    Step 2: Marco pastes into GitHub Spark¶

    1. Go to https://github.com/spark
    2. Paste the prompt
    3. Review generated React app in preview
    4. Click "Create Repository"

    Step 3: Spark expert implements¶

    • Creates React app with TypeScript
    • Implements GC-compliant UI
    • Adds tests
    • Opens PR to eva-orchestrator

    Step 4: Optional - Compile to Web Components¶

    If the component needs to be framework-agnostic: 

    npm run build:wc  # Compiles React → Web Components

    Benefits of This Approach¶

    For Autonomous Crew¶

    ✅ Better AI Code Generation: Spark, Copilot, ChatGPT all produce higher-quality React code
    ✅ Reliable Prompts: DevMaster prompts work consistently with React context
    ✅ Fewer Bugs: Less AI hallucination with React patterns

    For Visual Builder (WYSIWYG IDE)¶

    ✅ TeleportHQ Integration: Register React components in visual builder
    ✅ Plasmic Integration: Use EVA components as custom blocks
    ✅ Locofy Integration: Design-to-code from Figma → React → EVA

    For Framework Flexibility¶

    ✅ React Apps: Use components directly (best DX)
    ✅ Vue/Angular/Svelte Apps: Use Web Components build
    ✅ Plain HTML: Import WC bundle via <script> tag

    For GC Departments¶

    ✅ Familiar Stack: React is most widely used in government
    ✅ Hiring: Largest talent pool
    ✅ Integrations: More third-party components available

    For Developers: Quick Start¶

    Use React Components Directly¶

    import { GovHeader, HeroBanner, QuickActionsGrid } from '@eva-suite/sovereign-ui';

function App() {
  return (
    <>
      <GovHeader 
        appNameKey="eva.da.appName"
        profile="canada_gc_intranet"
      />
      <HeroBanner
        titleKey="eva.da.home.hero.title"
        descriptionKey="eva.da.home.hero.description"
        showVoiceButton={true}
      />
      <QuickActionsGrid
        items={[
          { labelKey: 'eva.da.home.quick.recent', action: 'openRecentQueries' },
          { labelKey: 'eva.da.home.quick.upload', action: 'openUploadDialog' },
        ]}
      />
    </>
  );
}

    Use Web Components (Any Framework)¶

    <!DOCTYPE html>
<html>
<head>
  <script type="module" src="https://unpkg.com/@eva-suite/sovereign-ui-wc"></script>
</head>
<body>
  <eva-gov-header 
    app-name-key="eva.da.appName"
    profile="canada_gc_intranet">
  </eva-gov-header>

  <eva-hero-banner
    title-key="eva.da.home.hero.title"
    description-key="eva.da.home.hero.description"
    show-voice-button="true">
  </eva-hero-banner>
</body>
</html>
