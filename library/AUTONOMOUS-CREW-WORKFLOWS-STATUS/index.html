    Lessons Learned (This Morning)¶

    What Worked Well ‚úÖ¶

    1. Error Handling: Comprehensive try-catch blocks caught issues early
    2. Incremental Testing: Test one workflow at a time, fix, then move to next
    3. Git Commits: Small, frequent commits made it easy to identify which fix worked
    4. Continue-on-Error: Prevented cascading failures (git push conflict doesn't block cost tracking)
    5. Logging: Detailed console.log statements helped debug OpenAI API issues

    What Didn't Work ‚ùå¶

    1. Assumptions: Assumed GitHub token had all permissions (Discussions write missing)
    2. Git Race Conditions: Multiple workflows pushing to eva-lwo simultaneously ‚Üí conflicts
    3. Free Tier Underestimation: Calculated 2,880 min/month, forgot other workflows also consume minutes
    4. OpenAI Cost Underestimation: $2/call √ó multiple agents = budget exceeded quickly

    What to Do Differently üîÑ¶

    1. Permission Audit: Check all required GitHub token permissions upfront (Discussions, Projects, Packages)
    2. Git Locking: Use GitHub Actions concurrency groups to prevent simultaneous eva-lwo pushes
    3. Cost Calculator: Build spreadsheet to forecast costs before deploying workflows
    4. Smoke Tests: Run all workflows in test mode first (dry-run, no actual API calls) before enabling schedules
    5. Self-Hosted Runner: Set up Marco's machine as runner to eliminate GitHub Actions costs
