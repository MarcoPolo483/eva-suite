    Chapter 03: Components Overview¶

    EVA Sovereign UI provides a comprehensive library of production-ready Web Components organized into five functional categories. Each component follows the Lit reactive lifecycle, uses TypeScript decorators for properties, and exposes a consistent API through props, slots, and events. This chapter explores component organization, lifecycle patterns, composition strategies, and real-world examples.

    Component Categories¶

    EVA Sovereign UI components are organized into five categories based on their primary function: Forms, Navigation, Feedback, Layout, and Data Display. This taxonomy helps developers discover components and understand their intended usage.

    Forms¶

    Form components handle user input, validation, and submission workflows. All form components implement bilingual labels, accessible error messaging, and WCAG 2.1 AA keyboard navigation.

    Available Components:

    • eva-gc-input - Text input with validation
    • eva-gc-textarea - Multi-line text input
    • eva-gc-select - Dropdown selector
    • eva-gc-radio-group - Radio button group
    • eva-gc-checkbox - Checkbox with label
    • eva-gc-date-picker - Accessible date picker
    • eva-gc-file-upload - File upload with drag-drop

    Key Features:

    • Automatic bilingual label rendering based on lang context
    • Real-time validation with accessible error announcements
    • Support for required, pattern, minlength, maxlength constraints
    • Integration with native form submission and FormData API

    Example:

    <eva-gc-input
  name="email"
  label-en="Email Address"
  label-fr="Adresse courriel"
  type="email"
  required
  error-en="Please enter a valid email"
  error-fr="Veuillez saisir un courriel valide"
></eva-gc-input>

    Navigation¶

    Navigation components provide wayfinding, routing, and structural organization. These components enforce GC Design System navigation patterns and Federal Identity Program compliance.

    Available Components:

    • eva-gc-header - GC-branded page header with language toggle
    • eva-gc-footer - GC-compliant footer with mandatory links
    • eva-gc-breadcrumbs - Hierarchical navigation trail
    • eva-gc-tabs - Tabbed content switcher
    • eva-gc-menu - Dropdown menu with keyboard navigation
    • eva-gc-pagination - Page navigation for data sets

    Key Features:

    • Automatic language toggle integration
    • RBAC-aware navigation (hide/show items based on user role)
    • Keyboard navigation with arrow keys and Home/End
    • ARIA landmarks and navigation semantics

    Example:

    <eva-gc-header
  title-en="My Application"
  title-fr="Mon application"
  show-language-toggle
  show-search
>
  <eva-gc-menu slot="nav">
    <eva-gc-menu-item href="/" label-en="Home" label-fr="Accueil"></eva-gc-menu-item>
    <eva-gc-menu-item href="/about" label-en="About" label-fr="À propos"></eva-gc-menu-item>
  </eva-gc-menu>
</eva-gc-header>

    Feedback¶

    Feedback components communicate system status, validation results, and user notifications. All feedback components use ARIA live regions for screen reader announcements.

    Available Components:

    • eva-gc-alert - Contextual alert messages (info, warning, error, success)
    • eva-gc-toast - Auto-dismissing notifications
    • eva-gc-modal - Modal dialog with focus trapping
    • eva-gc-loading-spinner - Loading indicator
    • eva-gc-progress-bar - Progress indicator
    • eva-gc-ai-disclaimer - AI-generated content warning

    Key Features:

    • Automatic ARIA live region management
    • Bilingual messaging with language-aware formatting
    • Focus management and keyboard trapping for modals
    • Accessible dismiss patterns (Escape key, close button)

    Example:

    <eva-gc-alert
  type="error"
  message-en="Invalid file format. Please upload a PDF."
  message-fr="Format de fichier non valide. Veuillez téléverser un PDF."
  dismissible
></eva-gc-alert>

    Layout¶

    Layout components provide structural scaffolding and responsive grid systems. These components use CSS Grid and Flexbox with mobile-first breakpoints.

    Available Components:

    • eva-gc-container - Max-width content container
    • eva-gc-grid - 12-column responsive grid
    • eva-gc-stack - Vertical spacing manager
    • eva-gc-card - Content card with optional header/footer
    • eva-gc-accordion - Collapsible content sections
    • eva-gc-split-view - Two-column layout with resizable divider

    Key Features:

    • Mobile-first responsive behavior
    • GC Design System spacing tokens
    • Accessibility landmarks (main, aside, section)
    • Nested grid support

    Example:

    <eva-gc-grid>
  <eva-gc-card span="12" span-md="6" span-lg="4">
    <h3 slot="header">Card Title</h3>
    <p>Card content goes here.</p>
  </eva-gc-card>
  <eva-gc-card span="12" span-md="6" span-lg="4">
    <h3 slot="header">Another Card</h3>
    <p>More content here.</p>
  </eva-gc-card>
</eva-gc-grid>

    Data Display¶

    Data display components present structured information, tables, and AI-generated content with citations and confidence indicators.

    Available Components:

    • eva-gc-table - Accessible data table with sorting and filtering
    • eva-gc-citation-card - RAG citation with confidence score
    • eva-gc-ai-content-indicator - Visual marker for AI-generated content
    • eva-gc-badge - Status badge with semantic colors
    • eva-gc-avatar - User avatar with initials fallback
    • eva-gc-timeline - Chronological event display

    Key Features:

    • Accessible table markup with <thead>, <tbody>, ARIA roles
    • RAG citation linking to source documents
    • Confidence score visualization (0-100%)
    • Bilingual column headers and data formatting

    Example:

    <eva-gc-citation-card
  title-en="Legal Precedent - Smith v. Canada"
  title-fr="Précédent juridique - Smith c. Canada"
  confidence="87"
  source-url="/documents/smith-v-canada.pdf"
  excerpt-en="The court ruled that..."
  excerpt-fr="La cour a statué que..."
></eva-gc-citation-card>

    Component Lifecycle¶

    EVA Sovereign UI components follow the Lit reactive lifecycle, extending the standard Web Components lifecycle with reactive property updates and efficient rendering. Understanding the lifecycle is essential for composition patterns, dynamic updates, and performance optimization.

    Lifecycle Methods¶

    constructor() - Component instantiation. Initialize private state, bind event handlers. Do NOT access DOM or attributes here.

    constructor() {
  super();
  this._isOpen = false;
  this._handleEscape = this._handleEscape.bind(this);
}

    connectedCallback() - Component added to DOM. Add event listeners, fetch initial data, start animations. Called each time component is inserted.

    connectedCallback() {
  super.connectedCallback();
  document.addEventListener('keydown', this._handleEscape);
  this._fetchUserPreferences();
}

    disconnectedCallback() - Component removed from DOM. Clean up event listeners, cancel timers, abort requests. Essential for preventing memory leaks.

    disconnectedCallback() {
  super.disconnectedCallback();
  document.removeEventListener('keydown', this._handleEscape);
  this._abortController?.abort();
}

    updated(changedProperties) - Reactive properties changed. Access updated DOM, trigger side effects, sync external state.

    updated(changedProperties: Map<string, any>) {
  super.updated(changedProperties);
  if (changedProperties.has('lang')) {
    this._updateAriaLabels();
  }
}

    render() - Return component template. Pure function that returns Lit template result. Called on every property change.

    render() {
  return html`
    <button @click=${this._handleClick}>
      ${this._getLabel()}
    </button>
  `;
}

    Lifecycle Order¶

    1. constructor() - Component created
    2. connectedCallback() - Added to DOM
    3. render() - Initial render
    4. updated() - After first render
    5. (properties change) → render() → updated()
    6. disconnectedCallback() - Removed from DOM

    Props vs Slots vs Events¶

    EVA Sovereign UI uses three mechanisms for component communication: props (data in), slots (content projection), and events (data out). Understanding when to use each pattern is critical for composable, maintainable components.

    Props (Data In)¶

    Props are reactive attributes that accept primitive values (string, number, boolean) or complex objects (arrays, objects). Lit's @property() decorator makes props reactive.

    When to Use Props:

    • Configuration options (size, variant, disabled state)
    • Bilingual content (label-en, label-fr)
    • Data binding (value, checked, selected)

    Example:

    import { LitElement, html } from 'lit';
import { property } from 'lit/decorators.js';

export class EvaGcButton extends LitElement {
  @property({ type: String, attribute: 'label-en' })
  labelEn = '';

  @property({ type: String, attribute: 'label-fr' })
  labelFr = '';

  @property({ type: Boolean })
  disabled = false;

  @property({ type: String })
  variant: 'primary' | 'secondary' = 'primary';
}

    Usage:

    <eva-gc-button
  label-en="Submit"
  label-fr="Soumettre"
  variant="primary"
  disabled
></eva-gc-button>

    Slots (Content Projection)¶

    Slots allow content insertion into component templates. Shadow DOM slots enable composition without prop explosion.

    When to Use Slots:

    • Rich content (formatted text, images, nested components)
    • User-defined templates (card headers, modal bodies)
    • Extensibility points (custom actions, footers)

    Example:

    render() {
  return html`
    <div class="card">
      <header>
        <slot name="header">Default Header</slot>
      </header>
      <div class="body">
        <slot>Default content</slot>
      </div>
      <footer>
        <slot name="footer"></slot>
      </footer>
    </div>
  `;
}

    Usage:

    <eva-gc-card>
  <h3 slot="header">Custom Title</h3>
  <p>This is card content with <strong>formatting</strong>.</p>
  <eva-gc-button slot="footer" label-en="Action"></eva-gc-button>
</eva-gc-card>

    Events (Data Out)¶

    Events communicate state changes from child to parent. EVA Sovereign UI follows a consistent event naming convention: eva-{component}-{action}.

    When to Use Events:

    • User interactions (button clicks, input changes)
    • State transitions (modal opened, tab selected)
    • Validation results (form submitted, error occurred)

    Example:

    _handleClick() {
  this.dispatchEvent(new CustomEvent('eva-button-click', {
    bubbles: true,
    composed: true,
    detail: { variant: this.variant }
  }));
}

    Usage:

    <eva-gc-button
  label-en="Save"
  @eva-button-click=${this.handleSave}
></eva-gc-button>

<script>
  class MyApp extends LitElement {
    handleSave(e) {
      console.log('Button clicked:', e.detail.variant);
    }
  }
</script>

    Composition Examples¶

    Real-world EVA Suite applications require complex UI patterns composed from atomic components. The following examples demonstrate composition strategies for common use cases.

    AI Response Card¶

    An AI response card displays RAG-retrieved content with citations, confidence scores, and bilingual disclaimers.

    import { LitElement, html, css } from 'lit';
import { customElement, property } from 'lit/decorators.js';

@customElement('eva-ai-response-card')
export class EvaAiResponseCard extends LitElement {
  @property({ type: String }) answer = '';
  @property({ type: Array }) citations = [];
  @property({ type: Number }) confidence = 0;
  @property({ type: String }) lang = 'en';

  render() {
    return html`
      <eva-gc-card>
        <div slot="header">
          <eva-gc-ai-content-indicator></eva-gc-ai-content-indicator>
          <eva-gc-ai-disclaimer
            lang=${this.lang}
          ></eva-gc-ai-disclaimer>
        </div>

        <div class="answer">
          ${this.answer}
        </div>

        <div class="citations">
          <h4>${this.lang === 'en' ? 'Sources' : 'Sources'}</h4>
          ${this.citations.map(cite => html`
            <eva-gc-citation-card
              title-en=${cite.titleEn}
              title-fr=${cite.titleFr}
              confidence=${cite.confidence}
              source-url=${cite.url}
              excerpt-en=${cite.excerptEn}
              excerpt-fr=${cite.excerptFr}
            ></eva-gc-citation-card>
          `)}
        </div>

        <div slot="footer">
          <eva-gc-badge
            label-en="Confidence: ${this.confidence}%"
            label-fr="Confiance : ${this.confidence} %"
            variant=${this.confidence > 80 ? 'success' : 'warning'}
          ></eva-gc-badge>
        </div>
      </eva-gc-card>
    `;
  }
}

    RBAC Navigation¶

    Navigation menu with role-based access control, showing/hiding items based on user permissions.

    import { LitElement, html } from 'lit';
import { customElement, property } from 'lit/decorators.js';

@customElement('eva-rbac-navigation')
export class EvaRbacNavigation extends LitElement {
  @property({ type: Object }) user = null;

  _hasRole(role) {
    return this.user?.roles?.includes(role);
  }

  render() {
    return html`
      <eva-gc-menu>
        <eva-gc-menu-item
          href="/"
          label-en="Home"
          label-fr="Accueil"
        ></eva-gc-menu-item>

        ${this._hasRole('viewer') ? html`
          <eva-gc-menu-item
            href="/reports"
            label-en="Reports"
            label-fr="Rapports"
          ></eva-gc-menu-item>
        ` : ''}

        ${this._hasRole('editor') ? html`
          <eva-gc-menu-item
            href="/editor"
            label-en="Editor"
            label-fr="Éditeur"
          ></eva-gc-menu-item>
        ` : ''}

        ${this._hasRole('admin') ? html`
          <eva-gc-menu-item
            href="/admin"
            label-en="Administration"
            label-fr="Administration"
          ></eva-gc-menu-item>
        ` : ''}
      </eva-gc-menu>
    `;
  }
}

    Best Practices¶

    Component Composition¶

    ✅ DO: Compose small, focused components into larger patterns ❌ DON'T: Create monolithic components with excessive props

    Prop Naming¶

    ✅ DO: Use bilingual prop pairs (label-en, label-fr) ❌ DON'T: Use single label prop with conditional logic

    Event Naming¶

    ✅ DO: Use eva-{component}-{action} pattern ❌ DON'T: Use generic names like click or change

    Slot Usage¶

    ✅ DO: Use named slots for multiple insertion points ❌ DON'T: Rely solely on default slot for complex layouts

    Summary¶

    EVA Sovereign UI provides five categories of components (Forms, Navigation, Feedback, Layout, Data Display) following the Lit reactive lifecycle. Components communicate via props (data in), slots (content projection), and events (data out). Real-world composition patterns like AI Response Cards, Bilingual Forms, RBAC Navigation, and Data Tables demonstrate how atomic components compose into production-ready interfaces.
