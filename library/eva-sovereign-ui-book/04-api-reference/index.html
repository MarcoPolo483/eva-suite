    Chapter 04: API Reference¶

    This chapter provides comprehensive API documentation for EVA Sovereign UI, including property decorators, event system conventions, CSS custom properties, component-specific APIs, global design tokens, and TypeScript integration patterns. Use this reference to understand component contracts, customize styling, and implement type-safe applications.

    Property Decorators¶

    EVA Sovereign UI uses Lit's @property() decorator to define reactive properties. Properties automatically trigger re-rendering when changed, enabling declarative state management without manual DOM manipulation.

    Basic Property Declaration¶

    import { LitElement, html } from 'lit';
import { customElement, property } from 'lit/decorators.js';

@customElement('eva-gc-button')
export class EvaGcButton extends LitElement {
  @property({ type: String, attribute: 'label-en' })
  labelEn = '';

  @property({ type: String, attribute: 'label-fr' })
  labelFr = '';

  @property({ type: Boolean })
  disabled = false;

  @property({ type: String })
  variant: 'primary' | 'secondary' | 'tertiary' = 'primary';

  @property({ type: String })
  size: 'small' | 'medium' | 'large' = 'medium';
}

    Decorator Options¶

    type - TypeScript type for property conversion (String, Number, Boolean, Array, Object)

    attribute - HTML attribute name (default: kebab-case of property name)

    reflect - Sync property changes back to attribute (default: false)

    converter - Custom serialization/deserialization logic

    hasChanged - Custom equality check (default: strict equality)

    Bilingual Property Pattern¶

    All text-bearing components use paired bilingual properties with -en and -fr suffixes:

    @property({ type: String, attribute: 'label-en' })
labelEn = '';

@property({ type: String, attribute: 'label-fr' })
labelFr = '';

@property({ type: String, attribute: 'placeholder-en' })
placeholderEn = '';

@property({ type: String, attribute: 'placeholder-fr' })
placeholderFr = '';

@property({ type: String, attribute: 'error-en' })
errorEn = '';

@property({ type: String, attribute: 'error-fr' })
errorFr = '';

    Usage:

    <eva-gc-input
  label-en="Email Address"
  label-fr="Adresse courriel"
  placeholder-en="your.email@example.com"
  placeholder-fr="votre.courriel@exemple.com"
  error-en="Invalid email format"
  error-fr="Format de courriel non valide"
></eva-gc-input>

    Language Detection¶

    Components detect active language using this hierarchy:

    1. Explicit lang prop on component
    2. Closest ancestor [lang] attribute
    3. Document <html lang="..."> attribute
    4. Browser navigator.language
    _getLanguage() {
  return this.lang ||
         this.closest('[lang]')?.getAttribute('lang') ||
         document.documentElement.lang ||
         navigator.language.split('-')[0] ||
         'en';
}

_getLabel() {
  const lang = this._getLanguage();
  return lang === 'fr' ? this.labelFr : this.labelEn;
}

    Event System¶

    EVA Sovereign UI uses CustomEvents for component communication. All events follow a consistent naming convention and bubble through Shadow DOM boundaries.

    Event Naming Convention¶

    Pattern: eva-{component}-{action}

    Examples:

    • eva-button-click - Button clicked
    • eva-input-change - Input value changed
    • eva-modal-open - Modal opened
    • eva-modal-close - Modal closed
    • eva-tab-select - Tab selected
    • eva-table-sort - Table sorted

    Event Dispatch¶

    _handleClick(e: MouseEvent) {
  if (this.disabled) return;

  this.dispatchEvent(new CustomEvent('eva-button-click', {
    bubbles: true,
    composed: true,
    detail: {
      variant: this.variant,
      size: this.size,
      timestamp: Date.now()
    }
  }));
}

    Options:

    • bubbles: true - Event propagates up DOM tree
    • composed: true - Event crosses Shadow DOM boundaries
    • detail: {...} - Custom event payload

    Event Listeners¶

    HTML Attribute:

    <eva-gc-button
  label-en="Save"
  @eva-button-click=${this.handleSave}
></eva-gc-button>

    JavaScript addEventListener:

    const button = document.querySelector('eva-gc-button');
button.addEventListener('eva-button-click', (e) => {
  console.log('Button clicked:', e.detail);
});

    React Synthetic Events:

    <eva-gc-button
  label-en="Save"
  ref={(el) => {
    if (el) {
      el.addEventListener('eva-button-click', this.handleSave);
    }
  }}
/>

    Event Detail Payloads¶

    Each event includes a detail object with relevant context:

    // eva-input-change
{
  value: string;
  name: string;
  valid: boolean;
}

// eva-modal-close
{
  reason: 'escape' | 'backdrop' | 'close-button' | 'action';
}

// eva-tab-select
{
  index: number;
  tabId: string;
}

// eva-table-sort
{
  column: string;
  direction: 'asc' | 'desc';
}

    CSS Custom Properties¶

    EVA Sovereign UI exposes CSS custom properties (CSS variables) for theming and customization. Properties follow a hierarchical token system with global design tokens and component-specific overrides.

    Token Hierarchy¶

    Level 1: Global Design Tokens - Defined in :root, used across all components

    Level 2: Component Tokens - Defined in component :host, reference global tokens

    Level 3: State Tokens - Defined in :host([state]), reference component tokens

    Global Design Tokens¶

    See Global Design Tokens section below for complete reference.

    Component-Specific Tokens¶

    Each component exposes CSS custom properties for granular customization:

    eva-gc-button {
  --eva-button-bg-color: var(--gc-color-primary);
  --eva-button-text-color: var(--gc-color-text-inverse);
  --eva-button-border-radius: var(--gc-border-radius-md);
  --eva-button-padding: var(--gc-spacing-sm) var(--gc-spacing-md);
  --eva-button-font-size: var(--gc-font-size-md);
  --eva-button-font-weight: var(--gc-font-weight-semibold);
  --eva-button-hover-bg-color: var(--gc-color-primary-dark);
}

    Usage:

    /* Customize all buttons */
eva-gc-button {
  --eva-button-border-radius: 0;
  --eva-button-padding: 12px 24px;
}

/* Customize specific button */
eva-gc-button.cta {
  --eva-button-bg-color: #ff6b35;
  --eva-button-hover-bg-color: #ff5722;
}

    Component API: eva-gc-button¶

    Primary action button with GC Design System styling and bilingual support.

    Properties¶

    Property Type Default Description
    labelEn string '' English button label
    labelFr string '' French button label
    variant 'primary' \| 'secondary' \| 'tertiary' 'primary' Visual style variant
    size 'small' \| 'medium' \| 'large' 'medium' Button size
    disabled boolean false Disable button interaction
    type 'button' \| 'submit' \| 'reset' 'button' Button type for forms
    fullWidth boolean false Expand to container width
    icon string undefined Icon name (optional)
    iconPosition 'start' \| 'end' 'start' Icon placement
    loading boolean false Show loading spinner
    lang string undefined Explicit language override

    Events¶

    Event Detail Description
    eva-button-click { variant: string, size: string, timestamp: number } Fired when button clicked

    Methods¶

    Method Signature Description
    focus() () => void Programmatically focus button
    blur() () => void Remove focus from button
    click() () => void Trigger click event

    CSS Custom Properties¶

    Token Default Description
    --eva-button-bg-color var(--gc-color-primary) Background color
    --eva-button-text-color var(--gc-color-text-inverse) Text color
    --eva-button-border-color transparent Border color
    --eva-button-border-radius var(--gc-border-radius-md) Corner radius
    --eva-button-padding var(--gc-spacing-sm) var(--gc-spacing-md) Internal spacing
    --eva-button-font-size var(--gc-font-size-md) Text size
    --eva-button-font-weight var(--gc-font-weight-semibold) Text weight
    --eva-button-hover-bg-color var(--gc-color-primary-dark) Hover background
    --eva-button-focus-outline-color var(--gc-color-focus) Focus indicator color

    Example¶

    <eva-gc-button
  label-en="Submit Application"
  label-fr="Soumettre la demande"
  variant="primary"
  size="large"
  type="submit"
  icon="check"
  @eva-button-click=${this.handleSubmit}
></eva-gc-button>

    Component API: eva-gc-input¶

    Text input field with validation, bilingual labels, and accessible error messaging.

    Properties¶

    Property Type Default Description
    labelEn string '' English label
    labelFr string '' French label
    name string '' Form field name
    value string '' Input value
    type 'text' \| 'email' \| 'password' \| 'tel' \| 'url' \| 'number' 'text' Input type
    placeholderEn string '' English placeholder
    placeholderFr string '' French placeholder
    required boolean false Mark as required
    disabled boolean false Disable input
    readonly boolean false Make read-only
    pattern string undefined Validation regex pattern
    minlength number undefined Minimum length
    maxlength number undefined Maximum length
    autocomplete string undefined Autocomplete hint
    errorEn string '' English error message
    errorFr string '' French error message
    helpEn string '' English help text
    helpFr string '' French help text
    showCharacterCount boolean false Display character counter
    lang string undefined Language override

    Events¶

    Event Detail Description
    eva-input-change { value: string, name: string, valid: boolean } Value changed
    eva-input-focus { name: string } Input focused
    eva-input-blur { name: string, valid: boolean } Input blurred

    Methods¶

    Method Signature Description
    focus() () => void Focus input
    validate() () => boolean Trigger validation
    reset() () => void Clear value

    CSS Custom Properties¶

    Token Default Description
    --eva-input-bg-color var(--gc-color-background) Background color
    --eva-input-text-color var(--gc-color-text) Text color
    --eva-input-border-color var(--gc-color-border) Border color
    --eva-input-border-radius var(--gc-border-radius-sm) Corner radius
    --eva-input-padding var(--gc-spacing-sm) Internal spacing
    --eva-input-font-size var(--gc-font-size-md) Text size
    --eva-input-focus-border-color var(--gc-color-focus) Focus border
    --eva-input-error-border-color var(--gc-color-error) Error border
    --eva-input-error-text-color var(--gc-color-error) Error text

    Example¶

    <eva-gc-input
  name="email"
  label-en="Email Address"
  label-fr="Adresse courriel"
  type="email"
  required
  placeholder-en="your.email@example.com"
  placeholder-fr="votre.courriel@exemple.com"
  error-en="Please enter a valid email"
  error-fr="Veuillez saisir un courriel valide"
  help-en="We'll never share your email"
  help-fr="Nous ne partagerons jamais votre courriel"
  @eva-input-change=${this.handleEmailChange}
></eva-gc-input>

    Component API: eva-gc-modal¶

    Accessible modal dialog with focus trapping and keyboard navigation.

    Properties¶

    Property Type Default Description
    titleEn string '' English modal title
    titleFr string '' French modal title
    open boolean false Modal visibility state
    size 'small' \| 'medium' \| 'large' \| 'full' 'medium' Modal size
    closeOnBackdrop boolean true Close on backdrop click
    closeOnEscape boolean true Close on Escape key
    showCloseButton boolean true Display close button
    preventScroll boolean true Prevent body scroll when open
    lang string undefined Language override

    Slots¶

    Slot Description
    (default) Modal body content
    footer Modal footer (typically action buttons)

    Events¶

    Event Detail Description
    eva-modal-open { timestamp: number } Modal opened
    eva-modal-close { reason: 'escape' \| 'backdrop' \| 'close-button' \| 'action' } Modal closed

    Methods¶

    Method Signature Description
    show() () => void Open modal
    hide() () => void Close modal
    toggle() () => void Toggle open state

    CSS Custom Properties¶

    Token Default Description
    --eva-modal-backdrop-color rgba(0, 0, 0, 0.5) Backdrop overlay
    --eva-modal-bg-color var(--gc-color-background) Modal background
    --eva-modal-border-radius var(--gc-border-radius-lg) Corner radius
    --eva-modal-padding var(--gc-spacing-lg) Internal spacing
    --eva-modal-max-width 600px (medium) Maximum width
    --eva-modal-shadow 0 4px 24px rgba(0,0,0,0.15) Drop shadow

    Example¶

    <eva-gc-modal

  open=${this.showConfirmModal}

  @eva-modal-close=${this.handleModalClose}
>

  <p>tes-vous sr de vouloir supprimer cet lment ?</p>

  <div slot="footer">
    <eva-gc-button
      label-en="Cancel"
      label-fr="Annuler"
      variant="secondary"
      @eva-button-click=${this.handleCancel}
    ></eva-gc-button>
    <eva-gc-button
      label-en="Delete"

    ></eva-gc-button>

    TypeScript Integration¶

    EVA Sovereign UI is built with TypeScript and ships with complete type definitions for properties, events, and methods. This enables type-safe component usage, IDE autocompletion, and compile-time error detection.

    Type Definitions¶

    // Import component types
import type { EvaGcButton } from '@eva-suite/sovereign-ui';

// Type-safe property access
const button: EvaGcButton = document.querySelector('eva-gc-button')!;
button.labelEn = 'Click Me';  //  Type-safe
button.variant = 'primary';   //  Type-safe
button.variant = 'invalid';   //  TypeScript error

    Event Typing¶

    import type { EvaButtonClickEvent } from '@eva-suite/sovereign-ui';

button.addEventListener('eva-button-click', (e: EvaButtonClickEvent) => {
  console.log(e.detail.variant);   //  Type-safe detail access
  console.log(e.detail.timestamp); //  Type-safe
  console.log(e.detail.invalid);   //  TypeScript error
});

    Component Inheritance¶

    import { LitElement, html } from 'lit';
import { customElement, property } from 'lit/decorators.js';
import type { EvaGcButton } from '@eva-suite/sovereign-ui';

@customElement('custom-button')
export class CustomButton extends LitElement {
  @property({ type: String })
  action = '';

  render() {
    return html`
      <eva-gc-button
        label-en="Submit"
        variant="primary"
        @eva-button-click=${this._handleClick}
      ></eva-gc-button>
    `;
  }

  private _handleClick(e: CustomEvent) {
    console.log('Action:', this.action);
  }
}

    React TypeScript Integration¶

    import React, { useRef, useEffect } from 'react';
import type { EvaGcButton, EvaButtonClickEvent } from '@eva-suite/sovereign-ui';

const MyComponent: React.FC = () => {
  const buttonRef = useRef<EvaGcButton>(null);

  useEffect(() => {
    const button = buttonRef.current;
    if (!button) return;

    const handleClick = (e: EvaButtonClickEvent) => {
      console.log('Clicked:', e.detail.variant);
    };

    button.addEventListener('eva-button-click', handleClick);
    return () => button.removeEventListener('eva-button-click', handleClick);
  }, []);

  return (
    <eva-gc-button
      ref={buttonRef}
      label-en="Save"
      label-fr="Enregistrer"
      variant="primary"
    />
  );
};

    Vue TypeScript Integration¶

    <template>
  <eva-gc-button
    ref="buttonRef"
    :label-en="labelEn"
    :label-fr="labelFr"
    variant="primary"
  />
</template>

<script setup lang="ts">
import { ref, onMounted, onUnmounted } from 'vue';
import type { EvaGcButton, EvaButtonClickEvent } from '@eva-suite/sovereign-ui';

const buttonRef = ref<EvaGcButton | null>(null);
const labelEn = ref('Save');
const labelFr = ref('Enregistrer');

const handleClick = (e: EvaButtonClickEvent) => {
  console.log('Clicked:', e.detail.variant);
};

onMounted(() => {
  buttonRef.value?.addEventListener('eva-button-click', handleClick);
});

onUnmounted(() => {
  buttonRef.value?.removeEventListener('eva-button-click', handleClick);
});
</script>

    Summary¶

    EVA Sovereign UI provides a comprehensive API with:

    • Property decorators using @property() for reactive state
    • Consistent event naming (eva-{component}-{action})
    • CSS custom properties for theming (component tokens + global design tokens)
    • Complete component APIs with properties tables, events, methods, CSS tokens
    • Global design tokens (color, typography, spacing, breakpoints, shadows)
    • TypeScript integration with full type definitions for type-safe development
