    10.2 Migration Guides¶

    From React Components¶

    If you're migrating from React, replace JSX components with Web Components:

    Before (React): 

    import { Button } from '@gc-design-system/react';

function App() {
  return <Button variant="primary" onClick={handleClick}>Submit</Button>;
}

    After (Web Components): 

    import '@eva-sovereign-ui/components/eva-gc-button';

function App() {
  return html`<eva-gc-button variant="primary" @click=${handleClick}>Submit</eva-gc-button>`;
}

    Key Differences:

    • Use kebab-case attributes instead of camelCase props
    • Use @event syntax for event listeners (Lit) or .addEventListener() (vanilla JS)
    • Access properties via el.variant instead of props.variant
    • Use <slot> for children instead of {children}

    From Bootstrap/Material UI¶

    Mapping common components:

    Bootstrap/Material UI EVA Sovereign UI Notes
    <Button> <eva-gc-button> Use variant prop
    <Form.Control> <eva-gc-text-input> Use type prop
    <Alert> <eva-gc-alert> Use severity prop
    <Card> <eva-gc-card> Use slots for header/footer
    <Modal> <eva-gc-modal> Use open property
    <Tabs> <eva-gc-tab-set> Use <eva-gc-tab> children

    Migration CLI¶

    Use eva-migrate CLI to automate migrations:

    npx eva-migrate react ./src
# Scans React components and suggests EVA Sovereign UI replacements

npx eva-migrate bootstrap ./src
# Scans Bootstrap HTML and suggests EVA Sovereign UI replacements

    Output: 

    Found 12 components to migrate:
- Button (8 instances) → eva-gc-button
- Alert (4 instances) → eva-gc-alert

Run 'npx eva-migrate react ./src --write' to apply changes.

    10.3 Troubleshooting¶

    Component Not Rendering¶

    Problem: Component appears in DOM but doesn't render visually.

    Solutions:

    1. Verify import: Ensure component is imported before use: 

      import '@eva-sovereign-ui/components/eva-gc-button';

    2. Check Shadow DOM: Open DevTools, find element, expand Shadow DOM tree.

    3. Verify styles: Ensure styles.ts is imported and adoptedStyleSheets is set.

    4. Check browser support: EVA Sovereign UI requires Chrome 120+, Edge 120+, Firefox 121+, Safari 17+.

    WCAG Violations¶

    Problem: axe-core reports WCAG violations.

    Common Issues:

    1. Missing labels: 

      // ❌ Bad
<eva-gc-text-input></eva-gc-text-input>

// ✅ Good
<eva-gc-text-input label="Email" label-fr="Courriel"></eva-gc-text-input>

    2. Low color contrast:

    3. Use --eva-color-text-primary (4.5:1 contrast) instead of --eva-color-text-muted (3:1 contrast) for body text.

    4. Missing focus indicators:

    5. Ensure --eva-focus-ring is applied to all interactive elements.

    6. Unlabeled form fields:

    7. Use <label> element or aria-label attribute.

    Bilingual Text Missing¶

    Problem: Component shows English text when lang="fr".

    Solutions:

    1. Set both labels: 

      <eva-gc-button label="Submit" label-fr="Soumettre"></eva-gc-button>

    2. Verify lang attribute: 

      <html lang="fr">

    3. Use slots for complex content: 

      <eva-gc-alert>
  <span slot="label-en">An error occurred</span>
  <span slot="label-fr">Une erreur s'est produite</span>
</eva-gc-alert>

    10.4 Browser Support¶

    EVA Sovereign UI targets modern browsers with native Web Components support:

    Browser Minimum Version Notes
    Chrome 120+ Full support
    Edge 120+ Full support (Chromium-based)
    Firefox 121+ Full support
    Safari 17+ Full support
    IE 11 ❌ NOT SUPPORTED Use polyfills or server-side rendering

    Polyfills (for older browsers):

    <script src="https://unpkg.com/@webcomponents/webcomponentsjs@2.8.0/webcomponents-loader.js"></script>

    Feature Detection:

    if ('customElements' in window) {
  // Web Components supported
  import('@eva-sovereign-ui/components/eva-gc-button');
} else {
  // Fallback to React/Vue components
  console.error('Web Components not supported');
}

    10.7 Glossary¶

    ACP (Agentic Communication Protocol): Protocol for agent-to-agent communication in EVA 2.0 ecosystem. Extends MCP with autonomy-aware routing.

    axe-core: Open-source accessibility testing engine. Detects WCAG violations in rendered HTML. Integrated into EVA Sovereign UI test suite.

    C0-C3 (Autonomy Levels): GC Agentic Framework classification for AI agent decision-making authority: - C0: Advisory only (human approves all actions) - C1: Conditional autonomy (pre-approved actions only) - C2: Supervised autonomy (human monitors, can override) - C3: Full autonomy (human oversight, post-audit)

    CMK (Customer-Managed Key): Encryption key stored in Azure Key Vault. Required for Protected B data. Supports 90-day automatic rotation.

    GC Design System: Canada.ca visual identity and UI patterns. EVA Sovereign UI implements GC Design System with canada-gc theme.

    Lit: Lightweight web component framework. Uses tagged template literals (html\...``) for declarative rendering. Powers EVA Sovereign UI components.

    MCP (Model Context Protocol): Protocol for AI model communication. EVA Foundation uses MCP for orchestrating Azure OpenAI, Cohere, and future LLM providers.

    Protected B: Canadian government security classification for sensitive information (e.g., SIN, medical records). Requires encryption at rest and in transit.

    RBAC (Role-Based Access Control): Authorization model where permissions are assigned to roles (Admin, Editor, Viewer). EVA Foundation enforces RBAC via Azure AD.

    Shadow DOM: Encapsulated DOM tree attached to Web Component. Scopes styles and prevents CSS leakage. All EVA Sovereign UI components use Shadow DOM.

    Slot: Web Component feature for content projection. Named slots (<slot name="header">) allow flexible composition. Replaces React {children}.

    WCAG (Web Content Accessibility Guidelines): W3C standard for web accessibility. WCAG 2.1 AA is mandatory for GC web applications. EVA Sovereign UI enforces WCAG 2.1 AA compliance.

    Web Components: Browser-native component model (Custom Elements + Shadow DOM + HTML Templates). Framework-agnostic, runs in any JavaScript environment.
