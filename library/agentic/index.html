    üéØ Key Concepts Defined¶

    Agentic Maturity Model¶

    • L0: Non-Agentic (RAG Q&A only)
    • L1: Bounded Tool Helpers (1-3 steps, approved tools)
    • L2: Workflow Agents (3-10 steps, HITL, audit trail) ‚Üê Production Ready
    • L3: Open Multi-Agent (high autonomy) ‚Üê Sandbox Only

    Agent Runtime (Shared 7-Step Pipeline)¶

    1. Trigger ‚Üí Intent Resolution
    2. Context Gathering (repo, docs, policies)
    3. Policy & Guardrail Check (kill switch, autonomy level)
    4. LLM Reasoning (EVA with role-specific prompts)
    5. Result Validation (schema enforcement)
    6. Effectors (PR comments, docs, reports)
    7. Logging & Metrics (full traceability)

    Project Space (GC Template)¶

    Logical container defining: - Business purpose & objectives - Data sources & security classification - Tools/APIs & RAG indices available - RBAC for users & services - Agents configured within constraints - Governance artifacts (privacy, risk)

    POD Structure¶

    • POD-F: Foundation (P01, P02, P04-P11, P15)
    • POD-X: Experience (P02, P03, P04, P06, P07, P10, P12, P15)
    • POD-O: Operations (P07-P11, P14, P15)
    • POD-S: Solutions (All P01-P15 available)
