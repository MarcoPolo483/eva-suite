    CDD: EVA Analytics - Government Data Integration Framework¶

    Last Updated: 2024-12-08 Status: Backlog Priority: HIGH Epic: EVA Analytics Pod: POD-F (Foundation Libraries + Data Services)

    Vision¶

    Integrate live government data sources (Statistics Canada, Open Canada, provincial APIs) across all EVA Analytics dashboards—enabling users to query real-time economic indicators, labour statistics, demographic data, and policy metrics using natural language, while maintaining full accessibility compliance and demonstrating measurable ROI with actual government data.

    Business Value: Transform static analytics dashboards into live, data-driven decision support systems. Reduce time to insight by 70% for policy analysis, employment planning, and program evaluation. Demonstrate $2M+ ROI with concrete government data. Enable 15% of workforce with disabilities to independently access analytics through screen readers and voice control.

    Core Capabilities¶

    Live Data Integration (CE: Context-Aware Government Data)¶

    • Statistics Canada WDS API: 8,022 datasets (employment, economic indicators, demographics)
    • Open Canada Portal: 90,000+ open datasets (healthcare, education, environment)
    • Provincial Data Sources: BC Data Catalogue, Ontario Data Catalogue, Quebec Données Québec
    • Real-Time Updates: Automated daily ingestion at 8:30 AM EST (StatCan), hourly for other sources
    • Hybrid RAG: Combine indexed historical data with live API calls for temporal queries
    • Multi-Source Federation: Query across multiple government sources in single query

    Accessibility Features (HK: High-Contrast & Keyboard Navigation)¶

    • Screen Reader Optimization: ARIA labels for all metrics, KPIs, charts
    • Voice Control: Hands-free dashboard navigation using Azure Speech Services
    • Keyboard Navigation: Full feature access without mouse (Alt+E, Tab, Enter, Escape)
    • High-Contrast Themes: 7:1 contrast ratio for all analytics visuals (WCAG AAA)
    • Alternative Text Generation: GPT-4 Vision describes all charts and trends
    • Narration Customization: Speed (0.5x-2x), verbosity (brief/standard/detailed)

    Natural Language Queries (WM: What-Why-How Methodology)¶

    • What Queries: "What is current unemployment?" → Live StatCan API → "6.5% Nov 2024"
    • Why Queries: "Why did platform costs spike?" → Analyzes Azure metering + usage patterns → "40% increase in RAG queries"
    • How Queries: "How does my space compare to others?" → Comparative analysis → Side-by-side charts
    • Trend Analysis: "Show employment trend 2020-2024" → Multi-year line chart with annotations
    • Predictive Queries: "Forecast next quarter costs" → Azure ML predictions with confidence intervals

    Integration Points Across EVA Analytics Dashboards¶

    1. Command Center Dashboard (Platform Health + Economic Context)¶

    Integration Scenarios:

    Scenario 2: Government IT Benchmarking - Query: "How does our uptime compare to government IT standards?" - Data Sources: - Command Center: 99.9% uptime - TBS Directive on Service: 99.5% minimum standard - Open Canada: Public service digital standards - Analysis: Benchmark against government IT performance standards - Output: Gauge chart (EVA uptime vs. government standard), compliance indicator

    Hybrid RAG Query Router:

    class GovernmentDataQueryRouter:
    def route_query(self, question: str):
        """Route query to appropriate data source"""

        # Detect temporal keywords
        if any(word in question.lower() for word in ['current', 'latest', 'now', 'today']):
            return self.route_to_live_api(question)

        # Detect historical keywords
        elif any(year in question for year in ['2008', '2020', 'recession', 'crisis']):
            return self.route_to_rag(question)

        # Detect comparative keywords
        elif any(word in question.lower() for word in ['compare', 'vs', 'versus', 'benchmark']):
            return self.route_to_hybrid(question)

        else:
            return self.route_to_hybrid(question)  # Default: both sources

    Data Federation Engine:

    class DataFederationEngine:
    async def federated_query(self, question: str):
        """Query across multiple government sources"""

        # Parse query intent
        sources = self.identify_sources(question)

        # Parallel API calls
        results = await asyncio.gather(*[
            self.query_statcan(question),
            self.query_open_canada(question),
            self.query_provincial(question)
        ])

        # Merge results
        merged = self.merge_results(results)

        # Generate answer
        return await self.llm.generate(question, context=merged)

    Azure Synapse Integration:

    -- Federated query combining EVA metrics + StatCan data
CREATE VIEW v_EmploymentAnalytics AS
SELECT
    -- EVA metrics
    e.date,
    e.tenantId,
    e.spaceId,
    e.activeUsers,
    e.totalQueries,
    e.avgLatency,

    -- StatCan labour force data (joined by date)
    s.unemployment_rate,
    s.employment_level,
    s.participation_rate,

    -- Calculated fields
    e.activeUsers / NULLIF(s.employment_level, 0) * 1000000 AS users_per_million_employed,
    e.totalQueries / NULLIF(e.activeUsers, 0) AS queries_per_user
FROM EVAUsageMetrics e
LEFT JOIN StatCanLFSData s ON e.date = s.date
WHERE e.date >= DATEADD(year, -2, GETDATE())

    ARIA Label Generator:

    class AccessibilityLabelGenerator {
  generateMetricLabel(metric: Metric, governmentData: GovernmentData): string {
    const label = `${metric.name}: ${metric.value} ${metric.unit}`;

    const comparison = this.compareToGovernment(metric, governmentData);
    const trend = this.describeTrend(metric);

    return `${label}, ${comparison}, ${trend}`;

    // Example: "Unemployment rate: 6.5 percent, 2.2 points below 2008 crisis peak, declining trend over 6 months"
  }

  generateChartDescription(chart: Chart): string {
    const chartType = chart.type; // "line", "bar", "pie"
    const dataPoints = chart.data.length;
    const trend = this.analyzeTrend(chart.data);
    const extremes = this.findExtremes(chart.data);

    return `${chartType} chart with ${dataPoints} data points. ${trend}. Highest: ${extremes.max} in ${extremes.maxDate}. Lowest: ${extremes.min} in ${extremes.minDate}.`;

    // Example: "Line chart with 48 data points. Declining trend over 12 months. Highest: 7.1% in August 2024. Lowest: 6.5% in November 2024."
  }
}

    Voice Control Engine:

    class VoiceControlEngine {
  async processVoiceCommand(audio: AudioStream): Promise<QueryResponse> {
    // Azure Speech Services: Speech-to-Text
    const text = await this.speechToText(audio);

    // Parse command intent
    const intent = await this.parseIntent(text);

    // Execute query
    const response = await this.executeQuery(intent);

    // Azure Speech Services: Text-to-Speech
    const tts = await this.textToSpeech(response.summary);

    return {
      text: response,
      audio: tts
    };
  }

  // Bilingual support (English/French code-switching)
  detectLanguage(text: string): 'en' | 'fr' {
    // Auto-detect language or use explicit command
    return text.toLowerCase().includes('montre-moi') ? 'fr' : 'en';
  }
}

    Open Questions¶

    Q1: Government Data Licensing (LEGAL)¶

    Question: Do we need licenses to cache/redistribute government data? Impact: Legal risk vs. data freshness Recommendation: All government data covered by Open Government License (no restrictions). Cache allowed. Cite sources in analytics. Decision Owner: Security Sentinel

    Q2: Data Update Frequency (TECH)¶

    Question: How often to refresh government data? Daily, hourly, real-time? Impact: Freshness vs. API load Recommendation: StatCan daily at 8:31 AM EST (1 min after release). Open Canada hourly. Provincial daily. Decision Owner: ai2

    Q3: Multi-Source Conflicts (DATA QUALITY)¶

    Question: What if StatCan and Open Canada have different values for same metric? Impact: Data accuracy vs. complexity Recommendation: Prioritize official statistical agencies (StatCan > Provincial > Open Canada). Flag conflicts in UI. Decision Owner: ai2

    Q4: Privacy for Comparative Analytics (PRIVACY)¶

    Question: Show space names in rankings or anonymize? Impact: Transparency vs. privacy Recommendation: Anonymize by default (Space A, Space B). Allow opt-in for friendly competition. Decision Owner: Security Sentinel

    Q5: Voice Control Language Detection (UX)¶

    Question: Auto-detect language or require explicit command? Impact: Convenience vs. accuracy Recommendation: Auto-detect with confidence threshold (>80%). Allow language switcher (say "English" or "Français"). Decision Owner: UX Partner

    Q6: Economic Indicators for All Dashboards? (SCOPE)¶

    Question: Should all 5 dashboards include economic context or just Command Center? Impact: Value vs. complexity Recommendation: Command Center (essential), Space Dashboard (optional), others (deferred to Sprint 15). Decision Owner: Marco (PO)
