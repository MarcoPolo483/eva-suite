    CDD: EVA Vision - Statistics Canada WDS Integration¶

    Last Updated: 2024-12-08 Status: Backlog Priority: MEDIUM Epic: EVA Vision (Power BI Plugin) Pod: POD-F (Foundation Libraries + Data Services)

    Vision¶

    Integrate Statistics Canada's Web Data Service (WDS) API with EVA Vision to provide real-time employment and economic data within Power BI dashboards—enabling users to query live labour force statistics, compare historical trends, and generate insights using natural language, while maintaining full accessibility compliance for users with disabilities.

    Business Value: Transform static employment dashboards into live, AI-powered analytics. Reduce time to insight by 70% for employment equity analysis, labour market research, and policy impact assessment. Enable 15% of workforce with disabilities to independently access employment data through screen readers, voice control, and keyboard navigation.

    Core Capabilities¶

    Data Integration (CE: Context-Aware Employment Data)¶

    • Live Labour Force Survey (LFS) Data: Real-time unemployment rate, employment level, participation rate (Vector IDs: 2062814, 2062815, 2062816)
    • Historical Analysis: 48+ years of LFS data (1976-2024) for trend analysis and policy impact studies
    • Multi-Dimensional Queries: Filter by Geography (11 provinces/territories), Gender (3), Age Group (9), Labour Force Characteristics (9)
    • Daily Data Updates: Automated ingestion at 8:31 AM EST (1 minute after StatCan release at 8:30 AM)
    • Hybrid RAG: Combine indexed historical documents with live WDS API calls for "current vs. 2008 recession" queries

    Accessibility Features (HK: High-Contrast & Keyboard Navigation)¶

    • Screen Reader Optimization: ARIA labels for employment metrics ("Unemployment rate: 6.5%, down 12% year-over-year")
    • Voice Control: Hands-free queries ("Show me unemployment trend for Ontario") using Azure Speech Services
    • Keyboard Navigation: Full feature access without mouse (Alt+E to open EVA chat, Tab through metrics)
    • High-Contrast Themes: 7:1 contrast ratio for charts, metrics, and text (WCAG AAA compliance)
    • Alternative Text Generation: Azure OpenAI GPT-4 Vision describes charts ("Line chart: Unemployment rate 2020-2024. Peak: 13.7% May 2020, Current: 6.5% Nov 2024")

    Natural Language Queries (WM: What-Why-How Methodology)¶

    • What Queries: "What is the current unemployment rate?" → Live API call → "6.5% as of November 2024"
    • Why Queries: "Why did unemployment spike in April 2020?" → Retrieves COVID-19 employment tables → "3 million jobs lost due to lockdowns"
    • How Queries: "How does Ontario employment compare to Quebec?" → Generates DAX measure → Side-by-side bar chart (Ontario: 7.2M, Quebec: 4.5M)
    • Trend Analysis: "Show me employment trend for women aged 25-54" → Applies filters (Gender=Female, Age=25-54) → Line chart
    • Comparative Analysis: "Compare current unemployment to 2008 financial crisis" → Hybrid RAG (live + historical data) → Comparison chart

    User Stories¶

    US-1: As an employment equity analyst, I want real-time disability employment data so that I monitor compliance¶

    Acceptance Criteria:

    • Open Power BI employment equity dashboard → EVA chat panel appears
    • Ask "What is the employment rate for persons with disabilities?" → EVA queries StatCan table 13100377
    • Response: "Employment rate for persons with disabilities (aged 25-64): 59.2% vs. 80.1% for persons without disabilities (2022 Canadian Survey on Disability)"
    • Click source reference → navigates to disability employment visual
    • Data auto-refreshes every 5 years (when new CSD data released)
    • Response time <3 seconds P95

    US-2: As a blind policy analyst, I want screen reader-optimized employment dashboards so that I analyze labour market independently¶

    Acceptance Criteria:

    • Open employment dashboard → Screen reader announces: "Labour Force Survey dashboard loaded. Unemployment rate: 6.5%, down 0.6 percentage points year-over-year"
    • Tab through metrics → Screen reader narrates each KPI (employment, unemployment, participation rate)
    • Ask "What are the top 3 provinces for employment?" → EVA responds with audio + high-contrast visual
    • Screen reader announces: "Ontario leads with 7.2 million employed, followed by Quebec at 4.5 million, British Columbia at 2.8 million"
    • Charts include ARIA labels and alternative text (auto-generated by GPT-4 Vision)
    • Full keyboard navigation (no mouse required)
    • Tested with JAWS, NVDA, VoiceOver (99% compatibility)

    US-3: As a labour market researcher, I want to compare current trends to historical crises so that I contextualize policy decisions¶

    Acceptance Criteria:

    • Ask "Compare current unemployment to 2008 financial crisis"
    • EVA retrieves: Live data (6.5% Nov 2024) + Indexed historical data (peak 8.7% June 2009)
    • Response: "Current unemployment (6.5%) is 2.2 percentage points lower than 2008 crisis peak (8.7%). Recovery patterns differ: 2008 recovery took 72 months, COVID recovery took 18 months."
    • Side-by-side line chart showing both periods
    • Click chart → drill down by province, age group, gender
    • Save analysis as bookmark (persist for future reference)

    US-4: As a business analyst, I want voice-controlled dashboard navigation so that I query data hands-free¶

    Acceptance Criteria:

    • Enable voice control (microphone icon in EVA chat)
    • Say "Show me employment trend for Ontario"
    • EVA applies filter (Geography=Ontario), retrieves LFS vector data, renders line chart
    • Voice response: "Ontario employment trend 2020-2024. Current: 7.2 million, up 8% from pandemic low of 6.6 million in May 2020."
    • Say "Compare to Quebec" → Chart updates to show Ontario vs. Quebec
    • Bilingual support (English/French code-switching): "Montre-moi le taux de chômage au Québec"
    • Azure Speech Services (Neural TTS, voice recognition)

    US-5: As a dashboard author, I want EVA to generate employment-specific DAX measures so that I build reports faster¶

    Acceptance Criteria:

    • Ask "Create a measure for year-over-year employment growth"
    • EVA generates DAX: YoY Employment Growth = DIVIDE([Employment], [Employment PY] - 1, 0)
    • Preview: "For November 2024: 1,477,800 vs. 1,505,500 in November 2023 = -1.8% decline"
    • Validate → Measure added to model (appears in Fields pane)
    • Explanation: "This measure calculates employment growth by comparing current month to same month last year. Negative values indicate decline."
    • Supports StatCan table dimensions (Geography, Gender, Age Group)

    Functional Requirements¶

    Data Integration (CE)¶

    1. FR-CE-1: SHALL connect to Statistics Canada WDS API (base URL: https://www150.statcan.gc.ca/t1/wds/rest/)
    2. FR-CE-2: SHALL retrieve live data for 3 core employment vectors (V2062814: Employment, V2062815: Unemployment Rate, V2062816: Participation Rate)
    3. FR-CE-3: SHALL query 765 active employment-related datasets (filtered from 8,022 total StatCan tables)
    4. FR-CE-4: SHALL support multi-dimensional queries (Geography, Gender, Age Group, Labour Force Characteristics)
    5. FR-CE-5: SHALL implement daily data ingestion at 8:31 AM EST (1 minute after StatCan release)
    6. FR-CE-6: SHALL cache frequently queried vectors (unemployment rate) with 24-hour TTL
    7. FR-CE-7: SHALL handle API rate limits (50 req/sec global, 25 per IP) with exponential backoff
    8. FR-CE-8: SHALL implement hybrid RAG (indexed historical data + live API calls)
    9. FR-CE-9: SHALL download full LFS table (14100287) monthly for offline analysis (25 MB zipped CSV)
    10. FR-CE-10: SHALL validate data quality codes (symbolCode, statusCode) and flag preliminary/revised data

    Accessibility Features (HK)¶

    1. FR-HK-1: SHALL provide ARIA labels for all employment metrics (e.g., "Unemployment rate, 6.5 percent, down 12 percent year-over-year")
    2. FR-HK-2: SHALL implement full keyboard navigation (no mouse required, logical tab order)
    3. FR-HK-3: SHALL support high-contrast themes with 7:1 contrast ratio (WCAG AAA)
    4. FR-HK-4: SHALL generate alternative text for charts using Azure OpenAI GPT-4 Vision
    5. FR-HK-5: SHALL implement voice control using Azure Speech Services (TTS + voice recognition)
    6. FR-HK-6: SHALL provide screen reader announcements for dynamic content (ARIA live regions)
    7. FR-HK-7: SHALL support keyboard shortcuts (Alt+E: open EVA chat, Escape: close)
    8. FR-HK-8: SHALL implement skip links ("Skip to main content", "Skip to EVA chat")
    9. FR-HK-9: SHALL support narration customization (speed 0.5x-2x, verbosity: brief/standard/detailed)
    10. FR-HK-10: SHALL test with JAWS, NVDA, VoiceOver, TalkBack (target 99% compatibility)

    Natural Language Queries (WM)¶

    1. FR-WM-1: SHALL respond to "What" queries with current data ("What is unemployment rate?" → "6.5% Nov 2024")
    2. FR-WM-2: SHALL respond to "Why" queries with explanatory analysis ("Why did unemployment spike?" → "COVID-19 lockdowns, 3M jobs lost")
    3. FR-WM-3: SHALL respond to "How" queries with comparative analysis ("How does Ontario compare to Quebec?" → Side-by-side chart)
    4. FR-WM-4: SHALL generate DAX measures from natural language ("Create YoY growth measure")
    5. FR-WM-5: SHALL validate generated DAX (syntax + semantic checks against LFS table schema)
    6. FR-WM-6: SHALL maintain session context (last 5 queries, current filters, active page)
    7. FR-WM-7: SHALL support bilingual queries (en-US, fr-FR) with bilingual responses
    8. FR-WM-8: SHALL provide drill-down options (by province, age group, gender, raw data)
    9. FR-WM-9: SHALL create ad-hoc visuals from queries (bar chart, line chart, table)
    10. FR-WM-10: SHALL cite data sources in responses ("Based on Labour Force Survey, Table 14100287")

    Non-Functional Requirements¶

    Performance¶

    • Query Response: <3s P95 (natural language → answer with visual)
    • Live Data Retrieval: <1s (WDS API call → data returned)
    • Historical Data Search: <2s (RAG query → indexed documents retrieved)
    • DAX Generation: <2s (natural language → validated DAX measure)
    • Screen Reader Narration: <500ms (element focus → announcement)
    • Voice Control Latency: <1s (speech input → query execution)

    Reliability¶

    • API Uptime: 99.9% SLA (aligned with StatCan WDS availability)
    • Data Freshness: 100% (daily ingestion at 8:31 AM EST, no missed updates)
    • DAX Validation Accuracy: >95% (syntax + semantic correctness)
    • Screen Reader Compatibility: >99% (JAWS, NVDA, VoiceOver, TalkBack)
    • Failover: Graceful degradation if WDS API unavailable (show cached data + staleness warning)

    Security¶

    • Authentication: Power BI workspace-level authentication (existing RLS applies)
    • Authorization: Users can only query data they have access to (RLS enforcement)
    • Data Privacy: Query logs anonymized (no PII, aggregated metrics only)
    • Audit Logging: All DAX measure creations logged to Cosmos DB (1-year retention)
    • API Rate Limiting: Client-side enforcement (25 req/sec per IP, exponential backoff)

    Accessibility¶

    • WCAG 2.1 AAA: Full compliance (automated + manual audit, accessibility statement published)
    • Screen Readers: Tested with JAWS, NVDA, VoiceOver, TalkBack (99% compatibility)
    • Keyboard Navigation: 100% feature coverage (all features accessible without mouse)
    • High-Contrast: Text contrast ≥7:1, visual elements visible in all themes
    • Narration: Adjustable speed (0.5x-2x), verbosity (brief, standard, detailed)

    Internationalization¶

    • Locales: en-US (default), fr-FR
    • DAX Generation: Supports English and French function names (CALCULATE vs. CALCULER)
    • Number Formatting: Respects Power BI report locale (currency, dates)
    • Screen Reader Languages: Support English and French narration
    • Voice Control: Bilingual commands (code-switching supported)

    Observability¶

    • Metrics: Application Insights tracks queries, WDS API calls, DAX generations, visual creations, accessibility feature usage
    • Alerts: Query latency >5s, WDS API errors >2%, DAX validation failures >10%, screen reader errors >1%
    • Dashboards: Power BI dashboard for plugin usage, top queries, WDS API health, accessibility adoption

    Scalability¶

    • Concurrent Users: Support 1K+ simultaneous Power BI users
    • Query Volume: Handle 10K+ queries/day (cached responses for unemployment rate, employment level)
    • WDS API Load: Batch requests where possible (max 25 vectors per query)
    • Data Volume: Index 765 employment tables (~2 GB total, incremental updates)

    Technical Architecture¶

    Frontend¶

    • Framework: Power BI Custom Visual SDK (TypeScript, D3.js)
    • UI Components: Chat interface (message list, input box, visual preview), accessibility controls
    • State Management: React hooks (local state), WebSocket (real-time updates)
    • Rendering: D3.js for ad-hoc visuals, Power BI SDK for dashboard interactions
    • Accessibility: ARIA attributes, semantic HTML, focus management, keyboard shortcuts

    Backend¶

    • API Gateway: Azure APIM (rate limiting, JWT validation)
    • Functions: Azure Functions (Node.js 20) for query processing, DAX generation, context extraction
    • StatCan Connector: Python service (statcan_connector.py) for WDS API integration
    • AI Service: Azure OpenAI GPT-4o (query understanding, DAX generation), GPT-4 Vision (alternative text)
    • Speech Service: Azure Speech Services (Neural TTS, voice recognition)
    • RAG Service: eva-rag (Azure AI Search, Cosmos DB) for historical data indexing
    • Validation: DAX.js library (syntax checking), Power BI Tabular Model (semantic validation)

    Implementation Phases¶

    Phase 1: StatCan Connector & Core Integration (Sprint 15, Week 1-3)¶

    • Create statcan_connector.py (WDS API wrapper)
    • Methods: get_latest_data(vector_id, periods), get_table_csv(product_id, lang), get_changed_cubes(date)
    • Rate limiting: 25 req/sec per IP, exponential backoff
    • Caching: Redis TTL 24 hours for unemployment rate, employment level
    • Index 765 employment tables in Azure AI Search (incremental daily updates)
    • Full LFS table (14100287): 25 MB zipped CSV, monthly refresh
    • Employment Insurance tables (14100004-14100006): 15 MB each
    • Disability employment (13100377): 8 MB
    • Implement hybrid RAG (live WDS API + indexed historical data)
    • Query router: "current" → live API, "2008" → indexed docs, "compare" → both
    • Context merging: Combine live + historical results in single response
    • Test with 3 core vectors (2062814, 2062815, 2062816)

    Phase 2: Natural Language Queries & DAX Generation (Sprint 15-16, Week 4-6)¶

    • Natural language → DAX translation (GPT-4o fine-tuned on LFS table schema)
    • Training data: 50 example queries + DAX measures for employment metrics
    • Validation: Syntax (DAX.js) + Semantic (check Geography, Gender, Age Group dimensions exist)
    • Context-aware query processing (detect current filters, page, visuals)
    • Power BI API: Read active slicers (Geography=Ontario), current page metadata
    • Session context: Maintain last 5 queries for multi-turn conversations
    • What/Why/How query handling
    • What: "What is unemployment rate?" → Live API call
    • Why: "Why did unemployment spike?" → Retrieve COVID-19 tables (PIDs 331002**)
    • How: "How does Ontario compare?" → Generate DAX, render side-by-side chart
    • Bilingual support (en-US, fr-FR) with code-switching

    Phase 3: Accessibility Features (Sprint 16-17, Week 7-9)¶

    • ARIA labels for employment metrics (KPI cards, charts, filters)
    • Example: <div role="region" aria-label="Unemployment rate, 6.5 percent, down 12 percent year-over-year">
    • Keyboard navigation (Alt+E to open chat, Tab through metrics, Enter to activate)
    • Logical tab order: KPIs → Charts → Filters → EVA Chat
    • Visible focus indicators: 3px high-contrast outline
    • High-contrast themes (7:1 contrast ratio, WCAG AAA)
    • Custom color palette: Dark mode, light mode, Windows High Contrast
    • Alternative text generation (GPT-4 Vision for charts)
    • Example: "Line chart showing unemployment rate 2020-2024. Peak: 13.7% May 2020. Current: 6.5% Nov 2024."
    • Voice control (Azure Speech Services)
    • Voice input → Text query → EVA processing → TTS response
    • Bilingual: English/French commands

    Phase 4: Testing, Compliance & Launch (Sprint 17-18, Week 10-12)¶

    • Screen reader testing (JAWS, NVDA, VoiceOver, TalkBack)
    • Test employment dashboard with blind users (internal accessibility team)
    • Fix ARIA issues, improve narration quality
    • WCAG 2.1 AAA audit (automated: axe-core, Lighthouse; manual: accessibility consultant)
    • Publish accessibility statement (conformance report)
    • Performance testing (1K+ concurrent users, 10K+ queries/day)
    • Load test WDS API integration (rate limiting, failover)
    • Power BI dashboard for adoption metrics
    • Queries per day, top vectors, WDS API health, accessibility feature usage
    • User guides & training
    • Video: "Querying Employment Data with EVA Vision"
    • Documentation: Keyboard shortcuts, voice commands, screen reader tips

    Success Criteria¶

    • ✅ Query response <3s P95 (natural language → answer with live StatCan data)
    • ✅ WDS API uptime >99.9% (failover to cached data if unavailable)
    • ✅ DAX validation >95% accuracy (syntax + semantic checks)
    • ✅ Screen reader compatibility >99% (JAWS, NVDA, VoiceOver, TalkBack)
    • ✅ WCAG 2.1 AAA compliance (automated + manual audit passed)
    • ✅ User adoption >40% of employment analysts within 6 months
    • ✅ Accessibility adoption >50% of users with disabilities within 6 months
    • ✅ Time to insight reduction >70% (vs. manual dashboard exploration)
    • ✅ User satisfaction >80% (post-query survey)
