    CDD: EVA Data Analysis Copilot Tool¶

    Last Updated: 2025-11-19 Status: Backlog Priority: MEDIUM Epic: EVA Tools Suite

    Vision¶

    Provide an AI-powered data analysis assistant that generates SQL queries from natural language, creates visualizations, identifies insights, and automates reportingâ€”enabling business users to analyze data without SQL expertise.

    Business Value: Reduce data analysis time by 60%, democratize data access, enable self-service analytics, and accelerate data-driven decision-making.

    Core Capabilities¶

    • Natural Language to SQL: Convert questions to SQL queries ("Show top 10 customers by revenue")
    • Query Execution: Run queries against connected databases (Azure SQL, Synapse, Databricks)
    • Auto-Visualization: Generate charts (bar, line, pie, scatter) based on query results
    • Insight Detection: Identify trends, outliers, correlations, anomalies
    • Report Generation: Create automated reports (PDF, PowerPoint) with charts and commentary
    • Data Exploration: Browse schema, preview tables, understand relationships
    • Query History: Save and reuse queries, share with team members
    • Export Options: Download results as CSV, Excel, JSON, or sync to Power BI

    User Stories¶

    US-1: As a business analyst, I want SQL from questions so that I query without coding¶

    Acceptance Criteria:

    • Ask: "Show top 10 customers by revenue in 2025"
    • EVA generates SQL: SELECT TOP 10 CustomerName, SUM(Revenue) FROM Sales WHERE Year=2025 GROUP BY CustomerName ORDER BY SUM(Revenue) DESC
    • Preview query before execution (confirm or edit)
    • Execute query and display results in table
    • Export results to Excel

    US-2: As a manager, I want auto-visualizations so that I see trends quickly¶

    Acceptance Criteria:

    • Query returns sales by month (12 rows)
    • EVA auto-suggests chart type: Line chart (time series)
    • Generate chart with customizable labels, colors
    • Toggle chart types (bar, line, area)
    • Export chart as PNG or embed in report

    US-3: As a data scientist, I want insight detection so that I find anomalies¶

    Acceptance Criteria:

    • Query returns 1000 rows of user activity
    • EVA identifies: "Spike in signups on Nov 15 (+120%)", "Outlier: User ID 789 with 500x avg sessions"
    • Highlight anomalies in table (red background)
    • Provide explanations with confidence scores
    • Allow user to investigate anomalies (drill-down queries)

    US-4: As an executive, I want automated reports so that I get weekly summaries¶

    Acceptance Criteria:

    • Define report: "Weekly sales summary with top products, revenue trend, customer growth"
    • Schedule report (every Monday 9am)
    • EVA generates PDF with charts, commentary, insights
    • Email report to distribution list
    • Archive reports in SharePoint

    US-5: As a developer, I want schema exploration so that I understand database structure¶

    Acceptance Criteria:

    • Browse database schema (tables, columns, types)
    • View sample data (first 10 rows per table)
    • Visualize table relationships (foreign keys)
    • Search for tables/columns by keyword
    • Export schema diagram as PNG

    Functional Requirements¶

    FR-1: Natural Language to SQL¶

    System SHALL generate SQL queries from natural language using GPT-4o with schema context (tables, columns, types).

    FR-2: Database Connections¶

    System SHALL connect to Azure SQL Database, Azure Synapse Analytics, Databricks SQL Warehouse using secure connection strings.

    FR-3: Query Execution¶

    System SHALL execute SQL queries with timeout (30 seconds), row limit (10K rows), and error handling.

    FR-4: Visualization¶

    System SHALL auto-generate charts (bar, line, pie, scatter, heatmap) using Chart.js based on query result types.

    FR-5: Insight Detection¶

    System SHALL analyze results using GPT-4o to identify trends, outliers, correlations, anomalies with confidence scores.

    FR-6: Report Generation¶

    System SHALL create automated reports (PDF, PowerPoint) with charts, tables, and AI-generated commentary.

    FR-7: Schema Exploration¶

    System SHALL provide UI to browse database schema, preview data, and visualize relationships (ER diagram).

    FR-8: Query History¶

    System SHALL save query history per user (30-day retention) with ability to star favorites and share with team.

    FR-9: Export Options¶

    System SHALL allow exporting results as CSV, Excel, JSON, or syncing to Power BI dataset.

    FR-10: Security¶

    System SHALL enforce RBAC: users query only databases they have permission to access (Entra ID integration).

    FR-11: Query Validation¶

    System SHALL validate SQL queries for safety (no DROP, TRUNCATE, DELETE) before execution.

    FR-12: Performance Optimization¶

    System SHALL optimize generated queries (add indexes hints, limit rows) and cache results (5-minute TTL).

    FR-13: Bilingual Support¶

    System SHALL accept questions in en-US or fr-FR and generate queries with localized column aliases.

    FR-14: Scheduled Reports¶

    System SHALL allow scheduling reports (daily, weekly, monthly) with email delivery and SharePoint archiving.

    FR-15: Audit Logging¶

    System SHALL log all queries, exports, and report generations to Application Insights.

    Non-Functional Requirements¶

    Performance¶

    • SQL generation: <5 seconds (P95)
    • Query execution: <10 seconds for 10K rows (P95)
    • Visualization: <3 seconds (P95)
    • Insight detection: <10 seconds (P95)
    • Support 500 concurrent queries

    Reliability¶

    • Uptime: 99.9% SLA
    • Auto-retry failed queries (3 attempts)
    • Graceful degradation (return partial results if timeout)
    • Data durability: 99.999999999% (Azure SQL)

    Security¶

    • Encrypt connection strings at rest (Azure Key Vault)
    • RBAC: users query only authorized databases
    • PII masking in query results (SIN, email, phone)
    • Compliance: PIPEDA, TBS Directive on Privacy
    • Audit log all queries and exports

    Accessibility¶

    • WCAG 2.1 AA compliance (UI, charts)
    • Keyboard navigation (query builder, chart controls)
    • Screen reader support (table results, chart descriptions)
    • High-contrast mode for charts

    i18n¶

    • Bilingual support (en-US/fr-FR)
    • Auto-detect question language or manual selection
    • ICU formatting for dates, numbers, currency
    • Bilingual chart labels and report commentary

    Observability¶

    • Application Insights custom events (query generated, executed, visualized)
    • Track query types, execution times, error rates
    • Alerts for high query failures or slow executions
    • Metrics: queries/day, avg execution time, insight detection accuracy

    Scalability¶

    • Horizontal scaling (Azure Functions for query execution)
    • Connection pooling (Azure SQL elastic pool)
    • Result caching (Azure Cache for Redis, 5-min TTL)
    • Support 10K queries/day per tenant

    Technical Architecture¶

    Backend (Query Engine)¶

    • SQL Generation: Azure OpenAI GPT-4o (prompt with schema context)
    • Execution: Azure Functions (SQL client libraries for Azure SQL, Synapse, Databricks)
    • Caching: Azure Cache for Redis (query results, 5-min TTL)
    • Scheduling: Azure Logic Apps (scheduled report generation)

    Frontend (UI)¶

    • Query Builder: React component with natural language input
    • Results Table: AG Grid (virtualized scrolling for large datasets)
    • Charts: Chart.js or Recharts (interactive visualizations)
    • Schema Browser: Tree view with search and preview

    Integration Points¶

    • Azure SQL Database: ODBC connection (Entra ID auth)
    • Azure Synapse Analytics: JDBC connection (Entra ID auth)
    • Databricks SQL Warehouse: JDBC connection (PAT token)
    • eva-openai: GPT-4o for SQL generation and insights
    • eva-auth: RBAC for database access
    • eva-api: REST API for query, visualize, export

    Open Questions¶

    Q1: Query Safety (SECURITY)¶

    Question: Block all mutating queries (INSERT, UPDATE, DELETE) or allow with approval? Impact: Safety vs. usability Recommendation: Block by default, allow with admin approval Decision Owner: Security Sentinel

    Q2: Schema Access (PRIVACY)¶

    Question: Show full schema to all users or filter by permissions? Impact: Discoverability vs. data sovereignty Recommendation: Filter schema by user's database permissions (Entra ID RBAC) Decision Owner: Security Sentinel

    Q3: SQL Accuracy (TECH)¶

    Question: How to validate generated SQL is correct? Impact: User trust vs. complexity Recommendation: Show query preview, allow editing, track user corrections for learning Decision Owner: ai2

    Q4: Result Limits (SCALABILITY)¶

    Question: 10K row limit reasonable or increase? Impact: Performance vs. usability Recommendation: 10K default, allow power users to request higher (up to 100K) Decision Owner: ai2

    Q5: Database Support (SCOPE)¶

    Question: Support Azure SQL only or include PostgreSQL, MySQL? Impact: Time-to-market vs. reach Recommendation: Azure SQL/Synapse MVP (Sprint 9), PostgreSQL/MySQL Sprint 13 Decision Owner: Marco
