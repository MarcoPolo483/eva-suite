    CDD: EVA Meeting Transcription Tool¶

    Last Updated: 2025-11-19 Status: Backlog Priority: MEDIUM Epic: EVA Tools Suite

    Vision¶

    Provide real-time meeting transcription with AI-powered action item extraction, speaker identification, and automated meeting notesâ€”enabling teams to focus on discussions instead of note-taking and ensuring no decisions are lost.

    Business Value: Save 30 minutes per meeting on note-taking, capture 100% of action items, improve meeting accountability, and create searchable meeting archives.

    Core Capabilities¶

    • Real-Time Transcription: Live speech-to-text during Teams/Zoom meetings
    • Speaker Identification: Recognize and label different speakers (diarization)
    • Action Items Extraction: Auto-detect commitments, tasks, decisions, deadlines
    • Meeting Summary: Generate concise recap (attendees, topics, outcomes)
    • Searchable Archive: Full-text search across meeting transcripts
    • Bilingual Support: Transcribe English/French with auto-detection
    • Export Options: PDF, Word, Teams channel post, email digest
    • Integration: Works with Teams, Zoom, Google Meet (via bot)

    User Stories¶

    US-1: As a meeting organizer, I want real-time transcription so that participants focus on discussion¶

    Acceptance Criteria:

    • Join Teams meeting, activate EVA transcription bot
    • See live transcription with <3 second latency
    • Speaker names labeled automatically (if permissions granted)
    • Transcript syncs to Cosmos DB in real-time
    • Download transcript as PDF after meeting

    US-2: As a project manager, I want action items extracted so that I follow up on commitments¶

    Acceptance Criteria:

    • EVA detects phrases: "Marco will...", "Action item:", "By Friday..."
    • Generate action items list with owner, task, deadline
    • Post action items to Teams channel after meeting
    • Send email digest to attendees with action items
    • Track completion status in EVA dashboard

    US-3: As an attendee, I want meeting summaries so that I catch up on missed meetings¶

    Acceptance Criteria:

    • Generate summary: purpose, key topics, decisions, action items, next steps
    • Summary max 1 page (500 words)
    • Include timestamp links to transcript sections
    • Bilingual summaries (en-US/fr-FR) based on meeting language
    • Export to Word or share via link

    US-4: As a compliance officer, I want searchable archives so that I retrieve historical decisions¶

    Acceptance Criteria:

    • Search across all meeting transcripts (full-text)
    • Filter by date range, attendees, meeting title
    • Highlight search terms in transcript results
    • Export search results to Excel
    • Retain transcripts for 7 years (compliance requirement)

    US-5: As a user, I want bilingual transcription so that French meetings are transcribed accurately¶

    Acceptance Criteria:

    • Auto-detect meeting language (en-US or fr-FR)
    • Transcribe with language-specific models (Azure Speech)
    • Handle code-switching (English/French in same meeting)
    • Generate bilingual action items if meeting mixed
    • Export transcript in both languages

    Functional Requirements¶

    FR-1: Real-Time Transcription¶

    System SHALL transcribe speech to text with <3 second latency using Azure Speech Services (Batch Transcription API).

    FR-2: Speaker Diarization¶

    System SHALL identify and label speakers (Speaker 1, Speaker 2) or use attendee names if permissions granted.

    FR-3: Action Items Detection¶

    System SHALL extract action items using GPT-4o with prompts for commitments, deadlines, owners.

    FR-4: Meeting Summary¶

    System SHALL generate summaries with sections: Purpose, Topics, Decisions, Action Items, Next Steps.

    FR-5: Export Options¶

    System SHALL allow exporting transcripts as PDF, DOCX, or posting to Teams channel.

    FR-6: Search API¶

    System SHALL provide full-text search across transcripts stored in Azure Cognitive Search.

    FR-7: Bilingual Support¶

    System SHALL support en-US and fr-FR transcription with auto-language detection.

    FR-8: Integration¶

    System SHALL integrate with Microsoft Teams (bot), Zoom (bot), Google Meet (bot).

    FR-9: Privacy Controls¶

    System SHALL require organizer consent before recording/transcribing meeting (PIPEDA compliance).

    FR-10: Access Control¶

    System SHALL enforce RBAC: only meeting attendees can access transcript.

    FR-11: Retention Policy¶

    System SHALL retain transcripts for 7 years (configurable per tenant) with Azure Storage Archive tier.

    FR-12: Notifications¶

    System SHALL send email digest after meeting with summary and action items.

    FR-13: Transcript Editing¶

    System SHALL allow post-meeting editing of transcripts (correct misheard words, redact PII).

    FR-14: Timestamp Links¶

    System SHALL provide clickable timestamps in summaries that jump to transcript section.

    FR-15: Audit Logging¶

    System SHALL log all transcription starts, exports, edits, and deletions to Application Insights.

    Non-Functional Requirements¶

    Performance¶

    • Transcription latency: <3 seconds (P95)
    • Summary generation: <30 seconds post-meeting
    • Search response time: <2 seconds for 10K transcripts
    • Support 100 concurrent meeting transcriptions
    • Real-time sync to Cosmos DB (<5 second delay)

    Reliability¶

    • Uptime: 99.9% SLA
    • Auto-retry failed audio uploads (3 attempts)
    • Graceful degradation (store audio, transcribe later if backend unavailable)
    • Data durability: 99.999999999% (Azure Storage)

    Security¶

    • Encrypt audio at rest (AES-256) and in transit (TLS 1.3)
    • RBAC: only attendees access transcript
    • PII masking in summaries (SIN, email, phone)
    • Compliance: PIPEDA, TBS Directive on Privacy
    • Watermark exported transcripts (user, timestamp)

    Accessibility¶

    • WCAG 2.1 AA compliance (transcript viewer)
    • Keyboard navigation (play, pause, skip)
    • Screen reader support (transcript text, timestamps)
    • High-contrast mode for transcript viewer

    i18n¶

    • Bilingual support (en-US/fr-FR)
    • Auto-detect meeting language or manual selection
    • ICU formatting for dates, times
    • Bilingual action items and summaries

    Observability¶

    • Application Insights custom events (transcription started, completed, exported)
    • Track meeting duration, attendees, action items count
    • Alerts for high failure rates or slow processing
    • Metrics: transcriptions/day, avg meeting duration

    Scalability¶

    • Horizontal scaling (Azure Functions for processing)
    • Partition storage by /tenantId/meetingId (HPK)
    • Azure CDN for exported transcripts
    • Support 10K meetings/month per tenant

    Technical Architecture¶

    Backend (Transcription Engine)¶

    • Audio Capture: Teams/Zoom bot records audio stream
    • Storage: Azure Blob Storage (audio files)
    • Transcription: Azure Speech Services (Batch Transcription API, diarization)
    • Processing: Azure Functions (Event Grid trigger for post-meeting processing)
    • Action Items: GPT-4o prompt for extraction

    Frontend (UI)¶

    • Live View: React component with real-time transcript (SignalR)
    • Viewer: Transcript viewer with timestamp navigation
    • Search: Azure Cognitive Search integration
    • Export: Download buttons for PDF/DOCX

    Integration Points¶

    • Microsoft Teams: Bot SDK for meeting join, audio capture
    • Zoom: Zoom Apps SDK for bot integration
    • eva-openai: GPT-4o for action items and summaries
    • eva-auth: RBAC for transcript access
    • eva-api: REST API for search, export

    Implementation Phases¶

    Phase 1: Core Transcription (Sprint 6)¶

    • Teams bot for meeting join and audio capture
    • Azure Speech Services integration (Batch API)
    • Cosmos DB schema for meetings and transcripts
    • Real-time transcript display (SignalR)
    • Basic export (PDF, DOCX)

    Phase 2: Action Items & Summaries (Sprint 7)¶

    • GPT-4o prompt for action items extraction
    • Summary generation (purpose, topics, decisions, action items)
    • Email digest after meeting
    • Teams channel post with summary
    • Timestamp links to transcript sections

    Phase 3: Search & Archive (Sprint 8)¶

    • Azure Cognitive Search indexing
    • Full-text search UI (filter by date, attendees)
    • 7-year retention policy (Archive tier)
    • Transcript editing (post-meeting corrections)
    • PII redaction tool

    Phase 4: Zoom/Meet & Polish (Sprint 9)¶

    • Zoom bot integration (Zoom Apps SDK)
    • Google Meet bot (Web Speech API fallback)
    • WCAG 2.1 AA audit (transcript viewer)
    • Load testing (100 concurrent meetings)
    • UAT with pilot teams

    Open Questions¶

    Q1: Recording Consent (COMPLIANCE)¶

    Question: Require explicit consent from all attendees or just organizer? Impact: Compliance vs. UX friction Recommendation: Organizer consent + banner notification to attendees Decision Owner: Security Sentinel

    Q2: Speaker Identification (PRIVACY)¶

    Question: Auto-label speakers with names or use generic labels (Speaker 1)? Impact: Usability vs. privacy Recommendation: Generic labels by default, allow organizer to map names Decision Owner: UX Partner

    Q3: Action Items Format (UX)¶

    Question: Structured (owner, task, deadline) or free-form text? Impact: Automation vs. flexibility Recommendation: Structured format with manual editing allowed Decision Owner: ai2

    Q4: Retention Policy (COMPLIANCE)¶

    Question: 7-year retention for all meetings or only flagged ones? Impact: Storage cost vs. compliance Recommendation: 7 years for compliance-sensitive tenants, 1 year default Decision Owner: Marco

    Q5: Zoom/Meet Priority (SCOPE)¶

    Question: Launch with Teams only or support Zoom/Meet in MVP? Impact: Time-to-market vs. reach Recommendation: Teams MVP (Sprint 6), Zoom/Meet Sprint 9 Decision Owner: Marco
