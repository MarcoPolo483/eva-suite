    CDD: EVA Document Summarizer Tool¶

    Last Updated: 2025-11-19 Status: Backlog Priority: MEDIUM Epic: EVA Tools Suite

    Vision¶

    Provide an AI-powered document summarizer that allows users to upload documents (PDF, Word, PowerPoint), extract key insights, and generate concise summariesâ€”saving hours of reading time and enabling faster decision-making.

    Business Value: Reduce document review time by 70%, accelerate policy analysis, enable executives to digest 100-page reports in minutes, and improve knowledge retention.

    Core Capabilities¶

    • Multi-Format Upload: PDF, DOCX, PPTX, TXT (up to 50MB per file)
    • Intelligent Extraction: Extract text, tables, images with OCR support
    • Key Insights: Identify main themes, action items, recommendations, risks
    • Summary Lengths: Executive summary (1 paragraph), standard (1 page), detailed (3 pages)
    • Q&A on Documents: Ask questions about uploaded document content
    • Citation Links: Click to jump to original section in source document
    • Batch Processing: Summarize multiple documents simultaneously
    • Export Options: Save summaries as PDF, Word, or share via link

    User Stories¶

    US-1: As a PS-A, I want to summarize policy documents so that I quickly understand changes¶

    Acceptance Criteria:

    • Upload 100-page PDF policy document (<50MB)
    • Select summary length: Executive (1 paragraph), Standard (1 page), Detailed (3 pages)
    • System generates summary within 60 seconds
    • Summary includes key themes, action items, and risks
    • Citations link back to original page numbers

    US-2: As an executive, I want executive summaries so that I digest reports before meetings¶

    Acceptance Criteria:

    • Upload board report (PowerPoint, 30 slides)
    • Generate 1-paragraph executive summary (5 sentences max)
    • Highlight: purpose, key findings, recommendations, next steps
    • Email summary as PDF attachment
    • Summary respects bilingual content (en-US/fr-FR)

    US-3: As a researcher, I want to ask questions so that I find specific information fast¶

    Acceptance Criteria:

    • Upload research paper (PDF, 50 pages)
    • Ask: "What methodology was used?"
    • System returns answer with citation (e.g., "See Section 3.2, Page 12")
    • Follow-up questions maintain conversation context
    • Export Q&A transcript as Word document

    US-4: As a manager, I want batch processing so that I summarize team reports efficiently¶

    Acceptance Criteria:

    • Upload 10 Word documents (team reports)
    • Select batch summarization mode
    • System processes all documents in parallel (<5 minutes)
    • Generate combined summary table (one row per document)
    • Download as Excel with links to individual summaries

    US-5: As a compliance officer, I want table extraction so that I analyze structured data¶

    Acceptance Criteria:

    • Upload PDF with embedded tables (budget reports)
    • System extracts tables as structured data (JSON/CSV)
    • Preview tables inline before export
    • Export tables to Excel with formatting preserved
    • Handle multi-page tables (merge rows)

    Functional Requirements¶

    FR-1: File Upload¶

    System SHALL accept PDF, DOCX, PPTX, TXT files up to 50MB via drag-and-drop or file picker.

    FR-2: Text Extraction¶

    System SHALL extract text using Azure AI Document Intelligence with OCR support for scanned documents.

    FR-3: Summary Generation¶

    System SHALL generate summaries using GPT-4o with configurable lengths: Executive (150 words), Standard (500 words), Detailed (1500 words).

    FR-4: Key Insights¶

    System SHALL identify and extract: main themes, action items, recommendations, risks, deadlines.

    FR-5: Q&A Interface¶

    System SHALL allow users to ask questions about uploaded documents with cited answers (page numbers, sections).

    FR-6: Citation Links¶

    System SHALL provide clickable citations that open source document at referenced page/section.

    FR-7: Batch Processing¶

    System SHALL process up to 10 documents simultaneously with parallel processing (Azure Functions).

    FR-8: Table Extraction¶

    System SHALL extract tables from PDFs using Azure AI Document Intelligence and export as CSV/Excel.

    FR-9: Export Options¶

    System SHALL allow exporting summaries as PDF, DOCX, or shareable link (expires after 30 days).

    FR-10: Version Control¶

    System SHALL maintain history of uploaded documents and generated summaries per user (30-day retention).

    FR-11: Security¶

    System SHALL scan uploaded files for malware (Azure Defender) and reject suspicious files.

    FR-12: Privacy¶

    System SHALL delete uploaded documents after 30 days unless user opts to keep (up to 1 year).

    FR-13: Bilingual Support¶

    System SHALL detect document language (en-US/fr-FR) and generate summaries in same language or translate if requested.

    FR-14: Access Control¶

    System SHALL enforce RBAC: users upload to personal library, share with space members if permitted.

    FR-15: Audit Logging¶

    System SHALL log all uploads, summarizations, exports, and shares to Application Insights.

    Non-Functional Requirements¶

    Performance¶

    • Summary generation: <60 seconds for 100-page document (P95)
    • Batch processing: <5 minutes for 10 documents
    • File upload: <10 seconds for 50MB file
    • Q&A response time: <5 seconds per question
    • Support 1000 concurrent summarizations

    Reliability¶

    • Uptime: 99.9% SLA
    • Auto-retry failed summarizations (3 attempts)
    • Graceful degradation (queue if backend overloaded)
    • Data durability: 99.999999999% (Azure Storage)

    Security¶

    • Malware scanning (Azure Defender for Storage)
    • Encrypt files at rest (AES-256) and in transit (TLS 1.3)
    • RBAC: users access own uploads only (unless shared)
    • PII masking in summaries (SIN, email, phone)
    • Compliance: PIPEDA, TBS Directive on Privacy

    Accessibility¶

    • WCAG 2.1 AA compliance (upload UI, viewer)
    • Keyboard navigation (Tab, Enter, Esc)
    • Screen reader support (file upload status, summary results)
    • High-contrast mode for document viewer

    i18n¶

    • Bilingual support (en-US/fr-FR)
    • Auto-detect document language or manual selection
    • ICU formatting for dates, numbers
    • Bilingual error messages

    Observability¶

    • Application Insights custom events (upload, summarize, export)
    • Track document types, sizes, processing times
    • Alerts for high failure rates or slow processing
    • Metrics: summarizations/day, avg processing time

    Scalability¶

    • Horizontal scaling (Azure Functions for processing)
    • Partition storage by /tenantId/userId (HPK)
    • Azure CDN for exported summaries
    • Support 50K users, 100K documents/month

    Technical Architecture¶

    Backend (Processing Engine)¶

    • Upload: Azure Blob Storage (Hot tier for 30 days, Cool tier archive)
    • Extraction: Azure AI Document Intelligence (OCR, layout analysis)
    • Summarization: Azure OpenAI GPT-4o (prompt engineering for summaries)
    • Processing: Azure Functions (Blob trigger for async processing)

    Frontend (UI)¶

    • Upload: React file upload component with progress bar
    • Viewer: PDF.js for inline document viewing
    • Q&A: Chat interface with cited answers
    • Export: Download buttons for PDF/DOCX/Excel

    Integration Points¶

    • eva-openai: GPT-4o for summarization and Q&A
    • eva-auth: RBAC for document access control
    • eva-safety: Content moderation for summaries
    • eva-api: REST API for upload, summarize, export

    Implementation Phases¶

    Phase 1: Core Summarization (Sprint 5)¶

    • Azure Blob Storage for file uploads
    • Azure AI Document Intelligence integration
    • GPT-4o summarization prompt templates
    • React upload UI with progress bar
    • Basic export (PDF, DOCX)

    Phase 2: Q&A & Citations (Sprint 6)¶

    • Q&A chat interface on documents
    • Citation extraction and linking
    • PDF.js viewer with jump-to-page
    • Conversation history per document
    • Export Q&A transcript

    Phase 3: Batch & Tables (Sprint 7)¶

    • Batch processing (parallel Azure Functions)
    • Table extraction (Document Intelligence)
    • Export tables to Excel/CSV
    • Combined summary report for batch
    • Processing queue UI

    Phase 4: Polish & Security (Sprint 8)¶

    • Malware scanning (Azure Defender)
    • WCAG 2.1 AA audit (upload UI, viewer)
    • 30-day auto-delete with reminders
    • Load testing (1000 concurrent uploads)
    • UAT with pilot departments

    Open Questions¶

    Q1: File Size Limit (TECH)¶

    Question: 50MB limit reasonable or increase to 100MB? Impact: Cost vs. usability Recommendation: Start 50MB, monitor user feedback, increase if needed Decision Owner: ai2 + Marco

    Q2: Retention Policy (COMPLIANCE)¶

    Question: 30-day auto-delete or user-controlled? Impact: Storage cost vs. UX Recommendation: 30-day default, allow user to extend (up to 1 year) Decision Owner: Security Sentinel

    Q3: Malware Scanning (SECURITY)¶

    Question: Scan all files or only suspicious ones? Impact: Security vs. latency Recommendation: Scan all uploads with Azure Defender (async, no blocking) Decision Owner: Security Sentinel

    Q4: Summary Quality (UX)¶

    Question: Fixed templates or customizable prompts? Impact: Consistency vs. flexibility Recommendation: Fixed templates (Sprint 5), add custom prompts in Sprint 9 Decision Owner: UX Partner

    Q5: Batch Processing Limit (SCALABILITY)¶

    Question: 10 documents limit or allow unlimited? Impact: User freedom vs. resource contention Recommendation: 10 limit per batch, allow multiple batches Decision Owner: ai2
