    Hub Automation Troubleshooting Guide¶

    Last Updated: November 19, 2025
    Owner: Scrum Master (GitHub Copilot)

    Quick Reference¶

    Error Code Symptom Solution
    ERR-001 Workflow not running on schedule Check workflow enabled, verify cron syntax
    ERR-002 MARCO-HUB.md not updating Verify metrics file exists, check script logs
    ERR-003 EVA Matrix site not updating Check eva-matrix repo access, verify GitHub Pages
    ERR-004 Git push failures Verify GH_ORCHESTRATOR_TOKEN permissions
    ERR-005 Script execution errors Test locally, check Node.js version

    Common Issues¶

    1. Workflow Not Running on Schedule (ERR-001)¶

    Symptoms: - No workflow runs at scheduled times - Last run was manual trigger only - Workflow shows as "disabled"

    Diagnosis: 

    # Check workflow status
gh workflow list

# View workflow definition
gh workflow view hub-update.yml

# Check recent runs
gh run list --workflow=hub-update.yml --limit 5

    Solutions:

    1. Enable workflow:
    2. Go to: Actions → Workflows → Hub Update (4x Daily)

    3. Click "Enable workflow" if disabled

    4. Verify cron syntax: 

      schedule:
  - cron: "0 11,17,23 * * *"  # 11:00, 17:00, 23:00 UTC
  - cron: "0 5 * * *"          # 05:00 UTC

    5. Test at crontab.guru

    6. Check branch protection:

    7. Settings → Branches → Branch protection rules

    8. Ensure github-actions[bot] can push to main

    9. Wait for first scheduled run:

    10. Workflows may take up to 15 minutes after creation to start

    Workaround: 

    # Trigger manually
gh workflow run hub-update.yml

    2. MARCO-HUB.md Not Updating (ERR-002)¶

    Symptoms: - Workflow runs successfully - "No changes" reported for MARCO-HUB.md - Content appears stale or outdated

    Diagnosis: 

    # Check metrics file
Get-Content metrics/latest-metrics.json | Select-String "generatedAt"

# Run script locally
node scripts/generate-hub-update.mjs

# Check git status
git status

    Solutions:

    1. Verify metrics file exists: 

      if (!(Test-Path "metrics/latest-metrics.json")) {
  Write-Host "❌ Metrics file missing"
}

    2. Check metrics file format: 

      {
  "generatedAt": "2025-11-19T15:30:40.838Z",
  "sprint": { "name": "Sprint 04", ... },
  "board": { "totalItems": 44, ... }
}

    3. Test script locally: 

      node scripts/generate-hub-update.mjs
git diff MARCO-HUB.md  # Should show changes

    4. Check workflow logs: 

      gh run view --log | Select-String "hub-update"

    Workaround: 

    # Manual update
node scripts/generate-hub-update.mjs
git add MARCO-HUB.md
git commit -m "chore: manual hub update"
git push origin main

    3. EVA Matrix Site Not Updating (ERR-003)¶

    Symptoms: - Workflow reports "updated" but site shows old data - Site returns 404 or fails to load - Dashboard displays default/placeholder content

    Diagnosis: 

    # Check eva-matrix repo
gh repo view MarcoPolo483/eva-matrix

# Test site generation locally
node scripts/generate-matrix-site.mjs
Get-Content dist/index.html | Select-String "EVA 2.0"

# Check GitHub Pages status
gh api repos/MarcoPolo483/eva-matrix/pages

    Solutions:

    1. Verify eva-matrix repo exists: 

      gh repo view MarcoPolo483/eva-matrix
# Should return: name, url, owner

    2. Enable GitHub Pages:

    3. Go to: https://github.com/MarcoPolo483/eva-matrix/settings/pages
    4. Source: "Deploy from a branch"
    5. Branch: "main"
    6. Folder: "/ (root)"

    7. Click "Save"

    8. Check deployment status: 

      # View Pages deployments
gh api repos/MarcoPolo483/eva-matrix/pages/builds/latest

    9. Clear browser cache:

    10. Hard refresh: Ctrl+Shift+R (Windows/Linux) or Cmd+Shift+R (Mac)

    11. Clear site data: DevTools → Application → Clear storage

    12. Wait for propagation:

    13. GitHub Pages can take 1-2 minutes to deploy
    14. CDN cache may take up to 10 minutes to update

    Workaround: 

    # Manual deployment
node scripts/generate-matrix-site.mjs
cd ../eva-matrix  # Clone if needed
Copy-Item ../eva-orchestrator/dist/* . -Recurse -Force
git add .
git commit -m "chore: manual dashboard update"
git push origin main

    4. Git Push Failures (ERR-004)¶

    Symptoms: - Workflow fails at "Commit" or "Push" steps - Error: "remote: Permission to MarcoPolo483/... denied" - Error: "fatal: Authentication failed"

    Diagnosis: 

    # Check workflow logs
gh run view --log | Select-String "git push"

# Verify token secret
gh secret list --repo MarcoPolo483/eva-orchestrator

    Solutions:

    1. Verify GH_ORCHESTRATOR_TOKEN secret:
    2. Go to: Settings → Secrets and variables → Actions
    3. Ensure GH_ORCHESTRATOR_TOKEN exists

    4. Token must have repo scope (full control)

    5. Regenerate token if expired:

    6. Go to: GitHub Settings → Developer settings → Personal access tokens
    7. Generate new token with repo scope

    8. Update secret in repo settings

    9. Check git config in workflow: 

      - run: |
    git config user.name "github-actions[bot]"
    git config user.email "github-actions[bot]@users.noreply.github.com"

    10. Verify branch protection:

    11. Settings → Branches → main
    12. Allow: "github-actions" in push allowlist

    Workaround: 

    # Push manually
git pull origin main
node scripts/generate-hub-update.mjs
git add MARCO-HUB.md
git commit -m "chore: manual hub update"
git push origin main

    5. Script Execution Errors (ERR-005)¶

    Symptoms: - Workflow fails at "Generate Hub Update" or "Generate EVA Matrix Dashboard HTML" - Error: "Cannot find module" or "SyntaxError" - Script exits with non-zero code

    Diagnosis: 

    # Test locally
node --version  # Must be v20+
npm ci
node scripts/generate-hub-update.mjs

# Check for errors
$LASTEXITCODE  # Should be 0

    Solutions:

    1. Verify Node.js version: 

      node --version  # v20.x.x required

    2. Install dependencies: 

      npm ci  # Clean install

    3. Check script syntax: 

      node --check scripts/generate-hub-update.mjs
node --check scripts/generate-matrix-site.mjs

    4. Review error logs: 

      gh run view --log | Select-String "Error"

    5. Test with defaults: 

      # Temporarily rename metrics file to test default handling
Rename-Item metrics/latest-metrics.json metrics/latest-metrics.json.bak
node scripts/generate-hub-update.mjs
Rename-Item metrics/latest-metrics.json.bak metrics/latest-metrics.json

    Workaround: 

    # Debug locally with verbose logging
$env:DEBUG="*"
node scripts/generate-hub-update.mjs

    Advanced Troubleshooting¶

    Workflow Stuck in "Queued"¶

    Cause: GitHub Actions runner availability

    Solution: 1. Wait 5-10 minutes for runner to become available 2. Check GitHub Status 3. Cancel and re-run if stuck >15 minutes

    File Not Found Errors¶

    Cause: Missing or moved files

    Solution: 

    # Verify all required files exist
Test-Path .github/workflows/hub-update.yml
Test-Path scripts/generate-hub-update.mjs
Test-Path scripts/generate-matrix-site.mjs
Test-Path metrics/latest-metrics.json

    Merge Conflicts¶

    Cause: Concurrent updates to MARCO-HUB.md

    Solution: 

    # Pull latest changes
git pull origin main

# Rerun workflow
gh workflow run hub-update.yml

    GitHub Pages 404 Error¶

    Cause: Pages not enabled or deploying from wrong branch

    Solution: 1. Go to: Settings → Pages 2. Verify Source: main branch, / (root) 3. Wait 1-2 minutes for deployment 4. Check build status: gh api repos/MarcoPolo483/eva-matrix/pages/builds/latest

    Cache Issues¶

    Cause: Browser or CDN caching stale content

    Solution: 

    # Add cache-busting to HTML
# In generate-matrix-site.mjs:
const timestamp = Date.now();
html += `<!-- Cache bust: ${timestamp} -->`;

    Escalation Procedures¶

    When to Escalate¶

    Escalate to Marco (PO) if: - ❌ Workflow fails 3+ consecutive times - ❌ Site unavailable for >1 hour - ❌ Data integrity issues (incorrect metrics displayed) - ❌ Security concerns (token exposure, unauthorized access)

    Escalation Channels¶

    1. GitHub Issues:
    2. Create issue in eva-orchestrator repo
    3. Label: priority:high, type:bug, component:automation

    4. Include: workflow run URL, error logs, steps to reproduce

    5. Direct Contact:

    6. Tag: @MarcoPolo483 in issue or PR
    7. Email: (if available)
    8. Teams/Slack: (if configured)

    Emergency Procedures¶

    If automation is completely broken:

    # Disable workflow temporarily
gh workflow disable hub-update.yml

# Perform manual updates
node scripts/generate-hub-update.mjs
git add MARCO-HUB.md
git commit -m "chore: emergency manual update"
git push origin main

node scripts/generate-matrix-site.mjs
cd ../eva-matrix
Copy-Item ../eva-orchestrator/dist/* . -Recurse -Force
git add .
git commit -m "chore: emergency manual update"
git push origin main

# Re-enable when fixed
gh workflow enable hub-update.yml

    Logging and Monitoring¶

    View Workflow Logs¶

    # Latest run
gh run view --log

# Specific run
gh run view <run-id> --log

# Filter logs
gh run view --log | Select-String "error"

    Check Workflow Success Rate¶

    # Last 20 runs
gh run list --workflow=hub-update.yml --limit 20 --json conclusion,status

    Monitor Site Uptime¶

    # Check site HTTP status
curl -I https://marcopolo483.github.io/eva-matrix/

# Automated monitoring
while ($true) {
  $response = Invoke-WebRequest -Uri "https://marcopolo483.github.io/eva-matrix/" -UseBasicParsing
  Write-Host "$(Get-Date) - Status: $($response.StatusCode)"
  Start-Sleep -Seconds 300  # Check every 5 minutes
}

    Prevention Tips¶

    1. Test locally before pushing: 

      npm ci
node scripts/generate-hub-update.mjs
node scripts/generate-matrix-site.mjs

    2. Use test workflow for validation: 

      gh workflow run hub-update-test.yml

    3. Monitor first run after changes: 

      gh run watch

    4. Keep dependencies updated: 

      npm outdated
npm update

    5. Review workflow logs regularly: 

      gh run list --workflow=hub-update.yml --limit 5
