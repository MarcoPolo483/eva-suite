    EVA 2.0 Cost Analysis - November 2025¶

    Date: November 28, 2025
    Product Owner: Marco Presta
    Context: Autonomous AI Development Crew - Budget Planning
    Session: VS IDE Session 0 (morning chat)

    Current Budget: $100 CAD/month¶

    Infrastructure Breakdown¶

    Azure Services: $36/month (64% cost reduction)¶

    Previous Architecture: $99/month - 3 separate Static Web Apps (dev, test, prod) - 3 separate Cosmos DB instances - Isolated environments

    New Shared Architecture: $36/month ✅ - 1 Static Web App (hosting all 3 environments) - 1 Cosmos DB (3 logical containers: dev, test, prod) - Savings: $63/month → reinvested in AI agent capacity

    Deployment Strategy: - Terraform IaC (Windows Guy ownership) - Smoke test required before production use - Blue/green deployment for zero downtime

    OpenAI API: $50/month¶

    Agent Spawning Costs: - P02 (Senior Advisor): \(0.50-\)2.00 per requirements analysis task - DevMaster: \(0.50-\)2.00 per implementation task - QA Master: \(0.50-\)2.00 per test generation task - P10 (Metrics): \(0.50-\)2.00 per analytics task

    Optimization: - On-demand spawning only (no persistent agents) - Shared 3-year memory context (ChatGPT API) - Cost tracking per agent spawn - Frequency optimization (avoid unnecessary calls)

    Safety Buffer: $14/month¶

    • Unexpected overages
    • API rate limit spikes
    • Terraform plan validation runs

    GitHub Actions: FREE (2,000 minutes/month)¶

    Autonomous Workflows: - Master Scrum Master (MSM) daily standup: ~5 min/day - PSM-1 to PSM-4 progress tracking: ~10 min/day - DevMaster code generation: ~20 min/day - QA test runs: ~30 min/day - Heartbeat monitoring: ~15 min/day

    Total Estimated Usage: ~80 min/day × 30 days = 2,400 min/month

    Mitigation: - Use self-hosted runner on Marco's machine (offline resilience advantage noted) - Optimize workflow triggers (on-push for critical paths only) - Cache dependencies aggressively

    1-Week Sprint Budget: $23 CAD¶

    Target: Prove 1 POD model before scaling
    Duration: 168 hours (24/7 autonomous operation)
    Calculation: $100/month ÷ 4.33 weeks = $23.09/week

    Success Metrics: 1. Velocity: Story points delivered per day 2. Cost: Actual spend vs $23 budget 3. Quality: Test coverage >80%, zero critical bugs 4. Autonomy: 90% decisions made without Marco escalation 5. Recovery: Mean time to recovery from workflow failures

    Decision Point: After 1 week, scale to 2 PODs ($200/month) or optimize

    Local Model Evaluation - NEW REQUEST¶

    Marco's Budget Increase: Up to $3,000 CAD for hardware OR increased cloud services

    Option 1: Self-Hosted on Marco's Machine¶

    Hardware Requirements for Llama 3.2 70B¶

    Recommended Specs: - GPU: NVIDIA RTX 4090 (24GB VRAM) × 2 = ~\(5,000 CAD ❌ (over budget) - **Alternative:** NVIDIA RTX 4080 (16GB VRAM) × 2 = ~\)3,200 CAD ⚠️ (slightly over) - Best Fit: NVIDIA RTX 4070 Ti (12GB VRAM) × 2 = ~$2,400 CAD ✅ - CPU: AMD Ryzen 9 7950X or Intel i9-13900K - RAM: 128GB DDR5 (for model loading + context windows) - Storage: 2TB NVMe SSD (models + datasets)

    Performance: - Llama 3.2 70B (quantized to 4-bit): ~15-20 tokens/sec on 2× RTX 4070 Ti - Context window: 8K tokens (sufficient for most agent tasks) - Latency: ~50-100ms (vs 200-500ms OpenAI API)

    Cost Analysis: - Upfront: $2,400 CAD (hardware only, assuming Marco has base system) - Operating: \(0/month (OpenAI API eliminated) - **Electricity:** ~\)30-40/month (500W × 24/7 × \(0.12/kWh) - **Break-even:** Month 3 (\)50/month savings × 3 = $150 vs $2,400 upfront = 16 months actual break-even)

    Pros: - ✅ Full control (no rate limits, no API outages) - ✅ Offline resilience (Marco's priority after internet outage this morning) - ✅ Privacy (no data leaves machine) - ✅ Unlimited inference (no per-token costs)

    Cons: - ❌ Upfront capital expense (\(2,400) - ❌ Power consumption (\)30-40/month) - ❌ Noise/heat (dual GPU workstation) - ❌ Maintenance (driver updates, cooling, etc.) - ❌ Limited to Llama models (no GPT-4 level reasoning yet)

    Option 2: Dedicated Cloud GPU Instances¶

    RunPod / Vast.ai / Lambda Labs Pricing¶

    Instance: 2× RTX 4090 (48GB total VRAM) - Cost: \(1.50-\)2.00/hour - Monthly (24/7): \(1.50 × 24 × 30 = **\)1,080/month** ❌ (10× current budget)

    Instance: 1× RTX 4090 (24GB VRAM, quantized Llama 3.2 70B) - Cost: \(0.80-\)1.00/hour - Monthly (24/7): \(0.80 × 24 × 30 = **\)576/month** ❌ (5.7× current budget)

    Instance: 1× RTX 4080 (16GB VRAM, Llama 3.2 70B 8-bit quantization) - Cost: \(0.50-\)0.70/hour - Monthly (24/7): \(0.50 × 24 × 30 = **\)360/month** ❌ (3.6× current budget)

    Conclusion: Cloud GPUs NOT cost-effective for 24/7 autonomous operation

    Option 3: Hybrid Model (Best of Both Worlds)¶

    Strategy: 1. Phase 1 (Months 1-3): Continue with OpenAI API ($50/month) - Validate autonomous crew model works - Gather performance metrics (tokens/day, task types) - Identify which agents need GPT-4 vs can use Llama

    1. Phase 2 (Month 4): Invest $2,400 in local hardware
    2. Move 70% of tasks to local Llama 3.2 70B (DevMaster, QA, P10)
    3. Keep 30% on OpenAI API for complex reasoning (P02 requirements analysis)

    4. New monthly cost: $15 OpenAI + $35 electricity = $50/month (same budget!)

    5. Phase 3 (Month 6): Evaluate Llama 3.3 or Llama 4 releases

    6. If local models match GPT-4 reasoning, eliminate OpenAI entirely
    7. Final cost: $35/month (electricity only) + $36 Azure = $71/month total

    Break-Even Analysis: - Investment: $2,400 hardware - Savings: \(35/month (from Month 4 onward) - **Break-even:** Month 4 + (\)2,400 ÷ $35) = Month 72 ⚠️

    REVISED (accounting for no OpenAI cost elimination in Phase 2): - Actually, in Phase 2 we still pay \(50/month total (\)15 API + $35 power) - Real savings start in Phase 3: $50 - $35 = $15/month saved - Break-even: $2,400 ÷ $15 = 160 months ❌ (not economical)

    BETTER HYBRID: - Don't buy hardware yet - Use \(3K budget for **increased cloud services** instead: - Upgrade Azure Cosmos DB to Autoscale (better performance) - Add Azure Monitor + Application Insights (\)50/month) - Increase OpenAI API budget to $150/month (3× agent capacity) - New total: $36 Azure + $50 monitoring + \(150 OpenAI = **\)236/month** - But with \(3K one-time budget → covers (\)236 - $100) = $136/month overage × 22 months

    Recommendation: Hybrid Cloud Burst Strategy¶

    Baseline (Business as Usual): $100/month - Azure: $36/month - OpenAI API: $50/month - Buffer: $14/month

    Sprint Burst Mode (When Needed): Use $3K budget - Temporarily increase OpenAI API to \(200/month (4× agent spawns) - Add Azure Monitor for real-time debugging (\)50/month) - Burst cost: $250/month - Duration: \(3,000 ÷ (\)250 - $100) = 20 months of burst capacity

    Use Cases for Burst: - Sprint deadlines (e.g., EVA-Sovereign-UI 53 components in 16 sprints) - Production incidents (spawn extra QA agents for regression testing) - Experimentation (test new autonomous workflows without budget anxiety)

    Advantages: - ✅ No upfront hardware commitment - ✅ Flexible (burst only when needed) - ✅ Cloud resilience (Marco's priority after internet outage) - ✅ No maintenance overhead - ✅ Can still evaluate local models later (Llama 4, Mistral Large 2)

    When to Revisit Local Hardware: - Llama 4 or Mistral Large 3 matches GPT-4 reasoning (expected Q2 2026) - Marco's internet reliability improves (or gets Starlink backup) - Autonomous crew proves valuable enough to justify dedicated hardware

    • Mitigation:

    • Feature flags for patches (can disable without rollback)
    • Automated rollback on failed health checks (within 5 minutes)
    • Canary releases (5% traffic, monitor, then 100%)
