    CDD: EVA OS Context Awareness¶

    Last Updated: 2025-11-19 Status: Backlog Priority: LOW (Moonshot) Epic: EVA OS Integration (Windows Copilot+ Replacement)

    Vision¶

    Enable EVA to passively understand user context through Windows Accessibility APIs, OCR, clipboard monitoring, and file system watchers—creating a comprehensive picture of user activity without explicit input, allowing EVA to proactively offer relevant assistance, automate repetitive tasks, and anticipate user needs in real-time.

    Business Value: Reduce manual input by 95% (ambient intelligence), enable context-aware automation (save 2 hours/day per user), establish EVA as the intelligent context layer for Windows workloads, and deliver enterprise-grade AI that understands not just what users say, but what they're doing.

    Core Capabilities¶

    • Accessibility API Hooks: Monitor UI controls, text fields, button clicks (UIA, MSAA)
    • OCR Text Extraction: Read text from images, PDFs, legacy apps (Azure AI Vision OCR)
    • Clipboard Monitoring: Track copy/paste activity, detect patterns (PII filtering)
    • File System Watchers: Detect file opens, edits, saves (recent documents, work context)
    • Browser Tab Tracking: Monitor active tabs, URLs (optional, privacy-conscious)
    • Application State Detection: Identify app modes (e.g., Outlook: reading vs. composing)
    • Meeting Context: Detect Teams/Zoom meetings, participants, topics (calendar integration)
    • Contextual Triggers: Fire automation based on context (e.g., "User opened budget Excel → offer analysis")

    User Stories¶

    US-1: As a user, I want EVA to understand my UI interactions so that I get relevant help¶

    Acceptance Criteria:

    • User focuses on text field in Word → EVA detects: "Editing document 'Q4 Report'"
    • Proactive suggestion: "Need grammar check?" (based on context: Word, editing mode)
    • User clicks suggestion → EVA runs grammar check (Word add-in integration)
    • Context persists across apps (last 5 activities tracked in-memory)
    • Privacy: UI interactions not logged (metadata only: app name, activity type)

    US-2: As a user, I want EVA to read text from images so that I interact with non-text content¶

    Acceptance Criteria:

    • User opens PDF with scanned document (no text layer)
    • EVA detects: "PDF contains image-based text"
    • Proactive suggestion: "Extract text from this PDF?"
    • User clicks → EVA runs OCR (Azure AI Vision), displays extracted text (editable)
    • Error handling: Low-quality image → suggest re-scan

    US-3: As a user, I want EVA to learn from my clipboard so that I get smart paste suggestions¶

    Acceptance Criteria:

    • User copies email address from Outlook → switches to Excel
    • EVA detects: "Clipboard contains email address"
    • Proactive suggestion: "Paste as hyperlink?"
    • User clicks → EVA pastes as mailto: hyperlink (not plain text)
    • Pattern learning: If user always pastes emails as hyperlinks, auto-apply (opt-in)

    US-4: As a user, I want EVA to track my recent files so that I quickly access work context¶

    Acceptance Criteria:

    • User opens "Budget.xlsx" → edits → saves
    • EVA adds to recent context: "Working on Budget.xlsx"
    • Ask "What was I working on?" → EVA responds: "Budget.xlsx, last edited 5 minutes ago"
    • File context displayed in EVA sidebar (recent docs, quick open)
    • Privacy: File names tracked, content not read (unless user grants permission)

    US-5: As a user, I want EVA to detect meetings so that I get context-aware assistance¶

    Acceptance Criteria:

    • User joins Teams meeting → EVA detects: "In meeting: Q4 Planning, 5 participants"
    • Proactive suggestion: "Take notes?" (meeting transcription, action items)
    • During meeting: EVA listens for questions, provides answers (meeting co-pilot)
    • Post-meeting: EVA generates summary (action items, decisions, follow-ups)
    • Privacy: Transcription opt-in (not enabled by default)

    Functional Requirements¶

    1. FR-1: SHALL monitor UI controls using Windows Accessibility APIs (UIA, MSAA)
    2. FR-2: SHALL extract text from images and PDFs using Azure AI Vision OCR
    3. FR-3: SHALL monitor clipboard activity (copy/paste events, content types)
    4. FR-4: SHALL filter PII from clipboard content (emails, phone numbers, SINs)
    5. FR-5: SHALL watch file system for file opens, edits, saves (recent documents)
    6. FR-6: SHALL track browser tabs and URLs (optional, user opt-in)
    7. FR-7: SHALL detect application states (e.g., Outlook: reading vs. composing)
    8. FR-8: SHALL detect meeting context (Teams, Zoom, calendar integration)
    9. FR-9: SHALL fire contextual triggers (automation based on detected activity)
    10. FR-10: SHALL maintain context history (last 10 activities, in-memory only)
    11. FR-11: SHALL encrypt all captured context data in transit (TLS 1.3) and at rest (AES-256)
    12. FR-12: SHALL enforce tenant isolation (per-user context boundaries)
    13. FR-13: SHALL allow user opt-out (disable specific context types)
    14. FR-14: SHALL log context capture events to Application Insights (anonymized metadata only)
    15. FR-15: SHALL provide diagnostic mode (verbose logging for troubleshooting)

    Non-Functional Requirements¶

    Performance¶

    • Context Capture: <100ms (activity detected → context updated)
    • OCR Extraction: <2s (image → text extracted)
    • Clipboard Monitoring: <50ms (copy event → context updated)
    • File System Watch: <100ms (file save → context updated)

    Reliability¶

    • Context Accuracy: >90% (correct activity detection)
    • OCR Accuracy: >85% (clean, high-contrast images)
    • Failover: Graceful degradation if API unavailable (skip context update)
    • Data Integrity: Context data not corrupted (validation on capture)

    Security¶

    • Data Encryption: TLS 1.3 (in transit), AES-256 (at rest, temporary storage)
    • PII Filtering: Detect and redact emails, phone numbers, SINs from clipboard
    • Tenant Isolation: Per-user context boundaries (no cross-tenant access)
    • Audit Logging: All context capture events logged to Cosmos DB (user ID, activity type, timestamp, 1-year retention)

    Accessibility¶

    • WCAG 2.1 AA: Context-aware features keyboard-accessible
    • Screen Reader Support: Proactive suggestions announced (ARIA live regions)
    • High Contrast: UI elements respect Windows theme (dark/light mode)

    Internationalization¶

    • Locales: en-US (default), fr-FR
    • OCR Languages: Support English and French text extraction
    • Error Messages: Translated (e.g., "OCR failed" → "Échec de l'OCR")

    Observability¶

    • Metrics: Application Insights tracks context capture events, OCR requests, clipboard activity, file access
    • Alerts: Context capture latency >200ms, OCR accuracy <80%, error rate >2%
    • Dashboards: Power BI dashboard for context capture stats, OCR usage, error trends

    Scalability¶

    • Concurrent Users: Support 10K+ users per Windows Server (VDI scenario)
    • Context Volume: Handle 1K+ context captures/hour per user (peak activity)
    • OCR Volume: Handle 100+ OCR requests/day per user (batch processing)

    Technical Architecture¶

    Context Capture¶

    Accessibility API Hooks (UIA):

    // Monitor UI element focus changes
Automation.AddAutomationFocusChangedEventHandler(
  (sender, e) => {
    AutomationElement element = sender as AutomationElement;
    string controlType = element.Current.ControlType.ProgrammaticName;
    string value = element.Current.Name;

    // Send context to EVA API
    SendContextUpdate(new Context {
      Type = "ui_focus",
      ControlType = controlType,
      Value = value,
      Application = GetProcessName(element),
      Timestamp = DateTime.UtcNow
    });
  }
);

    OCR Text Extraction:

    // Azure AI Vision OCR
async Task<string> ExtractText(byte[] imageBytes) {
  var client = new ComputerVisionClient(credentials);
  var result = await client.ReadInStreamAsync(new MemoryStream(imageBytes));

  // Poll for completion
  while (result.Status == OperationStatusCodes.Running) {
    await Task.Delay(1000);
    result = await client.GetReadResultAsync(result.OperationLocation);
  }

  // Concatenate text lines
  return string.Join("\n", result.AnalyzeResult.ReadResults
    .SelectMany(r => r.Lines)
    .Select(l => l.Text));
}

    Clipboard Monitoring:

    // Monitor clipboard changes
AddClipboardFormatListener(hwnd);

void OnClipboardUpdate() {
  if (Clipboard.ContainsText()) {
    string text = Clipboard.GetText();
    string filtered = FilterPII(text); // Redact emails, phone numbers

    SendContextUpdate(new Context {
      Type = "clipboard",
      ContentType = "text",
      ContentLength = text.Length,
      HasPII = filtered != text,
      Timestamp = DateTime.UtcNow
    });
  }
}

    Implementation Phases¶

    Phase 1: Accessibility API Integration (Sprint 19, Week 1-3)¶

    • UIA/MSAA setup (monitor UI element focus changes)
    • Application state detection (active app, window title)
    • Text field monitoring (editable controls, value changes)
    • Context encryption (TLS 1.3, AES-256)
    • Tenant isolation (per-user context boundaries)

    Phase 2: OCR & Clipboard (Sprint 20, Week 4-6)¶

    • Azure AI Vision OCR integration (image/PDF text extraction)
    • Clipboard monitoring (copy/paste events, content types)
    • PII filtering (detect/redact emails, phone numbers, SINs)
    • Context triggers (automation based on detected activity)
    • Error handling (OCR failure, clipboard access denied)

    Phase 3: File System & Browser (Sprint 21, Week 7-9)¶

    • File system watchers (recent documents, file open/edit/save events)
    • Browser tab tracking (optional, user opt-in, Chrome/Edge extensions)
    • Recent context sidebar (EVA UI, quick access to recent files/tabs)
    • Context search (find past activities by keyword)
    • User preferences (opt-out per context type)

    Phase 4: Meeting Context & Automation (Sprint 22, Week 10-12)¶

    • Teams/Zoom meeting detection (calendar integration, API hooks)
    • Meeting transcription (optional, user opt-in, Azure Speech Services)
    • Contextual automation (triggers based on detected patterns)
    • Power BI dashboard for context capture stats, OCR usage, automation success
    • Documentation and user guides

    Success Criteria¶

    • ✅ Context capture <100ms (activity → context updated)
    • ✅ OCR extraction <2s (image → text)
    • ✅ Context accuracy >90% (correct activity detection)
    • ✅ OCR accuracy >85% (clean images)
    • ✅ User adoption >35% of Windows users within 6 months
    • ✅ Automation trigger success >80% (relevant suggestions)
    • ✅ PIPEDA compliance (consent, transparency, data deletion)
