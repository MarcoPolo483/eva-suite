    Phase 1: Specification & Setup (Day 1-2)¶

    Agent 1: Project Setup¶

    Similar to Track A, but admin-focused dependencies: - Role-based access control (RBAC) - Data tables (react-table or AG Grid) - Admin-specific routing

    Agent 2: Architecture Documentation¶

    Features: - User management (CRUD) - Tenant management (spaces, documents) - Usage analytics dashboard - Audit log viewer - System health monitoring - RBAC configuration

    Phase 2: Core Implementation (Day 3-10)¶

    Agent 1: Admin Authentication¶

    Deliverables: 1. Azure Entra ID authentication (admin-only) 2. RBAC checks (admin role required) 3. Audit logging (all admin actions)

    API Endpoints (eva-auth): - POST /auth/admin/login - GET /auth/admin/permissions

    Agent 2: User Management¶

    Deliverables: 1. User list with search/filter 2. Create user form 3. Edit user details 4. Delete user (with confirmation) 5. Role assignment (admin, user, viewer) 6. User activity logs

    API Endpoints (eva-api): - GET /api/admin/users - POST /api/admin/users - PUT /api/admin/users/:id - DELETE /api/admin/users/:id - GET /api/admin/users/:id/activity

    Agent 3: Tenant Management¶

    Deliverables: 1. Tenant list with search 2. Create tenant form 3. Edit tenant settings 4. Space management (within tenant) 5. Document access controls 6. Usage limits configuration

    API Endpoints (eva-api): - GET /api/admin/tenants - POST /api/admin/tenants - PUT /api/admin/tenants/:id - GET /api/admin/tenants/:id/spaces - GET /api/admin/tenants/:id/usage

    Agent 4: Analytics Dashboard¶

    Deliverables: 1. Usage metrics overview 2. Active users chart 3. Query volume graph 4. Storage usage pie chart 5. API response times 6. Error rate tracking

    API Endpoints (eva-metering): - GET /api/metrics/usage - GET /api/metrics/performance - GET /api/metrics/errors

    Agent 5: Audit Logs & System Health¶

    Deliverables: 1. Audit log viewer (paginated table) 2. Log filtering (user, action, date range) 3. Log export (CSV, JSON) 4. System health dashboard (CPU, memory, latency) 5. Alert notifications

    API Endpoints (eva-api): - GET /api/admin/audit-logs - GET /api/admin/system-health

    Phase 2: Core Implementation (Day 3-14)¶

    Agent 1: Usage Metrics Dashboard¶

    Deliverables: 1. Active users chart (line graph) 2. Query volume graph (bar chart) 3. Storage usage pie chart 4. API response times (area chart) 5. Date range selector 6. Real-time updates (optional)

    API Endpoints (eva-metering): - GET /api/metrics/users - GET /api/metrics/queries - GET /api/metrics/storage - GET /api/metrics/performance

    Agent 2: Query Analytics¶

    Deliverables: 1. Query frequency table (most common queries) 2. Query latency histogram 3. Query accuracy metrics (if available) 4. Failed queries list 5. Query sources breakdown (chat, API, SDK)

    API Endpoints (eva-api): - GET /api/analytics/queries - GET /api/analytics/query-performance - GET /api/analytics/query-failures

    Agent 3: Document Analytics¶

    Deliverables: 1. Most queried documents (top 10) 2. Least queried documents (bottom 10) 3. Document upload trends (over time) 4. Document size distribution 5. Document type breakdown (PDF, DOCX, TXT)

    API Endpoints (eva-api): - GET /api/analytics/documents - GET /api/analytics/document-usage

    Agent 4: User Behavior Analytics¶

    Deliverables: 1. Session duration histogram 2. User engagement score (queries per session) 3. Feature usage breakdown (chat, search, upload) 4. Returning users vs new users 5. Peak usage times (heatmap)

    API Endpoints (eva-metering): - GET /api/analytics/sessions - GET /api/analytics/engagement - GET /api/analytics/feature-usage

    Agent 5: Cost Analytics¶

    Deliverables: 1. Token usage chart (over time) 2. API costs breakdown (OpenAI, Azure Search, Cosmos DB) 3. Cost per query average 4. Budget alerts (if threshold exceeded) 5. Cost projection (next month)

    API Endpoints (eva-metering): - GET /api/metrics/costs - GET /api/metrics/token-usage - GET /api/metrics/budget
