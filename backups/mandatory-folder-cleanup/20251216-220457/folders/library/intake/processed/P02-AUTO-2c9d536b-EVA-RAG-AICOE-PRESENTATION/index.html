    EVA-RAG: Retrieval-Augmented Generation Engine¶

    AI Center of Excellence (AICOE) Technical Briefing

    Date: December 8, 2025 Presenter: Marco Presta Repository: https://github.com/MarcoPolo483/eva-rag Status: âœ… Production Ready (97.88% test coverage, 129/129 tests passing)

    Executive Summary¶

    EVA-RAG is a production-ready Retrieval-Augmented Generation engine that powers intelligent document search and question-answering for the EVA Suite. It transforms unstructured documents (PDF, DOCX, Excel, PowerPoint, CSV) into semantically searchable knowledge bases using Azure OpenAI embeddings and hybrid search.

    Key Capabilities¶

    • 13 document formats supported (PDF, DOCX, Excel, PowerPoint, MS Project, CSV, XML, HTML, TXT, Markdown)
    • Hybrid search (vector similarity + keyword BM25) with cross-encoder reranking
    • Bilingual support (EN-CA, FR-CA) with automatic language detection
    • Citation extraction with source documents and page numbers
    • Multi-tenant isolation for enterprise deployments
    • 97.88% test coverage (129 tests, <60s execution)
    • 4 deployment methods (Docker, Kubernetes, Azure App Service, Terraform IaC)

    20. Zero-Code Migration Strategy: Swap RAG Providers Instantly¶

    The Power of Abstraction Layers¶

    EVA-RAG is built on eva-core's repository pattern, which means zero code changes are required to swap underlying RAG providers. You can migrate from Azure AI Search to OpenAI Assistants, AWS Bedrock, Google Vertex AI, or even open-source solutions like Chroma/Weaviate without touching a single line of application code.

    How It Works: Repository Pattern in Action¶

    Domain Layer (eva-core) defines the contract:

    # eva-core/src/eva_core/domain/repositories/base.py
from abc import ABC, abstractmethod
from typing import Generic, TypeVar
from pydantic import BaseModel

T = TypeVar("T", bound=BaseModel)

class Repository(ABC, Generic[T]):
    """Abstract base repository - ALL implementations must follow this."""

    @abstractmethod
    async def get(self, id: str) -> T | None:
        """Retrieve entity by ID."""

from chromadb import AsyncClient
from eva_core.domain.repositories import DocumentRepository
from eva_core.domain.entities import Document

class ChromaDocumentRepository(DocumentRepository):
    """ChromaDB implementation of DocumentRepository."""

    def __init__(self, chroma_client: AsyncClient, collection_name: str):
        self.client = chroma_client
        self.collection = self.client.get_or_create_collection(name=collection_name)

    async def get(self, id: str) -> Document | None:
        result = await self.collection.get(ids=[id], include=['metadatas', 'documents'])
        if not result['ids']:
            return None
        return Document(
            id=result['ids'][0],
            content=result['documents'][0],
            **result['metadatas'][0]
        )

    async def save(self, entity: Document) -> Document:
        # Chroma auto-embeds on add
        await self.collection.add(
            ids=[entity.id],
            documents=[entity.content],
            metadatas=[{
                "space_id": entity.space_id,
                "tenant_id": entity.tenant_id,
                "filename": entity.filename
            }]
        )
        return entity

    async def list_by_space(
        self, space_id: str, tenant_id: str, skip: int = 0, limit: int = 100
    ) -> list[Document]:
        # Query Chroma with metadata filters
        results = await self.collection.get(
            where={
                "$and": [
                    {"space_id": space_id},
                    {"tenant_id": tenant_id}
                ]
            },
            limit=limit,
            offset=skip,
            include=['metadatas', 'documents']
        )
        return [
            Document(id=id, content=doc, **meta)
            for id, doc, meta in zip(
                results['ids'], results['documents'], results['metadatas']
