    CDD: EVA Learning Progress Tracking¶

    Last Updated: 2025-11-19 Status: Backlog Priority: HIGH Epic: EVA Learning

    Vision¶

    Provide comprehensive user journey analytics that track learning progress, completion rates, skill development, and ROI metrics—enabling organizations to measure EVA's impact on productivity, knowledge acquisition, and digital transformation goals.

    Business Value: Demonstrate 25% productivity gains, track $2M+ cost savings, enable data-driven training decisions, and prove ROI for executive stakeholders.

    Core Capabilities¶

    • Journey Analytics: Track user progression from onboarding to expert
    • Completion Tracking: Monitor lesson, module, and certification completion rates
    • Skill Badges: Award achievements for milestones (First Query, Power User, Expert)
    • ROI Metrics: Calculate time saved, cost avoided, productivity gains
    • Engagement Scoring: Daily/weekly active users, session duration, feature adoption
    • Learning Paths: Personalized recommendations based on role and gaps
    • Comparative Analytics: Benchmark against peer groups and teams
    • Export & Reporting: Generate executive summaries, compliance reports

    User Stories¶

    US-1: As a Learning Admin, I want to track completion rates so that I identify knowledge gaps¶

    Acceptance Criteria:

    • Dashboard shows completion % for each learning module
    • Filter by role, department, tenure
    • Identify users stuck on specific lessons (>7 days no progress)
    • Export report to Excel/PDF (bilingual)
    • Set target completion thresholds (e.g., 80% within 30 days)

    US-2: As a user, I want skill badges so that I feel motivated to learn¶

    Acceptance Criteria:

    • Award badges: First Query (1 query), Explorer (10 queries), Power User (100 queries), Expert (500 queries)
    • Display badges on user profile and space dashboard
    • Send congratulatory email when badge earned (bilingual)
    • Leaderboard shows top badge earners per space
    • Badges visible to peers (optional privacy setting)

    US-3: As an executive, I want ROI metrics so that I justify EVA's budget¶

    Acceptance Criteria:

    • Calculate time saved: avg query response vs. manual search (baseline 15 min)
    • Cost avoidance: (time saved) × (hourly rate) × (# users)
    • Productivity gain: queries/user/week trend over 6 months
    • Generate executive summary (1-page PDF, bilingual)
    • Real-time dashboard with YTD savings ($2M+ target)

    US-4: As a manager, I want team analytics so that I coach underperforming members¶

    Acceptance Criteria:

    • View team's aggregate learning progress (anonymized)
    • Identify team members below 50% completion
    • Compare team's metrics vs. department average
    • Schedule nudges/reminders for inactive users
    • Export team report for 1-on-1 reviews

    US-5: As a user, I want personalized learning paths so that I focus on relevant skills¶

    Acceptance Criteria:

    • System recommends next module based on role (PS-A, Manager, Executive)
    • Highlight skill gaps compared to peers
    • Show estimated time to complete recommended path
    • Allow user to customize path (skip modules, reorder)
    • Track path completion % on dashboard

    Functional Requirements¶

    FR-1: Journey Tracking¶

    System SHALL capture user milestones: account creation, first login, first query, module completion, badge earned, certification passed.

    FR-2: Completion Metrics¶

    System SHALL calculate completion rates: (completed modules / total modules) × 100, per user, team, role, department.

    FR-3: Skill Badges¶

    System SHALL award badges based on thresholds: First Query (1), Explorer (10), Power User (100), Expert (500), Master (1000 queries).

    FR-4: ROI Calculation¶

    System SHALL compute time saved: (# queries) × (avg time saved per query) × (hourly rate), aggregated per user, team, organization.

    FR-5: Engagement Scoring¶

    System SHALL track DAU (Daily Active Users), WAU (Weekly Active Users), avg session duration, avg queries/session.

    FR-6: Learning Paths¶

    System SHALL recommend modules based on user role, completion history, and skill gap analysis (compare to peer cohort).

    FR-7: Comparative Analytics¶

    System SHALL benchmark user's progress vs. department average, role average, and top 10% performers.

    FR-8: Export Reports¶

    System SHALL generate reports in Excel, PDF, PowerPoint formats (bilingual en-US/fr-FR).

    FR-9: Privacy Controls¶

    System SHALL anonymize data for team/dept analytics (no individual names unless manager has permission).

    FR-10: Target Thresholds¶

    System SHALL allow Learning Admins to set completion targets (e.g., 80% within 30 days) and alert if not met.

    FR-11: Leaderboards¶

    System SHALL display leaderboards (top badge earners, most queries, longest streak) with opt-out option for privacy.

    FR-12: Nudges & Reminders¶

    System SHALL send automated reminders to inactive users (no activity >7 days) via email/Teams notification.

    FR-13: Real-Time Dashboards¶

    System SHALL update metrics in real-time (max 5-minute delay) on Power BI dashboards.

    FR-14: Historical Trends¶

    System SHALL store historical data (24 months minimum) for trend analysis and year-over-year comparisons.

    FR-15: API Access¶

    System SHALL expose REST API for exporting metrics to external systems (HR, LMS, Tableau).

    Non-Functional Requirements¶

    Performance¶

    • Query response time: <2 seconds (P95)
    • Dashboard load time: <3 seconds (P95)
    • Report generation: <30 seconds for 10K users
    • Real-time data refresh: <5 minutes latency
    • Support 50K+ concurrent users

    Reliability¶

    • Uptime: 99.95% SLA (Business Critical)
    • Data durability: 99.999999999% (Cosmos DB)
    • Auto-retry failed event captures (3 attempts)
    • Backup metrics to Azure Storage (daily snapshots)

    Security¶

    • RBAC: Learning Admins see all data, Managers see team only, Users see self only
    • Encrypt PII at rest (AES-256) and in transit (TLS 1.3)
    • Audit log all data exports (Application Insights)
    • PII masking in shared analytics (anonymized user IDs)
    • Compliance: PIPEDA, TBS Directive on Privacy

    Accessibility¶

    • WCAG 2.1 AA compliance (dashboards, reports)
    • Keyboard navigation for all interactive elements
    • Screen reader support (ARIA labels, live regions)
    • High-contrast mode for Power BI visuals

    i18n¶

    • Bilingual support (en-US/fr-FR)
    • ICU formatting for dates, numbers, currency
    • Localized badge names, emails, reports
    • Language switcher on all dashboards

    Observability¶

    • Application Insights custom events (badge earned, report generated, API call)
    • Distributed tracing for API calls (OpenTelemetry)
    • Alerts for anomalies (sudden drop in DAU, spike in errors)
    • Metrics: API latency, error rate, DAU/WAU trends

    Scalability¶

    • Horizontal scaling (Azure Functions for API)
    • Partition Cosmos DB by /tenantId/userId (HPK)
    • Azure CDN for static reports
    • Support 100K users per tenant, 10M+ events/day

    Technical Architecture¶

    Backend (Analytics Engine)¶

    • Event Capture: Azure Functions (Event Hub trigger) for real-time event ingestion
    • Storage: Cosmos DB (transactional), Azure Data Lake Gen2 (analytical)
    • Processing: Azure Synapse Analytics for aggregations, Azure Databricks for ML (recommendations)
    • API: Azure Functions HTTP triggers (REST API for exports)

    Frontend (Dashboards)¶

    • Visualization: Power BI Embedded (real-time dashboards)
    • Reports: SSRS / Power BI Paginated Reports (Excel/PDF export)
    • Admin UI: React SPA (set thresholds, manage badges)

    Integration Points¶

    • eva-api: Event ingestion API (POST /events)
    • eva-auth: RBAC for data access (Learning Admin, Manager, User roles)
    • eva-i18n: Localized strings for badges, emails
    • eva-metering: Cost attribution per query

    Implementation Phases¶

    Phase 1: Event Capture (Sprint 4)¶

    • Event Hub + Azure Functions for event ingestion
    • Cosmos DB schema for events and user progress
    • API endpoints: POST /events, GET /progress/:userId
    • Integration with eva-api for query events
    • Application Insights tracking

    Phase 2: Badges & Leaderboards (Sprint 5)¶

    • Badge logic (thresholds, award, notify)
    • Email notifications (bilingual templates)
    • Leaderboard UI (React component)
    • User profile badge display
    • Privacy opt-out for leaderboards

    Phase 3: Analytics & ROI (Sprint 6)¶

    • Azure Synapse for aggregations
    • Power BI dashboards (Learning Admin, Executive)
    • ROI calculation engine (time saved, cost avoidance)
    • Export to Excel/PDF (paginated reports)
    • Comparative analytics (peer benchmarks)

    Phase 4: Recommendations & Polish (Sprint 7)¶

    • ML model for personalized learning paths (Azure Databricks)
    • Skill gap analysis (compare user vs. cohort)
    • Nudges & reminders (Azure Logic Apps)
    • WCAG 2.1 AA audit for dashboards
    • UAT with pilot departments

    Success Criteria¶

    • ✅ Track 50K+ users' learning journeys (100% coverage)
    • ✅ Award badges with <5 minute latency (real-time)
    • ✅ ROI dashboard shows $2M+ YTD cost savings
    • ✅ Completion rates increase by 30% (vs. no tracking baseline)
    • ✅ Engagement scores: 60% DAU/WAU ratio
    • ✅ Power BI dashboards load in <3 seconds
    • ✅ WCAG 2.1 AA compliance (axe DevTools passes)

    Open Questions¶

    Q1: Privacy Threshold (COMPLIANCE)¶

    Question: Show individual names in team reports or anonymize? Impact: Manager coaching vs. privacy Recommendation: Anonymize by default, require explicit permission for named reports Decision Owner: Security Sentinel + Marco

    Q2: Badge Inflation (UX)¶

    Question: Award too many badges (dilutes value) or too few (demotivates)? Impact: Engagement vs. meaningfulness Recommendation: 5 badge tiers (First Query, Explorer, Power User, Expert, Master), exponential thresholds Decision Owner: UX Partner

    Q3: ROI Baseline (BUSINESS)¶

    Question: Assume 15 min saved per query or measure actual? Impact: Accuracy vs. complexity Recommendation: Use 15 min baseline (industry standard), allow per-tenant override Decision Owner: Marco

    Q4: Learning Paths Logic (TECH)¶

    Question: Rule-based (if-then) or ML-driven recommendations? Impact: Accuracy vs. maintenance Recommendation: Start rule-based (Sprint 5), add ML in Sprint 7 Decision Owner: ai2

    Q5: Historical Data Retention (COMPLIANCE)¶

    Question: Store events indefinitely or archive after 24 months? Impact: Compliance vs. storage cost Recommendation: Hot storage 24 months, archive to Azure Storage (cold tier) after Decision Owner: windowsGuy
