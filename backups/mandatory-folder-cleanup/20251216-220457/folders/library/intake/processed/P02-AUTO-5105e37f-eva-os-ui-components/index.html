    CDD: EVA OS UI Components¶

    Last Updated: 2025-11-19 Status: Backlog Priority: LOW (Moonshot) Epic: EVA OS Integration (Windows Copilot+ Replacement)

    Vision¶

    Embed EVA directly into Windows 11 UI with native components—Win + C quick access panel, taskbar widget, search bar integration, and notification center cards—providing seamless OS-level AI assistance without requiring users to open separate applications, replacing Windows Copilot with an enterprise-grade, tenant-aware alternative.

    Business Value: Reduce friction to AI assistance by 99% (no app launch required), increase user adoption by 3x (OS-native vs. standalone app), establish EVA as the default AI layer for Windows 11+ enterprises, and deliver Microsoft's Copilot+ vision with ESDC's data sovereignty.

    Core Capabilities¶

    • Win + C Quick Panel: Slide-in panel from right (similar to Windows Copilot) with EVA chat
    • Taskbar Widget: EVA icon in taskbar with notification badge (unread suggestions)
    • Search Bar Integration: EVA results in Windows Search (Start menu, Taskbar search)
    • Notification Center Cards: Proactive suggestions as actionable Windows notifications
    • Context Menu Integration: Right-click files/text → "Ask EVA" (contextual queries)
    • Quick Actions: Taskbar flyout with common tasks (summarize, translate, explain)
    • System Tray Icon: EVA status, settings, quick access (battery saver mode)
    • Widgets Board: EVA widget in Windows 11 Widgets Board (news, weather, EVA suggestions)

    User Stories¶

    US-1: As a user, I want to open EVA with a keyboard shortcut so that I access it instantly¶

    Acceptance Criteria:

    • Press Win + C → EVA panel slides in from right (300ms animation)
    • Panel displays EVA chat (recent queries, input box)
    • Panel stays on top of all windows (always accessible)
    • Press Win + C again (or Escape) → panel closes (animation out)
    • Panel state persists across app switches (stays open if user left it open)

    US-2: As a user, I want EVA in my taskbar so that I see notifications at a glance¶

    Acceptance Criteria:

    • Taskbar displays EVA icon (system tray area)
    • Notification badge appears for unread suggestions (red dot, count)
    • Click icon → EVA panel opens (Win + C equivalent)
    • Right-click icon → context menu (Settings, About, Exit)
    • Icon animates on new notification (pulse effect, 1 second)

    US-3: As a user, I want EVA results in Windows Search so that I search across all sources¶

    Acceptance Criteria:

    • Open Start menu → type "Q4 budget"
    • Results include: Files (Budget.xlsx), Web (Bing results), EVA (recent queries, suggestions)
    • EVA results have distinct icon (purple "EVA" badge)
    • Click EVA result → opens EVA panel with context pre-populated
    • Search history: Recent EVA queries appear in suggestions

    US-4: As a user, I want proactive suggestions as notifications so that I stay informed¶

    Acceptance Criteria:

    • EVA detects context (e.g., user opens email about Q4 budget)
    • Windows notification appears: "Summarize this email?" (actionable button)
    • Click notification → EVA panel opens with summary (within 3 seconds)
    • Notification includes preview (first 2 lines of summary)
    • Notification respects Focus Assist (silent during meetings)

    US-5: As a user, I want to right-click text and ask EVA so that I get contextual help¶

    Acceptance Criteria:

    • Select text in any app (Word, browser, PDF) → right-click
    • Context menu includes "Ask EVA" option (below Copy, above Search)
    • Click "Ask EVA" → EVA panel opens with text pre-populated: "Explain: [selected text]"
    • Response within 3 seconds (query → answer displayed)
    • Works across all apps (Word, Excel, browser, Notepad)

    Functional Requirements¶

    1. FR-1: SHALL provide Win + C keyboard shortcut (open/close EVA panel)
    2. FR-2: SHALL display slide-in panel from right (300ms animation, stays on top)
    3. FR-3: SHALL place EVA icon in taskbar (system tray, notification badge)
    4. FR-4: SHALL integrate with Windows Search (EVA results in Start menu, Taskbar search)
    5. FR-5: SHALL send proactive suggestions as Windows notifications (actionable buttons)
    6. FR-6: SHALL add "Ask EVA" to context menus (right-click text/files)
    7. FR-7: SHALL provide quick actions flyout (taskbar icon click: summarize, translate, explain)
    8. FR-8: SHALL display EVA widget in Windows 11 Widgets Board
    9. FR-9: SHALL persist panel state across sessions (open/closed, position)
    10. FR-10: SHALL respect Focus Assist (silent notifications during meetings)
    11. FR-11: SHALL support dark/light themes (Windows theme detection)
    12. FR-12: SHALL provide keyboard shortcuts for all actions (accessibility)
    13. FR-13: SHALL log UI interactions to Application Insights (usage analytics)
    14. FR-14: SHALL enforce tenant isolation (per-user UI state, no cross-tenant leakage)
    15. FR-15: SHALL provide settings UI (configure notifications, shortcuts, appearance)

    Non-Functional Requirements¶

    Performance¶

    • Panel Open: <300ms (Win + C → panel visible)
    • Notification Display: <500ms (context detected → notification shown)
    • Search Integration: <200ms (query typed → EVA results appear)
    • Context Menu: <100ms (right-click → menu displayed)

    Reliability¶

    • Panel Stability: No crashes when opening/closing (99.99% uptime)
    • Notification Delivery: >99% (notifications always delivered)
    • Search Integration: >95% (EVA results always included)
    • Failover: Graceful degradation if EVA service unavailable (show cached results)

    Security¶

    • Data Privacy: UI state not logged (only metadata: panel opened, notification clicked)
    • Tenant Isolation: Per-user UI state (no cross-tenant access)
    • Authorization: UI respects user permissions (RLS enforcement)
    • Audit Logging: All UI interactions logged to Cosmos DB (user ID, action type, timestamp, 1-year retention)

    Accessibility¶

    • WCAG 2.1 AA: All UI components keyboard-accessible (no mouse required)
    • Screen Reader Support: ARIA labels, semantic HTML, live regions
    • Keyboard Shortcuts: Win + C (panel), Alt+E (focus input), Escape (close)
    • High Contrast: UI respects Windows High Contrast themes

    Internationalization¶

    • Locales: en-US (default), fr-FR
    • UI Text: Translated (panel title, buttons, notifications)
    • Keyboard Shortcuts: Localized (Win + C standard across locales)

    Observability¶

    • Metrics: Application Insights tracks panel opens, notification clicks, search queries, context menu uses
    • Alerts: Panel crash rate >0.1%, notification failure >1%, search integration down
    • Dashboards: Power BI dashboard for UI usage, top actions, error trends

    Scalability¶

    • Concurrent Users: Support 10K+ users per Windows Server (VDI scenario)
    • Notification Volume: Handle 100+ notifications/day per user (rate limiting)
    • UI Responsiveness: <16ms frame time (60 FPS, smooth animations)

    Technical Architecture¶

    UI Implementation¶

    • Framework: WinUI 3 (C# .NET 8, native Windows 11 components)
    • Panel: Window with AlwaysOnTop flag, slide-in animation (WinRT APIs)
    • Taskbar Integration: Shell Notification Area API (system tray icon)
    • Search Integration: Windows Search Protocol Handler (custom URI scheme)
    • Notifications: Windows.UI.Notifications API (toast notifications, action buttons)

    Win + C Panel¶

    // Register global hotkey
RegisterHotKey(hwnd, HOTKEY_ID, MOD_WIN, VK_C);

// On hotkey press
if (msg == WM_HOTKEY && wParam == HOTKEY_ID) {
  if (IsPanelOpen()) {
    ClosePanel(); // Slide out animation
  } else {
    OpenPanel(); // Slide in animation
  }
}

void OpenPanel() {
  // Create WinUI 3 window
  var window = new MainWindow {
    Title = "EVA",
    Width = 420,
    Height = 800,
    Topmost = true
  };

  // Position at right edge of screen
  var screen = Screen.PrimaryScreen.WorkingArea;
  window.Left = screen.Right - window.Width;
  window.Top = screen.Top;

  // Slide in animation
  window.RenderTransform = new TranslateTransform { X = 420 };
  window.RenderTransform.BeginAnimation(
    TranslateTransform.XProperty,
    new DoubleAnimation(420, 0, TimeSpan.FromMilliseconds(300))
  );

  window.Show();
}

    Taskbar Widget¶

    // Create system tray icon
var icon = new NotifyIcon {
  Icon = new Icon("eva-icon.ico"),
  Visible = true,
  Text = "EVA Assistant"
};

// Notification badge
icon.BalloonTipTitle = "EVA";
icon.BalloonTipText = "3 new suggestions";
icon.ShowBalloonTip(5000);

// Context menu
icon.ContextMenuStrip = new ContextMenuStrip();
icon.ContextMenuStrip.Items.Add("Open EVA", null, (s, e) => OpenPanel());
icon.ContextMenuStrip.Items.Add("Settings", null, (s, e) => OpenSettings());
icon.ContextMenuStrip.Items.Add("Exit", null, (s, e) => Application.Exit());

    Implementation Phases¶

    Phase 1: Win + C Panel (Sprint 20, Week 1-3)¶

    • WinUI 3 project setup (C# .NET 8, native Windows 11 components)
    • Global hotkey registration (Win + C, Windows API)
    • Slide-in panel (animation, always on top, keyboard accessible)
    • EVA chat integration (embed web view or native controls)
    • Panel state persistence (open/closed, position)

    Phase 2: Taskbar Integration (Sprint 21, Week 4-6)¶

    • System tray icon (notification badge, context menu)
    • Notification delivery (Windows toast notifications, action buttons)
    • Quick actions flyout (summarize, translate, explain)
    • Focus Assist integration (silent during meetings)
    • Settings UI (configure notifications, shortcuts, appearance)

    Phase 3: Search & Context Menu (Sprint 22, Week 7-9)¶

    • Windows Search integration (protocol handler, EVA results)
    • Context menu integration (right-click text/files → "Ask EVA")
    • Search history (recent EVA queries in Windows Search suggestions)
    • Context menu customization (admin-configurable menu items)
    • Cross-app support (Word, Excel, browser, Notepad)

    Phase 4: Widgets & Polish (Sprint 23, Week 10-12)¶

    • Windows 11 Widgets Board integration (EVA widget, recent queries)
    • Dark/light theme support (Windows theme detection, custom themes)
    • Accessibility audit (WCAG 2.1 AA, screen reader support)
    • Power BI dashboard for UI usage, top actions, error trends
    • Documentation and user guides

    Success Criteria¶

    • ✅ Panel open <300ms (Win + C → visible)
    • ✅ Notification delivery >99% (always shown)
    • ✅ Search integration >95% (EVA results always included)
    • ✅ User adoption >50% of Windows users within 6 months
    • ✅ Hotkey usage >60% (users prefer Win + C vs. manual launch)
    • ✅ WCAG 2.1 AA compliance (accessibility audit passed)
    • ✅ User satisfaction >85% (UI/UX survey)
