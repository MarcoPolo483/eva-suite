    CDD: EVA Learning Lab Environments¶

    Last Updated: 2025-11-19 Status: Backlog Priority: MEDIUM Epic: EVA Learning Platform

    Vision¶

    Provide hands-on sandbox environments where learners practice AI/ML skills with real tools—Azure OpenAI Playground, prompt engineering labs, RAG experiments—without risking production systems or incurring uncontrolled costs.

    Business Value: Accelerate skill development by 60%, enable safe experimentation, reduce training costs by $50K/year, and build confident AI practitioners.

    Core Capabilities¶

    • Isolated Environments: Personal sandbox per learner
    • Pre-Configured Tools: Azure OpenAI Playground, Jupyter notebooks, VS Code
    • Guided Labs: Step-by-step tutorials with checkpoints
    • Cost Controls: $10/month budget per learner, auto-pause at limit
    • Progress Tracking: Lab completion, checkpoints passed, time spent
    • Collaboration: Share experiments with peers or instructors
    • Templates: Pre-built labs (prompt engineering, RAG, function calling)
    • Auto-Cleanup: Environments reset after 7 days inactivity

    User Stories¶

    US-1: As a learner, I want a personal sandbox so that I can experiment safely¶

    Acceptance Criteria:

    • Click "Launch Lab" → personal environment provisioned <2 minutes
    • Sandbox includes: Azure OpenAI Playground, Jupyter notebook, sample datasets
    • Isolated from production (no risk to live systems)
    • Cost budget: $10/month (auto-pause at limit, email warning at $8)
    • Auto-cleanup after 7 days inactivity (email warning at 5 days)

    US-2: As a learner, I want guided labs so that I learn step-by-step¶

    Acceptance Criteria:

    • Lab curriculum: Prompt Engineering, RAG Deep Dive, Function Calling, Safety Filters
    • Each lab: learning objectives, step-by-step instructions, checkpoints, quiz
    • Checkpoints auto-verify (e.g., "Prompt must include 3 examples")
    • Progress saved (resume where left off)
    • Completion certificate (PDF) with QR verification

    US-3: As a learner, I want to share experiments so that I get feedback¶

    Acceptance Criteria:

    • Click "Share" → generate shareable link (expires in 30 days)
    • Recipient views notebook read-only (can't edit original)
    • Recipient can fork (create own copy to modify)
    • Commenting on shared experiments (threaded discussions)
    • Instructor can review and grade submissions

    US-4: As an instructor, I want to create custom labs so that I tailor content¶

    Acceptance Criteria:

    • Lab authoring UI (WYSIWYG editor, code blocks, checkpoints)
    • Upload sample datasets (CSV, JSON, PDF)
    • Define checkpoints (auto-grading criteria)
    • Preview lab as learner (test before publishing)
    • Publish to course catalog (assign to cohorts or make public)

    US-5: As a learner, I want cost visibility so that I manage my budget¶

    Acceptance Criteria:

    • Cost dashboard: allocated ($10/month), consumed, remaining
    • Cost breakdown: Azure OpenAI calls (by model), storage, compute
    • Real-time cost updates (every API call logged)
    • Email alerts at $8 (80%), $10 (100%) budget
    • Pause environment manually or automatically at limit

    Functional Requirements¶

    FR-1: Sandbox Provisioning¶

    System SHALL provision personal sandbox (Azure OpenAI, Jupyter, VS Code) in <2 minutes per learner.

    FR-2: Cost Controls¶

    System SHALL enforce $10/month budget per learner with auto-pause at limit and email warnings at 80%.

    FR-3: Guided Labs¶

    System SHALL provide lab templates (Prompt Engineering, RAG, Function Calling) with step-by-step instructions and checkpoints.

    FR-4: Checkpoint Verification¶

    System SHALL auto-verify checkpoints (e.g., prompt quality, RAG accuracy) and provide immediate feedback.

    FR-5: Progress Tracking¶

    System SHALL track lab completion, checkpoints passed, time spent, and display on learner dashboard.

    FR-6: Sharing¶

    Learners SHALL share experiments via shareable link (read-only, expires 30 days) with commenting and forking.

    FR-7: Lab Authoring¶

    Instructors SHALL create custom labs (WYSIWYG editor, code blocks, checkpoints, datasets) and publish to catalog.

    FR-8: Auto-Cleanup¶

    System SHALL auto-cleanup inactive environments (7 days inactivity) with email warning at 5 days.

    FR-9: Collaboration¶

    Learners SHALL collaborate (comment on shared experiments, fork notebooks, peer review).

    FR-10: Certificates¶

    System SHALL issue completion certificates (PDF with QR verification) for finished labs.

    FR-11: Templates¶

    System SHALL provide lab templates (starter notebooks, sample datasets, checkpoints) for common use cases.

    FR-12: Cost Visibility¶

    Learners SHALL view cost dashboard (allocated, consumed, remaining) with breakdown by service.

    FR-13: Bilingual Support¶

    Labs SHALL support bilingual (en-US/fr-FR) instructions and UI.

    FR-14: Accessibility¶

    Lab UI SHALL comply with WCAG 2.1 AA (keyboard navigation, screen reader).

    FR-15: Audit Trail¶

    System SHALL log all lab activities (launch, checkpoint, completion, sharing) for analytics.

    Non-Functional Requirements¶

    Performance¶

    • Sandbox provisioning: <2 minutes
    • Lab load: <3 seconds
    • Checkpoint verification: <1 second
    • Support 1000+ concurrent learners

    Reliability¶

    • Uptime: 99.9% SLA
    • Auto-recovery from failures
    • Daily backups (learner notebooks)
    • Progress auto-saved every 30 seconds

    Security¶

    • Sandbox isolation (no cross-contamination)
    • Cost quotas enforced (prevent runaway costs)
    • PII masked in logs
    • Access control (learners see own sandboxes only)

    Accessibility¶

    • WCAG 2.1 AA compliance
    • Keyboard navigation, screen reader support
    • High-contrast mode

    i18n¶

    • Bilingual labs (en-US/fr-FR)
    • ICU formatting for dates, numbers

    Observability¶

    • Application Insights (lab usage, checkpoint pass rate, cost per learner)
    • Alerts for budget exceeded, failed provisioning
    • Dashboard for instructor (learner progress, completion rate)

    Scalability¶

    • Support 1000+ learners
    • 10K+ lab environments
    • Horizontal scaling (Azure Container Instances)

    Technical Architecture¶

    Frontend¶

    • Framework: React 18 + TypeScript
    • Code Editor: Monaco Editor (VS Code online)
    • Notebooks: JupyterLab embedded

    Backend¶

    • API: Azure APIM + Azure Functions
    • Sandbox: Azure Container Instances (pre-built images)
    • Storage: Blob Storage (notebooks, datasets), Cosmos DB (progress, HPK /userId/labId)
    • Cost Tracking: Azure Cost Management API

    Implementation Phases¶

    Phase 1: Sandbox Infrastructure (Sprint 5)¶

    • Azure Container Instances setup
    • Cost quota enforcement ($10/month)
    • Auto-cleanup (7 days inactivity)
    • Lab provisioning (<2 minutes)

    Phase 2: Guided Labs (Sprint 6)¶

    • Lab templates (Prompt Engineering, RAG, Function Calling)
    • Checkpoint auto-verification
    • Progress tracking dashboard
    • Completion certificates

    Phase 3: Collaboration (Sprint 7)¶

    • Sharing (links, read-only, forking)
    • Commenting on experiments
    • Instructor grading UI
    • Peer review workflow

    Phase 4: Custom Labs (Sprint 8)¶

    • Lab authoring UI (instructors)
    • Dataset upload
    • Custom checkpoints
    • Publish to catalog

    Success Criteria¶

    • ✅ Sandbox provisioning <2 minutes
    • ✅ Cost per learner: <$10/month
    • ✅ Lab completion rate: >70%
    • ✅ Learner satisfaction: CSAT >4.0/5.0
    • ✅ 1000+ learners trained
    • ✅ Zero cost overruns

    Open Questions¶

    Q1: Sandbox Lifecycle (COST)¶

    Question: Keep sandboxes alive 24/7 or start/stop on demand? Impact: Convenience vs. cost Recommendation: Start on demand, auto-stop after 1 hour inactivity Decision Owner: windowsGuy

    Q2: GPT Model Access (FEATURES)¶

    Question: Allow GPT-4o or restrict to GPT-4o-mini (cheaper)? Impact: Learning experience vs. cost Recommendation: GPT-4o-mini for MVP, GPT-4o for advanced labs Decision Owner: Marco

    Q3: Dataset Privacy (SECURITY)¶

    Question: Allow learners to upload their own datasets? Impact: Flexibility vs. PII risk Recommendation: Curated datasets only for MVP, evaluate Q2 2026 Decision Owner: Security Sentinel

    Q4: Collaboration Limits (FEATURES)¶

    Question: Limit number of shared experiments per learner? Impact: Collaboration vs. storage cost Recommendation: 10 active shares max per learner Decision Owner: ai2

    Q5: Instructor Access (SECURITY)¶

    Question: Can instructors access learner sandboxes for troubleshooting? Impact: Support vs. privacy Recommendation: Read-only access with learner consent Decision Owner: Security Sentinel
