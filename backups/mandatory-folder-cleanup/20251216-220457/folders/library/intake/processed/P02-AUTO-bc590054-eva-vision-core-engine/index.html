    CDD: EVA Vision Core Engine¶

    Last Updated: 2025-11-19 Status: Backlog Priority: LOW (Moonshot) Epic: EVA Vision (Power BI Plugin)

    Vision¶

    Provide a context-aware AI chat interface embedded within Power BI dashboards—enabling users to query visualizations, explore data insights, and drill down into metrics using natural language, leveraging EVA's proven AskMe architecture adapted for business intelligence workflows.

    Business Value: Transform static dashboards into interactive AI-powered analytics, reduce time to insight by 70%, democratize data access for non-technical users, and establish EVA as the intelligent layer for enterprise BI platforms.

    Core Capabilities¶

    • Context-Aware Chat: Understands current dashboard (page, visuals, filters) automatically
    • Natural Language Queries: "Show me top 5 regions by sales" → generates DAX, renders visual
    • Visual Interpretation: Click chart → ask "Why did Q3 spike?" → EVA analyzes data + provides explanation
    • Drill-Down Assistance: "Show me details for Canada" → navigates to detail page or creates ad-hoc visual
    • Insight Discovery: Proactive suggestions ("Sales dropped 15% in Ontario last week")
    • DAX Generation: User query → EVA generates DAX measure → validates → adds to report
    • Embedded Widget: Sidebar panel in Power BI Desktop and Service (similar to EVA chat widget)
    • Session Memory: Maintains last 5 queries in context (multi-turn conversations)

    User Stories¶

    US-1: As a business analyst, I want to ask questions about my dashboard so that I get instant insights¶

    Acceptance Criteria:

    • Open Power BI report → EVA chat panel appears in right sidebar
    • Ask "What were total sales last quarter?" → EVA analyzes visuals, returns answer with number + chart
    • Answer includes source visual reference (e.g., "Based on Sales by Quarter chart")
    • Click reference → visual highlights in report (focus mode)
    • Response time <3 seconds P95 (query → answer with visual)

    US-2: As a user, I want EVA to understand my dashboard context so that I don't repeat myself¶

    Acceptance Criteria:

    • EVA auto-detects current page, visible visuals, active filters
    • Ask "Show me sales" → EVA uses current filters (e.g., FY2025, Canada region)
    • Follow-up: "What about last year?" → EVA recalls context (compares FY2024 vs. FY2025)
    • Page navigation → EVA updates context ("Now viewing Customer Analysis page")
    • Context visible in chat ("Context: FY2025, Canada, Sales Dashboard")

    US-3: As a user, I want to drill down into data so that I understand anomalies¶

    Acceptance Criteria:

    • Click spike in line chart → ask "Why did this spike?"
    • EVA analyzes underlying data, identifies drivers (e.g., "3 large orders from Ontario")
    • Provides drill-down options: "View by customer", "View by product", "View raw data"
    • Click option → new visual created or navigate to detail page
    • Save drill-down as bookmark (persist analysis)

    US-4: As a report author, I want EVA to generate DAX measures so that I create metrics faster¶

    Acceptance Criteria:

    • Ask "Create a measure for year-over-year growth"
    • EVA generates DAX: YoY Growth = DIVIDE([Sales], [Sales PY] - 1, 0)
    • Preview measure (sample calculation shown)
    • Validate → measure added to model (appears in Fields pane)
    • Measure explanation provided (how it's calculated, when to use)

    US-5: As a user, I want proactive insights so that I discover hidden trends¶

    Acceptance Criteria:

    • EVA analyzes dashboard data (background, non-intrusive)
    • Insight notification appears: "Sales dropped 15% in Ontario last week"
    • Click notification → drill-down view with explanation
    • Dismiss or snooze insight (configurable frequency: daily, weekly)
    • Insight types: anomalies, trends, correlations, outliers

    Functional Requirements¶

    1. FR-1: SHALL embed chat widget in Power BI Desktop and Service (sidebar panel)
    2. FR-2: SHALL auto-detect dashboard context (page, visuals, filters, slicers)
    3. FR-3: SHALL respond to natural language queries within 3 seconds P95
    4. FR-4: SHALL generate visualizations from queries (bar chart, line chart, table)
    5. FR-5: SHALL generate and validate DAX measures from natural language
    6. FR-6: SHALL provide drill-down options for anomalies (by dimension, raw data)
    7. FR-7: SHALL maintain session memory (last 5 queries for multi-turn conversations)
    8. FR-8: SHALL highlight source visuals when citing data in answers
    9. FR-9: SHALL provide proactive insights (anomalies, trends, correlations)
    10. FR-10: SHALL support bookmark creation (save analysis state)
    11. FR-11: SHALL integrate with Power BI REST API (read report metadata, create visuals)
    12. FR-12: SHALL use Azure OpenAI GPT-4o for query understanding and DAX generation
    13. FR-13: SHALL validate generated DAX (syntax check, semantic validation)
    14. FR-14: SHALL log all queries and interactions to Application Insights
    15. FR-15: SHALL support bilingual queries (en-US, fr-FR) with bilingual responses

    Non-Functional Requirements¶

    Performance¶

    • Query Response: <3s P95 (natural language → answer with visual)
    • Context Detection: <500ms (page load → context extraction)
    • DAX Generation: <2s (query → validated DAX measure)
    • Visual Rendering: <1s (data fetch → chart display)

    Reliability¶

    • Plugin Uptime: 99.9% SLA (aligned with Power BI Service)
    • DAX Validation: >95% accuracy (syntax + semantic correctness)
    • Failover: Graceful degradation if Azure OpenAI unavailable (show error message)
    • Retry Logic: Auto-retry on transient failures (exponential backoff)

    Security¶

    • Authentication: Power BI workspace-level authentication (existing RLS applies)
    • Authorization: User can only query data they have access to (RLS enforcement)
    • Data Privacy: Query logs anonymized (no PII, aggregated metrics only)
    • Audit Logging: All DAX measure creations logged to Cosmos DB (1-year retention)

    Accessibility¶

    • WCAG 2.1 AA: Chat widget keyboard-accessible, screen reader support
    • Keyboard Shortcuts: Alt+E to open EVA chat, Escape to close
    • Focus Management: Logical tab order (input → send → visual references)
    • High Contrast: Respects Power BI theme (dark/light mode)

    Internationalization¶

    • Locales: en-US (default), fr-FR
    • DAX Generation: Supports English and French function names (CALCULATE vs. CALCULER)
    • Number Formatting: Respects Power BI report locale (currency, dates)

    Observability¶

    • Metrics: Application Insights tracks queries, DAX generations, visual creations, errors
    • Alerts: Query latency >5s, DAX validation failures >10%, error rate >2%
    • Dashboards: Power BI dashboard for plugin usage, top queries, DAX generation success rate

    Scalability¶

    • Concurrent Users: Support 1K+ simultaneous Power BI users
    • Query Volume: Handle 10K+ queries/day (cached responses for common queries)
    • DAX Complexity: Support measures with up to 1000 characters

    Technical Architecture¶

    Frontend¶

    • Framework: Power BI Custom Visual SDK (TypeScript, D3.js)
    • UI Components: Chat interface (message list, input box, visual preview)
    • State Management: React hooks (local state), WebSocket (real-time updates)
    • Rendering: D3.js for ad-hoc visuals, Power BI SDK for dashboard interactions

    Backend¶

    • API Gateway: Azure APIM (rate limiting, JWT validation)
    • Functions: Azure Functions (Node.js 20) for query processing, DAX generation, context extraction
    • AI Service: Azure OpenAI GPT-4o (query understanding, DAX generation)
    • Power BI Integration: Power BI REST API (read report metadata, create visuals)
    • Validation: DAX.js library (syntax checking), Power BI Tabular Model (semantic validation)

    Implementation Phases¶

    Phase 1: Core Chat Interface (Sprint 10, Week 1-3)¶

    • Power BI Custom Visual SDK setup (chat widget sidebar)
    • Context extraction (read current page, visuals, filters via Power BI API)
    • Natural language query processing (Azure OpenAI GPT-4o)
    • Answer generation with visual references (cite source charts)
    • Session memory (last 5 queries, multi-turn context)

    Phase 2: DAX Generation (Sprint 11, Week 4-6)¶

    • Natural language → DAX translation (GPT-4o fine-tuned on DAX examples)
    • DAX syntax validation (DAX.js library)
    • Semantic validation (check table/column names exist in model)
    • Measure preview (sample calculation, test against data)
    • Add measure to Power BI model (via Tabular Model API)

    Phase 3: Drill-Down & Visuals (Sprint 12, Week 7-9)¶

    • Anomaly detection (identify spikes, drops, outliers)
    • Drill-down options (by dimension, raw data)
    • Ad-hoc visual creation (bar chart, line chart, table from query)
    • Bookmark creation (save analysis state)
    • Visual highlighting (focus mode on source chart)

    Phase 4: Proactive Insights (Sprint 13, Week 10-12)¶

    • Background data analysis (scheduled, non-intrusive)
    • Insight generation (anomalies, trends, correlations)
    • Notification system (inline badge in chat widget)
    • Insight drill-down (click → detailed analysis)
    • Power BI dashboard for plugin adoption, top queries, DAX success rate

    Success Criteria¶

    • ✅ Query response <3s P95 (natural language → answer)
    • ✅ DAX validation >95% accuracy (syntax + semantic)
    • ✅ User adoption >40% of Power BI users within 6 months
    • ✅ Time to insight reduction >70% (vs. manual exploration)
    • ✅ DAX generation success >85% (valid measures on first attempt)
    • ✅ User satisfaction >80% (post-query survey)
    • ✅ WCAG 2.1 AA compliance (accessibility audit passed)
