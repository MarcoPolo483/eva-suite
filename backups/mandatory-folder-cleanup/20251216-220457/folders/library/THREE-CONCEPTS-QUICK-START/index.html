    The Three Concepts Pattern - Quick Start¶

    What: Daily operating system for EVA Suite development Why: Prevents "vibe coding", silent failures, blind searching Impact: Exponential quality improvement across 33 repos

    üéØ The Three Concepts¶

    1Ô∏è‚É£ Context Engineering¶

    Prevents: "Vibe coding" (code that compiles but violates EVA standards) How: 4-block prompt pattern (Context ‚Üí Task ‚Üí Output ‚Üí Self-check) When: EVERY code generation, refactor, new feature Reference: docs/context-engineering.md

    Quick Template: 

    üß© CONTEXT
- Mission: [1-line purpose]
- Constraints: WCAG 2.1 AA, GC Design System, bilingual, RBAC-aware
- Specs: [link to requirements.md, architecture-notes.md]
- Reuse: [existing components from feature-map.md]

üß© TASK
- What: [specific component/function]
- Scope: [one file, narrow focus]
- Don't change: [public APIs, contracts]

üß© OUTPUT
- Code: [full implementation]
- Explanation: [how it aligns with specs/constraints]
- TODOs: [assumptions, future work]

üß© SELF-CHECK
- ‚úÖ WCAG compliance? (keyboard nav, screen reader, contrast)
- ‚úÖ Bilingual? (i18n catalogs, no hardcoded text)
- ‚úÖ GC Design System? (theme variables, no hardcoded colors)
- ‚úÖ RBAC-aware? (receives context via props, no UI authorization)
- ‚úÖ Reuses existing patterns? (checked feature-map.md)

    2Ô∏è‚É£ Workspace Management & Housekeeping¶

    Prevents: Silent failures, technical debt accumulation How: Automated health checks (session start, nightly, pre-commit) When: Session start (MANDATORY), post-implementation, nightly automation Reference: scripts/Invoke-EvaHousekeeping.ps1, PATTERN-003

    Session Start Checklist: 

    # 1. Check directory map freshness
Test-Path "c:\Users\marco\Documents\_AI Dev\EVA Suite\eva-lwo\eva-suite-full-tree.txt"
(Get-Item "c:\Users\marco\Documents\_AI Dev\EVA Suite\eva-lwo\eva-suite-full-tree.txt").LastWriteTime
# Should be < 48 hours old

# 2. Check workflow health
gh run list --limit=10
# Look for failures in: autonomous-queue-executor, p02-intake-pipeline, lesson-sync

# 3. Check housekeeping status
.\scripts\Invoke-EvaHousekeeping.ps1 -SkipTests -SkipHealthCheck
# Should show: 0 attention items

# 4. Quick tree (for session context)
.\scripts\Start-AISession.ps1

    3Ô∏è‚É£ Directory Mapping Pattern¶

    Prevents: Slow file searching (1600x speedup) How: Pre-computed workspace/repo trees When: ALWAYS check FIRST before file_search/semantic_search Reference: LESSON-001, TOOL-004

    Tree Files: - Global: eva-lwo/eva-suite-full-tree.txt (nightly refresh, 33 repos) - Per-repo: .eva-cache/repo-tree.txt (bootstrap + on-demand) - Quick: Temp trees at session start (depth-limited for speed)

    Usage: 

    # Search global tree (INSTANT)
grep_search -pattern "APIM|API Management" -includePattern "eva-lwo/eva-suite-full-tree.txt"

# Search per-repo tree
grep_search -pattern "eva-button" -includePattern ".eva-cache/repo-tree.txt"

# Refresh if stale
.\scripts\refresh-eva-suite-tree.ps1  # Global (30s)
.\scripts\Quick-WorkspaceTree.ps1     # Session start (3s)

    During Work (Feature Development)¶

    1. Check directory map for relevant files
   - grep_search on eva-suite-full-tree.txt
   - Find existing components to reuse

2. Use Context Engineering 4-block prompt
   - Context: Mission, constraints, specs, reuse
   - Task: What, scope, preserve
   - Output: Code, explanation, TODOs
   - Self-check: WCAG/i18n/GC DS/RBAC/reuse

3. Validate with housekeeping
   - Run tests (npm test)
   - Check coverage (80% minimum)
   - Verify no violations (lint, type-check)

    End of Day (Commit)¶

    # 1. Run housekeeping check
.\scripts\Invoke-EvaHousekeeping.ps1

# 2. Update memory (if new lessons learned)
# Edit .eva-memory.json ‚Üí Add LESSON-XXX or TOOL-XXX

# 3. Commit + push
git add .

git push

    üìä How to Know It's Working¶

    Context Engineering¶

    • Before: Generic code, no constraints, "looks right"
    • After: Spec-aligned code, explicit constraints, "provably correct"
    • Metric: 0 "vibe coded" components in PRs

    Housekeeping¶

    • Before: Silent failures accumulate for days
    • After: Issues caught within hours
    • Metric: 0 attention items on session start

    Directory Mapping¶

    • Before: 1600 seconds to traverse workspace manually
    • After: <1 second to search pre-computed tree
    • Metric: Session start < 10 seconds (tree cached)

    ‚ö†Ô∏è Common Mistakes (and Fixes)¶

    Mistake 1: Skipping Context Engineering¶

    Symptom: Code compiles but violates WCAG/GC DS/bilingual Fix: ALWAYS use 4-block prompt. No shortcuts.

    Mistake 2: Ignoring Housekeeping Warnings¶

    Symptom: "Just this once" turns into technical debt Fix: Address attention items IMMEDIATELY. Don't accumulate.

    Mistake 3: Using file_search Instead of Trees¶

    Symptom: Slow file discovery, incomplete results Fix: Check eva-suite-full-tree.txt FIRST. Always.
