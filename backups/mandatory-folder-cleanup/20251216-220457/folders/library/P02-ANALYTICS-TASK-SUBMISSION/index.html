    Task Overview¶

    Submitted comprehensive requirements gathering task to P02 Requirements Engine for EVA Suite public website analytics implementation.

    Scope¶

    • Google Analytics 4 (GA4) - User tracking, traffic metrics, conversion goals
    • Microsoft Clarity - Session recordings, heatmaps, frustration signals
    • Cookie Consent Banner - GDPR/CCPA compliant, granular controls
    • Privacy Policy - Government sector standards, plain language
    • Weekly Automated Reports - GitHub Actions workflow, trend analysis
    • Voice Accessibility Integration - Future-ready analytics for voice features
    • Compliance - GDPR, CCPA, WCAG 2.1 AAA

    Agentic Patterns Applied¶

    Primary: P02 – Requirements Agent¶

    • Recursive refinement from high-level objectives to detailed specifications
    • Component Design Document (CDD) methodology
    • Clear acceptance criteria for each requirement
    • Phased implementation approach

    Supporting Patterns:¶

    • P10 – Metrics & Insight Agent: Dashboard design, weekly reporting, trend analysis
    • P11 – Security & Compliance Helper: GDPR/CCPA mapping, privacy requirements
    • P12 – UX, Copy & Accessibility Agent: Cookie banner UX, WCAG 2.1 AAA compliance

    Task Structure¶

    Inputs Provided¶

    1. Context - Background, current state, target audience, compliance level
    2. Functional Requirements (7 total):
    3. ANLY-F-001: GA4 Integration
    4. ANLY-F-002: Microsoft Clarity
    5. ANLY-F-003: Traffic Metrics
    6. ANLY-F-004: Cookie Consent Banner
    7. ANLY-F-005: Privacy Policy
    8. ANLY-F-006: Weekly Automated Reports

    9. ANLY-F-007: Voice Accessibility Integration

    10. Non-Functional Requirements (4 total):

    11. ANLY-NF-001: Performance (page load <100ms impact)
    12. ANLY-NF-002: Privacy & Security (GDPR/CCPA/Government)
    13. ANLY-NF-003: Accessibility (WCAG 2.1 AAA)

    14. ANLY-NF-004: Maintainability (configuration, documentation)

    15. Technical Specifications:

    16. GA4: Measurement ID, events, user properties, conversions
    17. Clarity: Project ID, session recordings, heatmaps, PII masking

    18. Cookie Consent: Categories, storage, expiry, revocation

    19. Implementation Phases (5 phases):

    20. Phase 1: Foundation Setup (GA4 + Clarity accounts, basic tracking)
    21. Phase 2: Cookie Consent & Privacy (banner UI, privacy policy)
    22. Phase 3: Advanced Tracking (custom events, heatmaps, conversions)
    23. Phase 4: Reporting & Automation (weekly reports workflow)

    24. Phase 5: Accessibility & Compliance Audit (WCAG, GDPR, CCPA)

    25. Compliance Checklist:

    26. GDPR: Articles 5, 7, 12, 13, 15, 17
    27. CCPA: Sections 1798.100, 1798.105
    28. WCAG 2.1 AAA: Success Criteria 1.4.4, 1.4.6, 2.1.1, 2.4.7, 4.1.3
