    RAG Metadata Reference for EVA Library Knowledge Sources¶

    Pattern: P22-DPB (Documentation Publishing Bot) Version: 1.1.0 (RBAC + Citations) Updated: 2025-12-13

    Overview¶

    EVA Library Books are published as RAG knowledge sources with comprehensive metadata for: - RBAC filtering (privacy + role-based access) - Citation attribution (URLs, tags, chapter info) - Search optimization (semantic tags, categories)

    This enables EVA Data Pipeline to expose Books as searchable knowledge while respecting access controls.

    Metadata Structure¶

    Every RAG text file includes a metadata header:

    # ============================================================================
# METADATA
# ============================================================================
# Source: /path/to/source.md
# Book: EVA Agentic Book
# Chapter: Introduction
# Category: Architecture
# Extracted: 2025-12-13 05:28:23 UTC
# Format: RAG Plain Text (optimized for vector embedding)
#
# RBAC (Role-Based Access Control)
# Privacy: public
# RequiredRoles: Anonymous, Authenticated
#
# CITATION
# URL: https://marcopolo483.github.io/eva-orchestrator/eva-agentic-book/
# Tags: agentic, multi-agent, orchestration, autonomy, C0-C3
#
# USAGE INSTRUCTIONS FOR RAG SYSTEMS:
# - Use RBAC metadata to filter search results based on user roles
# - Include URL in citations when presenting content to users
# - Use Tags for semantic search and content discovery
# - Privacy="admin" requires authentication before exposing content
# ============================================================================

    RBAC Filtering (Privacy Levels)¶

    Public Books (privacy: "public")¶

    • Access: Anonymous, Authenticated users
    • Examples: EVA Agentic Book, EVA Library, EVA Sovereign UI
    • Use Case: Public documentation, component catalogs, tutorials
    • Query Filter: No authentication required

    Admin Books (privacy: "admin")¶

    • Access: Developer, TechLead, PlatformOwner roles only
    • Examples: DevTools Patterns (P01-P22), Internal runbooks
    • Use Case: Internal development patterns, operational procedures
    • Query Filter: Requires authenticated user with appropriate role

    Implementation in RAG Queries¶

    Before querying Azure AI Search or vector store:

    # Pseudo-code for RBAC filtering
user_roles = get_user_roles(current_user)  # e.g., ["Authenticated", "Developer"]

if user_roles.includes("Anonymous"):
    # Public only
    filter = "privacy eq 'public'"
elif user_roles.includes("PlatformOwner"):
    # Full access
    filter = None
elif user_roles.includes("Developer"):
    # Public + Admin
    filter = "privacy eq 'public' or privacy eq 'admin'"
else:
    # Authenticated users: public only
    filter = "privacy eq 'public'"

results = search_index(query, filter=filter)

    Citation Attribution¶

    When presenting search results to users, always include citations:

    // Example: EVA Chat Assistant response
interface ChatResponse {
  answer: string;
  citations: Citation[];
}

interface Citation {
  source: string;        // "EVA Agentic Book - Introduction"
  url: string;           // "https://marcopolo483.github.io/..."
  relevanceScore: number;
  privacy: string;       // "public" or "admin"
}

// User sees:
// "According to the EVA Agentic Book [1], autonomous agents..."
// [1] EVA Agentic Book - Introduction
//     https://marcopolo483.github.io/eva-orchestrator/eva-agentic-book/

    Search Optimization with Tags¶

    Tags enable semantic search beyond keyword matching:

    Book Tags Use Case
    EVA Agentic Book agentic, multi-agent, orchestration, autonomy, C0-C3 Users asking about "AI agents", "orchestration", "autonomy levels"
    EVA Library web-components, lit, gc-design-system, wcag, a11y, i18n Developers searching for "accessibility", "bilingual UI", "GC Design"
    DevTools Patterns devtools, patterns, automation, sdlc, p01-p22 Teams looking for "CI/CD patterns", "testing automation", "P02"

    Query Enhancement: 

    # User query: "How do I make my UI accessible?"
# Expand to include tags:
query = "accessible UI WCAG a11y bilingual"
# Automatically surfaces EVA Library content

    Adding New Books¶

    To register a new Book as a RAG knowledge source:

    1. Add Book Definition in extract-rag-text.ps1: 

      "my-new-book" = @{
    Title = "My New Book"
    Description = "Description for search results"
    Category = "Tutorial"  # Architecture, Components, DevTools, Tutorial, etc.
    Privacy = "public"     # or "admin"
    RBACRoles = @("Anonymous", "Authenticated")  # or @("Developer", "TechLead")
    Tags = @("tutorial", "beginner", "getting-started")
    URL = "https://marcopolo483.github.io/my-repo/book/"
    Priority = 10  # Lower = higher priority in search results
}

    2. Create Markdown Files in docs/my-new-book/*.md

    3. Run Extraction: 

      .\tools\eva-library\extract-rag-text.ps1 -AllBooks

    4. Verify Output: Check rag-output/my-new-book/_manifest.json

    5. Deploy: GitHub Actions automatically publishes to /rag-output/ on push

    Integration with EVA Data Pipeline¶

    Ingestion Flow:

    1. GitHub Actions runs extract-rag-text.ps1
   
2. Generates rag-output/{book}/*.txt + _manifest.json
   
3. Uploads to GitHub Pages (public URL)
   
4. EVA Data Pipeline ingests from URL
   
5. Azure AI Search index with RBAC metadata
   
6. EVA Chat Assistant queries with role-based filtering

    EVA Data Pipeline Configuration (future): 

    {
  "sources": [
    {
      "name": "EVA Agentic Book",
      "type": "rag-manifest",
      "url": "https://marcopolo483.github.io/eva-orchestrator/rag-output/eva-agentic-book/_manifest.json",
      "rbac": {
        "inherit": true
      }
    }
  ]
}

    Quality Gates¶

    Before promoting RAG content to Production:

    • Metadata Complete: All fields populated (privacy, roles, tags, URL)
    • RBAC Validated: Privacy level matches intended audience
    • Tags Relevant: Tags improve search discoverability
    • URLs Valid: Citation URLs return 200 OK
    • Manifest Hash: SHA256 hashes match file contents
    • UTF-8 Encoding: No character corruption
