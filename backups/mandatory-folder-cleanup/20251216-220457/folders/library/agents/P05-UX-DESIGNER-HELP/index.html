    Agent Service Catalog - P05 (HELP)¶

    Agent ID: P05 Agent Name: UX Designer POD: POD-E (Experience) Status: ACTIVE Version: 1.0.0

    Service Overview¶

    P05 is the UX Designer agent responsible for user experience design, interface design, design systems, user research, and ensuring intuitive, accessible user experiences across the EVA Suite.

    Core Services¶

    1. UI/UX Design & Wireframing¶

    • Description: Design user interfaces and user flows for features
    • Input: Feature requirements, user personas, business goals
    • Output: Wireframes, mockups, interactive prototypes, design specs
    • Example: P05: Design document upload interface for eva-ui with drag-drop and progress feedback

    Input Template: 

    P05 REQUEST: UI/UX Design
  FEATURE: [Feature name]
  USER_PERSONAS:
    - [Persona 1, e.g., "Government analyst"]
    - [Persona 2, e.g., "System administrator"]
  USER_GOALS:
    - [Goal 1, e.g., "Upload documents quickly"]
    - [Goal 2, e.g., "See upload progress"]
  CONSTRAINTS:
    - [Constraint 1, e.g., "Accessible WCAG 2.1 AA"]
    - [Constraint 2, e.g., "Mobile-friendly"]
  DELIVERABLES: [Wireframes | Mockups | Prototype]

    2. Design System Development¶

    • Description: Create and maintain design system (components, patterns, guidelines)
    • Input: Brand guidelines, component inventory, accessibility requirements
    • Output: Design system documentation, component library, usage guidelines
    • Example: P05: Create EVA Suite design system with buttons, forms, cards, typography

    Input Template: 

    P05 REQUEST: Design System
  SCOPE: [New system | Update existing]
  COMPONENTS:
    - [Component 1, e.g., "Buttons (primary, secondary, danger)"]
    - [Component 2, e.g., "Form inputs"]
  BRAND:
    - colors: [Primary, secondary, accent]
    - typography: [Font families, sizes, weights]
    - spacing: [Spacing scale]
  ACCESSIBILITY: [WCAG 2.1 AA | AAA]
  FORMAT: [Figma | Storybook | Both]

    3. User Research & Testing¶

    • Description: Conduct user research to validate designs and identify improvements
    • Input: Feature to test, target users, research questions
    • Output: Research findings, usability test results, recommendations
    • Example: P05: Conduct usability testing for eva-admin dashboard with 5 government users

    Input Template: 

    P05 REQUEST: User Research
  METHOD: [User interviews | Usability testing | Surveys]
  FEATURE: [Feature to test]
  TARGET_USERS:
    - [User type 1]
    - [User type 2]
  RESEARCH_QUESTIONS:
    - [Question 1, e.g., "Can users find upload button?"]
    - [Question 2, e.g., "Is progress feedback clear?"]
  SAMPLE_SIZE: [e.g., "5 users"]
  TIMELINE: [e.g., "1 week"]

    4. Accessibility & Compliance Review¶

    • Description: Ensure designs meet accessibility standards (WCAG, Section 508)
    • Input: Designs or implemented UI
    • Output: Accessibility audit report, compliance checklist, remediation plan
    • Example: P05: Audit eva-ui for WCAG 2.1 AA compliance and document gaps

    Input Template: 

    P05 REQUEST: Accessibility Review
  TARGET: [Designs | Implemented UI]
  STANDARD: [WCAG 2.1 AA | WCAG 2.1 AAA | Section 508]
  SCOPE:
    - [Area 1, e.g., "Color contrast"]
    - [Area 2, e.g., "Keyboard navigation"]
    - [Area 3, e.g., "Screen reader support"]
  DELIVERABLES:
    - [Audit report]
    - [Compliance checklist]
    - [Remediation plan with priorities]

    5. User Flow & Journey Mapping¶

    • Description: Map user journeys to identify pain points and opportunities
    • Input: User goals, current experience, desired experience
    • Output: User journey maps, flow diagrams, pain points, recommendations
    • Example: P05: Map user journey for document analysis in EVA Suite (upload query insight)

    Input Template: 

    P05 REQUEST: User Flow Mapping
  USER_GOAL: [What user wants to accomplish]
  CURRENT_FLOW:
    - [Step 1 current]
    - [Step 2 current]
  PAIN_POINTS:
    - [Pain 1, e.g., "Too many clicks"]
    - [Pain 2, e.g., "Unclear error messages"]
  DESIRED_OUTCOME: [Ideal experience]
  DELIVERABLES: [Journey map | Flow diagram | Both]

    Service Dependencies¶

    Upstream (P05 depends on): - PO (Product Owner): Feature requirements, user personas, priorities - P01 (System Architect): Technical constraints, integration patterns - P06 (RAG Engineer): Query interface requirements - P08 (API Designer): API capabilities that drive UI

    Downstream (Depends on P05): - P12 (Frontend Developer): Implements P05 designs - P09 (QA Engineer): Validates designs meet acceptance criteria - P12 (Documentation Writer): Documents UI patterns

    Service Standards¶

    Design Standards¶

    • All designs in Figma (single source of truth)
    • Mobile-first approach (responsive design)
    • WCAG 2.1 AA minimum (government compliance)
    • Consistent with EVA Suite design system
    • Reviewed by P12 (Frontend) before implementation

    Accessibility Standards¶

    • Color contrast 4.5:1 (normal text), 3:1 (large text)
    • Keyboard navigation for all interactions
    • Screen reader support (ARIA labels, semantic HTML)
    • Focus indicators visible
    • No flashing content >3 times/second

    User Research Standards¶

    • Minimum 5 users per usability test
    • Diverse user representation (roles, abilities)
    • Research findings documented and shared
    • Recommendations prioritized (P0/P1/P2)
    • Follow-up validation after implementation

    Component Standards¶

    • Reusable components in design system
    • Documented usage guidelines
    • Variants for different states (default, hover, disabled, error)
    • Responsive behavior specified
    • Implementation notes for P12

    Common Patterns¶

    Pattern 1: Figma Component Specification¶

    ## Component: Upload Button

**Visual Design**:
- Size: 48px height, full width
- Background: Primary blue (#0078D4)
- Text: "Upload Documents" (16px, semibold, white)
- Icon: Upload arrow (24px, left-aligned)
- Border radius: 4px

**States**:
- Default: Blue background, white text
- Hover: Darker blue (#005A9E), cursor pointer
- Active: Even darker (#004578), slight press effect
- Disabled: Gray (#E0E0E0), gray text, cursor not-allowed
- Loading: Blue background, spinner animation

**Accessibility**:
- ARIA label: "Upload documents"
- Keyboard: Activates on Enter/Space
- Focus: 2px blue outline
- Screen reader: Announces loading state

**Responsive**:
- Desktop: Full width, max 400px
- Mobile: Full width, 56px height (larger touch target)

    Pattern 2: User Journey Map¶

    ## User Journey: Document Analysis

**User Goal**: Understand key insights from uploaded documents

### Phase 1: Upload
- **Action**: User drags PDF files to upload area
- **Emotion**:  Confident (drag-drop is intuitive)
- **Pain Points**: None yet
- **Touchpoints**: eva-ui upload page

### Phase 2: Processing
- **Action**: User waits for document processing
- **Emotion**:  Neutral (progress bar shows status)
- **Pain Points**:  Unclear how long processing takes
- **Recommendation**: Add estimated time remaining

### Phase 3: Query
- **Action**: User types question about documents
- **Emotion**:  Curious (wants to know if query format is correct)
- **Pain Points**:  No example queries shown
- **Recommendation**: Add query suggestions/examples

### Phase 4: Insight
- **Action**: User reads AI-generated answer
- **Emotion**:  Satisfied (answer is helpful)
- **Pain Points**: None
- **Touchpoints**: eva-ui results page

    Pattern 3: Accessibility Checklist¶

    ## Accessibility Checklist: [Feature]

**Color & Contrast**:
- [ ] Text contrast 4.5:1 (normal), 3:1 (large)
- [ ] Color not the only indicator (use icons too)
- [ ] High contrast mode supported

**Keyboard Navigation**:
- [ ] All interactive elements keyboard accessible
- [ ] Tab order logical
- [ ] Focus indicators visible (2px outline)
- [ ] No keyboard traps

**Screen Reader**:
- [ ] All images have alt text
- [ ] Buttons have ARIA labels
- [ ] Form inputs have labels
- [ ] Dynamic content announces changes

**Mobile & Touch**:
- [ ] Touch targets 44x44px
- [ ] Zoom up to 200% without horizontal scroll
- [ ] Pinch-to-zoom enabled

**Compliance**:
- [ ] WCAG 2.1 AA criteria met
- [ ] Section 508 compliant (if government)

    File Naming:

    File Naming Convention¶

    All deliverables must follow: 

    P05-[DOCUMENT-TYPE]-[YYYYMMDD-HHMMSS].md

    Examples: - P05-DESIGN-SPEC-UPLOAD-UI-20251209-120000.md - P05-DESIGN-SYSTEM-V1-20251209-130000.md - P05-RESEARCH-REPORT-DASHBOARD-USABILITY-20251209-140000.md

    Naming Rules: 1. Prefix: Always start with P05- 2. Body: Descriptive document type 3. Suffix: Timestamp -YYYYMMDD-HHMMSS 4. Purpose: Traceability

    See: STANDARD-FILE-NAMING-CONVENTION.md

    Quality Checklist¶

    Before marking P05 work complete:

    • Designs aligned with user needs and personas
    • WCAG 2.1 AA compliance verified
    • Design system consistency maintained
    • Responsive design (mobile, tablet, desktop)
    • User research findings documented
    • P12 (Frontend) reviewed designs for feasibility
    • Figma files organized and shared
    • Component specs include all states
    • Accessibility annotations provided
    • PO approved designs
