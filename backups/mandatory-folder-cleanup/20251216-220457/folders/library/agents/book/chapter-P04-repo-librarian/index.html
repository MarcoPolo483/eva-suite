    Chapter P04: Repo Librarian¶

    Pattern Code: P04 Short Code: LIB Name: Repo Librarian Type: DevTools Pattern Scope: Documentation Quality & Consistency Maturity: Stable Autonomy Level: C1 (Human-on-the-loop)

    üìñ Chapter Overview¶

    The Repo Librarian (P04-LIB) serves as the documentation quality guardian for the EVA Suite, ensuring that all 10+ core repositories maintain consistent, up-to-date, and compliant documentation. Like a traditional librarian who catalogs books and ensures proper organization, P04-LIB systematically audits README files, Architecture Decision Records (ADRs), and bilingual documentation across the EVA ecosystem.

    In a multi-repository architecture with distributed teams, documentation drift is inevitable without active management. Files become outdated, architectural decisions go undocumented, and bilingual compliance erodes over time. P04-LIB addresses this through automated drift detection, standardized templates, and proactive maintenance workflows.

    The Problem P04 Solves: - Documentation Debt: READMEs missing critical sections (Quick Start, Architecture, Testing) - ADR Coverage Gaps: Major architectural decisions undocumented or poorly tracked - Bilingual Compliance: French-Canadian documentation missing or out of sync with English - Inconsistent Structure: Each repo invents its own documentation format - Drift Detection: No automated way to identify documentation quality issues

    The Value P04 Delivers: - Consistency: All repos follow same README structure and ADR format - Discoverability: Developers quickly find installation, testing, architecture info - Compliance: Bilingual requirements (EN-CA/FR-CA) systematically enforced - Quality Signals: Drift reports highlight repos needing documentation attention - Onboarding Speed: New team members navigate unfamiliar repos faster with predictable structure

    üéØ Role Definition and Purpose¶

    Primary Role¶

    P04-LIB serves as the Documentation Quality Assurance Agent for the EVA Autonomous Agile Command. It ensures:

    1. Structural Consistency: All repos follow standardized README and ADR templates
    2. Coverage Tracking: Architectural decisions are documented systematically
    3. Bilingual Compliance: EN-CA and FR-CA documentation requirements met
    4. Drift Detection: Automated identification of documentation quality issues
    5. Proactive Maintenance: Auto-fix capabilities for common documentation gaps

    Mission Statement¶

    "Maintain high-quality, consistent, and compliant documentation across all EVA Suite repositories through automated quality checks, standardized templates, and proactive drift remediation."

    Value Proposition¶

    For Product Owners: - Reduced onboarding time (predictable documentation structure) - Compliance confidence (bilingual requirements systematically enforced) - Quality visibility (drift reports surface documentation debt)

    For Developers: - Faster repo navigation (know where to find info) - Reduced documentation burden (templates and auto-fix) - Clear architectural context (ADR coverage)

    For Operations: - Consistent runbook structure across repos - Architectural decision history for troubleshooting - Bilingual support documentation for GoC compliance

    üìã Responsibilities¶

    Core Responsibilities¶

    1. README Standards Enforcement
    2. Validate presence of required sections (Quick Start, Architecture, Development, Documentation)
    3. Check formatting consistency (headings, code blocks, links)
    4. Ensure metadata accuracy (Purpose, POD, Owner, Status)

    5. Verify installation/usage examples are present and testable

    6. ADR Coverage Tracking

    7. Monitor docs/adr/ directory structure in all repos
    8. Count ADR files and flag repos with insufficient coverage
    9. Validate ADR format compliance (Status, Date, Context, Decision, Consequences)

    10. Ensure major architectural decisions are documented

    11. Bilingual Compliance Auditing

    12. Check for presence of README.fr.md (French-Canadian)
    13. Validate synchronization between EN-CA and FR-CA content
    14. Flag missing translations or outdated French documentation

    15. Support P12 (UX/Copy Agent) for translation quality review

    16. Documentation Drift Detection

    17. Run automated checks across all 10 core repos
    18. Generate drift reports highlighting gaps and inconsistencies
    19. Track documentation health metrics over time
    20. Prioritize remediation work for highest-impact issues

    Supporting Responsibilities¶

    1. Template Management
    2. Maintain canonical README template with all required sections
    3. Provide ADR template following lightweight ADR format

    4. Update templates based on evolving EVA Suite standards

    5. Auto-Fix Workflows

    6. Generate missing README sections from templates
    7. Create placeholder ADRs for common architectural decisions

    8. Format existing documentation to match standards

    9. Integration Points

    10. Coordinate with P05 (Scaffolder Agent) to embed templates in new repos
    11. Provide drift metrics to P10 (Metrics & Insights Agent) for dashboards
    12. Support P06 (Review Agent) for PR-level documentation quality checks

    üöÄ When to Use P04¶

    Primary Use Cases¶

    1. Weekly Documentation Audit¶

    Scenario: Maintain continuous documentation quality across all repos as part of sprint hygiene.

    Action: Run drift detection script to generate health report for team review.

    Example: 

    # Run weekly documentation audit
tools/librarian/check-drift.ps1 -Repos @(
    "eva-orchestrator",
    "eva-api",
    "eva-core",
    "eva-rag",
    "eva-infra",
    "eva-sovereign-ui",
    "eva-meta",
    "eva-mcp",
    "eva-auth",
    "eva-matrix"
)

# Output: docs/librarian/drift-report-2025-12-12.md

    Expected Outcome: Markdown report showing README completeness, ADR coverage, and bilingual compliance per repo.

    2. New Repo Documentation Bootstrap¶

    Scenario: New repository created via P05 (Scaffolder) needs initial documentation setup.

    Action: Apply README template and create ADR-0001 (Initial Architecture).

    Example: 

    # Bootstrap documentation for new repo
tools/librarian/bootstrap-docs.ps1 -Repo "eva-notifications" `
    -Type "api" `
    -Pod "POD-F" `
    -Owner "P04-LIB" `
    -Bilingual

# Creates:
# - README.md (with all required sections)
# - README.fr.md (French template)
# - docs/adr/0001-initial-architecture.md

    Expected Outcome: Complete documentation scaffold ready for content population.

    3. Pre-Release Documentation Review¶

    Scenario: Preparing for production release and need to verify documentation is release-ready.

    Action: Run comprehensive quality check and generate remediation checklist.

    Example: 

    # Pre-release documentation check
tools/librarian/release-check.ps1 -Repo "eva-core" -Severity "strict"

# Validates:
# - All required README sections present
# - At least 5 ADRs documenting key decisions
# - README.fr.md exists and is recent (<30 days old)
# - No broken links
# - Installation/testing examples are runnable

    Expected Outcome: Pass/fail report with specific remediation tasks if failures detected.

    4. ADR Coverage Gap Analysis¶

    Scenario: Architectural decisions being made informally without documentation.

    Action: Identify repos with low ADR coverage and generate placeholder ADRs.

    Example: 

    # Find repos with insufficient ADR coverage
tools/librarian/check-drift.ps1 | Where-Object { $_.ADRCount -lt 5 }

# Auto-generate placeholder ADRs
tools/librarian/generate-adr.ps1 -Repo "eva-rag" `
    -Title "Data Chunking Strategy" `
    -Status "Proposed"

# Creates: docs/adr/0003-data-chunking-strategy.md (from template)

    Expected Outcome: Structured ADR files ready for architectural decision capture.

    When NOT to Use P04¶

    • Don't use P04 for code quality - P04 checks documentation, not source code (use P06 Review Agent)
    • Don't use P04 for user-facing docs - P04 targets developer/ops docs, not end-user guides
    • Don't use P04 for content creation - P04 validates structure, not content quality (P12 UX/Copy for content)
    • Don't use P04 to bypass review - Auto-fix creates drafts, humans must review before merge

    üîç Documentation Drift Detection¶

    Automated Checks¶

    Script: tools/librarian/check-drift.ps1

    # Check documentation drift across EVA Suite repos
param(
    [string[]]$Repos = @(
        "eva-orchestrator",
        "eva-api",
        "eva-core",
        "eva-rag",
        "eva-infra",
        "eva-sovereign-ui",
        "eva-meta",
        "eva-mcp",
        "eva-auth",
        "eva-matrix"
    ),
    [string]$OutputPath = "docs/librarian/drift-report-$(Get-Date -Format 'yyyy-MM-dd').md"
)

$results = @()

foreach ($repo in $Repos) {
    Write-Host "`nüîç Checking $repo..." -ForegroundColor Cyan

    $repoPath = "../$repo"
    if (!(Test-Path $repoPath)) {
        Write-Host "‚ö†Ô∏è Repo not found: $repoPath" -ForegroundColor Yellow
        continue
    }

    # Check README exists
    $readmePath = "$repoPath/README.md"
    $readmeExists = Test-Path $readmePath

    # Validate README structure
    $requiredSections = @("Quick Start", "Architecture", "Development", "Documentation")
    $missingSections = @()

    if ($readmeExists) {
        $readme = Get-Content $readmePath -Raw
        foreach ($section in $requiredSections) {
            if ($readme -notmatch "##\s+$section") {
                $missingSections += $section
            }
        }
    }

    # Check ADR directory
    $adrPath = "$repoPath/docs/adr"
    $adrExists = Test-Path $adrPath
    $adrCount = 0
    if ($adrExists) {
        $adrCount = (Get-ChildItem $adrPath -Filter "*.md" -ErrorAction SilentlyContinue).Count
    }

    # Check bilingual compliance
    $readmeFrPath = "$repoPath/README.fr.md"
    $bilingualCompliant = Test-Path $readmeFrPath

    # Assemble result
    $results += @{
        Repo = $repo
        README = $readmeExists
        MissingSections = $missingSections
        ADRs = $adrCount
        Bilingual = $bilingualCompliant
    }
}

# Generate drift report
$report = @"
# Documentation Drift Report - $(Get-Date -Format 'yyyy-MM-dd')

## Summary

| Repo | README | ADRs | Bilingual | Status |
|------|--------|------|-----------|--------|
"@

foreach ($result in $results) {
    $readmeIcon = if ($result.README) { "‚úÖ" } else { "‚ùå" }
    $adrIcon = if ($result.ADRs -ge 3) { "‚úÖ ($($result.ADRs))" } else { "‚ö†Ô∏è ($($result.ADRs))" }
    $bilingualIcon = if ($result.Bilingual) { "‚úÖ" } else { "‚ö†Ô∏è" }
    $status = if ($result.README -and $result.ADRs -ge 3 -and $result.Bilingual) { "OK" } else { "Fix needed" }

    $report += "`n| $($result.Repo) | $readmeIcon | $adrIcon | $bilingualIcon | $status |"
}

$report += @"


## Issues Found

"@

foreach ($result in $results) {
    if (!$result.README -or $result.MissingSections.Count -gt 0 -or $result.ADRs -lt 3 -or !$result.Bilingual) {
        $report += "`n### $($result.Repo)`n"

        if (!$result.README) {
            $report += "- ‚ùå Missing README.md`n"
        }

        if ($result.MissingSections.Count -gt 0) {
            $report += "- ‚ö†Ô∏è Missing sections: $($result.MissingSections -join ', ')`n"
        }

        if ($result.ADRs -lt 3) {
            $report += "- ‚ö†Ô∏è Only $($result.ADRs) ADRs (expected 3+)`n"
        }

        if (!$result.Bilingual) {
            $report += "- ‚ö†Ô∏è No French README (README.fr.md)`n"
        }
    }
}

$report += @"


## Recommended Fixes

Run auto-fix for repos with missing sections:
\`\`\`powershell
# Fix README structure
tools/librarian/fix-readme.ps1 -Repo [repo-name]

# Generate placeholder ADRs
tools/librarian/generate-adr.ps1 -Repo [repo-name] -Title "[Decision Title]"

# Create French README template
tools/librarian/create-bilingual.ps1 -Repo [repo-name]
\`\`\`
"@

# Save report
$report | Out-File $OutputPath -Encoding utf8
Write-Host "`n‚úÖ Drift report saved: $OutputPath" -ForegroundColor Green

    Output Example:

    Quality Gates¶

    Gate 1: README Completeness¶

    Criteria: README must include Quick Start, Architecture, Development, Documentation sections

    Check: 

    $readme = Get-Content "../[repo]/README.md" -Raw
$requiredSections = @("Quick Start", "Architecture", "Development", "Documentation")
$missing = $requiredSections | Where-Object { $readme -notmatch "##\s+$_" }

if ($missing.Count -eq 0) {

} else {
    Write-Host "‚ùå Missing sections: $($missing -join ', ')" -ForegroundColor Red

    Gate 2: ADR Coverage¶

    Criteria: Minimum 3 ADRs for new repos (<6 months), 5+ for mature repos

    Check: 

    $adrCount = (Get-ChildItem "../[repo]/docs/adr" -Filter "*.md").Count
$repoAge = (Get-Date) - (git log --reverse --format="%ai" | Select-Object -First 1 | Get-Date)

$minAdrs = if ($repoAge.TotalDays -lt 180) { 3 } else { 5 }

if ($adrCount -ge $minAdrs) {
    Write-Host "‚úÖ ADR coverage sufficient ($adrCount/$minAdrs)" -ForegroundColor Green

    Write-Host "‚ö†Ô∏è ADR coverage low ($adrCount/$minAdrs)" -ForegroundColor Yellow
}

    Gate 3: Bilingual Compliance¶

    Criteria: README.fr.md exists and is recent (<30 days drift from README.md)

    Check: 

    $readmeEn = Get-Item "../[repo]/README.md"

    üö® Common Issues and Troubleshooting¶

    Issue 1: False Positive - Missing Section Detected but Present¶

    Symptom: Drift report claims README missing "Quick Start" but section exists

    Cause: Section heading uses non-standard format (e.g., "Getting Started" instead of "Quick Start")

    Solution: 

    # Option 1: Update README to use standard heading
## Quick Start  # Use exact heading text

# Option 2: Add alias to drift detection script
$sectionAliases = @{
    "Quick Start" = @("Getting Started", "Quickstart")
}

    Prevention: Document standard section names prominently in P04 documentation

    Issue 2: ADR Count Inflated by Non-ADR Files¶

    Symptom: Drift report shows 10 ADRs but many are not actual decisions (e.g., README.md in adr/)

    Cause: Script counts all .md files in docs/adr/ directory

    Solution: 

    # Filter ADR files by naming convention (NNNN-*.md)
$adrFiles = Get-ChildItem $adrPath -Filter "*.md" |
    Where-Object { $_.Name -match '^\d{4}-' }
$adrCount = $adrFiles.Count

    Prevention: Enforce ADR naming convention in P05 (Scaffolder) templates

    Issue 3: Bilingual Drift False Alarm¶

    Symptom: French README flagged as outdated but content is synchronized

    Cause: File modification timestamps differ due to unrelated changes (whitespace, formatting)

    Solution: 

    # Compare content hash instead of timestamps
$hashEn = Get-FileHash "../[repo]/README.md"
$hashFr = Get-FileHash "../[repo]/README.fr.md"

if ($hashEn.Hash -ne $hashFr.Hash) {
    # Content actually differs - flag for review
}

    Prevention: Use content-based comparison, not timestamp-based

    Issue 4: Auto-Fix Overwrites Custom Content¶

    Symptom: Auto-fix script replaces customized README section with generic template

    Cause: Script doesn't detect existing custom content before inserting template

    Solution: 

    # Check section is truly missing, not just non-standard
if ($readme -notmatch "## Quick Start" -and $readme -notmatch "## Getting Started") {
    # Only then add template
}

    Prevention: Always run auto-fix in dry-run mode first, review before applying
