    Pattern P13: Onboarding & Sandbox Coach¶

    Status: Draft Owner: POD-S (Sandbox) Version: 1.0.0 Last Updated: 2025-12-12

    Overview¶

    The Onboarding & Sandbox Coach guides new teams into EVA Sandbox environments, providing templates, guardrails, and best practices to accelerate self-service deployment and experimentation.

    Purpose¶

    • Onboard new teams to EVA Sandbox
    • Provide templates and scaffolding
    • Enforce sandbox guardrails (cost, security, compliance)
    • Coach teams on EVA Suite best practices

    Sandbox Environment Setup¶

    Automated Provisioning¶

    Script: tools/sandbox/provision-sandbox.ps1

    # Provision new sandbox environment
param(
    [Parameter(Mandatory)]
    [string]$TeamName,

    [Parameter(Mandatory)]
    [string]$TeamLead,

    [string]$Region = "Canada Central"
)

Write-Host "üèñÔ∏è Provisioning sandbox for $TeamName..." -ForegroundColor Cyan

# Generate unique resource group name
$timestamp = Get-Date -Format "yyyyMMdd-HHmm"
$rgName = "rg-eva-sandbox-$($TeamName.ToLower())-$timestamp"

# Create resource group
Write-Host "Creating resource group: $rgName" -ForegroundColor Yellow
az group create `
    --name $rgName `
    --location "$Region" `
    --tags Team="$TeamName" Lead="$TeamLead" Environment="Sandbox"

# Apply sandbox budget policy
Write-Host "Setting budget limit: CAD 500/month" -ForegroundColor Yellow
az consumption budget create `
    --budget-name "$rgName-budget" `
    --amount 500 `
    --time-grain Monthly `
    --resource-group $rgName

# Deploy sandbox infrastructure from template
Write-Host "Deploying sandbox infrastructure..." -ForegroundColor Yellow
az deployment group create `
    --resource-group $rgName `
    --template-file "templates/sandbox-env.bicep" `
    --parameters teamName="$TeamName" teamLead="$TeamLead"

# Create onboarding guide
$guidePath = "sandbox/$TeamName/onboarding-guide.md"
New-Item -Path $guidePath -ItemType File -Force | Out-Null

@"

### Budget Alerts
- **Warning**: 80% of monthly budget (CAD 400)
- **Critical**: 100% of monthly budget (CAD 500)
- **Action**: Auto-shutdown non-critical resources at 110%

- ‚ùå No PII without approval

    Template: Minimal API¶

    File: templates/samples/minimal-api/app.bicep

    // Minimal API sandbox template
param location string = resourceGroup().location
param appName string = 'eva-sandbox-api'

resource appServicePlan 'Microsoft.Web/serverfarms@2022-03-01' = {
  name: '${appName}-plan'
  location: location
  sku: {
    name: 'B1'
    tier: 'Basic'
  }
  kind: 'linux'
  properties: {
    reserved: true
  }
}

resource appService 'Microsoft.Web/sites@2022-03-01' = {
  name: appName
  location: location
  properties: {
    serverFarmId: appServicePlan.id
    siteConfig: {
      linuxFxVersion: 'PYTHON|3.11'
      appSettings: [
        {
          name: 'ENVIRONMENT'
          value: 'Sandbox'
        }
      ]
    }
    httpsOnly: true
  }
}

output appUrl string = 'https://${appService.properties.defaultHostName}'

    File: templates/samples/minimal-api/main.py

    """
EVA Sandbox Minimal API
A simple FastAPI starter for sandbox experimentation
"""
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
import logging

# Initialize logging
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

app = FastAPI(
    title="EVA Sandbox API",
    description="Minimal API template for EVA Sandbox",
    version="1.0.0"
)

class HealthResponse(BaseModel):
    status: str
    environment: str

@app.get("/health", response_model=HealthResponse)
async def health_check():
    """Health check endpoint"""
    return HealthResponse(
        status="healthy",
        environment="sandbox"
    )

@app.get("/")
async def root():
    """Root endpoint"""
    return {
        "message": "Welcome to EVA Sandbox API",
        "docs": "/docs",
        "health": "/health"
    }

if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000)

    Cost Monitoring¶

    Workflow: .github/workflows/sandbox-cost-monitor.yml

    name: Sandbox Cost Monitor

on:
  schedule:
    - cron: '0 9 * * *'  # Daily at 9 AM UTC

jobs:
  check-costs:
    runs-on: ubuntu-latest
    steps:
      - name: Check sandbox budgets
        run: |
          # Query all sandbox resource groups
          sandboxes=$(az group list --tag Environment=Sandbox --query "[].name" -o tsv)

          for rg in $sandboxes; do
            echo "Checking: $rg"

            # Get current month costs
            cost=$(az consumption usage list \
              --start-date $(date -d "1 month ago" +%Y-%m-%d) \
              --end-date $(date +%Y-%m-%d) \
              --query "[?resourceGroup=='$rg'].pretaxCost | sum(@)" -o tsv)

            budget=500
            threshold=$(echo "$budget * 0.8" | bc)

            if (( $(echo "$cost > $threshold" | bc -l) )); then
              echo "‚ö†Ô∏è WARNING: $rg at $(echo "$cost/$budget*100" | bc)% of budget"

              # Send notification (placeholder)
              curl -X POST $TEAMS_WEBHOOK \
                -H 'Content-Type: application/json' \
                -d "{\"text\": \"Sandbox $rg approaching budget limit: \$$cost / \$$budget\"}"
            fi
          done

    Security Policy Enforcement¶

    Azure Policy: policies/sandbox-security.json

    {
  "displayName": "EVA Sandbox Security Policy",
  "description": "Enforces security requirements for sandbox environments",
  "policyRule": {
    "if": {
      "allOf": [
        {
          "field": "tags['Environment']",
          "equals": "Sandbox"
        },
        {
          "anyOf": [
            {
              "field": "properties.publicNetworkAccess",
              "equals": "Enabled"
            },
            {
              "field": "properties.encryption.keySource",
              "notEquals": "Microsoft.Storage"
            }
          ]
        }
      ]
    },
    "then": {
      "effect": "deny"
    }
  }
}
