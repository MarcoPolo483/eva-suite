    Pattern P01: Agent Pattern Library¶

    Status: Draft Owner: POD-F (Foundation) Version: 1.0.0 Last Updated: 2025-12-12

    Overview¶

    The Agent Pattern Library provides a catalog, metamodel, and discovery workflow that enables any pod to spin up new agents from standardized patterns. This foundational pattern ensures consistency, reusability, and discoverability across all EVA DevTools agents (P01-P21) and Service Personas (SP00-SPxx).

    Purpose¶

    • Standardize agent definitions with consistent structure
    • Enable discovery of available patterns and personas
    • Accelerate delivery by reusing proven patterns
    • Ensure quality through documented templates and validation

    Metamodel¶

    Agent Pattern Structure¶

    pattern:
  code: P01-P21 | SP00-SPxx
  name: Human-readable pattern name
  short_code: 3-letter abbreviation (e.g., REQ, SCR, LIB)
  type: devtools | service_persona
  scope: Domain area (e.g., requirements, testing, ux)
  description: Brief purpose statement
  home: Path to pattern documentation
  maturity: stable | alpha | beta | concept | deprecated

  # Usage context
  primary_users:
    - Role 1
    - Role 2

  # Technical details
  invocation: How to trigger (for service personas)
  inputs: Expected inputs (for service personas)
  outputs: Produced artifacts (for service personas)
  limitations: Known constraints
  safety_rules: C0-C3 autonomy constraints

  # References
  related_repos:
    - repo-name
  related_patterns:
    - P01
    - P02
  related_frameworks:
    - Framework name

    Catalog Registry¶

    Location: docs/agents/registry.yaml

    The canonical registry contains: - DevTools Patterns (P00-P21): Internal development automation - Service Personas (SP00-SPxx): External-facing services - Maturity Levels: Development lifecycle tracking - Autonomy Levels: C0-C3 governance mapping

    Current Inventory¶

    DevTools Patterns (22 patterns): - P00: DevTools Agents Spec (meta-pattern) - P01: Agent Patterns & Toolbox - P02: Requirements Engine - P03: Scrum Agent - P04: Repo Librarian - P05: Scaffolder Agent - P06: Review Agent - P07: Testing & Failure Explainer - P08: CI/CD Guardian - P09: Runbook & SRE Agent - P10: Metrics & Insight Agent - P11: Security & Compliance Helper - P12: UX, Copy & Accessibility Agent - P13: Onboarding & Sandbox Coach - P14: DevOps LiveOps Companion - P15: Dev Master Orchestrator (Jarvis) - P16: Awareness Protocol - P17: Swarm PR Review - P18: Tool/API Auto-Onboarding - P19: Action Classification & Safety - P20: Continuous Self-Improvement - P21: Provenance & Fingerprinting

    Service Personas (3 personas): - SP00: Senior Advisor (ChatGPT bridge) - SP01: EVA Data Ingestion - SP02: ATO Auditor

    Discovery Workflow¶

    1. Browse Catalog¶

    # View all patterns
cat docs/agents/registry.yaml

# Filter by type
grep "type: devtools" docs/agents/registry.yaml
grep "type: service_persona" docs/agents/registry.yaml

# Filter by maturity
grep "maturity: stable" docs/agents/registry.yaml

    2. Read Pattern Documentation¶

    Each pattern has a home page: - DevTools: docs/agents/devtools/Pxx-*.md - Service Personas: docs/agents/personas/SPxx-*.md

    3. Check Dependencies¶

    # Example from P09 (Runbook Agent)
dependencies:
  - P07 (Testing hooks for failure detection)
  - TASK-034 (Telemetry instrumentation)

    4. Verify Maturity¶

    maturity_definitions:
  stable: Production-ready, fully documented, actively used
  alpha: Working implementation, documentation in progress
  beta: Feature-complete, broader testing
  concept: Design complete, implementation pending
  deprecated: No longer recommended, migration path available

    5. Understand Autonomy Level¶

    autonomy_levels:
  C0: Human-in-the-loop (AI suggests, human approves every action)
  C1: Human-on-the-loop (AI acts, human monitors)
  C2: Human-over-the-loop (AI acts with constraints, periodic review)
  C3: Full autonomy (well-defined boundaries, extensive validation)

    Usage Patterns¶

    Creating a New Pattern¶

    1. Choose Pattern Code
    2. DevTools: Next available Pxx (P22+)

    3. Service Persona: Next available SPxx (SP03+)

    4. Draft Pattern Entry 

      - code: P22
  name: New Pattern Name
  short_code: NPT
  type: devtools
  scope: pattern-domain
  description: Brief purpose
  home: docs/agents/devtools/P22-New-Pattern.md
  maturity: concept

    5. Create Documentation

    6. Use template from P00 (DevTools Agents Spec)
    7. Include: Overview, Purpose, Acceptance Criteria, QA Definition

    8. Document dependencies and autonomy level

    9. Register in Catalog 

      # Add entry to docs/agents/registry.yaml
# Commit and push
git add docs/agents/registry.yaml docs/agents/devtools/P22-*.md
git commit -m "feat: Add P22 pattern"

    Spinning Up an Agent¶

    1. Select Pattern from registry
    2. Read Documentation to understand inputs/outputs
    3. Check Dependencies are satisfied
    4. Verify Autonomy Level matches governance approval
    5. Invoke Agent per documented method
    6. Track Execution using pattern's QA definition

    Acceptance Criteria¶

    1. Catalog metamodel documented in this file
    2. Registry file (docs/agents/registry.yaml) exists with all patterns
    3. Discovery workflow steps clear and actionable
    4. Usage patterns for creating/using agents defined

    QA Definition¶

    • Docs lint clean (markdownlint docs/agents/devtools/P01-*.md)
    • Bilingual review complete (EN-CA/FR-CA)
    • Registry YAML validates (yamllint docs/agents/registry.yaml)
    • All 22 DevTools patterns + 3 Service Personas documented

    • docs/agents/registry.yaml: Canonical registry (already exists)
