    Executive Summary for Senior Management¶

    Submitted by: Marco Presta, EVA Suite Product Owner
    Date: December 4, 2025
    Status: Draft for Senior Management Review

    üìã Executive Summary¶

    This package delivers a comprehensive, technology-neutral, federal-grade Agentic AI Template for Government of Canada departments to safely build, deploy, and govern agentic AI systems.

    What This Template Provides: - Complete lifecycle framework (Sandbox ‚Üí Pilot ‚Üí Production) - Agentic Maturity Model (Levels 0-3, with Level 4 prohibited) - GC policy compliance (TBS, ITSG-33, WCAG, OLA, PIPEDA, DADM, RA/IDE) - Governance patterns (RBAC, audit trails, kill switches, bounded autonomy) - Reference implementation (EVA Suite as proof-of-concept)

    Business Value: - Architecture Alignment: Operationalizes EARB-approved AI Reference Architecture (AIRA) agentic layer - Risk Reduction: Resolves "agentic fear" with clear maturity model and guardrails - Cost Savings: Reusable template saves 6-12 months governance work per project - Compliance: Pre-built mappings to ALL GC policies (ITSG-33, NIST AI RMF, AIRA, Privacy Act, security, accessibility, bilingualism) - Leadership: Positions ESDC/GC as global leader in responsible agentic AI - Scalability: One template, multiple departments, consistent governance

    üéØ Strategic Context¶

    The Opportunity¶

    Agentic AI = next phase of AI evolution (moving from single-step Q&A to multi-step coordinated workflows)

    ESDC Context: Agentic AI is already part of ESDC's approved architecture - defined as the highest capability layer in the AI Reference Architecture (AIRA) presented to the Enterprise Architecture Review Board (EARB). The AIRA explicitly frames AI evolution: Data ‚Üí Predictive ‚Üí Generative ‚Üí Agentic Execution.

    GC departments are asking: "How do we operationalize this safely?"

    Current state: AIRA defines the architectural layer, but no operational standard exists (agent levels, risk categories, governance patterns, kill-switch requirements, RACI model)

    This template: Provides the missing operational standard that AIRA requires (like ITSG-33 provides controls for IT security architecture)

    Recent Validation (December 4, 2025)¶

    Statistics Canada published "Approaching Generative AI: Recommendations and Lessons Learned from AgPal Chat" - documenting RAG-based agricultural chatbot deployment with guardrails, data governance, and policy compliance.

    International Context: - UK Government: "AI Insights: Agentic AI" - official guidance on characteristics, risks, and public sector considerations - NIST: AI Risk Management Framework 1.0 - explicit coverage of autonomous agents and agent-based threats - ISED Canada: Voluntary Code Implementation Guide - responsible AI development and management - Global Public Sector: Multiple governments (UK, US, OECD) publishing agentic AI frameworks

    Gap: EARB-approved AIRA architecture + existing guidance (NIST, UK Gov, ISED) cover capabilities and risks. This template provides operational implementation - agent levels, controls, maturity model, governance patterns, reference architecture (EVA Suite).

    Positioning: - ESDC: First federal department to operationalize EARB-approved agentic architecture layer - Canada: First comprehensive, ATO-ready agentic AI framework tied to ITSG-33/NIST AI RMF/AIRA

    The Urgency¶

    • ESDC: Jurisprudence agentic assistant (Shannon's team, launching Q1 2026)
    • AAFC: AI-driven policy analysis (need governance framework) - AgPal Chat already operational (published Dec 4, 2025)
    • SSC: Infrastructure automation (agentic orchestration)
    • DND: Intelligence analysis (high-risk, needs strong controls)
    • Multiple departments: Asking for same guidance simultaneously

    Global Context: UK Government published "AI Insights: Agentic AI" guidance, NIST AI RMF explicitly covers autonomous agents, ISED Canada released Voluntary Code Implementation Guide. Canada needs GC-specific framework NOW.

    Timeline: If not delivered soon, departments build inconsistent approaches ‚Üí governance debt, security risks, compliance gaps
