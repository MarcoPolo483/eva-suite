    POD Specifications - All 4 PODs¶

    Document Classification: Public Version: 1.0 Date: December 11, 2025

    Overview¶

    EVA Orchestrator organizes 21 specialized agent profiles (P01-P21) into 4 PODs (Parallel Operational Domains), each responsible for specific SDLC functions. This document provides complete specifications for all PODs, including POD-X (Experience).

    Agent Distribution: - POD-F (Foundation): 5 agents - POD-O (Operations): 7 agents - POD-Q (Quality): 7 agents - POD-X (Experience): 2 agents - Total: 21 unique agent profiles

    POD-F: Foundation¶

    Purpose¶

    Build and maintain foundational SDLC artifacts: architecture, requirements, libraries, scaffolding, and tool automation.

    Agents (5)¶

    P01-ARC: Architecture Decision Records¶

    • Responsibility: Document architectural decisions, design patterns, system diagrams
    • Deliverables: ADR documents, architecture diagrams (C4, UML)
    • Technology: Markdown, Mermaid, PlantUML
    • Example Tasks:
    • ADR-001: Multi-tenancy strategy
    • ADR-002: State management approach (Cosmos vs. Redis)
    • System architecture diagram updates

    P02-REQ: Requirements Validation¶

    • Responsibility: Validate requirements, acceptance criteria, backlog refinement
    • Deliverables: Requirement documents, validation reports
    • Technology: YAML, Markdown, Gherkin (BDD)
    • Example Tasks:
    • Q1 2026 roadmap validation
    • User story acceptance criteria review
    • Epic decomposition

    P04-LIB: Library & SDK Documentation¶

    • Responsibility: Document APIs, SDKs, libraries, code examples
    • Deliverables: API documentation, SDK guides, code samples
    • Technology: JSDoc, Sphinx, OpenAPI
    • Example Tasks:
    • EVA Core SDK documentation
    • API reference generation
    • Code sample creation

    P05-SCA: Project Scaffolding¶

    • Responsibility: Generate project templates, starter kits, boilerplate code
    • Deliverables: Project templates, scaffolding scripts
    • Technology: Yeoman, Cookiecutter, custom generators
    • Example Tasks:
    • Microservice starter template
    • React component library scaffold
    • Azure Functions project template

    P18-AUT: Tool Auto-Onboarding¶

    • Responsibility: Automate tool setup, configuration, integration
    • Deliverables: Setup scripts, configuration guides
    • Technology: PowerShell, Bash, Ansible
    • Example Tasks:
    • GitHub CLI auto-onboarding
    • Azure DevOps pipeline setup
    • VS Code extension pack installer

    Queue Configuration¶

    • Queue Name: pod-f-queue (Azure Service Bus)
    • Queue File: POD-F-QUEUE.json (GitHub Actions)
    • Priority: High (foundational work)
    • AutoStart: Enabled
    • Execution Frequency: Every 30 minutes

    POD-O: Operations¶

    Purpose¶

    Manage operational excellence: runbooks, metrics, SLOs, data management, workspace awareness, and continuous improvement.

    Agents (7)¶

    P03-SCR: Scrum Master¶

    • Responsibility: Sprint planning, retrospectives, metrics tracking
    • Deliverables: Sprint reports, retrospective notes, velocity charts
    • Technology: Markdown, YAML, Charts
    • Example Tasks:
    • Sprint 1 retrospective
    • Velocity analysis
    • Burndown chart generation

    P09-RUN: Runbook Creation¶

    • Responsibility: Create operational runbooks, incident response guides
    • Deliverables: Runbook documents, playbooks
    • Technology: Markdown, YAML
    • Example Tasks:
    • Incident response runbook
    • Deployment checklist
    • Rollback procedures

    P10-MET: Usage Metrics Dashboard¶

    • Responsibility: Build dashboards, metrics collection, KPI tracking
    • Deliverables: Dashboards (HTML, Power BI), metric configs
    • Technology: HTML/JS, Prometheus, Grafana
    • Example Tasks:
    • Usage metrics dashboard
    • Performance KPI tracking
    • Cost analysis dashboard

    P14-LVO: Live Operations & SLOs¶

    • Responsibility: Define SLOs, monitor service health, alerting
    • Deliverables: SLO definitions, monitoring configs, alert rules
    • Technology: Prometheus, YAML, Markdown
    • Example Tasks:
    • Core services SLO definition
    • Alert rule configuration
    • On-call rotation setup

    P15-DAT: Data Management¶

    • Responsibility: Data schemas, migrations, backup strategies
    • Deliverables: Schema definitions, migration scripts, backup configs
    • Technology: SQL, NoSQL, migration tools
    • Example Tasks:
    • Cosmos DB schema design
    • Data migration scripts
    • Backup retention policies

    P16-AWR: Workspace Awareness¶

    • Responsibility: Scan workspace, understand codebase, generate context
    • Deliverables: Workspace maps, dependency graphs, context reports
    • Technology: AST parsing, static analysis
    • Example Tasks:
    • Workspace dependency scan
    • Codebase complexity analysis
    • File organization recommendations

    P20-SLF: Self-Improvement¶

    • Responsibility: Analyze agent performance, recommend improvements
    • Deliverables: Performance reports, improvement recommendations
    • Technology: Python (analytics), Markdown
    • Example Tasks:
    • Agent performance analysis
    • Task success rate tracking
    • Optimization recommendations

    POD-Q: Quality¶

    Purpose¶

    Ensure code quality, security, testing coverage, CI/CD reliability, and audit compliance.

    Agents (7)¶

    P06-REV: Code Review¶

    • Responsibility: Automated code review, style checking, best practices
    • Deliverables: Code review reports, suggestion comments
    • Technology: ESLint, SonarQube, custom analyzers
    • Example Tasks:
    • eva-rag code review
    • Pull request analysis
    • Code smell detection

    P07-TST: Testing & Coverage¶

    • Responsibility: Generate tests, measure coverage, test reports
    • Deliverables: Test files, coverage reports
    • Technology: Jest, Pytest, coverage.py
    • Example Tasks:
    • Test coverage report (POD-F)
    • Unit test generation
    • Integration test suite

    P08-CICD: CI/CD Pipeline Management¶

    • Responsibility: Pipeline audits, workflow optimization, build health
    • Deliverables: Pipeline configs, audit reports
    • Technology: GitHub Actions, Azure DevOps, YAML
    • Example Tasks:
    • CI/CD pipeline audit
    • Workflow optimization
    • Build failure analysis

    P11-SEC: Security Scanning¶

    • Responsibility: Vulnerability scans, dependency audits, security reports
    • Deliverables: Security reports, vulnerability lists
    • Technology: Snyk, OWASP ZAP, npm audit
    • Example Tasks:
    • Security scan (dependencies)
    • Vulnerability assessment
    • OWASP compliance check

    P17-SWR: Swarm Review¶

    • Responsibility: Multi-agent collaborative review, consensus building
    • Deliverables: Swarm review reports, consensus recommendations
    • Technology: Custom swarm logic, voting systems
    • Example Tasks:
    • Swarm PR review
    • Multi-agent design review
    • Consensus on architecture decisions

    P19-ACL: Action Classification¶

    • Responsibility: Classify actions (READ/WRITE/DESTRUCTIVE), safety matrix
    • Deliverables: Classification matrix, safety guides
    • Technology: YAML, Markdown
    • Example Tasks:

    • Safety level definitions
    • Approval workflow mapping

    P21-PRV: Provenance & Audit Trail¶

    • Responsibility: Track all changes, audit logs, compliance verification
    • Deliverables: Audit logs, provenance reports
    • Technology: Git history, blockchain (future), logging
    • Example Tasks:

    • Change provenance tracking
    • Compliance report generation

    Azure Migration Impact on PODs¶

    Current (GitHub Actions)¶

    • 3 separate workflow files: pod-f-executor.yml, pod-o-executor.yml, pod-q-executor.yml
    • Cron-based execution (every 30 minutes)
    • Sequential task processing (one task per POD per run)
    • Direct OpenAI API calls

    Target (Azure)¶

    • 1 Durable Functions orchestrator with 4 activity functions
    • Timer trigger (every 30 minutes) â†’ fan-out to all PODs in parallel
    • Concurrent task processing (one task per POD simultaneously)
    • MAF agent pool (21 agents shared across PODs)

    Performance Improvements¶

    Metric Current Target Improvement
    Parallel Execution 3 PODs 4 PODs +33% capacity
    Task Throughput 3 tasks/30min 4 tasks/30min +33%
    Execution Time ~5 min/task ~3 min/task -40%
    Retry Logic Manual Automatic Reliable
    Observability GitHub logs App Insights Rich telemetry

    Troubleshooting¶

    POD Not Executing¶

    1. Check queue for READY tasks: az servicebus queue show --name pod-f-queue
    2. Verify Durable Functions status: az functionapp show --name func-eva-orch-dev
    3. Review Application Insights logs: Search for POD-F Activity

    Task Stuck in IN_PROGRESS¶

    1. Check execution timeout (default: 10 minutes)
    2. Review error logs in Application Insights
    3. Manually reset task status in Cosmos DB: status: "READY"

    POD-X Not Starting¶

    • By design: POD-X is paused per user request
    • To enable: Set autoStart: true in POD-X-QUEUE.json and redeploy
