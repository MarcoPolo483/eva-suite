    EVA Suite Automation Management Guide¶

    Overview¶

    EVA Suite uses multiple automation layers to maintain code quality, sync documentation, and track metrics. This guide explains how to monitor and maintain these systems without manual intervention.

    Automation Components¶

    1. GitHub Actions Workflows (Auto-monitored)¶

    Critical Workflows: - P02 Usage Snapshot: Daily OpenAI usage tracking + PR creation - EVA Audit Trail Update: Tracks all file changes across eva-orchestrator - Daily Metrics Snapshot: Refreshes public metrics dashboard - P02 Advisor Daily: Generates AI-powered status reports (weekdays) - Project Board Sync: Keeps GitHub Projects in sync (every 15 min)

    Health Monitoring: 

    # Quick check during login (automatic)
.\login.ps1

# Manual detailed check
.\scripts\Invoke-AutomationHealthCheck.ps1 -Detailed

# Check only failures
.\scripts\Invoke-AutomationHealthCheck.ps1 -FailedOnly

# Attempt automatic fixes
.\scripts\Invoke-AutomationHealthCheck.ps1 -AutoFix

    2. Workspace Housekeeping (On-demand)¶

    Purpose: Clean up node_modules noise, run tests, check git status across all EVA repos.

    # Full housekeeping with tests
.\scripts\Invoke-EvaHousekeeping.ps1

# Skip tests (faster)
.\scripts\Invoke-EvaHousekeeping.ps1 -SkipTests

# Skip external health check
.\scripts\Invoke-EvaHousekeeping.ps1 -SkipHealthCheck

# Generate report without tests
.\scripts\Invoke-EvaHousekeeping.ps1 -SkipTests

    What it does: - Removes node_modules/.package-lock.json noise - Runs npm test on configured repos (eva-api, eva-core, eva-ui, eva-i18n) - Checks git status for uncommitted changes - Generates timestamped reports in reports/ - Optionally runs health-check.ps1 for system validation

    3. System Health Checks (Login-triggered)¶

    Purpose: Verify git repos, master plan, GitHub Pages, documentation.

    # Part of login.ps1 (automatic)
.\login.ps1

# Standalone quiet check
.\health-check.ps1 -Quiet

# Verbose check
.\health-check.ps1

    Checks performed: - eva-orchestrator git status (uncommitted changes) - eva-suite git status - Master plan YAML validation - GitHub Pages deployment + content verification - Documentation completeness

    Daily Workflow¶

    Morning Login¶

    .\login.ps1
    What happens: 1. System health check (git status, plan validation, pages deployment) 2. GitHub Actions automation check (workflow failures, stale runs) 3. Context loading from previous session 4. Session time tracking starts

    If issues detected: - Red ❌ = Critical failure requiring immediate attention - Yellow ⚠️ = Warning, system functional but degraded - Green ✅ = All systems operational

    During Work¶

    No action needed - all automation runs in background: - Project Board Sync: every 15 minutes - Workflow failures: automatically detected at next login - Usage snapshots: daily at midnight UTC - Metrics refresh: daily at 5 PM UTC

    Weekly Maintenance (Optional)¶

    # Clean up workspace + run tests
.\scripts\Invoke-EvaHousekeeping.ps1

# Review automation health
.\scripts\Invoke-AutomationHealthCheck.ps1 -Detailed

    Troubleshooting¶

    Workflow Failures¶

    Symptom: Automation check reports critical failures

    Diagnosis: 

    .\scripts\Invoke-AutomationHealthCheck.ps1 -Detailed

    Common Issues & Fixes:

    Error Pattern Cause Fix
    ModuleNotFoundError Missing Python package Add to requirements.txt + pip install -e .
    GITHUB_TOKEN missing Env var not set Add to workflow env: section
    github.ref.*refs/heads Double-encoded ref Use github.ref_name instead
    script not found Missing file in commit Ensure script exists and is committed
    submodule error Stale .gitmodules Add cleanup step to workflow

    Manual Workflow Trigger: 

    # Trigger specific workflow
gh workflow run "P02 Usage Snapshot" --ref main

# List recent runs
gh run list --workflow "P02 Usage Snapshot" --limit 10

# View failure details
gh run view <run-id> --log-failed

    Stale Workflows¶

    Symptom: Workflow hasn't run in 24+ hours

    Cause: Scheduled trigger may have failed or been disabled

    Fix: 

    # Manually trigger
gh workflow run "<workflow-name>" --ref main

# Check if workflow is disabled
gh workflow list | Select-String "<workflow-name>"

    Uncommitted Changes Blocking Health Check¶

    Symptom: Health check reports "⚠️ uncommitted changes"

    Fix: 

    # Review changes
git status

# Commit or stash
git stash  # temporary
# or
git add . && git commit -m "wip: session work"

    Automated PR Management¶

    Usage Snapshot PRs¶

    Auto-created by: P02 Usage Snapshot workflow
    Frequency: Daily (if usage data changes)
    Branch: usage-snapshots
    Action required: Review and merge when convenient

    Files updated: - reports/usage/p02-usage-latest.json - reports/usage/p02-usage-latest.md

    Audit Trail PRs¶

    Auto-created by: EVA Audit Trail Update workflow
    Frequency: Nightly (if files changed)
    Action required: Auto-merged by workflow (no action needed)

    Files updated: - docs/_audit/*.jsonl - docs/_vault/EVA-New-Artifacts-Map.md

    Configuration Files¶

    Workflow Definitions¶

    .github/workflows/
├── p02-usage-snapshot.yml       # Usage tracking
├── eva-audit-update.yml         # Audit trail
├── daily-metrics.yml            # Metrics dashboard
├── p02-advisor-daily.yml        # AI status reports
└── project-board-sync.yml       # GitHub Projects sync

    Housekeeping Config¶

    scripts/
├── Invoke-AutomationHealthCheck.ps1   # Workflow monitoring
├── Invoke-EvaHousekeeping.ps1         # Workspace cleanup
└── health-check.ps1                   # System validation

    Critical Workflows List¶

    Edit scripts/Invoke-AutomationHealthCheck.ps1: 

    $CRITICAL_WORKFLOWS = @(
    "P02 Usage Snapshot",
    "EVA Audit Trail Update",
    # Add more workflows to monitor
)

    Best Practices¶

    ✅ Do:¶

    • Run .\login.ps1 at start of each session (automation checks included)
    • Review workflow failures when reported (takes 30 seconds)
    • Run housekeeping weekly or when workspace feels cluttered
    • Let automation handle PRs unless explicitly flagged

    ❌ Don't:¶

    • Skip health checks regularly (-SkipHealthCheck is for emergencies)
    • Ignore critical workflow failures (red ❌)
    • Manually trigger workflows unless troubleshooting
    • Modify workflow files without testing locally first

    Emergency Procedures¶

    All Workflows Failing¶

    Steps: 1. Check GitHub Status: https://www.githubstatus.com/ 2. Verify GitHub CLI auth: gh auth status 3. Review recent commits: git log --oneline -10 4. Check workflow YAML syntax: 

    # Install YAML validator
Install-Module -Name powershell-yaml

# Validate workflow
Import-Module powershell-yaml
$yaml = Get-Content .github/workflows/<workflow>.yml -Raw
ConvertFrom-Yaml $yaml

    Automation Check Not Running¶

    Cause: Script moved or GitHub CLI not available

    Fix: 

    # Verify script exists
Test-Path .\scripts\Invoke-AutomationHealthCheck.ps1

# Verify GitHub CLI
gh --version

# Re-authenticate if needed
gh auth login
