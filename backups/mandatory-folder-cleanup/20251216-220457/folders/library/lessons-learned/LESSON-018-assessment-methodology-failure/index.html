    1. DON'T stop reading after 100 lines
    2. DON'T trust "Phase X Complete" labels without verification
    3. DON'T assume code exists because docs mention it
    4. DON'T skip TODO scanning in codebase
    5. DON'T trust integration claims without checking actual code

    What TO Do ‚úÖ¶

    1. DO read complete specifications (all lines)
    2. DO scan codebase for TODOs and FIXMEs

    4. DO map promised features to actual code systematically

    6. DO check for architecture documents (ROADMAP, DESIGN)
    7. DO distinguish between "documented" and "implemented"
