    üß© Three Concepts Pattern¶

    This P02 feature follows the Three Concepts Pattern:

    1. Context Engineering (CE)¶

    • 4-Block Prompt: Context ‚Üí Task ‚Üí Output ‚Üí Self-check
    • Execution Evidence: All commands include safety instructions and expected results
    • Working Style: Testable, Incremental, Reversible
    • Documentation: Every script has execution evidence and rollback procedures

    2. Housekeeping (HK)¶

    • Quality Gates: Pre-deployment, deployment, post-deployment validation
    • Health Checks: Automated monitoring and alerting
    • Cleanup: Automated resource cleanup for failed deployments
    • Audit Trail: All operations logged to EVA-Audit-Log.jsonl

    3. Directory Mapping¶

    • Workspace Navigation: All paths relative to eva-orchestrator root
    • Cross-References: Links to related documentation (eva-api, eva-auth, eva-orchestrator)
