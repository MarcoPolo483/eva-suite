    Feature Map – EVA Artifacts Generator¶

    Feature ID: P02-ARTIFACTS-001
    Last updated: 2025-11-25

    Overview¶

    This feature map visualizes the EVA Artifacts Generator capabilities, data flows, integrations, and roadmap across 3 phases.

    Capability matrix¶

    Capability Phase 1 (MVP) Phase 2 (AI-Assisted) Phase 3 (Multi-Repo)
    Data sources ✅ EVA-New-Artifacts-Map.md ➕ EVA-Audit-Log.jsonl ➕ Multi-repo aggregation
    Gap detection ✅ 6 rule categories ➕ Custom rules (plugin) ➕ Cross-repo analysis
    Doc generation ✅ Template-based stubs ✅ AI-assisted (OpenAI) ➕ Multi-model support
    Doc types ✅ README, P02, CDD → Same ➕ ADR, RFC, API specs
    Telemetry None ✅ P02 integration (100%) → Same
    Reports ✅ Gap analysis (Markdown) ➕ JSON export ➕ HTML dashboards
    Review workflow Manual (no auto-commit) → Same ➕ PR automation
    Scope ✅ eva-orchestrator → Same ✅ 23 EVA Suite repos
    Automation Manual CLI ➕ GitHub Actions (scan) ➕ Real-time suggestions
    Cost tracking None ✅ Per-doc cost estimates ➕ Budget alerts

    Legend: ✅ Implemented | ⏳ Planned (this sprint) | ➕ Future enhancement | → No change

    Integration points¶

    Current integrations (Phase 1-2)¶

    1. EVA Audit Trail (P02-AUDIT-001)
    2. Input: EVA-New-Artifacts-Map.md (primary data source)
    3. Input: EVA-Audit-Log.jsonl (optional complementary data)

    4. Output: P02 telemetry events (generator's OpenAI usage)

    5. P02 Telemetry Helpers

    6. Uses: scripts/p02-telemetry.ps1 functions
    7. Functions: New-P02OpenAIUsageEvent, Write-P02AuditEvent

    8. Integration: Phase 2 (AI-assisted generation)

    9. P02 Usage Reporting

    10. Appears in: scripts/p02-usage-report.ps1 output
    11. Feature key: P02-ARTIFACTS-001

    12. Metrics: Tokens used, costs, docs generated

    13. Template Library

    14. Location: scripts/templates/
    15. Templates: README.md.tmpl, P02-Package.tmpl, CDD.tmpl
    16. Version tracking: Template headers include v1.0, v1.1, etc.

    Future integrations (Phase 3)¶

    1. GitHub Actions
    2. Trigger: Nightly cron (2 AM UTC), manual dispatch
    3. Action: Run scanner, commit gap reports

    4. Workflow: .github/workflows/eva-artifacts-scan.yml

    5. Multi-Repo Bootstrap

    6. Pattern: Similar to EVA Audit Trail bootstrap
    7. Script: eva-artifacts-bootstrap.ps1

    8. Extends to: 23 EVA Suite repos

    9. Power BI Dashboards

    10. Data source: P02 telemetry events (via p02-usage-report.ps1 CSV)
    11. Metrics: Docs generated per week, AI costs, gap trends

    12. Dashboards: Generator usage, documentation health

    13. OpenAI API

    14. Provider: Azure OpenAI Service (preferred) or OpenAI API
    15. Model: gpt-4.1 (primary), gpt-4.1-mini (fallback)
    16. Usage: README generation, P02 package authoring

    Feature dependencies¶

    Prerequisites (completed)¶

    • ✅ EVA Audit Trail: Must be running to populate EVA-New-Artifacts-Map.md
    • ✅ P02 Telemetry Helpers: scripts/p02-telemetry.ps1 exists with required functions
    • ✅ PowerShell 7.x: Required for script execution
    • ⏳ Template Library: Create scripts/templates/ with doc templates (Phase 1 task)

    Dependent features¶

    • P02 Usage Reporting (p02-usage-report.ps1): Generator appears in usage reports (Phase 2)
    • Power BI Dashboards: Visualize generator usage and doc health metrics (Phase 3)
    • Multi-Repo Bootstrap: Extend generator to all EVA Suite repos (Phase 3)

    Blocking issues¶

    • None currently (all prerequisites met for Phase 1 start)
