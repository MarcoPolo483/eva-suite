    üéì Lessons Learned¶

    Session Lessons¶

    1. Traceability is critical: Every document needs source, purpose, location tracking
    2. Fact-check everything: Repository count was 36, not 17 as documented
    3. Hidden features exist: P02 has 8 features, not just requirements gathering
    4. File-based coordination works: 5-minute response time adequate for autonomy
    5. Memory prevents re-work: Housekeeping stale because no daily reminder system
    6. Pattern recognition after 2 reps: Auto-create tools after seeing pattern twice
    7. Hierarchical SMs scale: Master SM + POD SMs + agents = manageable coordination
    8. Budget constraints force optimization: $0.15/spawn means smart agent selection

    Technical Lessons¶

    1. TypeScript readonly arrays: Cast to readonly string[] for .includes() operations
    2. P02 package structure: Minimum viable = README.md + backlog.md
    3. Backlog lane pattern: docs/features/<feature-slug>/backlog.md (96+ lanes)
    4. Agent memory schema: JSON with tasks, lessons, tools, patterns, quality standards
    5. GitHub Actions proven: Already have 38+ workflows, just need to enable them
    6. OpenWebUI + PubSec: Reference repos exist for pattern extraction (Svelte, Azure)

    Process Lessons¶

    1. Marco's vision is clear: Autonomous operation with minimal PO intervention
    2. Agents need memory: Persistent storage prevents repeating mistakes
    3. Lesson sharing accelerates: Agent A learns ‚Üí all agents benefit immediately
    4. Tool auto-creation: After 2 pattern repetitions, create reusable tool
    5. 3-strike escalation: Try, retry, escalate to SM, notify Marco (last resort)
    6. Self-healing is essential: Replace failed agents automatically
    7. Budget optimization matters: Stay under $100/month via caching, batching, smart routing
