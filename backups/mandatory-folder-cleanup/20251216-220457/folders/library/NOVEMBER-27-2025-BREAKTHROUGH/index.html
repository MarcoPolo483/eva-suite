    Rollback Capabilities¶

    Option 1: Azure Static Web Apps Slot Swap (Instant)¶

    Production: v1.1.0 (Blue slot)
Staging:    v1.0.9 (Green slot - previous version kept for 24 hours)

1. Marco (or MSM) triggers slot swap (instant)
2. Production now serves v1.0.9 (zero downtime)
3. Time to rollback: <30 seconds

    Option 2: Git Revert (5 minutes)¶

    # If slot swap not available, revert git commit
git revert <commit-hash>
git push origin main

# CI/CD pipeline auto-deploys reverted version
# Time to rollback: ~5 minutes

    Option 3: Redeploy Previous Tag (10 minutes)¶

    # Checkout previous release tag
git checkout v1.0.9

# Trigger manual deployment workflow
gh workflow run cd-prod.yml --ref v1.0.9

# Time to rollback: ~10 minutes (full build + deploy)
