    ğŸ§  EVA Bootstrap Memory System - "Never Lose Context Again"¶

    Date: November 28, 2025
    Architect: EVA-Sovereign-UI Architect (Copilot A)
    Purpose: CRITICAL - Ensure agents never lose context, never recreate reports, always know repo state

    The Problem Marco Identified¶

    "if you lose all you know, i am screwed again. i will have to start it over again."

    Current Pain: - Every time Marco opens a repo â†’ Copilot scans entire workspace (slow, expensive) - Every time agent spawns â†’ No memory of previous sessions - Every session â†’ Recreates reports from scratch - No persistent memory â†’ Agents repeat mistakes - No lesson sharing â†’ Each agent learns independently

    Marco's Requirements: 1. Bootstrap memory in every repo (immediate context on open) 2. No scanning - memory file pre-generated, always current 3. No recreating reports - reports cached, auto-updated 4. Agent memory - lessons, tools, patterns persist across sessions 5. Audit trail - all actions logged, reports saved 6. Guardrails - prevent memory loss, ensure consistency

    Component 3: GitHub Workflow (Auto-Update)¶

    Purpose: Keep .eva-memory.json and .eva-cache/ always current.

    File: .github/workflows/eva-memory-update.yml (in every repo)

    Full Implementation:

    name: EVA Memory Auto-Update

on:
  schedule:
    - cron: '0 3 * * *'  # 3am daily (nightly housekeeping)
  push:
    branches: [main, develop]  # After every push
  workflow_dispatch:           # Manual trigger

jobs:
  update-memory:
    runs-on: ubuntu-latest
    permissions:
      contents: write

    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0  # Full history for accurate stats

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install Dependencies
        run: npm ci || npm install

      - name: Run Tests (Coverage)
        run: |
          npm test -- --coverage --silent || echo "Tests failed"
          COVERAGE=$(cat coverage/coverage-summary.json | jq '.total.lines.pct')
          echo "COVERAGE=$COVERAGE" >> $GITHUB_ENV

      - name: Run Linter
        run: |
          npm run lint -- --format json --output-file .eva-cache/lint-report.json || echo "Linting failed"
          WARNINGS=$(cat .eva-cache/lint-report.json | jq '[.[].messages[]] | length')
          echo "WARNINGS=$WARNINGS" >> $GITHUB_ENV

      - name: Generate Housekeeping Report
        run: |
          pwsh scripts/Invoke-EvaHousekeeping.ps1 -OutputPath .eva-cache/reports/

      - name: Update .eva-memory.json
        run: |
          node scripts/update-eva-memory.js \
            --coverage ${{ env.COVERAGE }} \
            --warnings ${{ env.WARNINGS }} \
            --output .eva-memory.json

      - name: Backup Previous Memory
        run: |
          TIMESTAMP=$(date +%Y-%m-%d-%H-%M)
          cp .eva-memory.json ".eva-cache/backups/eva-memory-$TIMESTAMP.json"

      - name: Cleanup Old Backups
        run: |
          # Keep only last 24 backups (24 hours)
          ls -t .eva-cache/backups/ | tail -n +25 | xargs -I {} rm .eva-cache/backups/{}

      - name: Commit Changes
        run: |
          git config user.name "EVA Memory Bot"
          git config user.email "eva-bot@marcopolo483.github.io"
          git add .eva-memory.json
          git commit -m "ğŸ§  Auto-update EVA memory [skip ci]" || echo "No changes"
          git push || echo "Push failed"

      - name: Upload Cache Artifacts
        uses: actions/upload-artifact@v4
        with:
          name: eva-cache
          path: .eva-cache/
          retention-days: 7

    Triggers: 1. Nightly (3am) - Full housekeeping + memory update 2. Every push (main/develop) - Quick memory refresh 3. Manual - Marco can trigger anytime via GitHub UI

    Duration: ~3-5 minutes per repo (parallel across all repos).

    Memory Update Script¶

    File: scripts/update-eva-memory.js (in every repo)

    Implementation:

    #!/usr/bin/env node

const fs = require('fs');
const path = require('path');
const { execSync } = require('child_process');

// Read existing memory
const memoryPath = path.join(__dirname, '..', '.eva-memory.json');
const memory = JSON.parse(fs.readFileSync(memoryPath, 'utf8'));

// Get current repo stats
const gitLog = execSync('git log -1 --format="%ci"', { encoding: 'utf8' }).trim();
const openIssues = execSync('gh issue list --state open --json number --jq "length"', { encoding: 'utf8' }).trim();

// Update context
memory.repo.lastUpdated = new Date().toISOString();
memory.context.openIssues = parseInt(openIssues) || 0;

// Update quality metrics (from CLI args)
const args = process.argv.slice(2);
const coverageArg = args.find(a => a.startsWith('--coverage='));
const warningsArg = args.find(a => a.startsWith('--warnings='));

if (coverageArg) {
  memory.quality.testCoverage = parseFloat(coverageArg.split('=')[1]) || 0;
}
if (warningsArg) {
  memory.quality.lintWarnings = parseInt(warningsArg.split('=')[1]) || 0;
}

// Update reports metadata
memory.reports.lastHousekeeping = new Date().toISOString();
memory.reports.lastAudit = new Date().toISOString();
memory.reports.lastTestRun = new Date().toISOString();

// Write updated memory
fs.writeFileSync(memoryPath, JSON.stringify(memory, null, 2), 'utf8');

console.log('âœ… EVA memory updated successfully');
console.log(`   Coverage: ${memory.quality.testCoverage}%`);
console.log(`   Warnings: ${memory.quality.lintWarnings}`);
console.log(`   Open Issues: ${memory.context.openIssues}`);

node scripts/update-eva-memory.js --coverage=85.3 --warnings=2

    const vscode = require('vscode');
const fs = require('fs');
const path = require('path');

function activate(context) {
  console.log('EVA Memory extension activated');

  // Load .eva-memory.json on workspace open
  const workspaceFolders = vscode.workspace.workspaceFolders;
  if (workspaceFolders) {
    workspaceFolders.forEach(folder => {
      const memoryPath = path.join(folder.uri.fsPath, '.eva-memory.json');
      if (fs.existsSync(memoryPath)) {
        const memory = JSON.parse(fs.readFileSync(memoryPath, 'utf8'));

        // Display welcome message with context
        vscode.window.showInformationMessage(
          `ğŸ§  ${memory.repo.name} loaded | Coverage: ${memory.quality.testCoverage}% | Issues: ${memory.context.openIssues}`,
          'View Reports'
        ).then(selection => {
          if (selection === 'View Reports') {
            // Open reports folder
            vscode.commands.executeCommand('vscode.openFolder', 
              vscode.Uri.file(path.join(folder.uri.fsPath, '.eva-cache', 'reports'))
            );
          }
        });

        // Make memory available to Copilot context
        context.workspaceState.update('eva-memory', memory);
      }
    });
  }
}

function deactivate() {}
