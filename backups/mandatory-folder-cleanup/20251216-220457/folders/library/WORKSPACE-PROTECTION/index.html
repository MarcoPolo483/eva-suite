    EVA Workspace Protection System¶

    Created: 2025-11-28
    Purpose: Prevent accidental bulk deletion of EVA Suite repositories
    Trigger Event: Data loss incident on Nov 28, 2025 at ~11:00 AM

    Incident Summary (Nov 28, 2025)¶

    What Happened: - Time: ~11:00 AM EST - Lost Work: 5 hours (6:00 AM - 11:00 AM) - Folders Deleted: All EVA repos in workspace (~30+ folders) - Root Cause: Copilot session executed bulk deletion command - Recovery: Cloned fresh repos from GitHub at 12:50 PM

    Critical Data Lost: - ‚ùå eva-gc-look-n-feel repository (never pushed to GitHub) - ‚ùå EVA-Sovereign-UI: 5 Eyes country templates (Canada, USA, UK, Australia, New Zealand) - ‚ùå eva-i11y, eva-i18n enhancements - ‚ùå Autonomous workflow implementations - ‚ùå GitHub Spark integration code - ‚ùå All smoke test results and configurations

    Impact: ~5 hours of development work permanently lost

    Protection System¶

    1. Workspace Protection Script¶

    Location: scripts/Protect-Workspace.ps1

    Features: - ‚úÖ Intercepts Remove-Item commands with -Recurse flag - ‚úÖ Detects deletion of protected folders - ‚úÖ Requires explicit "DELETE" confirmation (case-sensitive) - ‚úÖ Shows uncommitted git changes before deletion - ‚úÖ Logs all protection events to logs/workspace-protection.log - ‚úÖ Protects 33 folders in EVA Suite workspace

    Protected Folders: 

    eva-orchestrator, EVA-Sovereign-UI, eva-i11y, eva-i18n, eva-suite, eva-ui,
eva-api, eva-core, eva-agent, eva-auth, eva-rag, eva-ops, eva-safety,

    4. Testing the Protection¶

    Safe test (will be caught): 

    Remove-Item -Recurse -Force "C:\Users\marco\Documents\_AI Dev\EVA Suite\eva-orchestrator\test-temp-folder"

    Dangerous test (will trigger protection): 

    Remove-Item -Recurse -Force "C:\Users\marco\Documents\_AI Dev\EVA Suite\eva-orchestrator"

    Type anything OTHER than "DELETE" (all caps) to cancel.

    2. Git Push Discipline¶

    Best Practice: Push to GitHub after EVERY meaningful change.

    Quick push script (scripts/quick-push.ps1): 

    param([string]$Message = "chore: save work in progress")

git add -A
git commit -m $Message
git push

    Usage: 

    .\scripts\quick-push.ps1 "feat: add 5 Eyes templates"

    Lessons Learned¶

    Critical Practices¶

    1. ‚úÖ ALWAYS push to GitHub after each meaningful change
    2. ‚úÖ Use git branches for experimental work
    3. ‚úÖ Enable workspace protection in PowerShell profile
    4. ‚úÖ Configure backup software to include entire EVA Suite
    5. ‚úÖ Log work sessions with detailed notes
    6. ‚úÖ Test deployments in separate branches first
    7. ‚úÖ Never trust AI alone - review generated commands before execution

    Warning Signs¶

    üö® RED FLAGS - Stop immediately: - Copilot suggests Remove-Item -Recurse in workspace root - Bulk deletion commands without explicit path - Scripts that modify parent directories - Wildcard deletions (Remove-Item *)

    Recovery Strategy¶

    If data loss occurs:

    1. STOP - Don't make more changes
    2. Check Recycle Bin - May recover some files
    3. Check OneDrive/File History - Windows backup features
    4. Check Nero BackItUp - Last backup restore point
    5. GitHub clone - Get last pushed state
    6. EVA Audit Trail - Reconstruct work from logs
    7. Work Sessions - Review what was done
    8. ChatGPT/Copilot history - Recover generated code from AI chat logs

    Incident Response Plan¶

    If Folders Disappear¶

    1. Immediate Actions (within 5 minutes):
    2. Don't clone fresh repos yet
    3. Check Recycle Bin immediately
    4. Check Windows File History

    5. Check backup software last run time

    6. Assessment (5-15 minutes):

    7. Identify which folders are missing
    8. Check PowerShell history for deletion commands
    9. Review work-sessions.json for today's tasks

    10. Check EVA Audit Trail for file activity

    11. Recovery (15-60 minutes):

    12. Restore from Nero BackItUp if available
    13. Clone missing repos from GitHub
    14. Review unpushed changes in backup

    15. Reconstruct lost work from logs/notes

    16. Post-Incident (1-2 hours):

    17. Document what was lost
    18. Update protection scripts if needed
    19. Add new folders to protected list
    20. Review and improve backup strategy

    Escalation¶

    If critical data loss (>2 hours of work): 1. Stop all work immediately 2. Take screenshots of current state 3. Export PowerShell history 4. Export work-sessions.json 5. Export EVA Audit Trail logs 6. Create incident report in docs/incidents/
