    üí∞ Business Model (Multi-Tenant Project Spaces)¶

    Sandbox vs Production¶

    EVA DA Accelerator Sandbox: - Centrally funded (IITB/AICOE) - Experimentation, proof-of-concept, UAT - Standard templates, quick setup - No cost to client during evaluation

    Production Project Space: - Client-funded (departmental cost centers) - IaC-applied on-demand (costs only when live) - Per-space customization (Project Space config, UI/UX theming, system prompt, semantic tuning) - SLA guarantees (99.9% uptime)

    Index Strategy (80 Capacity Architecture)¶

    Shared Indexes (Public/Unclassified): - 1-2 indexes (EN/FR) - Multi-tenant, cost-efficient - Examples: Public jurisprudence, benefits guides

    Individual Indexes (Protected A/B/C): - Client isolation for security - IaC-driven provisioning - Costs attributed per client

    Capacity Planning: 40 prospects √ó 2 headroom = 80 Azure AI Search indexes

    Roles & Governance¶

    EVA PS-A (Project Space Administrator): - Creates sandboxes, promotes to production - Binds cost centers, manages lifecycle

    EVA PS-BA (Production Space Business Analyst): - Designs RAG workflows, prompts, UI - Collaborates with client SMEs

    Client Production Space Admin: - Owns runtime operations - Monitors costs, usage, quality

    EVA Directory ("Key Keeper"): - Manages roles, permissions, assignments - Audit trails for all changes
