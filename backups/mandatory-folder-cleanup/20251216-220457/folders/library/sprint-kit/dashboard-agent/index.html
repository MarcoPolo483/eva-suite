    GitHub Projects Dashboard Agent¶

    Status: âœ… Implemented
    Location: src/agents/update-dashboard-agent.ts
    Priority: HIGH

    Quick Start¶

    # Query mode (read-only)
npm run dashboard -- --repo MarcoPolo483/eva-orchestrator --project "EVA Agile Sprint 1"

# Update mode (requires project setup)
npm run dashboard -- --repo MarcoPolo483/eva-orchestrator --project "EVA Agile Sprint 1" \
  --update --issue 123 --field "Status" --value "In Progress"

# Dry-run (no mutations)
$env:DRY_RUN="1"; npm run dashboard -- --repo MarcoPolo483/eva-orchestrator --project "EVA Agile Sprint 1" --update --issue 123 --field "Status" --value "Done"

    Features Implemented¶

    âœ… Query Mode (read-only):

    • Connect to GitHub Projects v2 via GraphQL
    • Find project by name (user or org scope)
    • Query repository metrics (issues, PRs, stars, forks, language)
    • List all project items (issues/PRs with titles and numbers)
    • Display formatted dashboard report

    âœ… Update Mode (mutations):

    • Update GitHub Projects v2 custom field values
    • Support for single-select fields (Status, Sprint, Priority)
    • Resolve field IDs and option IDs automatically
    • Dry-run mode for safe testing
    • Clear success/error feedback

    Create a PAT with repo and project scopes:

    4. Set environment variable:
    $env:GITHUB_TOKEN="ghp_your_token_here"

    Create a new GitHub Project (Projects v2):

    1. GitHub â†’ Your profile â†’ Projects â†’ New project

    GitHub Actions Integration¶

    Create .github/workflows/update-dashboard.yml:

    name: Update Dashboard

on:
  issues:
    types: [closed]

jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '18'
      - run: npm install
      - run: |
          npm run dashboard -- \
            --repo ${{ github.repository }} \
            --project "EVA Agile Sprint 1" \
            --update \
            --issue ${{ github.event.issue.number }} \
            --field "Status" \
            --value "Done"
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

    API Reference¶

    Command-Line Arguments¶

    • --repo <owner/repo> - Repository in format owner/repo (required)
    • --project "<Project Name>" - GitHub Project name (required)
    • --update - Enable mutation mode (optional, default: read-only)
    • --issue <number> - Issue number to update (required with --update)
    • --field "<Field Name>" - Custom field name (required with --update)
    • --value "<Option Name>" - New field value (required with --update)

    Environment Variables¶

    • GITHUB_TOKEN or GH_TOKEN - GitHub PAT with repo+project scopes (required)
    • DRY_RUN=1 - Enable dry-run mode, no mutations (optional)

    Exit Codes¶

    • 0 - Success
    • 1 - Error (missing args, project not found, update failed)

    Troubleshooting¶

    Error: "Missing GITHUB_TOKEN"¶

    Solution: Set environment variable:

    $env:GITHUB_TOKEN="ghp_your_token_here"

    Error: "Project not found"¶

    Causes:

    • Project name typo (case-sensitive)
    • Project is organization-level, but token only has user access
    • Token missing project scope

    Solution: Verify project name and token permissions.

    Error: "Field not found"¶

    Causes:

    • Field name typo (case-sensitive)
    • Field not created in project settings

    Solution: Go to Project â†’ Settings â†’ Custom fields, verify field exists.

    Error: "Option not found for field"¶

    Causes:

    • Option value typo (case-sensitive)
    • Option not added to single-select field

    Solution: Go to Project â†’ Settings â†’ Custom fields â†’ Field options, verify option exists.
