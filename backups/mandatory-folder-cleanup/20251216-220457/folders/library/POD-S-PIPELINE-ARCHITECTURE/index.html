    ğŸ—ï¸ POD-S Unified Ingestion Pipeline Architecture¶

    Date: November 28, 2025
    Architect: EVA-Sovereign-UI Architect (Copilot A)
    Scope: One pipeline, multiple applications (Juris, Assist, CANL), diverse sources/formats

    Executive Summary¶

    ARCHITECTURE: One unified Azure ingestion pipeline serving 3 POD-S solutions in parallel.

    Key Insight: "One pipeline, multiple applications, sources, formats" means: - Single infrastructure: Shared Azure Functions, Cosmos DB, AI Search, OpenAI - Multiple tenants: Juris, Assist, CANL isolated by Hierarchical Partition Keys (HPK) - Diverse sources: XML (AssistMe), PDF (legal), plain text (booklets) - Unified processing: Common chunking, embedding, indexing, retrieval logic

    Budget Impact: $0 additional infrastructure cost (reuse eva-foundation pipeline).

    POD-S Solutions Overview¶

    Solution Domain Source Format Size Demo Data Ready
    Juris Legal/Jurisprudence PDF (case law, statutes) ~500-1000 docs âœ… YES (folder ready)
    Assist Productivity/HR XML (AssistMe articles) 1,450+ articles âœ… YES (XML file ready)
    CANL Canada Life Insurance Plain text (booklets) ~20-30 booklets âœ… YES (booklets ready)

    Common Requirement: All 3 need: 1. Document ingestion (upload) 2. Content extraction (parse) 3. Chunking (split into passages) 4. Embedding (vectorization with OpenAI) 5. Indexing (store in AI Search + Cosmos DB) 6. RAG retrieval (search + GPT-4 synthesis)

    2. Document Processor Function¶

    Purpose: Parse diverse formats into plain text + metadata.

    Implementation:

    // functions/document-processor/index.ts

import { AzureFunction, Context } from "@azure/functions";
import { BlobServiceClient } from "@azure/storage-blob";
import { parseXML, parsePDF, parseText } from "./parsers";

const documentProcessor: AzureFunction = async function (
  context: Context,
  blob: Buffer
): Promise<void> {
  const blobName = context.bindingData.name; // e.g., "case-law-2024-001.pdf"
  const tenant = context.bindingData.container.split("-")[1]; // e.g., "juris"
  const extension = blobName.split(".").pop();

  context.log(`Processing ${blobName} for tenant ${tenant}`);

  let text: string;
  let metadata: Record<string, any>;

  // Route by file type
  switch (extension) {
    case "pdf":
      ({ text, metadata } = await parsePDF(blob, blobName));
      break;
    case "xml":
      ({ text, metadata } = await parseXML(blob, blobName));
      break;
    case "txt":
      ({ text, metadata } = await parseText(blob, blobName));
      break;
    default:
      throw new Error(`Unsupported file type: ${extension}`);
  }

  // Queue for chunking
  context.bindings.chunkingQueue = {
    tenant,
    projectId: metadata.projectId || "default",
    documentId: blobName,
    text,
    metadata,
  };

  context.log(`Queued ${blobName} for chunking (${text.length} chars)`);
};

export default documentProcessor;

    Parsers:

    • PDF: Use pdf-parse or Azure Form Recognizer (OCR for scanned docs)
    • XML: Use xml2js or fast-xml-parser (already implemented in eva-foundation)
    • Text: Simple UTF-8 read (no parsing needed)

    3. Chunker & Embedder Function¶

    Purpose: Split text into 512-token chunks, generate embeddings, store in Cosmos DB + AI Search.

    Implementation:

    // functions/chunker-embedder/index.ts

import { AzureFunction, Context } from "@azure/functions";
import { CosmosClient } from "@azure/cosmos";
import { OpenAIClient } from "@azure/openai";
import { chunk } from "./chunking";

const chunkerEmbedder: AzureFunction = async function (
  context: Context,
  queueItem: any
): Promise<void> {
  const { tenant, projectId, documentId, text, metadata } = queueItem;

  // Split into 512-token chunks (overlap 50 tokens)
  const chunks = chunk(text, { maxTokens: 512, overlap: 50 });

  context.log(`Split ${documentId} into ${chunks.length} chunks`);

  // Generate embeddings
  const openai = new OpenAIClient(process.env.OPENAI_ENDPOINT, ...);
  const embeddings = await openai.embeddings.create({
    model: "text-embedding-3-small",
    input: chunks.map((c) => c.text),
  });

  // Store in Cosmos DB + AI Search
  const cosmos = new CosmosClient(process.env.COSMOS_ENDPOINT);
  const container = cosmos
    .database("eva-knowledge")
    .container("knowledge-chunks");

  for (let i = 0; i < chunks.length; i++) {
    const chunkId = `${documentId}-chunk-${i}`;
    const doc = {
      id: chunkId,
      tenantId: tenant,
      projectId,
      documentId,
      chunkIndex: i,
      text: chunks[i].text,
      embedding: embeddings.data[i].embedding,
      metadata: {
        ...metadata,
        page: chunks[i].page,
        section: chunks[i].section,
      },
    };

    // Upsert to Cosmos DB
    await container.items.upsert(doc);

    // Index in AI Search (via Azure SDK)
    await indexInAISearch(doc);
  }

  context.log(`Indexed ${chunks.length} chunks for ${documentId}`);
};

export default chunkerEmbedder;

    Chunking Strategy:

    Tenant Strategy Rationale
    Juris Page-based (1 page = 1 chunk) Legal docs reference pages
    Assist Article-based (1 article = 1-3 chunks) XML already structured
    CANL Paragraph-based (2-3 paragraphs = 1 chunk) Booklets are narrative

    Token Limit: 512 tokens (~384 words, ~2000 chars) per chunk.

    6. RAG Query Service Function¶

    Purpose: Answer user questions using RAG (Retrieval Augmented Generation).

    Implementation:

    // functions/rag-query-service/index.ts

import { AzureFunction, Context, HttpRequest } from "@azure/functions";
import { OpenAIClient } from "@azure/openai";
import { SearchClient } from "@azure/search-documents";

const ragQueryService: AzureFunction = async function (
  context: Context,
  req: HttpRequest
): Promise<void> {
  const { query, tenantId } = req.body;

  // 1. Generate query embedding
  const openai = new OpenAIClient(process.env.OPENAI_ENDPOINT, ...);
  const embedding = await openai.embeddings.create({
    model: "text-embedding-3-small",
    input: query,
  });

  // 2. Vector search (filtered by tenant)
  const searchClient = new SearchClient(
    process.env.SEARCH_ENDPOINT,
    "knowledge-index",
    ...
  );
  const results = await searchClient.search("*", {
    vectorQueries: [
      {
        vector: embedding.data[0].embedding,
        k: 10,
        fields: ["embedding"],
      },
    ],
    filter: `tenantId eq '${tenantId}'`,
    select: ["id", "text", "metadata"],
  });

  // 3. Build context from top results
  const context = results
    .map((r, i) => `[${i + 1}] ${r.document.text}`)
    .join("\n\n");

  // 4. Synthesize answer with GPT-4
  const completion = await openai.chat.completions.create({
    model: "gpt-4",
    messages: [
      {
        role: "system",
        content: `You are a helpful assistant. Answer using ONLY the context below. Cite sources using [1], [2], etc.\n\nContext:\n${context}`,
      },
      { role: "user", content: query },
    ],
  });

  // 5. Return answer + citations
  context.res = {
    status: 200,
    body: {
      answer: completion.choices[0].message.content,
      citations: results.map((r) => r.document.metadata),
    },
  };
};

export default ragQueryService;
