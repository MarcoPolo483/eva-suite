    P23 Garbage Collection Archival Guide¶

    Pattern: P23-Garbage-Collection + P21-Provenance Version: 2.0.0 Last Updated: December 14, 2025

    Overview¶

    EVA Garbage Collection (P23) now includes formal archival with P21 provenance integration for full audit trail and ATO compliance. Every cleanup run creates a timestamped archive with complete metadata for government-grade traceability.

    Archive Location Structure¶

    eva-orchestrator/.eva-cache/garbage-archive/
└── 20251214-133212/                    # Timestamp: YYYYMMDD-HHMMSS
    ├── eva-api/                        # Preserved folder structure
    │   ├── chat-ui/
    │   │   ├── index.html              # (72 KB monolith chat demo)
    │   │   ├── styles.css
    │   │   └── [13 more files]
    │   └── load-tests/
    │       └── [load testing scripts]
    ├── eva-rag/
    │   ├── data/
    │   │   └── [test data CSVs]
    │   └── data-sources/
    │       └── [data source configs]
    ├── _provenance.json                # P21 metadata
    ├── _fingerprints.json              # SHA256 hashes
    ├── _execution-log.json             # Machine-readable log
    ├── _execution-log.md               # Human-readable report
    └── _manifest.json                  # Complete inventory

    Key Principle: Folder structure is PRESERVED, not flattened. Files stay in their original hierarchy for easy browsing and restoration.

    Metadata Files Explained¶

    1. _provenance.json (P21 Audit Trail)¶

    Purpose: Who, what, when, why, how for ATO compliance

    {
  "operation": "garbage-collection",
  "pattern": "P23-Garbage-Collection",
  "runId": "aceb9076-3b47-44c8-b866-f09bd1c412d2",
  "timestamp": "2025-12-14T13:32:12Z",
  "operator": "GitHub Copilot (SM)",
  "approver": "Marco Presta",
  "reason": "Cleanup eva-api and eva-rag repos - accumulated garbage during rapid AI development (P23)",
  "riskLevel": "LOW-MEDIUM",
  "backupStrategy": "Full folder structure preservation with fingerprinting",
  "reversible": true,
  "compliance": {
    "atoRelevant": true,
    "auditTrail": true,
    "p21Integration": true
  },
  "reposAffected": ["eva-api", "eva-rag"],
  "archivePath": "C:\\...\\garbage-archive\\20251214-133212",
  "itemsDeleted": 45,
  "itemsMoved": 15,
  "itemsArchived": 10
}

    Usage: - ATO audits: "Show me who approved this deletion and why" - Compliance checks: "Is this operation reversible?" - Risk assessment: "What was the risk level?"

    2. _fingerprints.json (Integrity Verification)¶

    Purpose: SHA256 hashes of all archived files for integrity checks

    {
  "runId": "aceb9076-3b47-44c8-b866-f09bd1c412d2",
  "timestamp": "2025-12-14T13:32:12Z",
  "algorithm": "SHA256",
  "files": [
    {
      "path": "eva-api/chat-ui/index.html",
      "sha256": "a3f5e1d7c2b8f4a9e6d3c1b2a5f8e7d4c3b2a1f9e8d7c6b5a4f3e2d1c0b9a8",
      "size": 72840,
      "lastModified": "2025-12-10T15:30:00Z"
    },
    {
      "path": "eva-api/chat-ui/styles.css",
      "sha256": "b4e6f2d8c3b9f5a0e7d4c2b3a6f9e8d5c4b3a2f0e9d8c7b6a5f4e3d2c1b0a9",
      "size": 15234,
      "lastModified": "2025-12-10T15:30:00Z"
    }
  ]
}

    Usage: - Integrity verification: "Has this archive been tampered with?" - Restoration validation: "Did the restore succeed correctly?" - Forensics: "Which files were modified after archival?"

    How to verify: 

    # Verify integrity of a restored file
$hash = Get-FileHash -Path "eva-api/chat-ui/index.html" -Algorithm SHA256
$expected = "a3f5e1d7c2b8f4a9e6d3c1b2a5f8e7d4c3b2a1f9e8d7c6b5a4f3e2d1c0b9a8"
if ($hash.Hash -eq $expected) {
    Write-Host "✅ File integrity verified"
} else {
    Write-Host "❌ File integrity check FAILED"
}

    How to Restore from Archive¶

    Full Restoration¶

    # 1. Navigate to archive
cd "C:\Users\marco\Documents\_AI Dev\EVA Suite\eva-orchestrator\.eva-cache\garbage-archive\20251214-133212"

# 2. Review what was archived
cat _execution-log.md

# 3. Copy files back (preserving structure)
Copy-Item -Path "eva-api\*" -Destination "C:\...\EVA Suite\eva-api\" -Recurse -Force
Copy-Item -Path "eva-rag\*" -Destination "C:\...\EVA Suite\eva-rag\" -Recurse -Force

# 4. Verify integrity
.\verify-integrity.ps1  # (uses _fingerprints.json)

    Selective Restoration¶

    # Restore only chat-ui
Copy-Item -Path "eva-api\chat-ui" -Destination "C:\...\EVA Suite\eva-api\chat-ui" -Recurse -Force

# Verify specific file
$hash = Get-FileHash -Path "C:\...\eva-api\chat-ui\index.html" -Algorithm SHA256
# Compare with _fingerprints.json

    Automation Examples¶

    Check if cleanup succeeded¶

    $log = Get-Content ".eva-cache\garbage-archive\20251214-133212\_execution-log.json" | ConvertFrom-Json
if ($log.exitCode -eq 0) {
    Write-Host "✅ Cleanup succeeded"
    Write-Host "Deleted: $($log.statistics.itemsDeleted)"
    Write-Host "Moved: $($log.statistics.itemsMoved)"
} else {
    Write-Host "❌ Cleanup failed - check logs"
}

    Find all archives from last 30 days¶

    $archives = Get-ChildItem ".eva-cache\garbage-archive" -Directory |
    Where-Object { $_.CreationTime -gt (Get-Date).AddDays(-30) } |
    Sort-Object CreationTime -Descending

foreach ($archive in $archives) {
    $provenance = Get-Content "$($archive.FullName)\_provenance.json" | ConvertFrom-Json
    Write-Host "$($archive.Name): $($provenance.itemsDeleted) deleted, $($provenance.itemsMoved) moved"
}

    Generate compliance report¶

    # Collect all provenance.json from last 90 days for ATO audit
$complianceReport = @()
Get-ChildItem ".eva-cache\garbage-archive\*\_provenance.json" | ForEach-Object {
    $provenance = Get-Content $_ | ConvertFrom-Json
    if ((Get-Date $provenance.timestamp) -gt (Get-Date).AddDays(-90)) {
        $complianceReport += $provenance
    }
}

$complianceReport | ConvertTo-Json -Depth 10 | Out-File "ATO-Compliance-Report-Garbage-Collection.json"

    Compliance Benefits¶

    For ATO Audits¶

    • ✅ Complete audit trail: Who approved, when, why
    • ✅ Reversibility proof: Full restoration capability with integrity verification
    • ✅ Risk documentation: Risk level assessment for each cleanup
    • ✅ Operator tracking: Who executed the cleanup (username, machine)

    For Security Reviews¶

    • ✅ Fingerprints: SHA256 hashes for tamper detection
    • ✅ Provenance chain: Traceable back to original approval
    • ✅ Integrity verification: Can prove archive hasn't been modified

    For Operational Continuity¶

    • ✅ Quick restoration: Preserved folder structure = easy restore
    • ✅ Selective recovery: Can restore individual files if needed
    • ✅ Disaster recovery: Archives are timestamped, can recover any point in time
