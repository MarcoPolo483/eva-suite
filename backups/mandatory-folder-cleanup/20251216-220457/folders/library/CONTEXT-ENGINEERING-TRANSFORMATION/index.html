    Three Concepts Transformation Plan: Context Engineering + Housekeeping + Workspace Management¶

    Status: üöÄ Ready to Execute Date: December 7, 2025 Owner: Marco Presta (PO) + GitHub Copilot (SM) Duration: 3 phases over 2-4 weeks

    üéØ The Three Concepts Pattern¶

    1Ô∏è‚É£ Context Engineering (CE)¶

    Prevents: "Vibe coding" - code that compiles but violates EVA standards How: 4-block prompt pattern (Context ‚Üí Task ‚Üí Output ‚Üí Self-check) Applied: Every code generation, refactor, new feature Reference: docs/context-engineering.md

    2Ô∏è‚É£ Housekeeping (HK)¶

    Prevents: Technical debt, silent failures, workspace rot How: Automated health checks, cleanup, quality gates Applied: Session start, nightly automation, pre-commit hooks Reference: scripts/Invoke-EvaHousekeeping.ps1

    3Ô∏è‚É£ Workspace Management (WM) - Directory Mapping Pattern (P16-DMP)¶

    Prevents: Expensive file searches, context loss, navigation delays How: Pre-computed directory trees (.eva-cache/repo-tree.txt) Applied: After file additions, nightly sync, session start verification Reference: docs/eva-patterns/P16-DMP.md

    üéØ Vision Statement¶

    Transform eva-orchestrator into a Three Concepts exemplar - every script, every output, every workflow follows all three principles: Context Engineering, Housekeeping, and Workspace Management. Then use this as the gold standard to transform EVA Sovereign UI (first client).

    Guiding Principles¶

    1. Context Engineering prevents 'vibe coding' - code that compiles but doesn't encode EVA's actual behavior, rules, and constraints
    2. Housekeeping prevents workspace rot - automated health checks catch silent failures before they compound
    3. Workspace Management prevents context loss - pre-computed directory trees eliminate expensive searches and enable instant navigation

    Example (chat-completion function with Three Concepts): 

    /**
 * Chat Completion with RAG
 *
 * @context_engineering
 * Mission: Provide secure, RBAC-aware chat with document retrieval
 * Constraints: RBAC (RBACUserContext), Protected B (encryption), Audit (all requests logged)
 * Reuses: CosmosClient (singleton), RBAC utilities, AI Search client
 * Validates: User permissions, input sanitization, output redaction
 *
 * @housekeeping
 * Health: /health endpoint, connection pool monitoring
 * Cleanup: Closes connections, clears temp caches, releases resources
 * Monitoring: Application Insights (latency, errors, RU consumption)
 * Recovery: Retry with exponential backoff, circuit breaker pattern
 *
 * @workspace_management
 * Location: functions/chat-completion/
 * Caching: LRU cache for recent queries (5min TTL)
 * SharedUtils: CosmosClient (singleton), RBACValidator, AISearchClient
 *
 * @param request - ChatCompletionRequest (validated)
 * @param context - InvocationContext (Azure Functions)
 * @returns ChatCompletionResponse (with citations, RBAC-filtered)
 */
export async function chatCompletion(
  request: HttpRequest,
  context: InvocationContext
): Promise<HttpResponseInit> {
  // CE: Input validation
  const validated = ChatCompletionRequestSchema.parse(await request.json());

  // CE: RBAC check
  const rbacContext = await validateRBAC(request.headers);

  // CE: Self-check before processing
  if (!hasPermission(rbacContext, 'chat:query')) {
    return { status: 403, body: 'Permission denied' };
  }

  // HK: Health check - verify connections
  await validateConnections();

  // WM: Check cache first
  const cacheKey = generateCacheKey(validated);
  const cached = await queryCache.get(cacheKey);
  if (cached) return { status: 200, body: JSON.stringify(cached) };

  // Implementation...

  // CE: Audit log
  await logAuditEvent({
    type: 'chat-completion',
    userId: rbacContext.userId,
    tenantId: rbacContext.tenantId,
    timestamp: new Date().toISOString()
  });

  // HK: Cleanup temp resources
  await cleanupTempFiles();

  // WM: Cache result
  await queryCache.set(cacheKey, response, 300); // 5min TTL

  // CE: Output validation
  return { status: 200, body: JSON.stringify(response) };
}
