    Issue Templates for Sprint 5 Story Map (2025-12-12)¶

    Each entry below mirrors a story map row. Copy the title/description block into a GitHub issue, add the listed labels, and attach docs/backlog/triage-2025-12-12-story-map.md as supporting context.

    Sprint 5: P01–P10 Patterns¶

    P01 | Agent Pattern Library¶

    • Title: Pattern P01 — Agent Pattern Library
    • Description:
    • Define the catalog, metamodel, and discovery workflow for agent patterns so any pod can spin up a new agent from the library.
    • Acceptance Criteria:
    • Catalog + metamodel + discovery workflow documented in a CDD.
    • Linked docs/agents/devtools/P01-*.md draft exists.
    • QA: Docs lint + bilingual review.
    • Dependencies: P02 refined backlog.
    • Labels: pod-f-priority, pattern-story, pattern-P01.

    P02 | Requirements Engine¶

    • Title: Pattern P02 — Requirements Engine
    • Description:
    • Package intake schema, templates, and auto-generated notes so requirements become executable.
    • Acceptance Criteria:
    • Templates regenerate automatically and link to sample notes.
    • Sample payload included in documentation.
    • QA: Regeneration script passes.
    • Dependencies: Sprint board context; no blocking tasks.
    • Labels: pod-f-priority, pattern-P02.

    P03 | Scrum Agent¶

    • Title: Pattern P03 — Scrum Agent
    • Description:
    • Automate stand-up outputs, runbook updates, and sprint board sync from the Scrum Agent.
    • Acceptance Criteria:
    • Stand-up automation documented.
    • Runbook + sprint board sync described and linked.
    • QA: Workflows pass and docs/sprints directory reflects updates.
    • Dependencies: Sprint config data.
    • Labels: pod-f-priority, pattern-P03.

    P04 | Repo Librarian¶

    • Title: Pattern P04 — Repo Librarian
    • Description:
    • Sync README and ADR coverage across 10 repos, capturing drift and fixes.
    • Acceptance Criteria:
    • README/ADR coverage status updated for all targeted repos.
    • Doc drift log maintained.
    • QA: Bilingual copy proofed and docs lint without errors.
    • Dependencies: Branch tracking data.
    • Labels: pattern-P04.

    P05 | Scaffolder¶

    • Title: Pattern P05 — Scaffolder
    • Description:
    • Deliver a scaffolder CLI that bootstraps EVA repo layout and supporting docs.
    • Acceptance Criteria:
    • CLI scaffolds repo + doc layout.
    • Usage documented in README.
    • QA: Scaffold smoke test passes.
    • Dependencies: P06 infrastructure work.
    • Labels: pattern-P05.

    P06 | Review Agent¶

    • Title: Pattern P06 — Review Agent
    • Description:
    • Publish the PR review checklist and automation for lint/tests.
    • Acceptance Criteria:
    • Checklist documented.
    • Automation triggers on new PRs.
    • QA: Test failures surfaced in logs.
    • Dependencies: Integration with P08 CI/CD.
    • Labels: pattern-P06.

    P07 | Testing & Failure Explainer¶

    • Title: Pattern P07 — Testing & Failure Explainer
    • Description:
    • Provide test suggestions and failure narratives with coverage context.
    • Acceptance Criteria:
    • Suggestions + narratives documented.
    • Coverage >80% noted.
    • QA: Coverage report generated.
    • Dependencies: Metrics definitions for P07.
    • Labels: pattern-P07.

    P08 | CI/CD Guardian¶

    • Title: Pattern P08 — CI/CD Guardian
    • Description:
    • Document pipeline templates, guardrails, and automation checks.
    • Acceptance Criteria:
    • Pipeline templates published.
    • Guardrail list captured.
    • QA: Pipeline smoke test + secrets scoped.
    • Dependencies: Repository secrets + token inventory.
    • Labels: pattern-P08, ci.

    P09 | Runbook / SRE Agent¶

    • Title: Pattern P09 — Runbook/SRE Agent
    • Description:
    • Generate runbooks + incident triggers and validate telemetry hooks.
    • Acceptance Criteria:
    • Runbook + incident flow published.
    • Observability hook validation documented.
    • QA: Task 034 telemetry verified.
    • Dependencies: Instrumentation data from TASK-034.
    • Labels: pattern-P09.

    P10 | Metrics & Insights Agent¶

    • Title: Pattern P10 — Metrics & Insights Agent
    • Description:
    • Publish daily metrics summaries and burn charts.
    • Acceptance Criteria:
    • Metrics summary updated.
    • Burn chart refreshed.
    • QA: Metrics API responses validated.
    • Dependencies: P09 instrumentation.
    • Labels: pattern-P10.

    P11 | Security & Compliance Helper¶

    • Title: Pattern P11 — Security & Compliance Helper
    • Description:
    • Produce the security checklist and quickscan automation.
    • Acceptance Criteria:
    • Checklist + automation documented.
    • Compliance report generated.
    • QA: Bandit and npm audit pass.
    • Dependencies: Azure secrets vault.
    • Labels: pattern-P11.

    P12 | UX, Copy & Accessibility Agent¶

    • Title: Pattern P12 — UX, Copy & Accessibility Agent
    • Description:
    • Validate UX flows, WCAG compliance, bilingual screens.
    • Acceptance Criteria:
    • Accessiblity checks documented.
    • Dual-language copy reviewed.
    • QA: axe-core scan clean.
    • Dependencies: GC design tokens.
    • Labels: pattern-P12.

    P13 | Onboarding & Sandbox Coach¶

    • Title: Pattern P13 — Onboarding & Sandbox Coach
    • Description:
    • Refresh onboarding docs, sandbox templates, runbooks.
    • Acceptance Criteria:
    • Onboarding checklist updated.
    • Sandbox runbook refreshed.
    • QA: Playbook walkthrough completed.
    • Dependencies: Docs team review.
    • Labels: pattern-P13.

    P14 | DevOps LiveOps Companion¶

    • Title: Pattern P14 — DevOps LiveOps Companion
    • Description:
    • Compile LiveOps runbook + incident response flows.
    • Acceptance Criteria:
    • Runbook synced with telemetry.
    • Incident procedure rehearsed.
    • QA: Runbook lint + simulated incident.
    • Dependencies: Observability stack.
    • Labels: pattern-P14.

    P15 | Dev Master Orchestrator (Jarvis)¶

    • Title: Pattern P15 — Dev Master Orchestrator
    • Description:
    • Orchestrate multi-agent workflows across repos.
    • Acceptance Criteria:
    • Distributed tasks scheduled.
    • Artifact checkpoints recorded.
    • QA: Workflow dry run shows expected outputs.
    • Dependencies: P07 testing hooks, P05 scaffolder.
    • Labels: pattern-P15.

    P16 | Awareness Protocol¶

    • Title: Pattern P16 — Awareness Protocol
    • Description:
    • Surface assumptions, uncertainties, awareness logs.
    • Acceptance Criteria:
    • Awareness artifacts captured.
    • Agent states logged.
    • QA: Awareness dashboard verified.
    • Dependencies: Telemetry service + logging.
    • Labels: pattern-P16.

    P17 | Swarm PR Review¶

    • Title: Pattern P17 — Swarm PR Review
    • Description:
    • Coordinate multi-agent review of PRs.
    • Acceptance Criteria:
    • Feedback aggregated.
    • Review scorecard generated.
    • QA: Sample PR flagged and resolved.
    • Dependencies: PR automation hooks.
    • Labels: pattern-P17.

    P18 | Tool/API Auto-Onboarding¶

    • Title: Pattern P18 — Tool/API Auto-Onboarding
    • Description:
    • Standardize onboarding for new APIs/tools.
    • Acceptance Criteria:
    • Template docs ready.
    • Validation checks pass.
    • QA: New tool onboarding rehearsal.
    • Dependencies: Integration playbook.
    • Labels: pattern-P18.

    P19 | Action Classification & Safety¶

    • Title: Pattern P19 — Action Classification & Safety
    • Description:
    • Classify actions by autonomy level C0-C3.
    • Acceptance Criteria:
    • Classification taxonomy updated.
    • Safety checks enforced.
    • QA: Safety audit run.
    • Dependencies: Agentic Framework reference.
    • Labels: pattern-P19.

    P20 | Continuous Self-Improvement¶

    • Title: Pattern P20 — Continuous Self-Improvement
    • Description:
    • Feed logs/metrics into retrospectives.
    • Acceptance Criteria:
    • Retro notes created.
    • Improvement backlog items raised.
    • QA: Metrics confirm improvements.
    • Dependencies: P10 metrics.
    • Labels: pattern-P20.

    P21 | Provenance & Fingerprinting¶

    • Title: Pattern P21 — Provenance & Fingerprinting
    • Description:
    • Capture provenance/fingerprint data for audits.
    • Acceptance Criteria:
    • Provenance metadata stored.
    • Audit snapshot available.
    • QA: Provenance query returns expected data.
    • Dependencies: Storage account + audit logs.
    • Labels: pattern-P21.

    Agentic Book | Operating Manual (Issue 81)¶

    • Title: Agentic Book Issue 81 — Operating Manual integration
    • Description:
    • Link every Pxx specification into the operating manual hosted in eva-meta and highlight bilingual safety disclaimers.
    • Acceptance Criteria:
    • Manual references all P01–P21 patterns.
    • Bilingual summary plus GC Agentic Framework alignment documented.
    • QA: Manual proofed for GC tone + bilingual compliance.
    • Dependencies: All pattern docs, GC Agentic Framework reference.
    • Labels: agentic-book, pod-f-priority.

    Agentic Book | EVA Agentic Book Drafts (Issue 82)¶

    • Title: Agentic Book Issue 82 — Chapter drafts
    • Description:
    • Draft chapters of the seven-part Agentic Book template from docs/agents/book/ with P02 (requirements) and P12 (UX/proof) ownership.
    • Acceptance Criteria:
    • Chapter drafts preserve GC Design System voice + bilingual text.
    • Template compliance and references to docs/backlog/triage-2025-12-12-story-map.md.
    • QA: Template checklist complete, accessibility review recorded.
    • Dependencies: P12 reviewers, docs batches (#83–#89).
    • Labels: agentic-book, pattern-P02, pattern-P12.

    • QA: QA reviews logged, bilingual proofs completed, GC Design System compliance noted.
    • Dependencies: P04-LIB cleanup.
