    P22-DPB 4-Phase Control Loop - Quick Reference¶

    Pattern: P22-DPB (Documentation Publishing Bot) Version: 2.0.0 Status: Beta Last Updated: December 14, 2025

    The 4 Phases (At a Glance)¶

    ┌──────────────────────────────────────────────────────────────┐
│                    P22-DPB CONTROL LOOP                      │
└──────────────────────────────────────────────────────────────┘

Phase 1: ASSESS IMPLEMENTATION
├─ Tool: assess-implementation.ps1
├─ Input: Source code (*.ts, *.js, *.py)
├─ Output: _implementation-inventory.json (ground truth)
└─ Purpose: Create inventory of what EXISTS (no hallucination)

Phase 2: GENERATE MARKDOWN
├─ Tool: generate-api-docs.ps1 (needs update)
├─ Input: _implementation-inventory.json + source code
├─ Output: docs/api/*.md + _metadata.json
└─ Purpose: Document FROM inventory (validated)

Phase 3: BUILD HTML + RAG
├─ Tool: mkdocs build + extract-rag-text.ps1
├─ Input: docs/**/*.md
├─ Output: site/*.html + rag-output/*.txt
└─ Purpose: Publish multi-format docs

Phase 4: AUDIT OUTPUTS
├─ Tool: audit-p22-output.ps1
├─ Input: Inventory + docs + site
├─ Output: _p22-audit-report.json + P22-AUDIT-REPORT.md
├─ Exit Code: 0 (PASS) or 1 (FAIL if accuracy < 95%)
└─ Purpose: Enforce quality gate

┌──────────────────────────────────────────────────────────────┐
│  If Phase 4 PASS → Deploy                                   │
│  If Phase 4 FAIL → Fix Phase 2 → Re-run (max 3 iterations)  │
└──────────────────────────────────────────────────────────────┘

    Quick Commands¶

    Run Complete Workflow:¶

    cd "C:\Users\marco\Documents\_AI Dev\EVA Suite\eva-sovereign-ui"

# Phase 1: Assess
pwsh ..\eva-orchestrator\tools\eva-library\assess-implementation.ps1 -ProjectPath "."

# Phase 2: Generate (needs inventory validation update)
pwsh ..\eva-orchestrator\tools\eva-library\generate-api-docs.ps1 -ProjectPath "."

# Phase 3: Build
mkdocs build --strict --verbose
pwsh ..\eva-orchestrator\tools\eva-library\extract-rag-text.ps1 -AllBooks

# Phase 4: Audit
pwsh ..\eva-orchestrator\tools\eva-library\audit-p22-output.ps1 -ProjectPath "." -MinAccuracy 95
# Exit code 0 = PASS (deploy), 1 = FAIL (fix and re-run)

    Troubleshooting¶

    Phase 1 Fails (No Exports Found):¶

    # Check source patterns in script
# Verify file extensions (.ts, .tsx, .js, .jsx, .py)
# Check exclusion patterns (node_modules, dist, build)

    Phase 4 Fails (Accuracy < 95%):¶

    Hallucinated Items (documented but not in code): - Remove from documentation - Re-run Phase 2

    Missing Items (in code but not documented): - Add to documentation - Re-run Phase 2

    Broken Links: - Fix empty hrefs (href="#") - Fix missing pages (404s) - Re-run Phase 3

    Infinite Loop Prevention:¶

    • Max 3 iterations before human escalation
    • Check CI/CD workflow logs for iteration count
    • If stuck, investigate root cause (broken TypeDoc config, missing files, etc.)

    Quality Gates¶

    Phase Metric Threshold Action if Fail
    Phase 1 Files scanned > 0 Check file patterns
    Phase 1 Exports found > 0 Check source code
    Phase 2 Markdown generated > 0 Check TypeDoc config
    Phase 2 Validation passed 100% Fix missing exports
    Phase 3 MkDocs build No warnings Fix markdown syntax
    Phase 3 RAG extraction > 0 files Check book definitions
    Phase 4 Accuracy >= 95% Fix hallucinations/missing items
    Phase 4 Broken links 0 critical Fix empty hrefs, 404s

    Files Reference¶

    Scripts:¶

    • tools/eva-library/assess-implementation.ps1 (Phase 1)
    • tools/eva-library/generate-api-docs.ps1 (Phase 2)
    • tools/eva-library/extract-rag-text.ps1 (Phase 3b)
    • tools/eva-library/audit-p22-output.ps1 (Phase 4)

    Documentation:¶

    • docs/agents/devtools/P22-Documentation-Publishing-Bot.md (Pattern docs)
    • docs/P22-4PHASE-IMPLEMENTATION-SUMMARY.md (Implementation summary)
    • docs/BROKEN-LINKS-AUDIT-REPORT.md (Broken links audit)

    Configuration:¶

    • mkdocs.yml (Phase 3a - MkDocs config)
    • tools/eva-library/api-metadata-schema.json (12-category metadata)

    Success Criteria¶

    ✅ Phase 1: Inventory created with > 0 exports ✅ Phase 2: Docs generated and validated against inventory ✅ Phase 3: HTML + RAG built without errors ✅ Phase 4: Accuracy >= 95% AND no critical broken links ✅ CI/CD: All 4 phases pass in < 60 seconds

    Common Pitfalls¶

    ❌ Don't skip Phase 1 - Causes hallucinations (infers from docs instead of code) ❌ Don't ignore Phase 4 failures - Accuracy < 95% means unreliable docs ❌ Don't run infinite loops - Max 3 iterations before human intervention ❌ Don't deploy without audit - Broken links and hallucinations reach production
