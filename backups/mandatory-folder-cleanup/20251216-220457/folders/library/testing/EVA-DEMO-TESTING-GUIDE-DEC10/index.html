    üö® Troubleshooting¶

    Chat UI Issues¶

    Issue: CORS errors in browser console 

    Access to fetch at 'http://127.0.0.1:8000/api/v1/chat/query' from origin 'null' has been blocked by CORS policy
    Solution: 1. Verify API server running: http://127.0.0.1:8000/health 2. Check CORS headers in eva-api/src/eva_api/main.py: 
    app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # Or specific origins
    allow_methods=["*"],
    allow_headers=["*"]
)
    3. Alternative: Use VS Code Live Server extension (serves via http://) - Install: Extensions ‚Üí "Live Server" (Ritwick Dey) - Right-click index.html ‚Üí "Open with Live Server" - Opens at http://127.0.0.1:5500/chat-ui/index.html

    Issue: "No Knowledge Spaces Found" Solution: 1. Check Cosmos DB connection (API server logs) 2. Create test space using API: 

    $headers = @{ "X-API-Key" = "your-api-key" }
$body = @{ name = "Test Space"; description = "Test knowledge space" } | ConvertTo-Json
Invoke-RestMethod -Uri "http://127.0.0.1:8000/api/v1/spaces" -Method POST -Headers $headers -Body $body -ContentType "application/json"
    3. Refresh chat UI

    Issue: API key invalid Solution: 1. Open eva-api/chat-ui/index.html in editor 2. Find API_KEY constant (around line 480): 

    const API_KEY = 'your-api-key-here';
    3. Update with valid key from eva-api backend 4. Save and refresh browser

    Portal Issues¶

    Issue: npm install fails Solution: 

    # Clear cache and retry
npm cache clean --force
rm -r node_modules
rm package-lock.json
npm install

# If still fails, check Node.js version
node --version  # Should be 18+

    Issue: Port 3000 already in use Solution: 

    # Option 1: Kill process using port 3000
Get-Process -Id (Get-NetTCPConnection -LocalPort 3000).OwningProcess | Stop-Process -Force

# Option 2: Use different port
npm run dev -- --port 3001

    Issue: TypeScript errors during build Solution: 

    # Check tsconfig.json strict mode
# Run type checker
npm run type-check

# View specific errors
npx tsc --noEmit

    Issue: Tailwind CSS not applying styles Solution: 1. Verify tailwind.config.js content paths: 

    content: [
  "./index.html",
  "./src/**/*.{js,ts,jsx,tsx}"
]
    2. Restart dev server (Vite hot reload issue) 3. Purge cache: 
    rm -r node_modules/.vite
npm run dev
