    Policy Engine Validation¶

    Decision Logic Verified¶

    The _should_use_api(task) function implements the following priority order:

    1. EVA_OPENAI_FORCE_OFFLINE → return False (skip API)
2. task.execution_hint == "offline" → return False
3. task.execution_hint == "online" → return True
4. EVA_OPENAI_FORCE_ONLINE → return True (increment counter)
5. _REAL_CALLS_USED >= max_calls → return False
6. Default → return True (increment counter)

    Test Coverage: - ✅ Priority 1: FORCE_OFFLINE=1 correctly returned offline decision - ✅ Priority 4/6: FORCE_OFFLINE=0 correctly allowed online decision - ⏸️ Priorities 2, 3, 5: Not tested in this run (would require YAML execution_hint or quota exhaustion)

    Counter Management¶

    The _REAL_CALLS_USED global counter tracks API calls per CLI run: - Offline Job: Counter never incremented (no calls made) - Online Job: Counter would increment to 1, then cap future calls - Reset: Counter resets on each python -m eva_suite_orchestrator.cli invocation

    Environment Variable Handling¶

    All policy env vars correctly parsed: - EVA_OPENAI_FORCE_OFFLINE: "1" → True, "0" → False - EVA_OPENAI_FORCE_ONLINE: "1" → True, "0" → False
    - EVA_OPENAI_MAX_REAL_CALLS_PER_RUN: "0" → no calls, "1" → 1 call, "" → unlimited - EVA_OPENAI_OFFLINE: Preserved for backward compatibility with agent-level flag

    Appendix A: Workflow YAML¶

    name: EVA Orchestrator – propose-patch test

on:
  workflow_dispatch:
    inputs:
      task_id:
        description: 'Task ID to generate proposal for'
        required: false
        default: 'suite-i18n-scaffold'
        type: string
      plan_path:
        description: 'Path to plan YAML file'
        required: false
        default: 'plans/eva-suite/eva-suite-polish-deploy-v1.yml'
        type: string

jobs:
  propose_patch_offline:
    runs-on: ubuntu-latest
    env:
      PYTHONPATH: src
      EVA_OPENAI_API_KEY: ${{ secrets.EVA_OPENAI_API_KEY }}
      EVA_OPENAI_OFFLINE: "0"
      EVA_OPENAI_FORCE_OFFLINE: "1"
      EVA_OPENAI_MAX_REAL_CALLS_PER_RUN: "0"
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install -r requirements.txt
      - run: python -m eva_suite_orchestrator.cli \
              --plan ${{ inputs.plan_path }} \
              --mode propose-patch \
              --task ${{ inputs.task_id }}
      - uses: actions/upload-artifact@v4
        if: always()
        with:
          name: proposals-offline
          path: out/proposals/

  propose_patch_online:
    runs-on: ubuntu-latest
    needs: propose_patch_offline
    env:
      PYTHONPATH: src
      EVA_OPENAI_API_KEY: ${{ secrets.EVA_OPENAI_API_KEY }}
      EVA_OPENAI_OFFLINE: "0"
      EVA_OPENAI_FORCE_OFFLINE: "0"
      EVA_OPENAI_MAX_REAL_CALLS_PER_RUN: "1"
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install -r requirements.txt
      - run: python -m eva_suite_orchestrator.cli \
              --plan ${{ inputs.plan_path }} \
              --mode propose-patch \
              --task ${{ inputs.task_id }}
      - uses: actions/upload-artifact@v4
        if: always()
        with:
          name: proposals-online
          path: out/proposals/
