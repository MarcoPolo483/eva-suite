    Chapter 01: Getting Started¶

    Installation¶

    EVA Sovereign UI can be consumed in three ways: npm package (recommended for production), CDN (rapid prototyping), or source build (contributing to library).

    Option 1: npm Package (Recommended)¶

    For production applications, install via npm:

    npm install @eva-suite/sovereign-ui

    Peer dependencies (install if not already present):

    npm install lit@^3.0.0

    Optional TypeScript definitions (included automatically):

    # No additional install needed - TypeScript definitions ship with package

    Option 2: CDN (Prototyping)¶

    For quick prototyping or static HTML pages:

    <!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>EVA Sovereign UI - CDN Example</title>

  <!-- Import EVA Sovereign UI from CDN -->
  <script type="module">
    import 'https://cdn.jsdelivr.net/npm/@eva-suite/sovereign-ui@latest/dist/eva-sovereign-ui.js';
  </script>
</head>
<body>
  <!-- Components available immediately -->
  <eva-gc-button label-en="Hello" label-fr="Bonjour"></eva-gc-button>
</body>
</html>

    ⚠️ Warning: CDN usage is NOT recommended for Protected B environments due to external dependency constraints. Use npm package for production deployments.

    Option 3: Source Build (Contributors)¶

    For library development:

    git clone https://github.com/MarcoPolo483/eva-sovereign-ui.git
cd eva-sovereign-ui
npm install
npm run dev  # Start Storybook dev server

    Quick Start Example¶

    Basic HTML Page¶

    Create index.html:

    <!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My GC Application</title>

  <!-- GC Design System base styles (optional but recommended) -->
  <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/@eva-suite/sovereign-ui/dist/themes/canada-gc.css">

  <style>
    body {
      font-family: Lato, sans-serif;
      padding: 2rem;
      max-width: 800px;
      margin: 0 auto;
    }
  </style>
</head>
<body>
  <!-- GC Header -->
  <eva-gc-header
    title-en="My Application"
    title-fr="Mon Application"
    lang="en">
  </eva-gc-header>

  <!-- Main content -->
  <main>
    <h1>Welcome to EVA Sovereign UI</h1>

    <!-- Button with loading state -->
    <eva-gc-button
      label-en="Submit Application"
      label-fr="Soumettre la demande"
      variant="primary"
      onclick="handleSubmit(event)">
    </eva-gc-button>

    <!-- Alert for feedback -->
    <eva-gc-alert
      id="feedback"
      type="success"
      title-en="Success"
      title-fr="Succès"
      message-en="Your application was submitted."
      message-fr="Votre demande a été soumise."
      hidden>
    </eva-gc-alert>
  </main>

  <!-- GC Footer -->
  <eva-gc-footer lang="en"></eva-gc-footer>

  <!-- Import components -->
  <script type="module">
    import '@eva-suite/sovereign-ui';

    // Event handler
    window.handleSubmit = async (event) => {
      const button = event.target;
      const alert = document.getElementById('feedback');

      button.setAttribute('loading', '');

      await new Promise(resolve => setTimeout(resolve, 2000));  // Simulate API call

      button.removeAttribute('loading');
      alert.removeAttribute('hidden');
    };
  </script>
</body>
</html>

    React Integration¶

    For React applications:

    // App.tsx
import { useEffect, useState } from 'react';
import '@eva-suite/sovereign-ui';

// TypeScript definitions for Web Components
declare global {
  namespace JSX {
    interface IntrinsicElements {
      'eva-gc-button': any;
      'eva-gc-alert': any;
    }
  }
}

export function App() {
  const [loading, setLoading] = useState(false);
  const [showAlert, setShowAlert] = useState(false);

  const handleSubmit = async () => {
    setLoading(true);
    await fetch('/api/submit', { method: 'POST' });
    setLoading(false);
    setShowAlert(true);
  };

  return (
    <div className="app">
      <eva-gc-button
        label-en="Submit"
        label-fr="Soumettre"
        variant="primary"
        loading={loading ? '' : undefined}
        onClick={handleSubmit}
      />

      {showAlert && (
        <eva-gc-alert
          type="success"
          title-en="Success"
          title-fr="Succès"
          message-en="Submitted successfully"
          message-fr="Soumis avec succès"
        />
      )}
    </div>
  );
}

    React + TypeScript (recommended):

    npm install @eva-suite/sovereign-ui
npm install --save-dev @types/react @types/react-dom

    TypeScript Setup¶

    Automatic Type Definitions¶

    EVA Sovereign UI ships with TypeScript definitions. Your IDE will automatically provide autocompletion:

    // VSCode will autocomplete props
const button = document.createElement('eva-gc-button');
button.labelEn = 'Submit';  // ✅ Type-safe
button.labelFr = 'Soumettre';
button.variant = 'primary';  // ✅ Autocomplete: 'primary' | 'secondary' | 'tertiary'

    Type-Safe Event Handlers¶

    Components emit strongly-typed custom events:

    import type { EvaGcButtonClickEvent } from '@eva-suite/sovereign-ui';

const button = document.querySelector('eva-gc-button');
button?.addEventListener('eva-button-click', (event: EvaGcButtonClickEvent) => {
  console.log(event.detail.timestamp);  // ✅ Type-safe access
  console.log(event.detail.lang);       // 'en' | 'fr'
});

    JSX Type Declarations¶

    For React/Preact TypeScript projects:

    // src/vite-env.d.ts or global.d.ts
import { EvaGcButton, EvaGcAlert } from '@eva-suite/sovereign-ui';

declare global {
  namespace JSX {
    interface IntrinsicElements {
      'eva-gc-button': EvaGcButton;
      'eva-gc-alert': EvaGcAlert;
      // ... add other components as needed
    }
  }
}

    Your First Component: eva-gc-button¶

    Let's explore eva-gc-button in detail—the most commonly used component.

    Basic Usage¶

    <eva-gc-button
  label-en="Submit"
  label-fr="Soumettre">
</eva-gc-button>

    All Props (Annotated)¶

    <eva-gc-button
  <!-- Required bilingual labels -->
  label-en="Submit Application"
  label-fr="Soumettre la demande"

  <!-- Optional ARIA labels (defaults to label-* if not provided) -->
  aria-label-en="Submit your passport application"
  aria-label-fr="Soumettre votre demande de passeport"

  <!-- Variant: primary (blue), secondary (white), tertiary (link-style) -->
  variant="primary"

  <!-- Size: small, medium, large -->
  size="medium"

  <!-- States (boolean attributes) -->
  disabled
  loading

  <!-- Icon (optional, from eva-gc-icon library) -->
  icon="arrow-right"
  icon-position="end"

  <!-- Language preference (defaults to document lang) -->
  lang="en"

  <!-- Type for form submission -->
  type="submit">
</eva-gc-button>

    Event Handling¶

    eva-gc-button emits custom events:

    const button = document.querySelector('eva-gc-button');

button.addEventListener('eva-button-click', (event) => {
  console.log('Button clicked!', event.detail);
  // { timestamp: 1702500000000, lang: 'en', variant: 'primary' }
});

    Dynamic States¶

    Toggle loading state during async operations:

    async function submitForm() {
  const button = document.querySelector('eva-gc-button');

  button.setAttribute('loading', '');  // Show spinner
  button.setAttribute('disabled', ''); // Prevent re-clicks

  try {
    await fetch('/api/submit', { method: 'POST' });
    alert('Success!');
  } catch (error) {
    console.error(error);
  } finally {
    button.removeAttribute('loading');
    button.removeAttribute('disabled');
  }
}

    Styling with CSS Variables¶

    Override GC Design System tokens for custom styling:

    eva-gc-button {
  --eva-button-bg-primary: #0066cc;      /* Custom primary color */
  --eva-button-border-radius: 0.5rem;    /* Rounded corners */
  --eva-button-padding: 1rem 2rem;       /* Larger padding */
}

    ⚠️ Caution: Custom styling may violate GC Design System guidelines. Consult with UX/accessibility teams before overriding defaults.

    Development Server¶

    For live component development with hot reloading:

    Storybook (Interactive Catalog)¶

    cd eva-sovereign-ui
npm run storybook

    Opens http://localhost:6006 with interactive component playground:

    graph LR
    A[Storybook UI] --> B[Component Catalog]
    B --> C[eva-gc-button]
    B --> D[eva-gc-alert]
    B --> E[eva-gc-form]
    C --> F[Knobs for Props]
    C --> G[A11y Checks]
    C --> H[Viewport Resizing]

    Features: - Interactive prop controls (Knobs) - Accessibility checks (a11y addon) - Viewport resizing (mobile/tablet/desktop) - Code snippets (copy-paste ready)

    Vite Dev Server (Component Development)¶

    For building new components or modifying existing ones:

    npm run dev

    Opens http://localhost:5173 with: - Hot module reloading (instant updates) - TypeScript type checking - Lit DevTools integration

    Project Structure¶

    When consuming EVA Sovereign UI in your project:

    my-gc-app/
├── node_modules/
│   └── @eva-suite/
│       └── sovereign-ui/
│           ├── dist/                # Bundled components
│           ├── themes/              # GC Design System themes
│           └── types/               # TypeScript definitions
├── src/
│   ├── index.html
│   ├── main.ts
│   └── components/                  # Your custom components
├── package.json
└── tsconfig.json

    Import strategy:

    // Option 1: Import all components (increases bundle size)
import '@eva-suite/sovereign-ui';

// Option 2: Import specific components (tree-shakable)
import '@eva-suite/sovereign-ui/eva-gc-button';
import '@eva-suite/sovereign-ui/eva-gc-alert';

    Troubleshooting¶

    Components Not Rendering¶

    Issue: Web components show as <eva-gc-button> without rendering.

    Solution: Ensure module is imported before components are used:

    <!-- ❌ Wrong order -->
<eva-gc-button label-en="Test"></eva-gc-button>
<script type="module">
  import '@eva-suite/sovereign-ui';
</script>

<!-- ✅ Correct order -->
<script type="module">
  import '@eva-suite/sovereign-ui';
</script>
<eva-gc-button label-en="Test"></eva-gc-button>

    TypeScript Errors in React¶

    Issue: Property 'eva-gc-button' does not exist on type 'JSX.IntrinsicElements'

    Solution: Add type declarations (see TypeScript Setup section above).

    Styles Not Applying¶

    Issue: Components appear unstyled or use wrong fonts.

    Solution: Import base theme:

    <link rel="stylesheet" href="node_modules/@eva-suite/sovereign-ui/dist/themes/canada-gc.css">

    Or in JavaScript:

    import '@eva-suite/sovereign-ui/dist/themes/canada-gc.css';
