    Chapter 04: Context Awareness in EVA Chat 2.0¶

    Overview¶

    Context awareness is the foundational capability that distinguishes enterprise AI assistants from generic chatbots. EVA Chat 2.0 maintains four orthogonal context dimensions that work together to produce accurate, personalized, and compliant responses:

    1. User Context: Identity, preferences, language, security clearance
    2. Conversation Context: Message history, topic continuity, intent tracking
    3. Document Context: RAG-sourced citations, source attribution, confidence scores
    4. Temporal Context: Time-based filtering, recency bias, versioned content

    Each dimension is implemented as a discrete service layer with clear boundaries, enabling independent scaling, testing, and compliance auditing. This chapter explores the architecture, implementation patterns, and security considerations for each context type.

    User Context: Identity and Preferences¶

    User Profile Schema¶

    EVA Chat 2.0 stores user profiles in Azure Cosmos DB (Protected B compliant) with the following schema:

    interface UserProfile {
  userId: string;                    // Azure AD objectId
  email: string;                     // Primary email (Protected B)
  displayName: string;               // Bilingual display name
  preferredLanguage: "en-CA" | "fr-CA";
  securityClearance: "unclassified" | "protected-a" | "protected-b";
  department: string;                // ESDC, SSC, TBS, etc.
  roleAssignments: string[];         // RBAC roles (reader, contributor, admin)
  preferences: UserPreferences;
  consentFlags: ConsentFlags;
  createdAt: Date;
  lastActiveAt: Date;
}

interface UserPreferences {
  theme: "light" | "dark" | "auto";
  citationStyle: "inline" | "footnote" | "sidebar";
  autoTranslate: boolean;            // Auto-translate FRâ†”EN responses
  responseLength: "concise" | "detailed" | "comprehensive";
  enableRAG: boolean;                // Allow document retrieval
  allowExternalSources: boolean;     // Allow non-GC sources
}

interface ConsentFlags {
  acceptedTermsVersion: string;      // Latest accepted ToS version
  analyticsConsent: boolean;         // Telemetry opt-in
  feedbackConsent: boolean;          // Allow feedback collection
  dataRetentionConsent: boolean;     // Conversation history retention
}

    User Context Provider¶

    The UserContextProvider component wraps the chat interface and provides user profile data via React Context:

    import { createContext, useContext, useState, useEffect } from 'react';
import { UserProfile } from './types';
import { fetchUserProfile } from './services/user-service';

interface UserContextType {
  profile: UserProfile | null;
  loading: boolean;
  error: Error | null;
  refreshProfile: () => Promise<void>;
}

const UserContext = createContext<UserContextType | undefined>(undefined);

export const UserContextProvider: React.FC<{ children: React.ReactNode }> = ({ children }) => {
  const [profile, setProfile] = useState<UserProfile | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<Error | null>(null);

  const refreshProfile = async () => {
    try {
      setLoading(true);
      const userProfile = await fetchUserProfile();
      setProfile(userProfile);
      setError(null);
    } catch (err) {
      setError(err as Error);
      console.error('Failed to load user profile:', err);
    } finally {
      setLoading(false);
    }
  };

  useEffect(() => {
    refreshProfile();
  }, []);

  return (
    <UserContext.Provider value={{ profile, loading, error, refreshProfile }}>
      {children}
    </UserContext.Provider>
  );
};

export const useUserContext = () => {
  const context = useContext(UserContext);
  if (!context) {
    throw new Error('useUserContext must be used within UserContextProvider');
  }
  return context;
};

    Protected B Compliance¶

    Critical Requirements: - User profiles MUST be stored in Canada Central or Canada East regions only - Email addresses and department affiliations are Protected B and MUST be encrypted at rest (CMK in Azure Key Vault) - User preferences MUST NOT be logged in Application Insights (PII leakage risk) - Security clearance MUST be validated via Azure AD claims (never client-provided)

    Encryption Pattern:

    import { SecretClient } from '@azure/keyvault-secrets';
import { DefaultAzureCredential } from '@azure/identity';

class UserProfileService {
  private secretClient: SecretClient;

  constructor() {
    const credential = new DefaultAzureCredential();
    this.secretClient = new SecretClient(
      process.env.AZURE_KEYVAULT_URL!,
      credential
    );
  }

  async encryptEmail(email: string): Promise<string> {
    const { value } = await this.secretClient.setSecret(
      `user-email-${userId}`,
      email,
      {
        contentType: 'application/octet-stream',
        tags: { classification: 'Protected-B' }
      }
    );
    return value!;
  }

  async decryptEmail(userId: string): Promise<string> {
    const { value } = await this.secretClient.getSecret(`user-email-${userId}`);
    return value!;
  }
}

    Conversation Context: History and Intent Tracking¶

    Conversation Storage Schema¶

    Conversations are stored in Cosmos DB with a partition key of userId for efficient retrieval:

    interface Conversation {
  id: string;                        // UUID
  userId: string;                    // Partition key
  title: string;                     // Auto-generated from first message
  messages: Message[];
  metadata: ConversationMetadata;
  createdAt: Date;
  lastMessageAt: Date;
}

interface Message {
  id: string;                        // UUID
  role: "user" | "assistant" | "system";
  content: string;
  language: "en-CA" | "fr-CA";
  timestamp: Date;
  sources?: Citation[];              // RAG citations (if applicable)
  autonomyLevel?: "C0" | "C1" | "C2" | "C3";  // GC Agentic Framework
  confidence?: number;               // 0.0-1.0
}

interface ConversationMetadata {
  topics: string[];                  // Extracted topics (e.g., ["EI", "CPP"])
  sentiment: "positive" | "neutral" | "negative";
  containsPII: boolean;              // Flagged by Azure Content Safety
  classification: "unclassified" | "protected-a" | "protected-b";
  retentionPolicy: "7-days" | "90-days" | "7-years";
}

    Intent Tracking with Azure OpenAI¶

    EVA Chat 2.0 uses GPT-4o to extract user intent and maintain topic continuity across multi-turn conversations:

    import { OpenAIClient } from '@azure/openai';
import { DefaultAzureCredential } from '@azure/identity';

class IntentTracker {
  private openai: OpenAIClient;

  constructor() {
    const credential = new DefaultAzureCredential();
    this.openai = new OpenAIClient(
      process.env.AZURE_OPENAI_ENDPOINT!,
      credential
    );
  }

  async extractIntent(messages: Message[]): Promise<string[]> {
    const systemPrompt = `
You are a topic extraction assistant for the Government of Canada.
Analyze the conversation and extract key topics (max 5).
Focus on government programs, policies, and services.

Return a JSON array of topics: ["topic1", "topic2", ...]
`;

    const chatHistory = messages.map(m => ({
      role: m.role,
      content: m.content
    }));

    const response = await this.openai.getChatCompletions(
      process.env.AZURE_OPENAI_DEPLOYMENT!,
      [
        { role: 'system', content: systemPrompt },
        ...chatHistory
      ],
      {
        temperature: 0.3,
        maxTokens: 200,
        responseFormat: { type: 'json_object' }
      }
    );

    const result = JSON.parse(response.choices[0].message.content);
    return result.topics || [];
  }
}

    Conversation Continuity Pattern¶

    To maintain context across page refreshes and sessions, EVA Chat 2.0 uses a sliding window approach:

    class ConversationManager {
  private readonly MAX_CONTEXT_TOKENS = 8000;  // GPT-4o context limit
  private readonly MAX_MESSAGES = 20;          // Hard limit

  async getRelevantContext(conversationId: string): Promise<Message[]> {
    const conversation = await this.fetchConversation(conversationId);
    const messages = conversation.messages;

    // Always include system message + last N messages
    const systemMessage = messages.find(m => m.role === 'system');
    const recentMessages = messages.slice(-this.MAX_MESSAGES);

    // Estimate token count (rough heuristic: 1 token â‰ˆ 4 chars)
    let totalTokens = this.estimateTokens(recentMessages);

    // If over limit, remove older messages (keep most recent)
    while (totalTokens > this.MAX_CONTEXT_TOKENS && recentMessages.length > 5) {
      recentMessages.shift();
      totalTokens = this.estimateTokens(recentMessages);
    }

    return systemMessage ? [systemMessage, ...recentMessages] : recentMessages;
  }

  private estimateTokens(messages: Message[]): number {
    const totalChars = messages.reduce((sum, m) => sum + m.content.length, 0);
    return Math.ceil(totalChars / 4);
  }
}

    Document Context: RAG Citations and Source Tracking¶

    Citation Schema¶

    Citations link assistant responses back to source documents in Azure Cognitive Search:

    interface Citation {
  id: string;                        // Search result ID
  title: string;                     // Document title (bilingual)
  url: string;                       // Source URL (SharePoint, Azure Blob)
  excerpt: string;                   // Relevant text snippet
  confidence: number;                // 0.0-1.0 (vector similarity score)
  source: "sharepoint" | "azure-blob" | "internal-kb";
  classification: "unclassified" | "protected-a" | "protected-b";
  lastModified: Date;
  metadata: Record<string, string>; // Custom metadata (author, department, etc.)
}

    Document Context Provider¶

    The DocumentContextProvider manages RAG citations and source attribution:

    interface DocumentContextType {
  citations: Citation[];
  addCitation: (citation: Citation) => void;
  clearCitations: () => void;
  findCitationById: (id: string) => Citation | undefined;
}

export const useDocumentContext = () => {
  const [citations, setCitations] = useState<Citation[]>([]);

  const addCitation = (citation: Citation) => {
    setCitations(prev => {
      // Avoid duplicates
      if (prev.find(c => c.id === citation.id)) return prev;
      return [...prev, citation];
    });
  };

  const clearCitations = () => setCitations([]);

  const findCitationById = (id: string) => citations.find(c => c.id === id);

  return { citations, addCitation, clearCitations, findCitationById };
};

    Security: Source Validation¶

    Critical: EVA Chat 2.0 MUST validate that users have access permissions to cited documents BEFORE displaying them:

    class CitationSecurityService {
  async validateUserAccess(userId: string, citation: Citation): Promise<boolean> {
    // Check user's security clearance vs document classification
    const user = await this.userService.getProfile(userId);

    if (citation.classification === 'protected-b' && user.securityClearance !== 'protected-b') {
      return false;
    }

    // Check SharePoint permissions via Microsoft Graph API
    if (citation.source === 'sharepoint') {
      const hasAccess = await this.graphClient.checkSharePointAccess(userId, citation.url);
      return hasAccess;
    }

    // Check Azure RBAC for Blob Storage
    if (citation.source === 'azure-blob') {
      const hasAccess = await this.checkBlobAccess(userId, citation.url);
      return hasAccess;
    }

    return true;  // Default allow for unclassified internal KB
  }
}

    Temporal Context: Time-Based Filtering and Recency Bias¶

    Temporal Filtering Strategy¶

    Government policies and programs evolve over time. EVA Chat 2.0 uses temporal filters to prioritize recent documents:

    interface TemporalFilter {
  startDate?: Date;                  // Filter documents after this date
  endDate?: Date;                    // Filter documents before this date
  recencyBias: number;               // Boost factor for recent docs (1.0-2.0)
  versionAware: boolean;             // Show only latest version
}

class TemporalSearchService {
  async searchWithTemporalFilter(
    query: string,
    filter: TemporalFilter
  ): Promise<Citation[]> {
    const searchClient = this.getSearchClient();

    const results = await searchClient.search(query, {
      filter: this.buildODataFilter(filter),
      orderBy: ['lastModified desc', 'confidence desc'],
      scoringProfile: 'recency-boost',  // Custom scoring profile
      scoringParameters: [`recencyBoost-${filter.recencyBias}`]
    });

    return results.map(r => this.mapToCitation(r));
  }

  private buildODataFilter(filter: TemporalFilter): string {
    const clauses: string[] = [];

    if (filter.startDate) {
      clauses.push(`lastModified ge ${filter.startDate.toISOString()}`);
    }

    if (filter.endDate) {
      clauses.push(`lastModified le ${filter.endDate.toISOString()}`);
    }

    if (filter.versionAware) {
      clauses.push(`isLatestVersion eq true`);
    }

    return clauses.join(' and ');
  }
}

    Recency Bias Scoring Profile¶

    Azure Cognitive Search supports custom scoring profiles to boost recent documents:

    {
  "name": "recency-boost",
  "text": {

        "boostingDuration": "P180D"
      }
    }
  ]
}

    Versioned Content Management¶

    For documents with multiple versions (e.g., policy updates), EVA Chat 2.0 tracks version history:

    interface VersionedDocument {
  documentId: string;
  versions: DocumentVersion[];
  currentVersion: string;            // Latest version ID
}

interface DocumentVersion {
  versionId: string;
  effectiveDate: Date;
  expiryDate?: Date;
  content: string;
  changeLog: string;                 // Summary of changes
  isLatest: boolean;
}

class VersionManager {
  async getEffectiveVersion(
    documentId: string,
    asOfDate: Date = new Date()
  ): Promise<DocumentVersion | null> {
    const doc = await this.fetchVersionedDocument(documentId);

    const effective = doc.versions.find(v =>
      v.effectiveDate <= asOfDate &&
      (!v.expiryDate || v.expiryDate > asOfDate)
    );

    return effective || null;
  }
}

    Context Orchestration: Bringing It All Together¶

    Context Aggregation Service¶

    The ContextAggregationService combines all four context dimensions into a unified request context for Azure OpenAI:

    interface RequestContext {
  user: UserProfile;
  conversation: Message[];
  documents: Citation[];
  temporal: TemporalFilter;
}

class ContextAggregationService {
  async buildContext(
    userId: string,
    conversationId: string,
    query: string
  ): Promise<RequestContext> {
    // Parallel context fetching
    const [user, conversation, documents] = await Promise.all([
      this.userService.getProfile(userId),
      this.conversationManager.getRelevantContext(conversationId),
      this.ragService.searchDocuments(query, { recencyBias: 1.5 })
    ]);

    // Validate document access
    const allowedDocs = await Promise.all(
      documents.map(async doc => {
        const hasAccess = await this.citationSecurity.validateUserAccess(userId, doc);
        return hasAccess ? doc : null;
      })
    );

    return {
      user,
      conversation,
      documents: allowedDocs.filter(d => d !== null) as Citation[],
      temporal: { recencyBias: 1.5, versionAware: true }
    };
  }
}

    Prompt Engineering with Context¶

    The aggregated context is injected into the system prompt for Azure OpenAI:

    function buildSystemPrompt(context: RequestContext): string {
  return `
You are EVA Chat Assistant, serving the Government of Canada.

User Profile:
- Name: ${context.user.displayName}
- Language: ${context.user.preferredLanguage}
- Department: ${context.user.department}

2. Cite sources using [Source #N] notation
3. Never cite documents above user's security clearance
4. Prioritize recent documents (last 180 days)
5. Flag uncertain information with confidence scores
`;
}

    Performance Optimization¶

    Caching Strategy¶

    User profiles and conversation histories are cached in Azure Redis Cache to reduce Cosmos DB latency:

    import { createClient } from 'redis';

class ContextCacheService {
  private redis = createClient({ url: process.env.REDIS_URL });

  async getCachedProfile(userId: string): Promise<UserProfile | null> {
    const cached = await this.redis.get(`profile:${userId}`);
    if (cached) {
      return JSON.parse(cached);
    }

    // Cache miss - fetch from Cosmos DB
    const profile = await this.cosmosService.getProfile(userId);
    await this.redis.setEx(`profile:${userId}`, 3600, JSON.stringify(profile));
    return profile;
  }
}

    Lazy Loading Citations¶

    Citations are loaded on-demand to reduce initial payload size:

    function MessageWithCitations({ message }: { message: Message }) {
  const [expanded, setExpanded] = useState(false);
  const [citations, setCitations] = useState<Citation[]>([]);

  const loadCitations = async () => {
    if (message.sources) {
      const loaded = await Promise.all(
        message.sources.map(id => fetchCitation(id))
      );
      setCitations(loaded);
      setExpanded(true);
    }
  };

  return (
    <div>
      <p>{message.content}</p>
      {message.sources && (
        <button onClick={loadCitations}>
          Show {message.sources.length} sources
        </button>
      )}
      {expanded && <CitationList citations={citations} />}
    </div>
  );
}

    Summary¶

    Context awareness transforms EVA Chat 2.0 from a stateless chatbot into a stateful, personalized AI assistant that understands:

    • Who the user is (identity, preferences, clearance)
    • What they're discussing (conversation history, intent)
    • Where to find answers (RAG citations, source validation)
    • When information was published (temporal filtering, recency bias)

    All four context dimensions are security-hardened for Protected B deployment, with encryption at rest, audit logging, and RBAC enforcement at every layer. The next chapter explores RAG integration in depth, covering vector search, document ingestion, and citation generation.
