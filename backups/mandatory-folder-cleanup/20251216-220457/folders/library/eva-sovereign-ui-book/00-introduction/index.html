    Chapter 00: Introduction¶

    What is EVA Sovereign UI?¶

    EVA Sovereign UI is an enterprise-grade, open-source web components library purpose-built for the Government of Canada digital ecosystem. Built on Lit (Web Components), it provides production-ready UI components that implement the GC Design System while enforcing WCAG 2.1 AA accessibility standards and mandatory bilingualism (EN-CA/FR-CA) by default.

    Unlike traditional component libraries that treat accessibility and bilingualism as optional add-ons, EVA Sovereign UI makes these first-class architectural constraints. Every component ships with:

    • Baked-in bilingualism: Language-aware props, automatic content switching, RTL/LTR support
    • WCAG 2.1 AA compliance: Keyboard navigation, screen reader support, ARIA semantics, color contrast validation
    • GC Design System alignment: canada-gc theme tokens, FIP (Federal Identity Program) compliance, GC branding guidelines
    • Protected B readiness: Designed for deployment in secure Government of Canada environments with private endpoints and data residency constraints

    EVA Sovereign UI is the UI foundation layer for the EVA Suite—a collection of 40+ AI-powered enterprise applications serving the Government of Canada. While EVA Foundation handles cloud services (Azure AD, Cosmos DB, Azure OpenAI), EVA Sovereign UI ensures a consistent, accessible, bilingual user experience across all products.

    Why Web Components and Lit?¶

    Framework Agnosticism¶

    Government of Canada departments use diverse technology stacks—React, Vue, Angular, Blazor, vanilla JavaScript. Web Components provide a framework-neutral foundation:

    <!-- Works in React -->
<eva-gc-button label-en="Submit" label-fr="Soumettre" />

<!-- Works in Vue -->
<eva-gc-button :label-en="submitLabel" />

<!-- Works in vanilla HTML -->
<eva-gc-button label-en="Submit" label-fr="Soumettre"></eva-gc-button>

    Future-Proof Standards¶

    Web Components are W3C standards supported natively in all modern browsers. No build toolchain required for consumption. No framework lock-in. No risk of vendor abandonment.

    Lit Advantages¶

    Lit is Google's Web Components framework chosen for EVA Sovereign UI because:

    1. Minimal overhead: 5KB runtime, faster than virtual DOM frameworks
    2. Reactive properties: Declarative state management with TypeScript decorators
    3. Scoped styles: Shadow DOM encapsulation prevents CSS leaks
    4. Developer experience: Hot module reloading, TypeScript first, excellent tooling
    5. Production proven: Used by Google, Adobe, Salesforce, Microsoft

    GC Design System Alignment¶

    EVA Sovereign UI implements the Canada.ca Design System as the authoritative visual and interaction standard for Government of Canada web applications. This includes:

    Visual Identity¶

    • FIP (Federal Identity Program): canada-gc theme with official Government of Canada wordmark, colors, and typography
    • Design tokens: Standardized color palettes, spacing scales, typography ramps extracted from canada.ca
    • Responsive grid: Mobile-first 12-column grid with GC-approved breakpoints

    Interaction Patterns¶

    • Navigation: canada-gc-header, canada-gc-footer, breadcrumbs, language toggle
    • Forms: Validated inputs, error messaging (EN/FR), accessible fieldsets
    • Feedback: Alerts, notifications, loading states with GC-approved styling

    Compliance¶

    All components pass the GC Digital Standards checklist:

    • ✅ Mobile-first design
    • ✅ WCAG 2.1 AA accessibility
    • ✅ Bilingual by default (EN-CA/FR-CA)
    • ✅ Progressive enhancement
    • ✅ Open standards (HTML5, CSS3, ES2020+)

    Target Audience¶

    Primary Users¶

    1. Government of Canada Developers: Building public-facing or internal applications requiring GC Design System compliance
    2. EVA Suite Product Teams: Consuming EVA Sovereign UI as the standard UI layer for EVA products
    3. Accessibility Specialists: Auditing and validating WCAG 2.1 AA compliance
    4. UX Designers: Prototyping GC-compliant interfaces with production-ready components

    Prerequisites¶

    • Basic web development: HTML, CSS, JavaScript (ES6+)
    • npm/yarn: Package manager experience for installation
    • TypeScript (optional but recommended): For type safety and IDE autocompletion
    • Lit familiarity (helpful but not required): Understanding of Web Components lifecycle

    Key Features¶

    1. Accessibility by Default¶

    Every component implements WCAG 2.1 AA out of the box:

    <eva-gc-button
  label-en="Submit Application"
  label-fr="Soumettre la demande"
  aria-label-en="Submit your passport application"
  aria-label-fr="Soumettre votre demande de passeport"
  disabled
  loading>
</eva-gc-button>

    Automatic features: - Keyboard navigation (Tab, Enter, Space, Arrow keys) - Screen reader announcements (ARIA live regions) - Focus management (visible focus indicators, focus trapping in modals) - Color contrast validation (4.5:1 for text, 3:1 for UI components)

    2. Mandatory Bilingualism¶

    All user-facing text requires both EN-CA and FR-CA:

    // TypeScript API enforces bilingual props
interface ButtonProps {
  labelEn: string;  // Required
  labelFr: string;  // Required
  lang?: 'en' | 'fr';  // Optional, defaults to user preference
}

    Components automatically switch content based on: 1. Explicit lang attribute 2. Closest [lang] ancestor 3. Document <html lang="..."> attribute 4. Browser navigator.language

    3. GC Design System Tokens¶

    Centralized design tokens ensure consistency:

    /* GC Design System tokens */
--gc-color-primary: #26374a;      /* Canada.ca blue */
--gc-color-accent: #e60a14;       /* Canada red */
--gc-color-focus: #303fc3;        /* Focus ring blue */
--gc-font-family: Lato, sans-serif;
--gc-spacing-unit: 0.25rem;       /* 4px base unit */

    4. Protected B Security Posture¶

    Components designed for deployment in secure Government of Canada environments:

    • No external dependencies: All assets bundled, no CDN calls to third-party services
    • CSP compliance: Content Security Policy compatible (no inline styles/scripts in Shadow DOM)
    • Audit logging: Emits custom events for user interactions (form submissions, button clicks) for compliance tracking
    • Data residency: No data leaves browser context (no analytics, no telemetry)

    5. Developer Experience¶

    Modern tooling and developer ergonomics:

    • TypeScript definitions: Full type safety and IDE autocompletion
    • Storybook integration: Interactive component catalog for exploration
    • Hot module reloading: Instant feedback during development
    • Unit tests: Vitest + Testing Library for behavior-driven testing
    • Visual regression tests: Automated screenshot diffing for UI changes

    What EVA Sovereign UI is NOT¶

    To set clear expectations:

    • Not a design system: It implements the GC Design System; it doesn't replace it
    • Not a framework: It's a component library consumed by frameworks (React, Vue, etc.)
    • Not canada.ca: It provides components for internal government applications, not the public canada.ca website
    • Not EVA Foundation: EVA Sovereign UI is UI only; backend services (auth, data, AI) are handled by EVA Foundation

    Architecture Context¶

    EVA Sovereign UI sits in the Presentation Layer of the EVA 2.0 architecture:

    graph TD
    A[User Browser] --> B[EVA Sovereign UI]
    B --> C[EVA Foundation API]
    C --> D[Azure Services]
    D --> E[Azure AD / Cosmos DB / OpenAI]

    B -.-> F[GC Design System]
    B -.-> G[WCAG 2.1 AA]
    B -.-> H[Bilingual Content]

    Separation of Concerns: - EVA Sovereign UI: Renders UI, handles user interactions, enforces accessibility/bilingualism - EVA Foundation: Enforces authentication/authorization, manages data, orchestrates AI services

    Components never make authorization decisions—they render based on RBACUserContext passed from EVA Foundation.
