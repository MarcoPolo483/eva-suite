    Chapter 07: Internationalization¶

    EVA Sovereign UI enforces mandatory bilingualism (EN-CA/FR-CA) at the type level—components cannot be instantiated without providing both English and French content. This architectural constraint ensures Government of Canada applications comply with the Official Languages Act by default.

    Bilingual-First Architecture¶

    Type-Level Enforcement¶

    Unlike libraries that treat internationalization as optional, EVA Sovereign UI makes bilingualism a compile-time requirement:

    // ❌ This will NOT compile
<eva-gc-button label="Submit"></eva-gc-button>

// ✅ This WILL compile
<eva-gc-button
  label-en="Submit"
  label-fr="Soumettre"
></eva-gc-button>

    Dual-Property Pattern¶

    Every text-bearing property requires both language variants:

    interface BilingualProps {
  'label-en': string;
  'label-fr': string;
  'placeholder-en'?: string;
  'placeholder-fr'?: string;
  'error-en'?: string;
  'error-fr'?: string;
  'description-en'?: string;
  'description-fr'?: string;
}

    This pattern prevents the common anti-pattern of "English-first, translate later" and ensures parity between language variants.

    Language Context Provider¶

    EVA Sovereign UI uses a global language context that all components observe:

    // Set application language
import { LanguageService } from '@eva-suite/sovereign-ui';

LanguageService.setLanguage('fr'); // Switch to French
const currentLang = LanguageService.getLanguage(); // 'en' or 'fr'

// Listen for language changes
LanguageService.onLanguageChange((newLanguage) => {
  console.log(`Language changed to ${newLanguage}`);
  // Update page title, fetch localized data, etc.
});

    All components automatically re-render when language changes—no manual prop updates required.

    i18n Service¶

    The i18n service provides centralized translation management beyond component properties.

    Translation Registry¶

    Register translations for application-specific content:

    import { i18n } from '@eva-suite/sovereign-ui';

i18n.register('en', {
  common: {
    save: 'Save',
    cancel: 'Cancel',
    delete: 'Delete',
    confirm: 'Are you sure?'
  },
  errors: {
    required: 'This field is required',
    invalidEmail: 'Please enter a valid email address',
    networkError: 'Unable to connect to server'
  },
  navigation: {
    dashboard: 'Dashboard',
    reports: 'Reports',
    settings: 'Settings'
  }
});

i18n.register('fr', {
  common: {
    save: 'Enregistrer',
    cancel: 'Annuler',
    delete: 'Supprimer',
    confirm: 'Êtes-vous sûr?'
  },
  errors: {
    required: 'Ce champ est obligatoire',
    invalidEmail: 'Veuillez entrer une adresse courriel valide',
    networkError: 'Impossible de se connecter au serveur'
  },
  navigation: {
    dashboard: 'Tableau de bord',
    reports: 'Rapports',
    settings: 'Paramètres'
  }
});

    Translation Lookup¶

    // Get translated string for current language
const saveLabel = i18n.t('common.save'); // 'Save' or 'Enregistrer'

// Translations with parameters
i18n.register('en', {
  messages: {
    greeting: 'Hello, {name}!'
  }
});

i18n.register('fr', {
  messages: {
    greeting: 'Bonjour, {name}!'
  }
});

const greeting = i18n.t('messages.greeting', { name: 'Marco' });
// EN: "Hello, Marco!"
// FR: "Bonjour, Marco!"

    Pluralization¶

    i18n.register('en', {
  items: {
    count: '{count} item | {count} items'
  }
});

i18n.register('fr', {
  items: {
    count: '{count} élément | {count} éléments'
  }
});

i18n.t('items.count', { count: 1 }); // "1 item" / "1 élément"
i18n.t('items.count', { count: 5 }); // "5 items" / "5 éléments"

    Missing Translation Handling¶

    // Development mode: Log warnings for missing translations
i18n.configure({
  mode: 'development',
  onMissingTranslation: (key, language) => {
    console.warn(`Missing ${language} translation for key: ${key}`);
    return `[${language}:${key}]`; // Fallback display
  }
});

// Production mode: Fail fast
i18n.configure({
  mode: 'production',
  onMissingTranslation: (key, language) => {
    throw new Error(`Missing required ${language} translation: ${key}`);
  }
});

    Language Switching¶

    EVA Sovereign UI provides seamless language switching without page reload.

    Language Toggle Component¶

    <eva-gc-language-toggle></eva-gc-language-toggle>

    Renders as:

    <!-- Current language: EN -->
<button
  class="eva-gc-language-toggle"
  aria-label="Français"
  lang="fr"
>
  <abbr title="Français">FR</abbr>
</button>

<!-- Current language: FR -->
<button
  class="eva-gc-language-toggle"
  aria-label="English"
  lang="en"
>
  <abbr title="English">EN</abbr>
</button>

    Custom Language Switcher¶

    <eva-gc-select
  id="languageSelect"
  label-en="Language"
  label-fr="Langue"
  value="en"
>
  <option value="en">English</option>
  <option value="fr">Français</option>
</eva-gc-select>

<script>
  const select = document.getElementById('languageSelect');

  select.addEventListener('eva-change', (event) => {
    const newLanguage = event.detail.value;
    LanguageService.setLanguage(newLanguage);

    // Update URL parameter
    const url = new URL(window.location.href);
    url.searchParams.set('lang', newLanguage);
    window.history.pushState({}, '', url);

    // Update HTML lang attribute
    document.documentElement.setAttribute('lang', newLanguage);
  });
</script>

    Persistence¶

    Store user language preference:

    // Save to localStorage
LanguageService.onLanguageChange((language) => {
  localStorage.setItem('preferred-language', language);
});

// Restore on page load
const savedLanguage = localStorage.getItem('preferred-language');
if (savedLanguage === 'en' || savedLanguage === 'fr') {
  LanguageService.setLanguage(savedLanguage);
}

    URL-Based Language Detection¶

    // Detect language from URL parameter (?lang=fr)
function detectLanguageFromURL(): 'en' | 'fr' {
  const params = new URLSearchParams(window.location.search);
  const lang = params.get('lang');

  if (lang === 'fr') return 'fr';
  return 'en'; // Default to English
}

// Initialize language on app load
const initialLanguage = detectLanguageFromURL();
LanguageService.setLanguage(initialLanguage);
document.documentElement.setAttribute('lang', initialLanguage);

    Translation Workflow¶

    Source Files¶

    Organize translations in JSON files:

    src/
  i18n/
    en.json       # English translations
    fr.json       # French translations
    index.ts      # Export combined translations

    en.json: 

    {
  "app": {
    "title": "EVA Domain Assistant",
    "description": "AI-powered research assistant for Government of Canada"
  },
  "navigation": {
    "dashboard": "Dashboard",
    "search": "Search",
    "documents": "Documents",
    "settings": "Settings"
  },
  "search": {
    "placeholder": "Enter your question...",
    "submit": "Search",
    "noResults": "No results found",
    "resultsCount": "{count} results"
  }
}

    fr.json: 

    {
  "app": {
    "title": "Assistant de domaine EVA",
    "description": "Assistant de recherche alimenté par l'IA pour le gouvernement du Canada"
  },
  "navigation": {
    "dashboard": "Tableau de bord",
    "search": "Recherche",
    "documents": "Documents",
    "settings": "Paramètres"
  },
  "search": {
    "placeholder": "Entrez votre question...",
    "submit": "Rechercher",
    "noResults": "Aucun résultat trouvé",
    "resultsCount": "{count} résultats"
  }
}

    index.ts: 

    import en from './en.json';
import fr from './fr.json';
import { i18n } from '@eva-suite/sovereign-ui';

// Register all translations
i18n.register('en', en);
i18n.register('fr', fr);

export { en, fr };

    Translation Keys Convention¶

    Use dot notation for hierarchical structure:

    // ✅ Good: Hierarchical, descriptive
i18n.t('search.form.placeholder')
i18n.t('errors.validation.required')
i18n.t('navigation.menu.settings')

// ❌ Bad: Flat, unclear
i18n.t('searchPlaceholder')
i18n.t('error1')
i18n.t('nav3')

    Professional Translation Process¶

    1. Extract translatable strings from source code
    2. Send to certified translators (Translation Bureau, external agency)
    3. Receive translated JSON files
    4. Validate completeness (all EN keys have FR equivalents)
    5. QA review (terminology, context, tone)
    6. Deploy to production

    Translation Validation Script¶

    // scripts/validate-translations.ts
import en from './src/i18n/en.json';
import fr from './src/i18n/fr.json';

function flattenObject(obj: any, prefix = ''): string[] {
  const keys: string[] = [];

  for (const [key, value] of Object.entries(obj)) {
    const fullKey = prefix ? `${prefix}.${key}` : key;

    if (typeof value === 'object' && value !== null) {
      keys.push(...flattenObject(value, fullKey));
    } else {
      keys.push(fullKey);
    }
  }

  return keys;
}

const enKeys = flattenObject(en);
const frKeys = flattenObject(fr);

const missingInFr = enKeys.filter(key => !frKeys.includes(key));
const missingInEn = frKeys.filter(key => !enKeys.includes(key));

if (missingInFr.length > 0) {
  console.error('Missing French translations:', missingInFr);
  process.exit(1);
}

if (missingInEn.length > 0) {
  console.error('Missing English translations:', missingInEn);
  process.exit(1);
}

console.log('✅ All translations validated');

    Run in CI/CD:

    # .github/workflows/i18n.yml
name: Translation Validation

on: [push, pull_request]

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
      - run: npm ci
      - run: npm run validate:translations

    Date and Time Formatting¶

    Locale-specific date/time formatting:

    import { formatDate, formatTime, formatDateTime } from '@eva-suite/sovereign-ui';

const date = new Date('2025-01-15T14:30:00Z');

// English formatting
formatDate(date, 'en'); // "January 15, 2025"
formatTime(date, 'en'); // "2:30 PM"
formatDateTime(date, 'en'); // "January 15, 2025 at 2:30 PM"

// French formatting
formatDate(date, 'fr'); // "15 janvier 2025"
formatTime(date, 'fr'); // "14 h 30"
formatDateTime(date, 'fr'); // "15 janvier 2025 à 14 h 30"

    Custom Date Formats¶

    import { DateFormatter } from '@eva-suite/sovereign-ui';

const formatter = new DateFormatter({
  en: {
    short: 'MMM D, YYYY',      // Jan 15, 2025
    medium: 'MMMM D, YYYY',    // January 15, 2025
    long: 'dddd, MMMM D, YYYY' // Wednesday, January 15, 2025
  },
  fr: {
    short: 'D MMM YYYY',       // 15 jan. 2025
    medium: 'D MMMM YYYY',     // 15 janvier 2025
    long: 'dddd D MMMM YYYY'   // mercredi 15 janvier 2025
  }
});

formatter.format(date, 'long'); // Auto-detects current language

    Number Formatting¶

    Locale-specific number formatting:

    import { formatNumber, formatCurrency, formatPercent } from '@eva-suite/sovereign-ui';

const amount = 1234567.89;

// English formatting
formatNumber(amount, 'en');        // "1,234,567.89"
formatCurrency(amount, 'en');      // "$1,234,567.89"
formatPercent(0.1234, 'en');       // "12.34%"

// French formatting
formatNumber(amount, 'fr');        // "1 234 567,89"
formatCurrency(amount, 'fr');      // "1 234 567,89 $"
formatPercent(0.1234, 'fr');       // "12,34 %"

    Currency Symbol Positioning¶

    Government of Canada style guide: - English: $1,234.56 (symbol before, no space) - French: 1 234,56 $ (symbol after, space separator)

    EVA Sovereign UI handles this automatically:

    <eva-gc-currency
  value="1234.56"
  currency="CAD"
></eva-gc-currency>

// Renders:
// EN: "$1,234.56 CAD"
// FR: "1 234,56 $ CAD"

    RTL (Right-to-Left) Support¶

    While EN-CA and FR-CA are both LTR (left-to-right), EVA Sovereign UI includes RTL support for future expansion:

    /* Automatically flips layout for RTL languages */
:host {
  direction: ltr;
}

:host([dir="rtl"]) {
  direction: rtl;
}

/* Use logical properties for RTL compatibility */
:host {
  margin-inline-start: 1rem;  /* margin-left in LTR, margin-right in RTL */
  margin-inline-end: 1rem;    /* margin-right in LTR, margin-left in RTL */
  padding-block-start: 0.5rem; /* padding-top */
  padding-block-end: 0.5rem;   /* padding-bottom */
}

    Content Management Integration¶

    Headless CMS Example¶

    Fetch bilingual content from external CMS:

    interface CMSContent {
  id: string;
  title: { en: string; fr: string };
  body: { en: string; fr: string };
  publishedAt: Date;
}

async function fetchContent(slug: string): Promise<CMSContent> {
  const response = await fetch(`/api/content/${slug}`);
  return response.json();
}

// Render bilingual content
const content = await fetchContent('welcome-message');

const titleElement = document.createElement('eva-gc-heading');
titleElement.setAttribute('text-en', content.title.en);
titleElement.setAttribute('text-fr', content.title.fr);
titleElement.setAttribute('level', '1');

const bodyElement = document.createElement('eva-gc-rich-text');
bodyElement.setAttribute('content-en', content.body.en);
bodyElement.setAttribute('content-fr', content.body.fr);

    Static Site Generation¶

    Pre-render bilingual pages at build time:

    // build-pages.ts
import { generatePage } from './generator';

const pages = [
  { slug: 'home', titleEn: 'Home', titleFr: 'Accueil' },
  { slug: 'about', titleEn: 'About', titleFr: 'À propos' },
  { slug: 'contact', titleEn: 'Contact', titleFr: 'Contactez-nous' }
];

for (const page of pages) {
  // Generate English version
  await generatePage({
    slug: page.slug,
    language: 'en',
    title: page.titleEn
  });

  // Generate French version
  await generatePage({
    slug: page.slug,
    language: 'fr',
    title: page.titleFr
  });
}

    Summary¶

    EVA Sovereign UI's i18n architecture ensures:

    • Type-safe bilingualism: Compile-time enforcement of EN-CA/FR-CA content
    • Centralized translation management: i18n service with validation
    • Seamless language switching: No page reload, automatic component updates
    • Professional translation workflow: JSON-based, CI/CD validation
    • Locale-specific formatting: Dates, times, numbers, currency
    • RTL support: Ready for future language expansion

    All Government of Canada applications built with EVA Sovereign UI comply with the Official Languages Act by default.
