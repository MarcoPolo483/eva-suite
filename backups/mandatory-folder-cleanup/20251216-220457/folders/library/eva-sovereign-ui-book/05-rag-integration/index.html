    Chapter 05: RAG Integration in EVA Chat 2.0¶

    Overview¶

    Retrieval-Augmented Generation (RAG) is the core capability that enables EVA Chat 2.0 to provide accurate, source-backed answers grounded in Government of Canada documents. Unlike pure generative models that hallucinate or provide outdated information, RAG combines:

    1. Retrieval: Finding relevant documents from trusted sources (SharePoint, Azure Storage, internal knowledge bases)
    2. Augmentation: Injecting retrieved content into LLM prompts as context
    3. Generation: Producing responses with explicit citations and confidence scores

    This chapter covers the complete RAG pipeline: - Azure Cognitive Search integration (indexing, vector search) - Document ingestion from SharePoint and Azure Blob Storage - Vector embeddings with Azure OpenAI text-embedding-ada-002 - Citation generation with source attribution and confidence scoring - eva-gc-citation-card component for compliance-ready UI

    Azure Cognitive Search Integration¶

    Index Schema¶

    EVA Chat 2.0 uses a hybrid index combining vector embeddings and keyword search:

    import { SearchIndexClient, SearchIndex } from '@azure/search-documents';
import { DefaultAzureCredential } from '@azure/identity';

const indexSchema: SearchIndex = {
  name: 'eva-documents',
  fields: [
    { name: 'id', type: 'Edm.String', key: true },
    { name: 'title', type: 'Edm.String', searchable: true },
    { name: 'titleFr', type: 'Edm.String', searchable: true },  // Bilingual
    { name: 'content', type: 'Edm.String', searchable: true },
    { name: 'contentVector', type: 'Collection(Edm.Single)',
      searchable: true,
      vectorSearchDimensions: 1536,  // text-embedding-ada-002
      vectorSearchProfileName: 'eva-vector-profile'
    },
    { name: 'url', type: 'Edm.String', filterable: true },
    { name: 'source', type: 'Edm.String', filterable: true },
    { name: 'classification', type: 'Edm.String', filterable: true },
    { name: 'department', type: 'Edm.String', filterable: true },
    { name: 'lastModified', type: 'Edm.DateTimeOffset', sortable: true },
    { name: 'isLatestVersion', type: 'Edm.Boolean', filterable: true },
    { name: 'metadata', type: 'Edm.ComplexType', fields: [
      { name: 'author', type: 'Edm.String', searchable: true },
      { name: 'tags', type: 'Collection(Edm.String)', filterable: true }

  ],
  vectorSearch: {
    profiles: [
      {
        name: 'eva-vector-profile',
        algorithm: 'eva-hnsw-algorithm'
      }

    algorithms: [
      {
        name: 'eva-hnsw-algorithm',
        kind: 'hnsw',
        hnswParameters: {
          metric: 'cosine',
          m: 4,
          efConstruction: 400,
          efSearch: 500
        }
      }
    ]
  },
  scoringProfiles: [
    {
      name: 'recency-boost',
      text: { weights: { title: 2.0, content: 1.0 } },
      functions: [
        {
          type: 'freshness',
          fieldName: 'lastModified',
          boost: 2.0,

async function createSearchIndex() {
  const credential = new DefaultAzureCredential();
  const client = new SearchIndexClient(
    process.env.AZURE_SEARCH_ENDPOINT!,
    credential
  );

  await client.createOrUpdateIndex(indexSchema);
  console.log('Search index created successfully');

    Vector Search Configuration¶

    HNSW (Hierarchical Navigable Small World) algorithm provides sub-linear search times for high-dimensional vectors:

    • Metric: Cosine similarity (ideal for text embeddings)
    • m: Number of bi-directional links (4 = balanced performance/memory)
    • efConstruction: Index build quality (400 = high accuracy)
    • efSearch: Query time accuracy (500 = prioritize recall)

    Document Ingestion Pipeline¶

    SharePoint Integration¶

    EVA Chat 2.0 ingests documents from SharePoint Online via Microsoft Graph API:

    import { Client } from '@microsoft/microsoft-graph-client';
import { TokenCredentialAuthenticationProvider } from '@microsoft/microsoft-graph-client/authProviders/azureTokenCredentials';
import { DefaultAzureCredential } from '@azure/identity';

class SharePointIngestionService {
  private graphClient: Client;

  constructor() {
    const credential = new DefaultAzureCredential();
    const authProvider = new TokenCredentialAuthenticationProvider(credential, {
      scopes: ['https://graph.microsoft.com/.default']
    });

    this.graphClient = Client.initWithMiddleware({ authProvider });
  }

  async ingestSite(siteId: string, libraryName: string): Promise<void> {
    // Get all documents in library
    const driveId = await this.getDriveId(siteId, libraryName);
    const items = await this.graphClient
      .api(`/drives/${driveId}/root/children`)
      .select(['id', 'name', 'webUrl', 'lastModifiedDateTime', 'file'])
      .filter("file ne null")  // Only files, not folders
      .get();

    for (const item of items.value) {
      await this.processDocument(driveId, item);
    }
  }

  private async processDocument(driveId: string, item: any): Promise<void> {
    // Download file content
    const contentStream = await this.graphClient
      .api(`/drives/${driveId}/items/${item.id}/content`)
      .getStream();

    // Extract text based on file type
    const text = await this.extractText(contentStream, item.file.mimeType);

    // Chunk and embed
    const chunks = this.chunkDocument(text, { maxTokens: 512, overlap: 50 });
    const embeddings = await this.embedChunks(chunks);

    // Index in Azure Cognitive Search
    await this.indexDocument({
      id: item.id,
      title: item.name,
      content: text,
      contentVector: embeddings[0],  // Use first chunk embedding as doc embedding
      url: item.webUrl,
      source: 'sharepoint',
      lastModified: new Date(item.lastModifiedDateTime),
      isLatestVersion: true
    });
  }

  private async extractText(stream: NodeJS.ReadableStream, mimeType: string): Promise<string> {
    // Use Azure Form Recognizer for PDF/images, textract for Office docs
    if (mimeType === 'application/pdf') {
      return await this.extractPdf(stream);
    } else if (mimeType.includes('word')) {
      return await this.extractWord(stream);
    } else if (mimeType.includes('excel')) {
      return await this.extractExcel(stream);
    } else {
      throw new Error(`Unsupported file type: ${mimeType}`);
    }
  }
}

    Azure Blob Storage Integration¶

    For documents stored in Azure Blob Storage (e.g., archived PDFs, compliance docs):

    import { BlobServiceClient } from '@azure/storage-blob';

class BlobStorageIngestionService {
  private blobClient: BlobServiceClient;

  constructor() {
    const credential = new DefaultAzureCredential();
    this.blobClient = new BlobServiceClient(
      process.env.AZURE_STORAGE_ACCOUNT_URL!,
      credential
    );
  }

  async ingestContainer(containerName: string): Promise<void> {
    const containerClient = this.blobClient.getContainerClient(containerName);

    // List all blobs
    for await (const blob of containerClient.listBlobsFlat()) {
      if (blob.properties.contentType?.includes('pdf') ||
          blob.properties.contentType?.includes('word')) {
        await this.processBlob(containerName, blob.name);
      }
    }
  }

  private async processBlob(containerName: string, blobName: string): Promise<void> {
    const blobClient = this.blobClient
      .getContainerClient(containerName)
      .getBlobClient(blobName);

    // Download blob
    const downloadResponse = await blobClient.download();
    const content = await this.streamToBuffer(downloadResponse.readableStreamBody!);

    // Extract and index (similar to SharePoint flow)
    const text = await this.extractText(content, blobClient.name);
    const chunks = this.chunkDocument(text);
    const embeddings = await this.embedChunks(chunks);

    await this.indexDocument({
      id: blobClient.name,
      title: blobClient.name,
      content: text,
      contentVector: embeddings[0],
      url: blobClient.url,
      source: 'azure-blob',
      lastModified: (await blobClient.getProperties()).lastModified!,
      isLatestVersion: true
    });
  }
}

    Document Chunking Strategy¶

    Why Chunking? LLMs have context limits (8K-128K tokens). Documents must be split into chunks for: 1. Manageable embedding size (text-embedding-ada-002 supports max 8191 tokens) 2. Precise retrieval (smaller chunks = more focused matches) 3. Citation granularity (cite specific paragraphs, not entire docs)

    interface ChunkingOptions {
  maxTokens: number;        // Target chunk size (512-1024 recommended)
  overlap: number;          // Overlap between chunks (50-100 tokens)
  preserveParagraphs: boolean;  // Don't split mid-paragraph
}

class DocumentChunker {
  chunk(text: string, options: ChunkingOptions): string[] {
    const { maxTokens, overlap, preserveParagraphs } = options;

    // Split into paragraphs first
    const paragraphs = text.split(/\n\n+/);
    const chunks: string[] = [];
    let currentChunk = '';
    let currentTokens = 0;

    for (const paragraph of paragraphs) {
      const paragraphTokens = this.estimateTokens(paragraph);

      // If adding this paragraph exceeds limit, start new chunk
      if (currentTokens + paragraphTokens > maxTokens && currentChunk.length > 0) {
        chunks.push(currentChunk.trim());

        // Create overlap by keeping last N tokens
        const overlapText = this.getLastNTokens(currentChunk, overlap);
        currentChunk = overlapText + '\n\n' + paragraph;
        currentTokens = this.estimateTokens(currentChunk);
      } else {
        currentChunk += '\n\n' + paragraph;
        currentTokens += paragraphTokens;
      }
    }

    if (currentChunk.length > 0) {
      chunks.push(currentChunk.trim());
    }

    return chunks;
  }

  private estimateTokens(text: string): number {
    // Rough heuristic: 1 token ≈ 4 characters
    return Math.ceil(text.length / 4);
  }

  private getLastNTokens(text: string, n: number): string {
    const words = text.split(/\s+/);
    return words.slice(-n).join(' ');
  }
}

    Vector Embeddings with Azure OpenAI¶

    Embedding Generation¶

    EVA Chat 2.0 uses text-embedding-ada-002 (1536 dimensions) for semantic search:

    import { OpenAIClient } from '@azure/openai';

class EmbeddingService {
  private openai: OpenAIClient;
  private readonly EMBEDDING_MODEL = 'text-embedding-ada-002';
  private readonly BATCH_SIZE = 16;  // Azure OpenAI supports batching

  constructor() {
    const credential = new DefaultAzureCredential();
    this.openai = new OpenAIClient(
      process.env.AZURE_OPENAI_ENDPOINT!,
      credential
    );
  }

  async embedChunks(chunks: string[]): Promise<number[][]> {
    const embeddings: number[][] = [];

    // Process in batches to optimize API calls
    for (let i = 0; i < chunks.length; i += this.BATCH_SIZE) {
      const batch = chunks.slice(i, i + this.BATCH_SIZE);
      const response = await this.openai.getEmbeddings(
        this.EMBEDDING_MODEL,
        batch
      );

      embeddings.push(...response.data.map(d => d.embedding));
    }

    return embeddings;
  }

  async embedQuery(query: string): Promise<number[]> {
    const response = await this.openai.getEmbeddings(
      this.EMBEDDING_MODEL,
      [query]
    );

    return response.data[0].embedding;
  }
}

    Embedding Best Practices¶

    1. Normalize input text: Remove extra whitespace, special characters
    2. Prepend context: For domain-specific embeddings, prepend domain keywords
    3. Cache embeddings: Store in Redis to avoid re-computing
    4. Monitor token usage: text-embedding-ada-002 costs $0.0001/1K tokens
    function normalizeText(text: string): string {
  return text
    .replace(/\s+/g, ' ')           // Collapse whitespace
    .replace(/[^\w\s.,!?-]/g, '')   // Remove special chars
    .trim()
    .toLowerCase();
}

function prependContext(text: string, domain: string): string {
  const contextMap: Record<string, string> = {
    'ei': 'Employment Insurance Canada',
    'cpp': 'Canada Pension Plan',
    'oas': 'Old Age Security'
  };

  const prefix = contextMap[domain] || '';
  return prefix ? `${prefix}: ${text}` : text;
}

    Hybrid Search: Vector + Keyword Fusion¶

    Reciprocal Rank Fusion (RRF)¶

    Azure Cognitive Search supports hybrid search combining vector and keyword results:

    import { SearchClient } from '@azure/search-documents';

class HybridSearchService {
  private searchClient: SearchClient<any>;

  constructor() {
    const credential = new DefaultAzureCredential();
    this.searchClient = new SearchClient(
      process.env.AZURE_SEARCH_ENDPOINT!,
      'eva-documents',
      credential
    );
  }

  async search(query: string, options: SearchOptions): Promise<SearchResult[]> {
    // Generate query embedding
    const queryVector = await this.embeddingService.embedQuery(query);

    // Execute hybrid search
    const results = await this.searchClient.search(query, {
      vectorSearchOptions: {
        queries: [
          {
            kind: 'vector',
            vector: queryVector,
            kNearestNeighborsCount: 50,  // Retrieve top 50 vector matches
            fields: ['contentVector']
          }
        ]
      },
      queryType: 'semantic',  // Use semantic search for keyword ranking
      semanticSearchOptions: {
        configurationName: 'eva-semantic-config',
        captions: { highlight: true },  // Generate highlighted snippets
        answers: { count: 3 }
      },
      filter: this.buildSecurityFilter(options),
      top: 10,
      scoringProfile: 'recency-boost'
    });

    return this.mapResults(results);
  }

  private buildSecurityFilter(options: SearchOptions): string {
    const filters: string[] = [];

    // Filter by user's security clearance
    if (options.userClearance === 'unclassified') {
      filters.push(`classification eq 'unclassified'`);
    } else if (options.userClearance === 'protected-a') {
      filters.push(`classification in ('unclassified', 'protected-a')`);
    }
    // protected-b users see everything

    // Filter by department (optional)
    if (options.department) {
      filters.push(`department eq '${options.department}'`);
    }

    // Temporal filter: last 2 years by default
    const twoYearsAgo = new Date();
    twoYearsAgo.setFullYear(twoYearsAgo.getFullYear() - 2);
    filters.push(`lastModified ge ${twoYearsAgo.toISOString()}`);

    return filters.join(' and ');
  }
}

    Ranking and Scoring¶

    Hybrid Search Score = α × VectorScore + (1-α) × KeywordScore

    Azure Cognitive Search automatically fuses results using RRF:

    RRF_Score(doc) = Σ (1 / (k + rank_i))

where:
  k = 60 (constant to reduce impact of high ranks)
  rank_i = rank of doc in result set i (vector or keyword)

    Citation Generation¶

    Citation Service¶

    The CitationService generates structured citations with source attribution:

    interface GeneratedCitation {
  id: string;
  title: string;
  titleFr: string;
  url: string;
  excerpt: string;
  confidence: number;
  source: 'sharepoint' | 'azure-blob' | 'internal-kb';
  classification: string;
  lastModified: Date;
  highlightedSnippet?: string;  // From semantic search captions
}

class CitationService {
  async generateCitations(
    query: string,
    searchResults: SearchResult[]
  ): Promise<GeneratedCitation[]> {
    return searchResults.map((result, index) => ({
      id: result.document.id,
      title: result.document.title,
      titleFr: result.document.titleFr,
      url: result.document.url,
      excerpt: this.extractExcerpt(result.document.content, query),
      confidence: this.normalizeScore(result.score),
      source: result.document.source,
      classification: result.document.classification,
      lastModified: new Date(result.document.lastModified),
      highlightedSnippet: result.captions?.[0]?.text  // From semantic search
    }));
  }

  private extractExcerpt(content: string, query: string, maxLength = 200): string {
    // Find best matching sentence containing query terms
    const sentences = content.split(/[.!?]+/);
    const queryTerms = query.toLowerCase().split(/\s+/);

    const scored = sentences.map(sentence => {
      const lowerSentence = sentence.toLowerCase();
      const matchCount = queryTerms.filter(term => lowerSentence.includes(term)).length;
      return { sentence, score: matchCount };
    });

    const best = scored.sort((a, b) => b.score - a.score)[0];
    const excerpt = best.sentence.trim();

    return excerpt.length > maxLength
      ? excerpt.substring(0, maxLength) + '...'
      : excerpt;
  }

  private normalizeScore(score: number): number {
    // Azure Search scores are unbounded - normalize to 0-1
    return Math.min(score / 10, 1.0);
  }
}

    Confidence Scoring¶

    Confidence scores help users assess citation reliability:

    function calculateConfidence(citation: GeneratedCitation): number {
  let confidence = citation.confidence;  // Base vector similarity score

  // Boost for recent documents
  const ageInDays = (Date.now() - citation.lastModified.getTime()) / (1000 * 60 * 60 * 24);
  if (ageInDays < 30) confidence *= 1.2;
  else if (ageInDays < 180) confidence *= 1.1;
  else if (ageInDays > 730) confidence *= 0.8;  // Penalize old docs

  // Boost for authoritative sources
  if (citation.source === 'internal-kb') confidence *= 1.15;

  // Cap at 1.0
  return Math.min(confidence, 1.0);
}

  private getConfidenceLevel(): string {
    if (this.confidence >= 0.7) return 'high';
    if (this.confidence >= 0.4) return 'medium';
    return 'low';
  }
}

    Performance Optimization¶

    Caching Strategy¶

    Problem: Embedding generation and search are expensive operations.

    Solution: Multi-tier caching with Redis and CDN.

    class CachedSearchService {
  private redis: RedisClient;
  private ttl = 3600;  // 1 hour cache

  async search(query: string, options: SearchOptions): Promise<SearchResult[]> {
    // Generate cache key
    const cacheKey = this.generateCacheKey(query, options);

    // Check cache
    const cached = await this.redis.get(cacheKey);
    if (cached) {
      return JSON.parse(cached);
    }

    // Cache miss - execute search
    const results = await this.executeSearch(query, options);

    // Store in cache
    await this.redis.setEx(cacheKey, this.ttl, JSON.stringify(results));

    return results;
  }

  private generateCacheKey(query: string, options: SearchOptions): string {
    const normalized = query.toLowerCase().trim();
    const optionsHash = this.hashOptions(options);
    return `search:${normalized}:${optionsHash}`;
  }
}

    Batch Processing for Ingestion¶

    Problem: Ingesting 10,000+ documents sequentially takes hours.

    Solution: Parallel processing with Azure Functions and Service Bus.

    class BatchIngestionService {
  private serviceBusClient: ServiceBusClient;

  async ingestBatch(documents: Document[]): Promise<void> {
    const sender = this.serviceBusClient.createSender('document-ingestion');

    // Split into batches of 100
    const batches = this.chunkArray(documents, 100);

    for (const batch of batches) {
      const messages = batch.map(doc => ({
        body: { documentId: doc.id, url: doc.url }
      }));

      await sender.sendMessages(messages);
    }

    console.log(`Queued ${documents.length} documents for ingestion`);
  }

  private chunkArray<T>(array: T[], size: number): T[][] {
    return Array.from({ length: Math.ceil(array.length / size) }, (_, i) =>
      array.slice(i * size, i * size + size)
    );
  }
}

    Access Control Enforcement¶

    CRITICAL: EVA Chat 2.0 MUST validate document access BEFORE returning citations:

    class SecureSearchService {
  async search(query: string, userId: string): Promise<SearchResult[]> {
    // Execute search
    const rawResults = await this.hybridSearch(query);

    // Filter by user permissions (parallel validation)
    const validatedResults = await Promise.all(
      rawResults.map(async result => {
        const hasAccess = await this.validateAccess(userId, result.document);
        return hasAccess ? result : null;
      })
    );

    return validatedResults.filter(r => r !== null) as SearchResult[];
  }

  private async validateAccess(userId: string, doc: Document): Promise<boolean> {
    const user = await this.userService.getProfile(userId);

    // Check security clearance
    if (doc.classification === 'protected-b' && user.securityClearance !== 'protected-b') {
      return false;
    }

    // Check SharePoint permissions
    if (doc.source === 'sharepoint') {
      return await this.graphClient.checkPermissions(userId, doc.url);
    }

    return true;
  }
}

    class AuditLogger {
  async logSearch(event: SearchEvent): Promise<void> {
    await this.cosmosClient.items.create({

      userId: event.userId,

  private sanitize(text: string): string {
    // Remove potential PII (emails, phone numbers, SINs)
    return text
      .replace(/\b[\w\.-]+@[\w\.-]+\.\w+\b/g, '[EMAIL]')
      .replace(/\b\d{3}-\d{3}-\d{4}\b/g, '[PHONE]')
      .replace(/\b\d{3}-\d{3}-\d{3}\b/g, '[SIN]');
  }
